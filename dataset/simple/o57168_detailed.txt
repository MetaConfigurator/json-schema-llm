The schema describes configuration settings for a file checking tool. It is an object type. The main properties of the schema are: 'start_path' is a string or an array of strings that specifies the path(s) of the source file to begin the checking process. It is required. Multiple paths are allowed and individually checked. 'package_path' is an optional string that represents a value of `package.path`, relative to the workspace directory. 'package_cpath' is an optional string that represents a value of `package.cpath`, also relative to the workspace directory. 'preload' is an optional object that contains properties 'open' and 'require', both of which are arrays of strings, listing libraries to be preloaded. No additional properties are allowed at the root level.
