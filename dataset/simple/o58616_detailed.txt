The schema describes a Football Player. It is an object type. It defines several reusable components under 'definitions': 'person' is an object, it requires the fields 'first_name' and 'last_name'. It allows an optional field 'age' of type integer. 'football_team' is an object, it requires the fields 'name' and 'league'. It allows an optional field 'year_founded' of type integer. The main properties of the schema are: 'current_club' is a football_team reference, and it is required. The schema requires an object which must validate against both definitions and the additional structure. No additional properties are allowed at the root level.
