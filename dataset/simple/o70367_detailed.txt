The schema describes an error response. It is an object type. It defines several reusable components under 'definitions': 'error-codeType' is a string, which must be one of the following values: 'not-open-status', 'subscription-used', 'investor-not-found', 'wrong-lisa-manager', 'cant-change-accountID', 'cant-change-investor', 'invalid-status-change', 'no-acccount-with-manager', 'investor-already-created', 'compliance-charge-wrong', 'compliance-bonus-wrong', 'compliance-too-many-subscriptions-in-FP', 'compliance-too-much-LISA-funds-credited', 'compliance-too-much-ISA-funds-credited', 'compliance-account-open-outside-age-range', or 'compliance-funds-credit-outside-age-range'. The main properties of the schema are: 'code' is a reference to 'error-codeType', required. 'message' is a string, required. No additional properties are allowed at the root level.
