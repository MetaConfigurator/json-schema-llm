The schema describes a configuration push response. It is an object type. The main properties of the schema are: 'configId' is a string that describes the identifier of the current configuration and is required. 'id' is a number that describes the ID of the message used to match client response to the request and is required. 'reasonPhrase' is a string that describes a human-readable string explaining the cause of an error (if any); in case the request was successful, it is `ok`, and it is required. 'statusCode' is a number that describes the status code based on HTTP status codes and is required. No additional properties are allowed at the root level.
