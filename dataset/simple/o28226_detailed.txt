The schema describes the utility configuration for a particular project. It is an object type. The main properties of the schema are: 'bundle_id' is a string that represents the workflow's unique bundle ID and is required. 'export_file' is a string that represents the path to the exported Alfred workflow file, must match the pattern '^(.*?)\.alfred3?workflow$', and is required. 'readme' is a string that represents the path to the README file to include in the workflow and is optional. 'resources' is an array of strings that contains a list of file patterns to include in the packaged workflow and is required. No additional properties are allowed at the root level.
