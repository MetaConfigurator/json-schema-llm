The schema describes a broker mapping file schema used to validate metadata map files. It is an object type. It defines several reusable components under 'definitions': 'Map' is an array, 'Settings' is an array that contains objects with properties: 'name' which is an array, 'map_description' which is a string, 'root' which is a string, 'outputfile' which is a string, and 'mode' which is a string. The 'Settings' component requires all of these fields. The main properties of the schema are: 'Map' is an array that is referenced from the 'Map' definition. 'Settings' is an array that is referenced from the 'Settings' definition. No additional properties are allowed at the root level.
