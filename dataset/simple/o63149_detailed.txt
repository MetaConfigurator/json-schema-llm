The schema describes a completion of a BEL expression. It is an object type. The main properties of the schema are: 'completion' is an object that is required. It has no additional properties and requires the fields 'type', 'label', 'value', 'highlight', and 'actions'. The 'type' is an enum with allowed values 'function', 'namespace_prefix', and 'namespace_value'. The 'label' is a string. The 'value' is a string. The 'highlight' can be either null or an object that is required to have 'start_position', 'end_position', and 'range_type', where 'start_position' and 'end_position' are integers and 'range_type' is an enum with allowed values 'exclusive' and 'inclusive'. The 'actions' is an array, where each item can be one of three types: an object with a required 'delete' field (which also requires 'start_position', 'end_position', and 'range_type'), an object with a required 'insert' field (which requires 'position' and 'value'), or an object with a required 'move_cursor' field (which requires 'position'). No additional properties are allowed at the root level.
