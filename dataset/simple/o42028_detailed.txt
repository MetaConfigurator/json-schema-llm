The schema describes the result of a BlackDuck worker. It is an object type. The main properties of the schema are: 'schema' is an object that contains the properties 'name', 'version', and 'url'. 'name' is a string that represents the name of the schema and must match the pattern '^[a-zA-Z0-9_]+$'. 'version' is a string that represents the version of the schema and must match the pattern '^[0-9]+-[0-9]+-[0-9]+$'. Both 'name' and 'version' are required fields, while 'url' is optional and must be a valid URI. The property '_release' is a string that describes a unique release id in the form of 'ecosystem:package:version'. Additional properties are allowed at the root level.
