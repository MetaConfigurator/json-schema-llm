The schema describes a configuration for a device. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'i2c' is an object that contains properties 'speed' of type integer and 'stretch_clk' of type boolean, both optional. 'name' is a string that must match the pattern '^particle_electron$', and it is required. 'serial' is an array of integers, and it is optional. 'systmr_cfg' is an object that contains properties 'freq_hz' of type integer, which is optional, and 'owner' of type string with an enum constraint, where allowed values are 'thecore' and 'user', both optional. 'use_console' is a boolean that is optional. No additional properties are allowed at the root level.
