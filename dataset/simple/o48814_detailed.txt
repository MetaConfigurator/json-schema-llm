The schema describes a node's health status and related information. It is an object type. The main properties of the schema are: 'health' is an integer that represents the node health status, which is required and must be between 0 and 3. 'help' is a string that provides the node health check output; it is optional. 'host_ip' is a string representing the host IP from the ip-detect script, which is required and must match the pattern '^[\w\.]+$'. 'role' is a string indicating the node type, which is required and must be one of the following: 'master', 'agent', 'unknown'. No additional properties are allowed at the root level.
