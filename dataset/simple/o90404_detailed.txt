The schema describes a configuration for taints for use in K2. It is an object type. The main properties of the schema are: 'key' is a string representing the taint key which must match toleration, required, with a pattern of '^([A-Za-z0-9][A-Za-z0-9-._]*)?$', minimum length of 0, and maximum length of 253. 'value' is a string representing the taint value, required, with a pattern of '^([A-Za-z0-9][A-Za-z0-9-._]*)?$', minimum length of 0, and maximum length of 63. 'effect' is a string indicating the effect which can be one of the following values: 'NoSchedule', 'PreferNoSchedule', or 'NoExecute', required. No additional properties are allowed at the root level.
