The schema describes an API authorization request. It is an object type. The main properties of the schema are: 'accessToken' is a string that must have a minimum length of 1 and is required. 'requestedScopes' is an array of unique items, containing strings that must match specific patterns defined in two groups of enum values, supporting various application, organization, and user scopes. 'tokenTTL' is an integer with a minimum value of 0 and is optional. No additional properties are allowed at the root level.
