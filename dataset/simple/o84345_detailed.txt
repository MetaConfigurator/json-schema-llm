The schema describes a comment that is attached to an object (supported objects: release). It is an object type. The main properties of the schema are: '_id' is a string unique identifier for a build. Gets generated automatically on creation. It has a pattern of '^[a-z0-9._-]{2,}$' and is readOnly. 'created_at' is a string timestamp when the comment was posted. It is readOnly. 'from' is an object user who wrote the comment. It is also readOnly. 'message' is a string message. It has a maxLength of 5000 and a minLength of 3. It is required. The root schema requires 'id' and 'message'. No additional properties are allowed at the root level.
