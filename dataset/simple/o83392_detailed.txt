The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': 'regionName' is a string that must match the pattern '^[a-z]+(?:-+[a-z]+)*-v\d+$'. The main properties of the schema are: 'name' is a string that is required. 'label' is a string that is required. 'version' is an integer that is required and has a default value of 1. 'regions' is an array of items that must conform to the 'regionName' definition, and it is required. No additional properties are allowed at the root level.
