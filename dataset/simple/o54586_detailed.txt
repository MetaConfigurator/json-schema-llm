The schema describes a user profile perspective. It is an object type. The main properties of the schema are: 'avatar' is an object which contains the details for an avatar upload, including a component type, helper text, service name, accepted file types (image/*), maximum file size of 0.5, an auto-process flag set to false, a storage path formatted as 'avatars/<%= id %>', and a parameters object indicating public access. 'name' is a string representing the user's name with a minimum length of 3 and a maximum length of 128. 'name' is required. No additional properties are allowed at the root level.
