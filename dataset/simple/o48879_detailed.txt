The schema describes the structure of a time tracking entity. It is an object type. It defines several reusable components under 'definitions': 'id' is a string, representing a UUID. 'time' is a string, required to match the pattern '^20[0-1][0-9]-[0-1][0-9]-[0-3][0-9]T[0-2][0-9]:[0-5][0-9]:[0-5][0-9]\.000Z$'. 'timezoneName' is a string, representing the name of the timezone. 'timezoneOffset' is a number, which must be between -720 and 840. 'trackedTime' is a string, resolving to a time definition. The main properties of the schema are: 'id' is a string, required. 'time' is a string, required. 'timezoneName' is a string, required. 'timezoneOffset' is a number, required. 'trackedTime' is a string, required. No additional properties are allowed at the root level.
