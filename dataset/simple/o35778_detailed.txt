The schema describes the representation of a deployed service from a Charm. It is an object type. The main properties of the schema are: '$schema-location' is a string that must be one of the allowed values: 'Service-Schema.json'. 'id' is a string and is required. 'exposed' is a boolean, is required, and has a default value of false, with the description 'whether the service is exposed to outside world or not'. No additional properties are allowed at the root level.
