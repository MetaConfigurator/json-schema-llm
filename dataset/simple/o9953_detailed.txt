The schema describes a data structure for managing destination information. It is an object type. The main properties of the schema are: 'destinationId' is a string matching the pattern '^[A-Fa-f\d]{24}$' and is required. 'destinationType' is a string and is required; it can be either 'user' or 'organization'. 'applicationIds' is an optional array containing strings, each matching the pattern '^[A-Fa-f\d]{24}$', with a maximum of 1000 items. 'dashboardIds' is an optional array containing strings, each matching the pattern '^[A-Fa-f\d]{24}$', with a maximum of 1000 items. 'strict' is an optional boolean. No additional properties are allowed at the root level.
