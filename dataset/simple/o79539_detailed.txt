The schema describes an object representation of a recruitment process bucket. It is an object type. It defines several reusable components under 'definitions': 'id' is an integer, representing a unique identifier. The main properties of the schema are: 'id' is an integer, required; 'label' is a string, required; 'phase' is a string describing the part of the recruitment process this bucket is for, required, with allowed values being 'new', 'screening', 'interview', 'selection', 'offer', 'approval', 'hire', 'onboard', 'decline', 'other'; 'key' is a string, optional. No additional properties are allowed at the root level.
