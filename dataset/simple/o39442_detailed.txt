The schema describes a user update request. It is an object type. It defines several reusable components under 'definitions': 'Details' is an object, representing the details of each update. It has required fields 'field' and 'value'. The 'field' property is a string with an enum constraint allowing the values 'USER_NAME' or 'DISPLAY_NAME'. The 'value' property is a string. The main properties of the schema are: 'updates' is an array describing a list of updates to apply to the user. Each item in the array must conform to the 'Details' definition. No additional properties are allowed at the root level.
