The schema describes a value effect for a performance table entry of absolute data. It is an object type. The main properties of the schema are: 'type' is a string that is required and must be exactly 'exact'. 'value' is a number that is required. 'input' is an object that is optional and does not allow additional properties. Within 'input', 'value' is a number that is optional, and 'scale' is a string that is optional and must be either 'percentage' or 'decimal'. No additional properties are allowed at the root level.
