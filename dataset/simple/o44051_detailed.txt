The schema describes a lax ingest+publish request. It is an object type. The main properties of the schema are: 'action' is a string that specifies the action to be performed; it is required and must be one of the following values: 'ingest', 'publish', 'ingest+publish'. 'location' is a string that indicates the location of the article to ingest/publish; it is required, must be a URI, and must match the pattern '^(https|file)://'. 'id' is a string that serves as the article identifier; it is required. 'version' is an integer representing the version and is required; it must be between 1 and 9. 'force' is a boolean that indicates if the ingest/publish action should be forced, intended for silent corrections; it is optional. 'token' is a string that contains data to be returned to the sender once the action is complete; it is required. No additional properties are allowed at the root level.
