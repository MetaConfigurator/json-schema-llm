The schema describes a CT transcode output specification. It is an object type. The main properties of the schema are: 'output_bucket' is a string that describes the S3 bucket and path where the output file will be uploaded. It is required and must be between 3 and 63 characters, matching the pattern '^[0-9a-zA-Z\.\-_]+$'. 'output_file' is a string that describes the filename where to upload the resulting file. It is required and must be between 1 and 1024 characters, matching the pattern '^[\w /-]+\.[0-9a-zA-Z]+$'. 'output_type' is a string that describes the type of the file to output. It is required and can be one of the following values: 'VIDEO', 'THUMB', 'AUDIO', 'IMG', 'DOC'. 's3_encrypt' is a boolean that activates backend storage encryption. 's3_rrs' is a boolean that activates reduced redundancy or not in S3 storage. No additional properties are allowed at the root level.
