The schema describes an LSN applications profile. It is an object type. The main properties of the schema are: 'appsprofilename' is a string representing the profile name. 'filtering' is a string that is optional and must be one of the following values: 'ENDPOINT-INDEPENDENT', 'ADDRESS-DEPENDENT', 'ADDRESS-PORT-DEPENDENT'. 'ippooling' is a string that is optional and must be one of the following values: 'PAIRED', 'RANDOM'. 'l2info' is a string that is optional and must be one of the following values: 'ENABLED', 'DISABLED'. 'mapping' is a string that is optional and must be one of the following values: 'ENDPOINT-INDEPENDENT', 'ADDRESS-DEPENDENT', 'ADDRESS-PORT-DEPENDENT'. 'tcpproxy' is a string that is optional and must be one of the following values: 'ENABLED', 'DISABLED'. 'td' is an integer that is optional. 'transportprotocol' is a string that is optional and must be one of the following values: 'TCP', 'UDP', 'ICMP'. No additional properties are allowed at the root level.
