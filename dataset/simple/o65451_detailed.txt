The schema describes a statutory filing. It is an object type. The main properties of the schema are: 'date' is a string representing a date and is required. 'description' is a string and is optional. 'filing_type_code' is a string and is optional. 'filing_type_name' is a string and is required. 'other_attributes' is an object and is optional. 'title' is a string and is required. 'uid' is a string and is optional. 'url' is a string and is optional. The schema has a compositional constraint that at least one of 'title', 'description', or 'filing_type_name' must be present. No additional properties are allowed at the root level.
