The schema describes an exception report. It is an object type. It defines several reusable components under 'definitions': 'exception' is an object, requiring a 'code' property. The 'code' property is a string with an enum that allows the following values: 'InvalidParameterValue', 'InvalidUpdateSequence', 'MissingParameterValue', 'NoApplicableCode', 'OperationNotSupported', 'OptionNotSupported', 'VersionNegotiationFailed', 'InvalidRequest', 'RequestExtensionNotSupported', 'ResponseExceedsSizeLimit', 'InvalidPropertyOfferingCombination'. The 'locator' property is an optional string. The 'text' property can be either a string or an array of strings, where the array must have at least one item. The main properties of the schema are: 'version' is a string and is required; 'exceptions' is an array, required, and must contain at least one item that references the 'exception' definition. No additional properties are allowed at the root level.
