The schema describes postal abbreviations for provinces and territories of Canada. It is an object type. It defines several reusable components under 'definitions': 'CAN_province' is a string, an enumeration of allowed values: 'AB', 'BC', 'MB', 'NB', 'NL', 'NS', 'NT', 'NU', 'ON', 'PE', 'QC', 'SK', 'YT'. 'CAN_postalCode' is a string, matching the pattern '^[A-Z][0-9][A-Z]( )?[0-9][A-Z][0-9]$'. The main properties of the schema are: 'CAN_province' is a string required. It references the 'CAN_province' definition. 'CAN_postalCode' is a string required. It references the 'CAN_postalCode' definition. No additional properties are allowed at the root level.
