The schema describes a PEG grammar structure. It is an object type. It defines several reusable components under 'definitions': 'node' is an object, representing a grammar node with properties 'ast' (a required string, allowing values: 'BUILD', 'SKIP', 'VOID', 'PRUNE'), 'type' (a required string, allowing values: 'RULE', 'CALL', 'SEQUENCE', 'CHOICE', 'RANGE', 'WILDCARD', 'STRING', 'STRINGI'), 'min' (an optional integer), and 'max' (an optional integer). The 'node' can be one of several types based on the 'anyOf' constraint, requiring either 'data' structured as follows: 'RULE' must have a 'data' that is a reference to another 'node', 'CALL', 'STRING', or 'STRINGI' must have a 'data' that is a string with a minimum length of 1, 'RANGE' must have a 'data' that is an array of integers, 'SEQUENCE' or 'CHOICE' must have a 'data' that is an array of nodes, or 'WILDCARD' which has no additional properties. The main properties of the schema are: Each property name matching the regex '^[a-zA-Z0-8_]+$' is a node, which follows the structure defined by 'node'. No additional properties are allowed at the root level.
