The schema describes a Field. It is an object type. The main properties of the schema are: 'name' is a string required field, 'definition' is a string required field, 'description' is a string optional field, 'type' is a string required field, 'identifier' is a string optional field, 'label' is a string optional field. The 'type' property must satisfy one of the following configurations: it can be one of the values: 'count', 'boolean', 'countRange', 'observableProperty', 'quality', 'text'; or if 'type' is one of 'quantity', 'quantityRange', 'time', or 'timeRange', then 'uom' is required; or if 'type' is 'category', then 'codespace' is required. No additional properties are allowed at the root level.
