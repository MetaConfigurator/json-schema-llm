The schema describes an API updater structure. It is an object type. The main properties of the schema are: 'emailAddress' is a string matching the pattern '^[^@^\s]+@[^@^\.^\s]+(\.[^@^\.^\s]+)+$', required. 'hashpw' is a boolean with a description 'set to False to skip password hashing', optional. 'name' is a string with a minimum length of 1 and a maximum length of 255, required. 'password' is a string with a minimum length of 10 and a maximum length of 255, required. 'phoneNumber' is a string matching the pattern '^$|^\+?([\d\s()-]){9,20}$', optional. 'role' is a string with an enum of allowed values: 'buyer', 'supplier', 'admin', 'admin-ccs-category', 'admin-ccs-sourcing', 'admin-manager', 'admin-framework-manager', 'admin-ccs-data-controller', required. 'supplierId' is an integer, optional. No additional properties are allowed at the root level.
