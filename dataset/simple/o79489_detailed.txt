The schema describes a set of SAML values that uniquely identify this person, consistently across TAS apps, and with any number of potential IdPs. It is an object type. It defines several reusable components under 'definitions': 'principalType' is an enum, a class of principal (typically a human) identifiable by SSO. In theory developer-defined but for now only TAS can create; allowed values are 'user', 'candidate'. The main properties of the schema are: 'pt' is a principalType principal type; it is required. 'samlKey' is a string key of the SAML assertion with the TAS core SAML database; it is required. 'nameID' is a string; it is required. 'entityID' is a string; it is required. No additional properties are allowed at the root level.
