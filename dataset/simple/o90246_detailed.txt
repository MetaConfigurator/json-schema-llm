The schema describes the structure of a multi-dimensional definition. It is an object type. The main properties of the schema are: 'dimensionName' is a string that describes the type of dimension being used (ex. StdTime). 'isSlice' is a boolean that indicates whether it is a slice, with a default value of true. 'values' is an array where every element is either a single number or an array of two numbers (lower and upper bound). Each inner array must contain exactly two items. 'variableName' is a string that specifies the name of the variable. The schema has compositional constraints where one of 'variableName' or both 'dimensionName' and 'values' is required, and 'dimensionName' is dependent on 'values' and vice versa. No additional properties are allowed at the root level.
