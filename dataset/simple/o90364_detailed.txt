The schema describes AWS Subnet Configuration. It is an object type. The main properties of the schema are: 'name' is a string the name of the subnet, for use within k2 configurations. It is required. 'az' is a string Name of the Availability Zone (AZ) in AWS. Regions and AZs are documented here. It is required and has a default value of 'us-east-1a'. 'cidr' is a string the cidr to associate to the subnet (within the VPC). It is required, has a format of cidr, and a default value of '10.0.0.0/18'. 'enablePublicIPs' is a boolean do we allow for Auto-assigned Public IPs, Our clusters default to true. It is optional and has a default value of 'true'. No additional properties are allowed at the root level.
