The schema describes a status notification request. It is an object type. The main properties of the schema are: 'connectorId' is an integer and required, 'errorCode' is a string and required with allowed values: 'ConnectorLockFailure', 'EVCommunicationError', 'GroundFailure', 'HighTemperature', 'InternalError', 'LocalListConflict', 'NoError', 'OtherError', 'OverCurrentFailure', 'PowerMeterFailure', 'PowerSwitchFailure', 'ReaderFailure', 'ResetFailure', 'UnderVoltage', 'OverVoltage', 'WeakSignal', 'status' is a string and required with allowed values: 'Available', 'Preparing', 'Charging', 'SuspendedEVSE', 'SuspendedEV', 'Finishing', 'Reserved', 'Unavailable', 'Faulted', 'info' is a string with a maxLength of 50 and optional, 'timestamp' is a string with a format of date-time and optional, 'vendorId' is a string with a maxLength of 255 and optional, 'vendorErrorCode' is a string with a maxLength of 50 and optional. No additional properties are allowed at the root level.
