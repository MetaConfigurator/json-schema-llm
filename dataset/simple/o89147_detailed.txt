The schema describes memo objects that represent arbitrary data that can be included in a transaction. It is an object type. The main properties of the schema are: 'type' is a string that is optional and must match the pattern '^[A-Za-z0-9\-._~:/?#\[\]@!$&'()*+,;=%]*$', describing a unique relation that defines the format of this memo. 'format' is a string that is optional and must match the pattern '^[A-Za-z0-9\-._~:/?#\[\]@!$&'()*+,;=%]*$', containing information on how the memo is encoded. 'data' is a string that is required in at least one of the valid configurations and conventionally contains the content of the memo. The schema requires that at least one of the fields 'data' or 'type' must be present. No additional properties are allowed at the root level.
