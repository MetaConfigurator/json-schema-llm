The schema describes a list of users. It is an object type. The main properties of the schema are: 'Name' is a string, the name of the user, required. 'ID' is an integer, the unique identifier for a user, required. 'Email' is a string, the email of the user, optional, with a format of email. 'Role' is a string, the role of the user, optional, with allowed values: null, 'Architect', 'Developer', 'Tech Lead', 'Product Owner', 'UX Designer', 'QE', 'Operations'. 'Active' is a boolean, indicating if the user is active, optional. No additional properties are allowed at the root level.
