The schema describes an array of savings accounts. It is an object type. The main properties of the schema are: 'savingsAccounts' is an array of objects representing savings accounts. It is a required property. Each item in the array must adhere to the 'UK savings account' structure. Items must include 'id' and 'accountName' as required fields. The 'id' is a string with a pattern that must match '^[A-Za-z0-9]{15}$'. The 'accountName' is a string with a pattern that must match '^[A-Za-z0-9 &'()*,-./@\u00a3]{1,32}$' and must be unique for each National Insurance number while being between 1 and 32 characters long. No additional properties are allowed at the root level.
