The schema describes a status report. It is an object type. The main properties of the schema are: 'status' is a string representing the status of the report. It is required and must be one of the following values: 'OK' or 'ERROR'. 'uptime' is a number indicating the uptime, which is required and must be at least 0. 'components' is an array of objects representing individual components of the status report. It is required. Each object in the array must contain the following properties: 'status' is a string representing the status of the component. It must be one of the following values: 'OK' or 'ERROR'. 'name' is a string representing the name of the component, required. 'date' is a string indicating the date, required. 'message' is a string providing additional information, optional. No additional properties are allowed at the root level.
