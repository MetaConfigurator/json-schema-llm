The schema describes the creation of data sources. It is an object type. The main properties of the schema are: 'config' is an object that defines the configuration details and is required. It contains a required property 'target' which is a string. Additional properties are allowed within 'config'. 'schoolId' is an optional string that must match the pattern '[a-f0-9]{24}'. 'name' is an optional string. 'protected' is an optional array of strings that represents fields in the config that should receive special protection; it is noted that a field named 'password' will always be protected. No additional properties are allowed at the root level.
