The schema describes a message structure for Choria requests. It is an object type. The main properties of the schema are: 'protocol' is a string that is required and must be one of the following values: 'choria:secure:request:1'. 'message' is a string that is required and describes a YAML encoded choria:request:1. 'signature' is a string that is required and describes a SHA256 based signature of the message using the sender's private key. 'pubcert' is a string that is required and describes a PEM format X509 certificate of the sender. No additional properties are allowed at the root level.
