The schema describes the configuration for a service and Spark environment. It is an object type. The main properties of the schema are: 'service' is an object that contains properties for the service configuration, which includes: 'name' is a string with a default value of 'zeppelin'; 'zeppelin_java_opts' is a string with a default value of '-Dspark.mesos.coarse=true -Dspark.mesos.executor.home=/opt/spark/dist' and sets arbitrary Spark configuration properties. 'spark' is an object that includes properties related to the Spark configuration, with: 'cores_max' is a string with an empty default value and sets spark.cores.max; 'executor_docker_image' is a string describing the Docker image for Spark executors; 'executor_memory' is a string with a default value of '4g' that sets spark.executor.memory; 'uri' is a string that specifies a URI serving the Spark distribution for the local Zeppelin Spark context. No additional properties are allowed at the root level.
