The schema describes a change monitoring configuration. It is an object type. It defines several reusable components under 'definitions': 'valuePath' is a string, JSON Path of where to find value to check in payload, with a minimum length of 1. '_changeTypeOthers' is an enum, allowing the values: 'percent', 'percentInc', 'percentDec', 'value', 'valueInc', 'valueDec'. '_changeTypeAny' is an enum, allowing the value: 'any'. 'changeThreshold' is a number, representing the amount of change required. 'prevValuePath' is a string, JSON path of where to put the previous triggering value and time, with a minimum length of 1. The main properties of the schema are: The schema supports two configurations, one defined by the first set of properties, which includes 'valuePath', 'changeType', 'changeThreshold', and 'prevValuePath'. In this configuration, 'valuePath', 'changeType', and 'changeThreshold' are required. The second configuration includes 'valuePath', 'changeType', and 'prevValuePath', with 'valuePath' and 'changeType' being required. No additional properties are allowed at the root level.
