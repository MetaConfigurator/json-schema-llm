The schema describes a configuration for a DC/OS service. It is an object type. The main properties of the schema are: 'service' is an object containing the configuration properties of the service. It is required and has the following properties: 'cpus' is a number representing the CPU shares to allocate to each service instance, with a default value of 0.1 and a minimum value of 0.1. 'mem' is a number representing the memory to allocate to each service instance, with a default value of 256.0 and a minimum value of 128.0. 'name' is a string representing the name of this service instance, with a default value of 'time-server'. No additional properties are allowed at the root level.
