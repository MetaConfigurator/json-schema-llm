The schema describes a structure that includes billing and shipping addresses. It is an object type. It defines several reusable components under 'definitions': 'address' is an object, which contains a required field 'street_address' of type string. The main properties of the schema are: 'billing_address' is an object that references the 'address' definition. 'shipping_address' is an object that combines the 'address' definition with an additional property 'type', which is an optional string limited to the enum values 'residential' or 'business'. No additional properties are allowed at the root level.
