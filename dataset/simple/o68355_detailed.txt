The schema describes a place search request. It is an object type. The main properties of the schema are: 'q' is a string optional search query. 'id' is an array optional identifiers. 'country' is an array optional countries. 'limit' is an integer optional limit on results, with a minimum value of 1 and a maximum value of 200. 'location' is a string optional location, following the pattern '@([\-\+]?\d+(\.\d+)?),([\-\+]?\d+(\.\d+)?)'. 'type' is an array optional types of places, where items must be one of the following enum values: 'neighbourhood', 'locality', 'localadmin', 'county', 'region', 'country', 'continent', 'country', 'planet', 'local', 'metro', 'major'. No additional properties are allowed at the root level.
