The schema describes a PayPal credit card token funding instrument. It is an object type. The main properties of the schema are: 'credit_card_id' is a string ID of credit card previously stored using /vault/credit-card. It is optional. 'payer_id' is a string A unique identifier that you can assign and track when storing a credit card or using a stored credit card. This ID can help to avoid unintentional use or misuse of credit cards. It is optional. 'last4' is a string Last four digits of the stored credit card number. It is optional. 'type' is a string Credit card type. Valid types are: visa, mastercard, discover, amex. It is required. 'expire_year' is a number 4-digit expiration year. It is required and must match the pattern ^([\d]{4})$. 'expire_month' is a number Expiration month with no leading zero. Acceptable values are 1 through 12. It is required and must be between 0 and 12. No additional properties are allowed at the root level.
