The schema describes a response for client errors. It is an object type. The main properties of the schema are: 'error' is a string representing a single ASCII error code string, required, with allowed values: 'invalid_redirect_uri', 'invalid_client_metadata', 'invalid_software_statement', 'unapproved_software_statement'. 'error_description' is a string providing a human-readable ASCII text description of the error used for debugging, optional. No additional properties are allowed at the root level.
