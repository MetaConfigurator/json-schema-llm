The schema describes engine version output. It is an array type. It defines several reusable components under 'definitions': 'result' is an object, represents install/version request result. It has required fields 'name' and 'installed'. The properties are: 'installed' is a boolean, installation status; 'message' is a string, error message if can't install engine; 'name' is a string, engine name; 'version' is a string, engine version; 'packages' is an array, list of dependencies, which contains items that reference 'result'. The main properties of the schema are: The root schema represents an array of objects, each of which resolves to the referenced definition. No additional properties are allowed at the root level.
