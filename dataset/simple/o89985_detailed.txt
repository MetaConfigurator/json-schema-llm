The schema describes a dependency pointing to a Package via a package name and a version constraint. It is an object type. The main properties of the schema are: 'name' is a string, Name of the package for this dependency. It is required. 'version' is a string or null, Version of this dependent package: The effective or concrete resolved and used version. It is optional. 'version_constraint' is a string or null, The version constraints (aka. possible versions) for this dependent package: The meaning of this constraints is package type-specific. It is optional. No additional properties are allowed at the root level.
