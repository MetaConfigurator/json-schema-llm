The schema describes a JSON RPC 2.0 response. It is an object type. It defines several reusable components under 'definitions': 'common', 'success', and 'error'. 'common' is an object, it requires the fields 'id' (which can be a string, integer, or null) and 'jsonrpc' (with the enum value '2.0'). It cannot have 'result' and 'error' at the same time. 'success' is an object, it requires the fields from 'common' and additionally requires 'result', which can be anything. 'error' is an object, it requires the fields from 'common' and additionally requires 'error', which is an object requiring 'code' (an integer) and 'message' (a string), with an optional field 'data' that can be anything.
