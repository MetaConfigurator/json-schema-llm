The schema describes subscription for a user. It is an object type. The main properties of the schema are: 'email' is a string representing the user's email, which is required and must match the email format. 'list_code' is a string which is optional. 'list_id' is an integer which is optional. 'user_id' is an integer representing the user's identifier, which is required. 'variant_id' is an integer which is optional. The schema requires that either 'list_id' or 'list_code' be present. No additional properties are allowed at the root level.
