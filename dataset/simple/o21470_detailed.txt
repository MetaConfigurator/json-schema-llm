The schema describes the utilization of consulate services. It is an object type. The main properties of the schema are: '_id' is a string that represents a unique url-friendly base64 encoded, utf8 encoded concatenation identifier, required. '_timestamp' is a string formatted as ISO8601 combined date and time in UTC, required. 'available_hours' is a number indicating the number of hours available, decimal, with a minimum value of 0, required. 'consulate' is a string representing the name or identity of the consulate providing the appointment, required. 'period' is a string which must be one of the following values: 'hour', 'day', 'week', 'month', 'quarter', required. 'service' is a string denoting the name of the service provided, required. 'used_hours' is a number indicating the number of hours used, decimal, with a minimum value of 0, required. No additional properties are allowed at the root level.
