The schema describes artifacts to be fetched by URL. It is an array type. The main properties of the schema are: Each item in the array is an object, which represents a specific artifact to be fetched by URL. 'url' is a string, required, describing the URL to be used to fetch artifact. 'md5' is a string, optional, describing the md5 checksum of the artifact, with a pattern validation of '^[a-f0-9]{32}$'. 'sha1' is a string, optional, describing the sha1 checksum of the artifact, with a pattern validation of '^[a-f0-9]{40}$'. 'sha256' is a string, optional, describing the sha256 checksum of the artifact, with a pattern validation of '^[a-f0-9]{64}$'. 'target' is a string, optional, describing the name to be used when saving artifact to disk. The object must include at least one of the following: 'md5', 'sha1', or 'sha256'. No additional properties are allowed at the root level.
