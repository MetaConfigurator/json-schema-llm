The schema describes a notification. It is an object type. The main properties of the schema are: 'id' is a string required field describing the notification's unique identifier. 'user_id' is a string required field representing the user's identifier. 'subject' is a string required field indicating the subject of the notification. 'content' is a string required field comprising the main text of the notification. 'status_id' is a string optional field, which can be one of the following values: 'unread', 'read', 'archived', and defaults to 'unread'. 'created' is a number optional field indicating the timestamp of creation. 'updated' is a number optional field representing the timestamp of the last update. No additional properties are allowed at the root level.
