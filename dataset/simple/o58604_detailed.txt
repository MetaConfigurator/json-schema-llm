The schema describes a population resource. It is an object type. The main properties of the schema are: 'description' is a string that provides a short description of the population. It is optional. 'id' is an integer representing the population's ID, such as 2673. It is optional. 'name' is a string that serves as a name allowing to identify the population. It is required. 'owner' is a string that references a single related resource, which can be either a URI or a set of nested resource data. It is required. 'taxa' is a string representing the identifier (or URI) of the taxa object to which the population belongs. It is required. No additional properties are allowed at the root level.
