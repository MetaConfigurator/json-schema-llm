The schema describes a user authentication request. It is an object type. The main properties of the schema are: 'email' is a string that must be a valid email format with a maximum length of 1024 characters and is required. 'password' is a string with a minimum length of 8 characters and a maximum length of 2048 characters, and is required. 'twoFactorCode' is a string with a maximum length of 2048 characters and is optional. 'requestedScopes' is an array of unique strings defined by a complex set of oneOf conditions. Each string within can be one of several predefined values, including permissions related to applications, devices, organizations, and users. 'tokenTTL' is an integer that must be greater than or equal to 0 and is optional. No additional properties are allowed at the root level.
