The schema describes a reward transaction. It is an object type. The main properties of the schema are: 'tx' is an object containing transaction details, which is required. It has properties: 'id' is a string that must match the pattern '^[0-9a-f]{64}$' and is required; 'to' is an object that is required and must contain 'address' (a string, required) and 'amount' (an integer, required, with an exclusive minimum of 0); 'msg' is a string with a maximum length of 254 characters, and is required. 'sign' is a string and is also required. No additional properties are allowed at the root level.
