The schema describes a token response. It is an object type. The main properties of the schema are: 'access_token' is a string that represents the access token issued by the server and is required. 'expires_in' is a number that indicates the lifetime of the access token, in seconds, and is required. 'id_token' is a string that represents an optional identity token, issued for the code and password grants, which is not provided for client credentials grants. 'refresh_token' is a string that represents an optional refresh token, which can be used to obtain new access tokens and is issued only for long-lived authorisations that permit it, and is required. 'scope' is a string that indicates the scope of the access token. 'token_type' is a string that is set to 'bearer' and is required, with an enum value of 'bearer'. No additional properties are allowed at the root level.
