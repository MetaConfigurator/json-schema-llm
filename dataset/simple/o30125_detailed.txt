The schema describes an authentication policy. It is an object type. The main properties of the schema are: 'action' is a string optional, 'comment' is a string optional, 'description' is a string readonly, 'hits' is an integer readonly, 'logaction' is a string optional, 'name' is a string optional, 'newname' is a string optional, 'policysubtype' is a string readonly, restricted to the values 'LOCAL', 'RADIUS', 'LDAP', 'TACACS', 'CERT', 'NEGOTIATE', 'SAML', 'PROFILE', 'DFA', 'SAMLIDP', 'WEBAUTH', 'OAUTH', 'NO_AUTHN', 'EPA', 'SFAUTH', 'AZUREVAULT', 'EMAIL', 'CAPTCHA', 'rule' is a string optional, and 'undefaction' is a string optional. No additional properties are allowed at the root level.
