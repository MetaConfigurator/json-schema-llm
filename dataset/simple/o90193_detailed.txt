The schema describes a dynamic map layer. It is an object type. The main properties of the schema are: 'gdbVersion' is a string that specifies, if applicable, an alternate geodatabase version (optional). 'mapLayerId' is an integer representing the current map layer's id (required). 'type' is a string indicating the type, with an enum value of 'mapLayer' (required). Compositional constraints require 'mapLayerId' and 'type' to be present for a valid configuration. No additional properties are allowed at the root level.
