The schema describes a unit configuration. It is an object type. The main properties of the schema are: 'description' is a string that represents the unit name and is required. 'health' is an integer indicating the node health status, which must be between 0 and 3, and is required. 'help' is a string that provides steps a user can take for more information and is optional. 'id' is a string that must match the pattern '^[\w]+$' and is required. 'output' is a string describing the log output and is optional. No additional properties are allowed at the root level.
