The schema describes a validation structure for regex fields. It is an object type. The main properties of the schema are: 'field' is a string that is required, describing the target field for the regex; 'regex' is a string that is required with a format of regex, specifying the regular expression pattern; 'caseInsensitive' is a boolean that is optional, indicating if the regex should be case insensitive; 'multiline' is a boolean that is optional, indicating if the regex should support multiline matches; 'extended' is a boolean that is optional, indicating if the regex supports extended syntax; 'dotall' is a boolean that is optional, indicating if the dot matches newline characters. No additional properties are allowed at the root level.
