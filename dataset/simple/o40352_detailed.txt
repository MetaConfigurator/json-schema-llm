The schema describes the structure of an error message. It is an object type. The main properties of the schema are: 'type' is a string that is required and must be one of the enum values: 'error'. 'error' is a string that is required and must be one of the enum values: 'malformed', 'unauthorized', 'serverInternal', 'nonSupported', 'unknown', 'badCSR'. 'message' is a string that is optional. 'moreInfo' is a string that is optional. No additional properties are allowed at the root level.
