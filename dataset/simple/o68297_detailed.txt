The schema describes parameters accepted by the Beachfront adapter. It is an object type. The main properties of the schema are: 'appId' is a string that represents the id of an inventory target and can only be used in requests that contain one media type, applying to all imps in the request; 'appIds' is an object that specifies appIds for specific media types, which can be used for either single or multiple media type requests, containing properties 'video' (a string for video requests) and 'banner' (a string for banner requests); 'bidfloor' is a number representing the price floor for the bid, and it is required; 'videoResponseType' is a string indicating the format of the video response, which defaults to a nurl URL, and can be set to 'adm', 'both', or remain unchanged. The schema enforces that either 'appId' or 'appIds' must be present alongside 'bidfloor' and supports compositional constraints such that at least one of 'appId' or 'appIds' is required. No additional properties are allowed at the root level.
