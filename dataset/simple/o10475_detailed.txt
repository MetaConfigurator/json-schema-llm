The schema describes a 400 error returned on user creation, if bad data was provided, or the user exists in the system. It is an object type. The main properties of the schema are: 'status' is an integer and is required. 'message' is a string and is required; it must be one of the following values: 'user already exists' or 'invalid request. Validate data using provided schema'. 'link' is a string and is required. No additional properties are allowed at the root level.
