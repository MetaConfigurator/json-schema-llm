The schema describes validation for file metadata fields that the OSF allows to be edited by the user. It is an object type. The main properties of the schema are: 'file_description' is a string that represents the description of the file. It is optional. 'funders' is an array that contains objects representing funders responsible for funding the resource. Each object must have a required 'funding_agency' string and may have an optional 'grant_number' string. 'related_publication_doi' is a string that must match the pattern '10\..+/.+'. It is optional. 'resource_type' is a string with an enum allowing the following values: 'Audio/Video', 'Dataset', 'Image', 'Model', 'Software', 'Book', 'Funding Submission', 'Journal Article', 'Lesson', 'Poster', 'Preprint', 'Presentation', 'Research Tool', 'Thesis', 'Other'. It is optional. No additional properties are allowed at the root level.
