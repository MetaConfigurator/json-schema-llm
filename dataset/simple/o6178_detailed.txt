The schema describes the structure of an object that contains device information. It is an object type. The main properties of the schema are: 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is required. 'deviceId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is required. 'deviceClass' is a string with an enum of allowed values: 'standalone', 'master', 'owned', 'floating', 'virtual' and is required. 'token' is a string and is required. No additional properties are allowed at the root level.
