The schema describes a specific product within a packing list. It is an object type. The main properties of the schema are: 'doc_type' is a string that must match the pattern '^pickedProduct$' and is required. 'version' is a string formatted as semver and is required. 'expectedQty' is an integer and is required. 'actualQty' is an integer and is required. 'productType' is a string and is required. No additional properties are allowed at the root level.
