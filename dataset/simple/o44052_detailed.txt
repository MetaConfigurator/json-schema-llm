The schema describes lax ingest+publish response schema. It is an object type. The main properties of the schema are: 'datetime' is a string the date and time in ISO 8601 UTC when this message was sent. if 'status' is either 'ingested' or 'published', then this is the date and time recorded in lax. This property is required. 'id' is a string the article identifier, potentially without leading zeroes. if original request could not be parsed, a null value will be returned. This property is required. 'message' is a string elaborates on the status. if invalid, what is invalid. if in error, what failed. This property is optional. 'requested-action' is a string an enum with allowed values: 'ingest', 'publish', 'ingest+publish'. This property is optional. 'status' is a string an enum with allowed values: 'ingested', 'published', 'invalid', 'error'. This property is required. 'token' is a string data from original request to be returned to the sender once action complete. if original request could not be parsed, a null value will be returned. This property is required. Compositional constraints require that 'status', 'id', 'token', and 'datetime' are present in each valid instance. No additional properties are allowed at the root level.
