The schema describes changes to state made by client. It is an object type. It defines several reusable components under 'definitions': 'Change' is an object, containing required properties 'new' and 'old', and disallows additional properties. 'Item' is an object with additional properties that reference the 'Change' definition. The main properties of the schema are: 'context' is an object with additional properties that reference the 'Item' definition. 'plugins' is also an object with additional properties that reference the 'Item' definition. No additional properties are allowed at the root level.
