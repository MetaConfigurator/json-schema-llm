The schema describes adding a new bird to the library. It is an object type. The main properties of the schema are: 'id' is a string that represents the object id from the database and is required. 'name' is a string that represents the English bird name and is required. 'family' is a string that represents the Latin bird family name and is required. 'continents' is an array that represents the continents the bird exists on, requires at least one item, and each item must be a string. It also enforces unique items within the array and is required. 'added' is a string that represents the date the bird was added to the registry, formatted as YYYY-MM-DD, and is required. 'visible' is a boolean that determines if the bird should be visible in lists and is required. No additional properties are allowed at the root level.
