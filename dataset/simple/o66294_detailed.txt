The schema describes the structure of a document that is submitted to the endpoint defined in a hook. It is an object type. The main properties of the schema are: 'date' is a string when was the request made, format=YYYYMMDD HHmmssSSSS in UTC, required. 'clientId' is a string identifier (principal) for the client that made the request, required. 'request' is an object the request, optional. 'response' is an object the response, optional. 'before' is an object data before the request is processed, optional. 'after' is an object data after the request is processed, optional. The schema requires a minimum of 3 properties. No additional properties are allowed at the root level.
