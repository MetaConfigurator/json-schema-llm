The schema describes an authorization error response. It is an object type. The main properties of the schema are: 'error' is a string that represents the OAuth 2.0 error code, with values: 'invalid_request', 'unauthorized_client', 'access_denied', 'unsupported_response_type', 'invalid_scope', 'server_error', 'temporarily_unavailable'. This field is required. 'error_description' is a string that provides additional human-readable information about the error. This field is also required. 'error_uri' is a string that offers a human-readable web page with information about the error, typically intended for the client developer. This field is optional. 'state' is a string that represents the state value, if one was passed with the request. Clients must validate the value before proceeding. This field is optional. No additional properties are allowed at the root level.
