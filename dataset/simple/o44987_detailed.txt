The schema describes a plugin configuration. It is an object type. The main properties of the schema are: '@id' is a string unique identifier for the plugin, usually comprised of the name of the plugin and the version. 'name' is a string the name of the plugin, which will be used in the algorithm detection phase. 'description' is a string a summary of what the plugin does, and pointers to further information. 'version' is a string the version of the plugin. 'plugin_type' is a string sub-type of plugin, e.g. sentimentPlugin. 'extra_params' is an object optional object for additional parameters. No additional properties are allowed at the root level.
