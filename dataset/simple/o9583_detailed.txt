The schema describes a request message structure. It is an object type. The main properties of the schema are: 'kind' is a string that defines that the message is a request, it is required and has a default value of 'req'. 'id' is a string that is a request ID generated by the client. It should be unique per WebSocket session in order to identify the request. This ID will be reflected by the server in the response to the request. It is required, has a minimum length of 1, a maximum length of 64, and a default value of '0'. 'type' is a string that contains one of the APIs request type names, and it is required. 'body' is an object and is optional. No additional properties are allowed at the root level.
