The schema describes a cloud-init plugin manifest. It is an object type. The main properties of the schema are: 'system' is an object containing properties related to the system; it is optional and does not have any required fields. Within 'system', 'release' is a string that is optional and can only take one of the following values: 'wheezy', 'stable', 'jessie', 'testing', 'sid', 'unstable'. 'plugins' is an object containing properties related to plugins; it is required. Within 'plugins', 'cloud_init' is an object that is required and contains properties related to cloud-init configuration; within 'cloud_init', 'username' is a string that is required, 'disable_modules' is an array of strings that is optional and must consist of unique items, and 'metadata_sources' is a string that is optional. No additional properties are allowed at the root level.
