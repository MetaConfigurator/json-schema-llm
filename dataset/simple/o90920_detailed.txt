The schema describes the format of workload that's sending from Gearman Client to Gearman Worker. It is an object type. The main properties of the schema are: 'key' is a string that serves as the request key for sign notifications and data in storage. It is optional. 'commands' is an array of commands that must contain at least one item. Each item in the array is an object with 'name', which is a string representing the name of the command and is required, and 'params', which is an object representing an array of command params with key => value pairs and is optional. No additional properties are allowed at the root level.
