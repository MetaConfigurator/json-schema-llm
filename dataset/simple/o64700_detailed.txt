The schema describes a project description file. It is an object type. The main properties of the schema are: 'require' is an object This is a hash of module name (keys) and version constraints (values) that are required to run this module. It is required, with additional properties allowed as strings. 'path' is an object path config. It is required and defines two properties: 'modules' is a string representing the modules path, and 'applications' is a string representing the applications path. No additional properties are allowed at the root level.
