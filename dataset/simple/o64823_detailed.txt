The schema describes information about revenue. It is an object type. The main properties of the schema are: 'scope' is a string that specifies the scope of the revenue result and is required. Allowed values are 'company', 'city', 'station', 'pump'. 'id' is a string that serves as a unique identifier for the scope of the revenue result and is required. 'fuel' is an array that contains fuel types included in the revenue result and is required. Allowed values for the items are 'any', 'regular', 'premium', 'diesel'. 'from' is a string that indicates the timestamp at the beginning of the interval for which revenue was collected and is optional. 'to' is a string that indicates the timestamp at the end of the interval for which revenue was collected and is optional. 'amount' is a number that represents the revenue amount being reported and is required. No additional properties are allowed at the root level.
