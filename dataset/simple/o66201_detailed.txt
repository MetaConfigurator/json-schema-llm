The schema describes a configuration object for panda-hook to delete a repository from the hook-server. It is an object type. The main properties of the schema are: 'cluster_address' is a string that represents the hostname of the cluster. It is required and must specify user and port if not 22 in the format <user>@<host_address>:<port>. 'repo_name' is a string that denotes the name of the repository to be manipulated on the hook server. It is required. 'hook_address' is a string that indicates the hostname of the hook-server. It is required and must specify user and port if not 22 in the format <user>@<host_address>:<port>. 'hook_source' is a string that specifies the path of the githook template. It is optional. 'hook_name' is a string that represents the name of the githook to be manipulated. It is optional and its allowed values are: 'applypatch-msg', 'pre-applypatch', 'post-applypatch', 'pre-commit', 'prepare-commit-msg', 'commit-msg', 'post-commit', 'pre-rebase', 'post-checkout', 'post-merge', 'pre-push', 'pre-receive', 'update', 'post-receive', 'post-update', 'pre-auto-gc', 'post-rewrite'. 'launch_path' is a string that indicates the relative path to the target directory to scan for service names. It is optional. No additional properties are allowed at the root level.
