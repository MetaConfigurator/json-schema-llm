The schema describes a generic user credentials schema. It is an object type. It defines several reusable components under 'definitions': 'login' is a string, with a minimum length of 1. 'nested' is an object, which must fulfill all conditions specified in 'login'. The main properties of the schema are: 'credentials' is an object, which is required. It contains properties: 'username', which is a string that references the 'login' definition, 'password', which is a string that also references the 'login' definition, and 'api', which is an object with a property 'key' that can either reference the 'nested' definition or be null. No additional properties are allowed at the root level.
