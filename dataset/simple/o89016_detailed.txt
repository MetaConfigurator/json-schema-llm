The schema describes a common object structure. It is an object type. It defines several reusable components under 'definitions': 'id' is a string, unique identifier with a format of uuid. 'created_at' is a string, object creation timestamp with a format of date-time. 'updated_at' is a string, object last updated timestamp with a format of date-time. 'config' is an object, allowing properties with string values and allowing additional properties. The main properties of the schema are: 'id' is a string required and referenced from the 'id' definition. 'created_at' is a string required and referenced from the 'created_at' definition. 'updated_at' is a string required and referenced from the 'updated_at' definition. 'config' is an object required and referenced from the 'config' definition. No additional properties are allowed at the root level.
