The schema describes a requirement for persistent, read-only or read-write storage that should be available to the containerapp on runtime. It is an object type. The main properties of the schema are: 'persistentVolume' is an object that describes the storage requirement. It is required and must include the fields 'name', 'accessMode', and 'size'. The 'name' is a string with a default value of 'null' that is associated with the storage requirement. The 'accessMode' is a string that must be one of the following values: 'ReadWrite', 'ReadOnly'. The 'size' is a number that must be greater than or equal to 0. No additional properties are allowed at the root level.
