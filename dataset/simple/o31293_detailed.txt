The schema describes define segments to insert into a jinja template. It is an object type. The main properties of the schema are: '$schema' is a string an optional field that specifies the schema reference. 'identifier' is a string a required field that denotes the identification of the segments. 'description' is a string a required field that indicates the purpose of this group of segments. 'segments' is an object a required field that contains the segments to insert, with the constraint that each segment must be defined under a unique key. Each segment can be either a string or an array of strings, where each string represents a line in the segment. 'overwrite' is an array an optional field that allows a list of segment keys that may overwrite previously defined entries. No additional properties are allowed at the root level.
