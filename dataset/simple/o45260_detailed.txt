The schema describes the configuration for API access. It is an object type. The main properties of the schema are: 'apikey' is a string representing an autogenerated API key for basic auth and is required. 'secret_hash' is a string that contains the hashed version of the API secret and is required. 'domain' is a string specifying the allowed domain for CORS calls and is required. 'acl' is an object that serves as an access control list and is required. It has 'allow' and 'disallow' as required fields, both of which are arrays with patterns for HTTP methods. 'sessions' is an optional object that contains active sessions belonging to the client, with a property that matches any key and has an array of exactly two items. No additional properties are allowed at the root level.
