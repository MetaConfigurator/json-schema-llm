The schema describes support for conda's environment.yml files. It is an object type. It defines several reusable components under 'definitions': 'channel' is a string, representing a conda channel URL or name. 'package' is a string, representing a package name. 'path' is a string, representing a file path. The main properties of the schema are: 'name' is a string, optional. 'channels' is an array of strings, representing conda channels, optional. 'dependencies' is an array, containing either strings (package names) or objects with a required 'pip' array of package names, optional. 'prefix' is a path, resolving to the referenced 'path' definition, optional. No additional properties are allowed at the root level.
