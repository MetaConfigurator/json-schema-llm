The schema describes an Action. It is an object type. The main properties of the schema are: 'target' is a string that represents the **generatedIdentifier** given by iFLUX Server. See the API `GET actionTargets` or GET `actionTargets/:id`. It is required. 'type' is a string that defines a valid URL defining the action type. It is required. 'properties' is an object that contains the transformation done by the rule evaluation. If no transformation, the event is sent as the properties. In this case, refers to `POST /events`. It contains a nested object where the properties can be any type of object. It is required. No additional properties are allowed at the root level.
