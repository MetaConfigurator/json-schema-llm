The schema describes a booking system. It is an object type. The main properties of the schema are: 'booking_reason' is a string, optional, which describes the reason for booking. 'slots' is an array, required, representing the start/end date of the bookings. It must contain at least one item and has no additional properties. Each item in the 'slots' array is an object with the following properties: 'start_date' is an integer, required, representing the Unix timestamp of the start date of the booking; 'end_date' is an integer, required, representing the Unix timestamp of the end date of the booking; 'iana' is a string, required, describing the IANA identifier of the booking owner. No additional properties are allowed at the root level.
