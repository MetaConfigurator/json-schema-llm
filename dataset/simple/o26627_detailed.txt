The schema describes a policy document containing permission statements. It is an object type. The main properties of the schema are: 'statements' is an array containing objects, which are required. Each object in the array must have: 'effect', which is a string matching the pattern '^allow|deny$'; 'actions', which is an array of strings, each matching the pattern '^[a-z]+:\w+$'; and 'resources', which is an array of strings, each matching the pattern '^brn:(\w+):(\w+):(\w*):(\*|[\w\-]*\*?)$'. No additional properties are allowed at the root level.
