The schema describes a configuration object for a device. It is an object type. The main properties of the schema are: 'deviceId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is required. 'key' is a string that is optional. 'secret' is a string that is optional. 'tokenTTL' is an integer that must be greater than or equal to 0 and is optional. 'requestedScopes' is an array of unique strings that must contain values from the enum: 'all.Device', 'all.Device.read', 'data.export', 'data.timeSeriesQuery', 'data.lastValueQuery', 'device.commandStream', 'device.get', 'device.getCompositeState', 'device.getState', 'device.stateStream', 'device.getLogEntries', 'device.getCommand', 'device.debug', 'device.sendState', 'device.sendCommand', 'device.setConnectionStatus', 'devices.get', 'devices.sendCommand' and is optional. No additional properties are allowed at the root level.
