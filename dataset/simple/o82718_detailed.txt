The schema describes a custom code configuration. It is an object type. The main properties of the schema are: 'custom_code' is an object that represents a map containing (HTML, JavaScript, CSS, etc.) injected into an application at a named insertion point, such as 'html_head' or 'footer'. This property is optional. It uses pattern properties, where keys must match the regex '^[a-zA-Z_]{1}[\w\.:-]+' and values are strings with a minimum length of 0 and a maximum length of 65535. The 'custom_code' object does not allow additional properties. No additional properties are allowed at the root level.
