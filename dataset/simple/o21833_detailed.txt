The schema describes RFC 3339 timestamps. It is an object type. It defines several reusable components under 'definitions': 'date_fullyear' is a string, representing a full year (YYYY) with a minimum and maximum length of 4 characters, validated by the pattern /^[0-9]{4}$/ and an example of '2001'. 'full_date' is a string, representing a full date (YYYY-MM-DD) with a minimum and maximum length of 10 characters, validated by the pattern /^[0-9]{4}-[0-9]{2}-[0-9]{2}$/ and an example of '2001-12-21'. The main properties of the schema are: 'definations.date_fullyear' is a string referencing the 'date_fullyear' definition, required. 'definations.full_date' is a string referencing the 'full_date' definition, required. No additional properties are allowed at the root level.
