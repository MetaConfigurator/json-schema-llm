The schema describes a token password grant request. It is an object type. The main properties of the schema are: 'code' is a string representing the code received with the authentication response; it is required. 'grant_type' is a string that must be set to 'password'; it is required and has an enum value of 'password'. 'password' is a string representing the resource owner password; it is optional. 'scope' is a string representing optional requested scope values for the access token; it is optional. 'username' is a string representing the resource owner username; it is optional. No additional properties are allowed at the root level.
