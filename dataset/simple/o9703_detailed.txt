The schema describes a request message format. It is an object type. The main properties of the schema are: 'kind' is a string that defines that the message is a request, with a default value of 'req' and marked as readonly. 'id' is a string that represents the request ID generated by the client, which should be unique per WebSocket session to identify the request. It must have a minimum length of 1 and a maximum length of 64, with a default value of '0'. 'type' is a string that contains one of the API's request type names. The properties 'id' and 'type' are required. No additional properties are allowed at the root level.
