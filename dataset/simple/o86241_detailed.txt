The schema describes a representation of a normalized element. It is an object type. The main properties of the schema are: 'type' is a string that is required and can only have the value 'reference'. 'referent' is an object that is required and contains several properties. 'type' is a string describing the ANS type that the provider should return. 'service' is a string describing the type of interaction the provider expects. 'id' is a string that serves as the id passed to the provider to retrieve an ANS document. 'provider' is a string that is a URL capable of resolving the id into an ANS element. The properties 'provider' and 'id' under 'referent' are required. No additional properties are allowed at the root level.
