The schema describes an object structure. It is an object type. The main properties of the schema are: 'id' is a string representing a unique identifier. 'type' is a string specifying the type of the object. 'timestamp' is a string in date-time format indicating when the object was created. 'data' is an object for storing associated data. 'source' is an object providing source information, which includes 'kinesisPartitionKey' as a string representing the partition key in Kinesis and 'producerId' as a string identifying the producer. No additional properties are allowed at the root level.
