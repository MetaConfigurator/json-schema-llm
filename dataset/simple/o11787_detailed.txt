The schema describes a configuration for a client token system. It is an object type. The main properties of the schema are: 'client_token' is a string and required. 'policies' is an array of strings and required, with no specific constraints on its items. 'metadata' is an object and required, containing: 'aws_region' as a string, 'username' as a string, 'is_admin' as a string, 'groups' as a string, 'aws_account_id' as a string, and 'aws_iam_role_name' as a string, all of which are optional. 'lease_duration' is an integer and required. 'renewable' is a boolean and required. No additional properties are allowed at the root level.
