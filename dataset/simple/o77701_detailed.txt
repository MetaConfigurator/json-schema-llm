The schema describes a client configuration. It is an object type. The main properties of the schema are: 'name' is a string representing the client name, required, with a minimum length of 3 and a maximum length of 64. 'queues' is an object containing the queue configurations, required, with properties: 'input' is a string representing the SQS input queue, required, matching the pattern '^https://.+$'; 'output' is a string representing the SQS output queue, required, also matching the pattern '^https://.+$'. No additional properties are allowed at the root level.
