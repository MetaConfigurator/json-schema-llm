The schema describes numina oblock files. It is an array type. It defines several reusable components under 'definitions': 'oblock_ext' is an object, which has required fields: 'id', 'instrument', and 'mode'. It contains properties: 'id' is a string, 'instrument' is a string, 'mode' is a string, 'frames' is an array of strings, 'enabled' is a boolean, and 'requirements' is an object. The main properties of the schema are: each item in the array is required to adhere to the 'oblock_ext' definition. No additional properties are allowed at the root level.
