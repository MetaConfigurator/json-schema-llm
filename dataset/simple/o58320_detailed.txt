The schema describes JSON schema for Web Application manifest files (schema extensions). It is an object type. It defines several reusable components under 'definitions': 'mjs_access_whitelist_rule' is an object, describing the rules for URL access. It contains two properties: 'url', which is a string that represents the URL pattern and must conform to the 'uri' format; and 'apiAccess', which is a string that defines the type of access to the runtime API, with allowed values being 'none', 'all', or 'AllowForWebOnly'. The main properties of the schema are: 'mjs_access_whitelist' is an array of rules indicating the type of access URLs matching the rules should have. Each item in the array is required to be of type 'mjs_access_whitelist_rule'. No additional properties are allowed at the root level.
