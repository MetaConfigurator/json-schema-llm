The schema describes a request for a service. It is an object type. It defines several reusable components under 'definitions': 'extension' is an object, representing an extension with required fields 'definition' (a string) and 'value' (which can be either a boolean or a string). The main properties of the schema are: 'request' is a string and is required; 'version' is a string and is optional, with allowed values '1.0.0' and '2.0.0'; 'service' is a string and is required, with the allowed value 'SOS'; 'extensions' is an array and is optional, with a minimum of 1 item, where each item must conform to the 'extension' definition. No additional properties are allowed at the root level.
