The schema describes configuration options for a Zeppelin application. It is an object type. The main properties of the schema are: 'zeppelin' is an object that configures the Zeppelin application, required 'spark_version' specifies the version of the Spark distribution to run with Zeppelin and has a default value of 'spark-1.6.0'; 'role' is an optional string representing accepted resource roles, with a default value of '*'; 'vhost' is an optional string for the domain name that marathon-lb will use to route requests, with a default value of ''. No additional properties are allowed at the root level.
