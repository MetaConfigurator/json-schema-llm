The schema describes a collection of Toronto Mesh Nodes. It is an array type. The main properties of the schema are: 'items' is an object which represents the details of each node. It is required and must include the properties: 'name', 'latitude', 'longitude', 'status', and 'dateAdded'. The properties are as follows: 'name' is a string, required; 'latitude' is a number, required; 'longitude' is a number, required; 'cardinalDirection' is a string with allowed values 'North', 'North East', 'East', 'South East', 'South', 'South West', 'West', 'North West', 'Omni', 'None', optional; 'cardinalDirectionAntenna' is a string with allowed values 'North', 'North East', 'East', 'South East', 'South', 'South West', 'West', 'North West', 'Omni', optional; 'IPV6Address' is a string formatted as an IPv6 address, optional; 'status' is a string with allowed values 'proposed', 'active', required; 'dateAdded' is a string formatted as a date-time, required. No additional properties are allowed at the root level.
