The schema describes a generic job output. It is an object type. The main properties of the schema are: 'status' is a string, representing the result status of the job. It is required and must be one of the following values: 'Success', 'Failure', 'Timeout'. 'message' is a string, intended to capture a summarized error message in case of Failures. It is optional. 'results' is an object, which holds the job specific output details. It is also optional. No additional properties are allowed at the root level.
