The schema describes a jurisdiction. It is an object type. It defines several reusable components under 'definitions': 'address' is an object, representing a postal address that includes required fields: 'street_address', 'city', and 'state'. The main properties of the schema are: 'billing_address' is an object that references the 'address' definition, and 'shipping_address' is an object that combines an 'address' definition with an additional property 'type', which is required and must be either 'residential' or 'business'. The 'shipping_address' must satisfy all of the constraints defined in the merged schemas. No additional properties are allowed at the root level.
