The schema describes a representation of one particular element of an accounts statement (e.g. profit or assets). It is an object type. The main properties of the schema are: 'currency' is a string that represents the ISO 4217 code for the currency, with a minimum length of 3 characters and a maximum length of 3 characters. It is optional. 'date' is a string that denotes a specific point in time, formatted as a date. It is required in one of the valid configurations. 'end_date' is a string formatted as a date. It is required in one of the valid configurations. 'start_date' is a string formatted as a date. It is optional. 'value' is a string with a minimum length of 1 character and is required. The schema allows for either 'date' or 'end_date' to be required based on the anyOf condition. No additional properties are allowed at the root level.
