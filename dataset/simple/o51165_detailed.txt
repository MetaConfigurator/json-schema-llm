The schema describes configuration files for testing. It is an object type. It defines several reusable components under 'definitions': 'includeDefinition' is an object, specifies a list of glob patterns that match files to be included in compilation with an 'include' property that is an array of strings formatted as URIs. 'excludeDefinition' is an object, specifies a list of globs that need to be excluded from compilation, applicable only to files included via the 'include' property, with an 'exclude' property that is an array of strings formatted as URIs. The main properties of the schema are: 'tests' is an object that describes the directory where the codelytics rules live. It is optional and must adhere to one of the following: either match the 'includeDefinition' or the 'excludeDefinition'. No additional properties are allowed at the root level.
