The schema describes a configuration object for a process. It is an object type. The main properties of the schema are: 'dryRun' is a boolean that indicates whether the process should be run in a dry run mode, with a default value of false. 'includeFiles' is a boolean that specifies if files should be included, with a default value of false. 'includeDataTableRows' is a boolean that determines if data table rows should be included, with a default value of false. 'includeDevices' is a boolean that signifies if devices should be included, with a default value of false. 'ownerId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' or can be null. 'ownerType' is a string that can be either 'user' or 'organization'. 'email' is a string formatted as an email address with a maximum length of 1024 characters. No additional properties are allowed at the root level.
