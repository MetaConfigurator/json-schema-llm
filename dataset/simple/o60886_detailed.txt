The schema describes common json schema for transaction volumes tagged by user channel and channel type (see DbD). It is an object type. The main properties of the schema are: '_id' is a string unique url-friendly base64 encoded, utf8 encoded concatenation identifier. It is required. '_timestamp' is a string ISO8601 combined date and time in UTC e.g. '2014-12-01T00:00:00+00:00'. It is required. 'channel' is a string user contact channel. It is required. 'channel_type' is a string class of channel for Dbd categorisation. It is required with allowed values: 'digital', 'non-digital', 'assisted-digital'. 'count' is an integer the number of completed transactions over the specific channel. It is required with a minimum value of 0. 'period' is a string The period covered by each data point from the _timestamp. It is required with allowed values: 'hour', 'day', 'week', 'month', 'quarter'. No additional properties are allowed at the root level.
