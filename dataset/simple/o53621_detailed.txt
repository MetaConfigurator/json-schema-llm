The schema describes the data structure for services rendered. It is an object type. The main properties of the schema are: 'serviceId' is a string representing the ID of the service, required, with a minimum length of 1. 'serviceName' is a string representing the name of the service, required, with a minimum length of 1. 'serviceType' is a string representing the type of the service, required, with allowed values 'Full', 'EFAP', 'Supplemental', 'Non-food', 'Baby', 'Other'. 'servicePounds' is a number representing the number of pounds of service rendered, required. No additional properties are allowed at the root level.
