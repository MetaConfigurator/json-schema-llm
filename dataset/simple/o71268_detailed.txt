The schema describes a simple REST response schema for a payload for a non-successful response. It is an object type. The main properties of the schema are: 'messages' is an array of message objects related to the failure, required and must contain at least 1 item. Each item in the 'messages' array is an object that contains fields related to a failure, which includes: 'message' as a string, required, providing a meaningful, end-user-readable message; 'field' as an optional string representing the input mapping; and 'code' which can be either a string or integer providing info about a failure. No additional properties are allowed at the root level.
