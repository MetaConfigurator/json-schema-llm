The schema describes a parsed date object. It is an object type. The main properties of the schema are: 'year' is a number year part as YYYY. 'month' is a number month part of date in numbers e.g. 11 for November, 4 for April. 'day' is a number or null day part of date in numbers. The schema requires that at least one of the following configurations is valid: either the 'year' and 'month' properties must be present, or the 'month' and 'day' properties must be present. No additional properties are allowed at the root level.
