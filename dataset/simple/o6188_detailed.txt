The schema describes a configuration for storing values using a key. It is an object type. The main properties of the schema are: 'keyName' is a string that represents the storage key and is required with a minimum length of 1. 'valuePath' is a string that describes the JSON path for getting the value to store; it is required but has no minimum length constraint. If 'valuePath' is empty, the current value at the provided 'Storage Key' is deleted. No additional properties are allowed at the root level.
