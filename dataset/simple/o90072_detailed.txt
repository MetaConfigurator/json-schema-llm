The schema describes objects defined by a [definitionEditor](definitionEditor.md) input. It is an object type. The main properties of the schema are: 'defaultValue' is a value that can either be a number or a string; it is optional and defaults based on the provided value type. 'fieldName' is a string representing the name of the field to query; it is optional. 'parameterId' is an integer used to uniquely identify the specified parameter; it is optional. 'type' is a string that must be one of the enum values: 'esriFieldTypeBlob', 'esriFieldTypeDate', 'esriFieldTypeDouble', 'esriFieldTypeGeometry', 'esriFieldTypeGlobalID', 'esriFieldTypeGUID', 'esriFieldTypeInteger', 'esriFieldTypeOID', 'esriFieldTypeRaster', 'esriFieldTypeSingle', 'esriFieldTypeSmallInteger', 'esriFieldTypeString', 'esriFieldTypeXML'; it is optional. 'utcValue' is an integer representing exact UNIX time used when 'defaultValue' is a date string; it is optional. No additional properties are allowed at the root level.
