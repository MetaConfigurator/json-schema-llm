The schema describes the format for describing the update frequency of a data source. It is an object type. The main properties of the schema are: 'split' is a string that specifies the final component of the data packet name, i.e. the dimension on which data source packets are split. This property is optional and can have values from the enum: 'segment', 'timestamp', 'version'. 'avg' is an integer that represents the average frequency (in ms) for data updates. This property is required and must be at least 0. 'max' is an integer that indicates the maximum expected frequency (in ms) for data updates. This property is required and must be at least 0. 'min' is an integer that denotes the minimum expected frequency (in ms) for data updates. This property is required and must be at least 0. No additional properties are allowed at the root level.
