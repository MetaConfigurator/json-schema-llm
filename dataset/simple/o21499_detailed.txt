The schema describes a representation of transactions by channel. It is an object type. The main properties of the schema are: 'channel' is a string that represents the name of the channel along which to segment the count. It is required. 'count' is an integer that represents the count of the number of events. It is required. 'period' is a string that describes the period represented by each data point. It has an enum with allowed values: 'hour', 'day', 'week', 'month', 'year', 'decade', and 'century'. No additional properties are allowed at the root level.
