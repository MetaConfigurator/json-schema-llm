The schema describes a validation structure for managing device configurations. It is an object type. The main properties of the schema are: 'resultPath' is a string with a minimum length of 1, and it is required. 'deviceTags' is an array of objects, with a minimum of 1 item. Each object in the array must have a 'key' (string) and a 'value' (string). 'deviceIds' is an array of strings, with a minimum of 1 item. The schema requires 'resultPath' and either 'deviceTags' or 'deviceIds'. No additional properties are allowed at the root level.
