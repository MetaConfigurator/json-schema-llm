The schema describes common JSON schema for transaction volumes tagged by user channel. It is an object type. The main properties of the schema are: '_id' is a string unique url-friendly base64 encoded, utf8 encoded concatenation identifier. It is required. '_timestamp' is a string ISO8601 combined date and time in UTC e.g. '2014-12-01T00:00:00+00:00'. It is required. 'channel' is a string user contact channel. It is required. 'count' is an integer the number of completed transactions over the specific channel. It has a minimum of 0 and is required. 'period' is a string which indicates the period covered by each data point from the _timestamp. It is required and must be one of the following values: 'hour', 'day', 'week', 'month', 'quarter'. No additional properties are allowed at the root level.
