The schema describes an OpenT2T Error. It is an object type. It defines several reusable components under 'definitions': 'opent2t.error' is an object, it contains the following properties: 'name' is a string, ReadOnly, Gets the name for the error; 'statusCode' is a number, ReadOnly, Gets the status code for the error; 'message' is a string, ReadOnly, Gets the message for the error; 'innerErrorMessage' is a string, ReadOnly, Gets the message for the inner error; 'innerErrorName' is a string, ReadOnly, Gets the name for the inner error; 'innerErrorStack' is a string, ReadOnly, Gets the server call stack for the inner error. The main properties of the schema are: 'name' is a string, required; 'statusCode' is a number, required. No additional properties are allowed at the root level.
