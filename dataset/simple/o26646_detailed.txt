The schema describes a Blockstack Token File. It is an object type. The main properties of the schema are: 'version' is a number that represents the unique identifier for a product and is required with a minimum value of 0. 'profile' is an object that represents a user profile and is required. It contains three properties: 'header', 'payload', and 'signature', all of which are required. 'keys' is an object that requires at least one property and is also required. No additional properties are allowed at the root level.
