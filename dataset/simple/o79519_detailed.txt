The schema describes configuration settings for an app. It is an object type. It defines several reusable components under 'definitions': 'domain' is a string, a complete domain name, used by this app (as installed by this tenant), with optional wildcard, e.g. https://*.myapp.com. If the wildcard * is used, it must be followed by a dot. The main properties of the schema are: 'landingPage' is a string formatted as a URI, a (probably tenant-specific) url of the landing page for this app. The landing page is where the user is led to after clicking [open] in the UI. It is optional. 'settingsPage' is a string formatted as a URI, a url of a settings page for this app. The setup page is where the user is led to after clicking [settings] in the UI. After setup is complete the app should redirect the user back to the passed in relayPage. It is optional. 'setupRequired' is a boolean, true if the app is unusable or degraded due to lack of some setup. When true, this causes the setup required indicator to appear next to the app in the UI. When true, the app should provide a value for settingsPage. It is required. 'domainsUsed' is an array, an array of domains that your app uses. The tenant's IT team can view this, rolled up with all other installed apps, in order to add to Trusted Sites in bulk. The array items must adhere to the 'domain' definition. No additional properties are allowed at the root level.
