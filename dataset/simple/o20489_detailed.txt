The schema describes a token revoke request. It is an object type. The main properties of the schema are: 'token' is a string that represents the token that the client wants to get revoked, and it is required. 'token_type_hint' is a string that provides an optional hint about the type of the submitted token; if omitted the server will use heuristics to determine the token type, with allowed values being 'access_token' and 'refresh_token', and it is required. No additional properties are allowed at the root level.
