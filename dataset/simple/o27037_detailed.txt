The schema describes a simple address schema. It is an object type. It defines several reusable components under 'definitions': 'address1' is a string, it represents the first line of the address. 'address2' is a string, it represents the second line of the address. 'city' is a string, it represents the city of the address. 'postalCode' is a string, it represents the postal code. 'state' is a string, it represents the state of the address. 'country' is a string, it represents the country of the address. The main properties of the schema are: 'address1' is a string, optional. 'address2' is a string, optional. 'city' is a string, optional. 'postalCode' is a string, optional. 'state' is a string, optional. 'country' is a string, optional. No additional properties are allowed at the root level.
