The schema describes a common structure for user application started and complete volumes used for MI data based save and return completion-rate calculation. It is an object type. The main properties of the schema are: '_id' is a string unique url-friendly base64 encoded, utf8 encoded concatenation identifier. It is required. '_timestamp' is a string ISO8601 combined date and time in UTC e.g. '2014-12-01T00:00:00+00:00'. It is required. 'count' is an integer with a minimum value of 0. It is required. 'period' is a string representing the period covered by each data point from the _timestamp. It is required, with allowed values being 'hour', 'day', 'week', 'month', 'quarter'. 'state' is a string representing the application state. It is required, with allowed values being 'started', 'completed'. No additional properties are allowed at the root level.
