The schema describes a variable in a named space. It is an object type. The main properties of the schema are: 'comment' is a string representing additional information about the variable. 'expires' is an integer indicating when the variable should expire. 'iffull' is a string that must be one of the following values: 'undef', 'lru', indicating behavior if the variable is full. 'ifnovalue' is a string that must be one of the following values: 'undef', 'init', indicating behavior if there is no value. 'ifvaluetoobig' is a string that must be one of the following values: 'undef', 'truncate', indicating behavior if the value is too big. 'init' is a string that represents the initial value. 'name' is a string that provides the variable's name. 'referencecount' is a readonly integer that counts references to the variable. 'scope' is a string that must be one of the following values: 'global', 'transaction', indicating the variable's scope. 'type' is a string representing the variable's type. No additional properties are allowed at the root level.
