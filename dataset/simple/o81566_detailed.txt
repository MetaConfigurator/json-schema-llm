The schema describes an address. It is an object type. It defines several reusable components under 'definitions': 'US_zipCode' is a string, which must match the pattern '^[0-9]{5}(-[0-9]{4})?$'. The main properties of the schema are: 'line1' is a string required field. 'city' is a string required field. 'state' is a string required field. 'zip' is a string required field that must conform to the 'US_zipCode' definition. 'country' is a string required field. No additional properties are allowed at the root level.
