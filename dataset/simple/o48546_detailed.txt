The schema describes an encryption format. It is an object type. The main properties of the schema are: 'cipher' is a string that is required and can only be 'aes-128-ctr'. 'cipherparams' is an object that is required and cannot have additional properties. It must contain 'iv', which is a string that must match the pattern '^([0-9]|[a-f]){32}$'. 'ciphertext' is a string that is required and must match the pattern '^(([0-9]|[a-f]){2}){0,32}$'. No additional properties are allowed at the root level.
