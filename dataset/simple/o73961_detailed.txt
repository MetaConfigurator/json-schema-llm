The schema describes a structured data format for various entities. It is an object type. It defines several reusable components under 'definitions': 'object' is an object, container for a top level object. 'address' is an object, address data of contract partner, department and stuff like that. 'timeentry' is an object, dates for something. 'maintenance' is an object, maintenance data for objects. 'disturbance' is an object, data of disturbance events. 'object_disturbance' is an object, data of disturbance events. The main properties of the schema are: 'object' is a referenced object addressing the definition for 'object'. 'address' is a referenced object addressing the definition for 'address'. 'timeentry' is a referenced object addressing the definition for 'timeentry'. 'maintenance' is a referenced object addressing the definition for 'maintenance'. 'disturbance' is a referenced object addressing the definition for 'disturbance'. 'object_disturbance' is a referenced object addressing the definition for 'object_disturbance'. All these properties are required at the root level. No additional properties are allowed at the root level.
