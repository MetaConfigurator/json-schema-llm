The schema describes an OAuth client configuration. It is an object type. The main properties of the schema are: 'clientId' is a string with a minimum length of 30, making it a required field. 'grantType' is a string, which is optional. 'redirectUris' is an array of strings, which is optional. 'responseType' is a string, which is optional. 'scopes' is an array of strings, which is optional. 'secret' is a string with a minimum length of 30, making it a required field. 'userId' is a string that must match the pattern '[0-9a-fA-F]+' and must have a minimum length of 12 and a maximum length of 12, making it optional. The schema has a oneOf constraint that requires one of two configurations: either 'grantType' must be 'authorization_code' and 'responseType' must be 'code', or 'grantType' must be 'client_credentials' and 'responseType' must be 'token'. No additional properties are allowed at the root level.
