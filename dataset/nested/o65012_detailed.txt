The schema describes a filter system. It is an object type. It defines several reusable components under 'definitions': 'field' is a string, representing the field name in filtering. 'value' is a type that can be a string, number, boolean, array, or object, representing the value to filter against. 'range' is an array, which can contain a list of items. 'filter_type' is a string with an enum of allowed values: 'bool', 'match', 'in', 'exist', 'compare'. 'filters_allowed' is a type that is oneOf four definitions: 'basic_filter', 'exist', 'bool', or 'compare'. 'basic_filter' is an object with properties 'filter_type', 'field', 'value', and 'range', and can be defined either with 'field', 'value', 'filter_type' or 'field', 'range', 'filter_type' as required fields. 'exist' is an object with properties 'filter_type' and 'field', both required. 'compare' is an object with properties 'filter_type', 'field1', 'field2', 'constant', and 'comparison_operator', requiring either 'field1', 'field2', 'comparison_operator', 'filter_type' or 'field1', 'constant', 'comparison_operator', 'filter_type' as required fields. 'constant' can be a string or number, and 'comparison_operator' is a string with an enum of allowed values: 'lt', 'gt', 'lte', 'gte', 'eq'. The main properties of the schema are: 'filter' is a reference to the 'bool' definition, which defines a filter with 'filter_type', and arrays 'must', 'must_not', and 'should' containing filters allowed. No additional properties are allowed at the root level.
