The schema describes an object structure for an event. It is an object type. The main properties of the schema are: 'meta' is an object containing details about the event; it is required and has four required fields: 'id', 'type', 'version', and 'time'. 'data' is an object with details related to the event; it is required and has one required field: 'name'. 'links' is an array of objects, each containing a 'type' and a 'target'; it is required. 'meta.id' is a string matching the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'. 'meta.type' is a string and must be 'EiffelCompositionDefinedEvent'. 'meta.version' is a string with a default value of '2.0.0', and must be '2.0.0'. 'meta.time' is an integer. 'meta.tags' is an optional array of strings. 'meta.source' is an optional object with five string properties: 'domainId', 'host', 'name', 'serializer' (matching the pattern '^pkg:'), and 'uri'; it does not allow additional properties. 'meta.security' is an optional object with a required 'sdm' property which is an object requiring 'authorIdentity' and 'encryptedDigest' strings; it does not allow additional properties. 'data.version' is an optional string. 'data.customData' is an optional array of objects, each with required fields 'key' and 'value' and does not allow additional properties. 'links' is an array where each item must have required 'type' and 'target' properties, and 'target' must match the same UUID pattern. No additional properties are allowed at the root level.
