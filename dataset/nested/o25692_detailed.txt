The schema describes a collection of file and link changes, along with their attributes. It is an object type. It defines several reusable components under 'definitions': 'file_changed' is an object type, requiring the field 'status', which must be 'changed'. It allows further details through oneOf conditions pointing to 'file_changed_modified', 'link_changed_modified', and 'file_changed_deleted'. 'file_changed_modified' is an object type, requiring fields 'changes', 'mode', 'user', 'group', and 'type'. The 'changes' field is an array of enumerated values including 'size', 'mode', 'md5', 'device_number', 'link_path', 'user', 'group', 'time', 'capabilities', 'replaced', and 'other_rpm_changes'; it must have a minimum of one item. 'mode' must match the pattern /^[0-7]{3,4}$/, while 'user' and 'group' must be non-empty strings. The 'type' field can be either 'file' or 'dir'. 'link_changed_modified' is an object type, requiring fields 'target', 'changes', 'mode', 'user', 'group', and 'type'. It follows similar rules as 'file_changed_modified' for 'changes', 'mode', 'user', and 'group', but 'type' must be 'link' and 'target' is a string. 'file_changed_deleted' is an object type, requiring the 'changes' field, which must be ['deleted']. 'file_error' is an object type, requiring fields 'status' and 'error_message', where 'status' must be 'error' and 'error_message' is a string. The main properties of the schema are: '_attributes' is an object that requires the field 'extracted', which is a boolean. '_elements' is an array of objects, where each item must have 'name', 'package_name', and 'package_version' as required fields. 'package_name' and 'package_version' must be non-empty strings, while 'name' is a string. Each element must conform to oneOf either 'file_changed' or 'file_error'. No additional properties are allowed at the root level.
