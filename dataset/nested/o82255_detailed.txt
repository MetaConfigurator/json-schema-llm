The schema describes input parameters for booking a package. It is an object type. The main properties of the schema are: 'package_id' is a string that identifies the package, required. 'contact_phone' is a string that serves as the contact phone for the reservation, optional. 'Room' is an array that contains arrays of passengers who will reserve in each room, required. Each passenger array contains objects: each object is a passenger with required fields 'name', 'surname', 'kind_doc', 'num_doc', 'gender', 'birthdate', 'residence', 'nationality', and 'mail'. The 'kind_doc' field is a string with an enum of allowed values: 'DNI', 'Pasaporte'. The 'gender' field is a string with an enum of allowed values: 'F', 'M'. The 'residence' and 'nationality' fields are strings with extensive enums of country codes. The 'Passport' field is an object or null, containing the required 'expired_date' string. No additional properties are allowed at the root level.
