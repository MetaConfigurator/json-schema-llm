The schema describes a log file entry for ingestion by tsd aggregator. It is an object type. It defines several reusable components under 'definitions': 'data' is an object, it requires 'annotations' and 'version', and 'annotations' contains required fields 'initTimestamp' and 'finalTimestamp' with additional properties of type string. 'dataElement' is an object, it requires 'values', which is an array of items of type 'sampleObject'. 'metricsList' is an object with additional properties referencing 'dataElement'. 'sampleObject' is an object, it requires 'value' of type number, and includes an enum field 'unit' with allowed values: 'nanosecond', 'microsecond', 'millisecond', 'second', 'minute', 'hour', 'day', 'week', 'bit', 'byte', 'kilobit', 'kilobyte', 'megabit', 'megabyte', 'gigabit', 'gigabyte', 'terabyte', 'petabyte'. The main properties of the schema are: 'context' is an object, optional, without required properties. 'data' is a reference to the 'data' definition, required. 'id' is a string, optional. 'level' is a string, required, with a pattern of '^info$'. 'name' is a string, required, with a pattern of '^aint.metrics$'. 'time' is a string, required, with a format of 'date-time'. No additional properties are allowed at the root level.
