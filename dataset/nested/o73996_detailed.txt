The schema describes a region as a logical group for entry visibility restrictions. It is an object type. The main properties of the schema are: 'active' is a boolean indicating if the region is active, optional with a default value of true. 'comment' is a string providing comments about the tab, optional. 'created' is a string representing the creation date in datetime format, optional. 'group' is a string that describes the tag, optional. 'guid' is a string in uuid format that uniquely identifies the region, required. 'locked' is a boolean indicating if the entry should not be deleted, optional with a default value of false. 'name' is a string that serves as the name of the tag, required. 'tenantId' is a string in uuid format that indicates the related domain, optional, and references a definition from './tenant.json'. Additionally, the root schema has a compositional constraint specifying that both 'guid' and 'text' are required fields. No additional properties are allowed at the root level.
