The schema describes a top-level domain (TLD). It is an object type. The main properties of the schema are: 'tld' is an object that contains details about the TLD. It is required and defines several properties: 'created_at' is a string that represents the date and time of TLD creation, it has a format of date-time, is read-only, and cannot be modified. 'description' is a string or null that provides a description for the TLD, with a maximum length of 160 characters. 'id' is a string that serves as the TLD identifier, adhering to a specific UUID pattern, it is also read-only. 'links' is an object that contains other properties, specifically 'self', which is a string in the format of a URL. 'name' is a string that represents the TLD name, has a maximum length of 255 characters, and is immutable. 'updated_at' is a string or null that indicates the date and time of the last TLD modification, with a format of date-time and is read-only. No additional properties are allowed at the root level.
