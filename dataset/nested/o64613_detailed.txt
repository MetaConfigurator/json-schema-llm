The schema describes a signal configuration for the IETF DOTS protocol. It is an object type. It defines several reusable components under 'definitions': 'SignalConfig' is an object, containing optional properties: 'mitigating-config' and 'idle-config', both referencing the 'Config' definition. 'Config' is an object, containing optional properties: 'session-id' (an integer), 'heartbeat-interval', 'missing-hb-allowed', and 'max-retransmit' (all referencing 'IntCurrent'), and 'ack-timeout' and 'ack-random-factor' (both referencing 'DecimalCurrent'). 'IntCurrent' is an object with a property 'current-value' (an integer). 'DecimalCurrent' is an object with a property 'current-value-decimal' (a number). The root schema resolves to the referenced definition 'SignalConfig'. The main properties of the schema are: 'ietf-dots-signal-channel:signal-config' is a reference to the 'SignalConfig' definition. No additional properties are allowed at the root level.
