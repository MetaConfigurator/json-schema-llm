The schema describes alerts configuration. It is an object type. The main properties of the schema are: 'subjectId' is a string optional. 'subjectType' is a string required, with enum values of 'device', 'service', 'deviceGroup', and 'serviceGroup'. 'subject' is a string or null optional. 'enabled' is a boolean required. 'section' is an object optional. 'field' is an object optional. 'comparison' is an object optional. 'value' is an object optional. 'fix' is a boolean required. 'group' is a string optional. 'recipients' is an array required, with items being objects that contain: 'type' which is a string required with enum values of 'user', 'pagerduty', 'webhook', and 'hipchat'; 'id' is a string required; and 'actions' which is an array of strings with items having enum values of 'pagerduty', 'webhook', 'slack', 'hipchat', 'sms', and 'email'. 'wait' is an object required with properties: 'seconds' which is a number required, 'enabled' which is a boolean required, and 'displayUnits' which is a string required with enum values of 's' and 'm'. 'repeat' is an object required with properties: 'seconds' which is a number required, 'enabled' which is a boolean required, and 'displayUnits' which is a string required with enum values of 's' and 'm'.
