The schema describes the configuration file for the osvr_server application. It is an object type. The main properties of the schema are: 'aliases' is an object that allows manually adding to the path tree, optionally with transforms. 'display' is an unspecified type, described as a display descriptor. 'drivers' is an array that contains objects with 'driver' (a string), 'params' (an object), and 'plugin' (a string) properties. 'externalDevices' is an object that contains entries defining path tree names and device descriptors for external VRPN devices, and allows additional properties, each being an object that contains 'descriptor' (an unspecified type), 'deviceName' (a string), and 'server' (a string). 'plugins' is an array of strings that represent manual-load plugins, conforming to the pattern '^[a-zA-Z]+[a-zA-Z0-9_]*$'. 'routes' is an array described as deprecated route specifications. 'server' is an object with properties: 'interface' (a string), 'local' (a boolean with a default value of true), 'port' (an integer with a default value of 3883 and a minimum of 1), and 'sleep' (a number with a default value of 1 and a minimum of 0). No additional properties are allowed at the root level.
