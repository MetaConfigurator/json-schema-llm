The schema describes a collection of experience groups along with associated metadata. It is an object type. The main properties of the schema are: 'items' is an array that contains experience group objects. Each experience group object has the following properties: 'id' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'experienceGroupId' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'creationDate' is a string in the date-time format, 'lastUpdated' is a string in the date-time format, 'name' is a string with a minimum length of 1 and a maximum length of 255, 'description' is a string with a maximum length of 32767, 'experienceUserIds' is an array of strings, each matching the pattern '^[A-Fa-f\d]{24}$' with a maximum of 1000 items, 'experienceEndpointIds' is an array of strings, each matching the pattern '^[A-Fa-f\d]{24}$' with a maximum of 1000 items, 'groupTags' is an object with properties matching the pattern '^[0-9a-zA-Z_-]{1,255}$' where each property value is a string with a minimum length of 1 and a maximum length of 255, 'deviceIds' is an array of strings, each matching the pattern '^[A-Fa-f\d]{24}$' with a maximum of 1000 items, 'deviceTags' is an array of objects, each containing a 'key' (string matching pattern '^[0-9a-zA-Z_-]{1,255}$') and a 'value' (string with a minimum length of 1 and a maximum length of 255) with a maximum of 100 items, 'deviceQueryJson' is a string or null with a maximum length of 8192, and 'parentId' can be either a string matching the pattern '^[A-Fa-f\d]{24}$' or null. 'count' is an integer, 'totalCount' is an integer, 'perPage' is an integer, 'page' is an integer, 'filter' is a string, 'filterField' is a string, 'sortField' is a string, and 'sortDirection' is a string with allowed values: 'asc', 'desc', 'ASC', 'DESC', or ''. No additional properties are allowed at the root level.
