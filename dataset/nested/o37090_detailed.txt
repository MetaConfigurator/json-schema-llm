The schema describes a representation of a normal distribution for the performance table entry of absolute data. It is an object type. The main properties of the schema are: 'type' is a string required field that must be one of the following values: 'dnorm'. 'parameters' is an object required field that defines the characteristics of the distribution and must contain the required fields 'mu' and 'sigma', both of which are numbers and have no additional properties. 'input' is an optional object that may contain properties including 'mu', 'sigma', 'value', 'stdErr', 'lowerBound', 'upperBound', 'events', 'sampleSize', and 'scale'. Among these, 'scale' must be a string and can be either 'percentage' or 'decimal', while all other fields are numbers. No additional properties are allowed at the root level.
