The schema describes options for a schematic. It is an object type. The main properties of the schema are: 'all' is a boolean when true, updates all packages in `package.json`. It has a default value of false. 'force' is a boolean when false (the default), reports an error if installed packages are incompatible with the update. It has a default value of false. 'from' is a string when using `--migrateOnly` for a single package, the version of that package from which to migrate. 'migrateExternal' is a boolean, which is hidden and has a default value of false. 'migrateOnly' is a boolean that performs a migration, but does not update the installed version. It has a default value of false. 'next' is a boolean that updates to the latest version, including beta and RCs. It has a default value of false. 'packageManager' is a string that defines the preferred package manager configuration files to use for registry settings. It has a default value of 'npm' and can be one of the following values: 'npm', 'yarn', 'cnpm', 'pnpm'. 'packages' is an array that contains the package or packages to update. Each item in the array is a string. 'registry' is a string that specifies the npm registry to use. It can be either a URI or a hostname. 'to' is a string when using `--migrateOnly` for a single package, the version of that package to which to migrate. 'verbose' is a boolean when true, displays additional details during the update process. No additional properties are allowed at the root level.
