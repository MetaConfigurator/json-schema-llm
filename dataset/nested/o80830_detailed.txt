The schema describes information needed for glossary term entity. It is an object type. The main properties of the schema are: 'anchor' is a string representing the anchor of the glossary term. 'category' is a string representing the category of the glossary term. 'definition' is a string representing the definition of the glossary term. 'experiment' is an array of objects representing associated experiments, where each object has a required property 'name' of type string. 'id' is a string representing the unique identifier for the glossary term. 'links' is an array of objects representing related links, where each object has required properties 'text' and 'url', both of type string. 'see_also' is an array of objects referencing related terms, where each object has a required property 'term' of type string. 'term' is an array of strings representing the glossary term itself. 'type' is an object that contains properties: 'primary', which is a string, and 'secondary', which is an array of strings. No additional properties are allowed at the root level.
