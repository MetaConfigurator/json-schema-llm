The schema describes a collection of transaction records. It is an array type. It defines several reusable components under 'definitions': 'ISO8601-Date' is a string, representing a date formatted as YYYY-MM-DD. 'ISO8601-Timestamp' is a string, representing a timestamp formatted as YYYY-MM-DDTHH:MM(:SS)?(([+-]HH(:MM)?)|Z). 'IDType' is a string, representing an identifier that must match the pattern of 10 digits. 'LISAManagerReferenceNumberType' is a string, representing a reference number that must match the pattern of 'Z' followed by 6 digits. 'transactionTypeType' is a string, representing the type of transaction with allowed values: 'bonus', 'charge'. The main properties of the schema are: 'hmrcTransactionID' is a string, required, referencing 'IDType'. 'accountID' is a string, required, referencing 'IDType'. 'paymentReference' is a string, required, referencing 'IDType'. 'transactionType' is a string, required, referencing 'transactionTypeType'. 'transactionTimestamp' is a string, required, referencing 'ISO8601-Timestamp'. 'LISAManagerReferenceNumber' is a string, required, referencing 'LISAManagerReferenceNumberType'. 'amount' is a number, required. 'paymentForTransaction' is a string, required, referencing 'IDType'. 'applicablePeriod' is an object, required, with properties 'startDate' (string, referencing 'ISO8601-Date') and 'endDate' (string, referencing 'ISO8601-Date'). '_links' is an object, optional. No additional properties are allowed at the root level.
