The schema describes CLI sync input. It is an object type. The main properties of the schema are: 'fabric_uuid' is a string Fabric UUID. 'devices_cli' is an array of device details dictionaries, which are objects containing required fields 'device_uuid' and 'cli_objects'. Each 'device_uuid' is a string representing the UUIDs of the device. 'cli_objects' is an array of objects, each requiring 'time' and 'status'. 'time' is a string representing the timestamp of the cli change, and 'status' is a string indicating whether to accept or reject the commit. No additional properties are allowed at the root level.
