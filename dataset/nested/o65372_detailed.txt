The schema describes the structure of air quality data. It is an object type. The main properties of the schema are: 'location' is a string with a minimum length of 1 and is required. 'parameter' is a string that must be one of the following values: 'pm25', 'pm10', 'no2', 'so2', 'o3', 'co', 'bc' and is required. 'unit' is a string that must be one of the following values: 'µg/m³', 'ppm' and is required. 'averagingPeriod' is an object that contains the required properties 'value' (number) and 'unit' (string with an enum value of 'hours'). 'attribution' is an array of objects, each requiring 'name' (string with a minimum length of 1) and having an optional 'url' (string that must match a specific pattern). 'coordinates' is an object that requires 'latitude' (number between -90 and 90) and 'longitude' (number between -180 and 180). 'value' is a number and is required. 'date' is an object that requires 'utc' (string matching a specified pattern) and 'local' (string matching a specified pattern). 'sourceName' is a string with a minimum length of 1 and is required. 'sourceType' is a string that must be one of the following values: 'government', 'research', 'other' and is required. 'mobile' is a boolean and is required. 'city' is a string with a minimum length of 1 and is required. 'country' is a string with a length of exactly 2 and is required. No additional properties are allowed at the root level.
