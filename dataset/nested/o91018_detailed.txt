The schema describes a configuration for a collection of projects. It is an object type. The main properties of the schema are: 'title' is a string that represents the title of the configuration and is required. 'header_contributing' is a string that provides a contributing header and is optional. 'header' is a string that serves as a header and is optional. 'ios_app_link' is a string that links to an iOS application and is optional. 'categories' is an array of unique items consisting of category objects, which are each required to have 'title' and 'id' properties. Each category object has 'title' as a string, 'id' as a string matching the pattern '^[^A-Z_ ]+$', 'description' as a string with a default value of an empty string, and 'parent' as a string or null with a default value of null. 'projects' is an array of unique items consisting of project objects, which are each required to have 'title', 'category', and 'homepage' properties. Each project object has 'title' as a string, 'category' as either a string or an array of strings, 'description' as a string, 'homepage' as a string or null matching the pattern '^https?:\/\/.*?\..*$', and a default value of null, 'tags' as an array with a default value of an empty array, and 'swift' as a number representing the supported Swift version. No additional properties are allowed at the root level.
