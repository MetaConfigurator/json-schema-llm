The schema describes a network interface configuration. It is an object type. The main properties of the schema are: 'interface_type' is a string that specifies the type of interface, it is optional with a default value of 'physical' and can be either 'physical' or 'logical'. 'interface_filters' is an array of objects that define filtering rules, it is optional. Each item in the array is an object with a maximum of 2 properties, and additional properties are not allowed. Each filter object contains 'op', which is an enum that can be either 'regex' or 'startwith', and 'expr', which is a string that defines the filter expression. No additional properties are allowed at the root level.
