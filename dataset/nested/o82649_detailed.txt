The schema describes a document block. It is an object type. The main properties of the schema are: 'etag' is a string that must match the pattern '^[\w\.:-]+$'. It is optional. 'css_class' is a string that must match the pattern '^[\w\s-]+$' and can be appended to the document's class attribute in rendering environments that support HTML. It is optional. 'updated_date' is a string formatted as date-time that represents an update to the node associated with this block. It is optional. 'aside' is a boolean indicating whether this block represents content only indirectly related to the main content, typically presented as sidebars or call-out boxes. It is optional. 'node_ref' is a string matching the pattern '^[\w\/\.:-]+$' and is required. 'title' is a string with a minimum length of 0 and a maximum length of 255, serving as an optional override for the node's title. 'launch_text' is a string with a minimum length of 0 and a maximum length of 255. It is optional. 'aspect_ratio' is a string that can be one of the following values: 'unknown', 'auto', '1by1', '2by3', '3by2', '3by4', '4by3', '4by5', '5by4', '9by16', '16by9', 'original', 'custom'. It is optional. 'image_ref' is a string matching the pattern '^[\w\/\.:-]+$' and serves as an optional reference to an image asset. 'fallback_src_url' is a string that must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$' and is used to store the old URL for imported document blocks. It is optional. No additional properties are allowed at the root level.
