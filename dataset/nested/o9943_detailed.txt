The schema describes a user role and permissions configuration. It is an object type. The main properties of the schema are: 'userId' is a string matching the pattern '^[A-Fa-f\d]{24}$', required; 'role' is a string with an enum of allowed values: 'admin', 'edit', 'collaborate', 'view', 'none', optional; 'applicationRoles' is an array with a maximum of 1000 items, where each item is an object containing: 'resourceId' as a string matching the pattern '^[A-Fa-f\d]{24}$', required; 'role' as a string with an enum of allowed values: 'collaborate', 'view', 'none', required; 'dashboardRoles' is an array with a maximum of 1000 items, with the same structure and requirements as 'applicationRoles'. No additional properties are allowed at the root level.
