The schema describes an event structure. It is an object type. The main properties of the schema are: 'meta' is an object containing required fields 'id', 'type', 'version', and 'time'. 'id' is a string matching the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'. 'type' is a string with an enum value of 'EiffelIssueDefinedEvent'. 'version' is a string with an enum value of '2.0.0' and a default of '2.0.0'. 'time' is an integer. 'tags' is an optional array of strings. 'source' is an object with optional properties 'domainId', 'host', 'name', 'serializer', and 'uri'. 'serializer' has a pattern '^pkg:'. 'security' is an object with a required 'sdm' object that contains required fields 'authorIdentity' and 'encryptedDigest', both of type string. 'data' is an object containing required fields 'type', 'tracker', 'id', and 'uri'. 'type' has an enum with values 'BUG', 'IMPROVEMENT', 'FEATURE', 'WORK_ITEM', 'REQUIREMENT', and 'OTHER'. 'customData' is an optional array of objects that require 'key' and 'value'. 'links' is an optional array of objects, each requiring 'type' and 'target', with 'target' matching the same pattern as 'id'. No additional properties are allowed at the root level.
