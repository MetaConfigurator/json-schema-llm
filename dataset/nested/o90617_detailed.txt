The schema describes configuration properties for a service. It is an object type. The main properties of the schema are: 'networking' is an object containing beakerx networking configuration properties. It is optional. It has an 'external_access' property, which is an object with an unencrypted access configuration, and an optional 'port_api' property that specifies the internal API communication port. 'external_access' contains 'enable', which is a boolean indicating if a VIP for external access is enabled, 'external_access_port', which is a number specifying the port for external access with a default of 18888, and 'external_public_agent_ip', which is a string for the public agent IP with a default of an empty string. The 'port_api' property has a default value of 8888. 'service' is an object containing DC/OS service configuration properties. It is required and has three properties: 'cpus', which is a number with a minimum of 1 and a default of 1; 'mem', which is a number with a minimum of 1024.0 and a default of 2048.0; and 'name', which is a string with a default value of 'beakerx'. The 'cpus' and 'mem' properties are required. 'storage' is an object containing beakerx storage configuration properties. It is optional and has a property 'host_volume_size', which is a number with a default of 4000, and a 'persistence' property that is an object with an 'enable' property, which is a boolean with a default of false. No additional properties are allowed at the root level.
