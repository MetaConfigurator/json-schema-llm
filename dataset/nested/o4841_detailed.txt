The schema describes a sample. It is an object type. The main properties of the schema are: 'alias' is a string that serves as a unique identifier in a submission and is required. 'attributes' is an object that contains an array of items, each requiring a 'value' field, and may include 'units' and 'terms', where 'terms' is an array of objects each requiring a 'url' field formatted as a URI. 'description' is a string that allows for a more extensive free-form description and is optional. 'releaseDate' is a string formatted as a date and is required. 'sampleRelationships' is an array of objects that may be structured in one of two ways: one configuration requires 'alias', 'team', and 'nature' fields, while another requires 'accession' and 'nature'. 'taxonomy' is an object requiring a 'taxonId' field and may also include 'taxonName', making it required. 'title' is a string describing the sample's title and is optional. No additional properties are allowed at the root level.
