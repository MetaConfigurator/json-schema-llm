The schema describes a collection of channels. It is an object type. The main properties of the schema are: 'channels' is an array of objects representing individual channels. It is required. Each channel object must include the required field 'id'. The 'channels' array must have unique items. Each channel object has the following optional properties: 'title' is a string, 'description' is a string, 'dj' is a string, 'djmail' is a string, 'genre' is a string, 'image' is a string, 'largeimage' is a string, 'xlimage' is a string, 'twitter' is a string, 'updated' is a string, 'listeners' is a string matching the pattern '[0-9]*', 'lastPlaying' is a string, 'lastPlayingImage' is a string formatted as a URL, 'lastPlayingAlbum' is a string, 'lastPlayingTrack' is a string, 'lastPlayingLabel' is a string, 'lastPlayingCountry' is a string, and 'lastPlayingYear' is a string matching the pattern '[0-9]*'. Each channel object may contain a 'playlists' property, which is an array of objects. Each playlist object must include the required fields 'url', 'format', and 'quality'. The 'format' must be one of the following values: 'mp3', 'aac', 'aacp'. The 'quality' must be one of the following values: 'low', 'high', 'highest'. The root schema resolves to the referenced definition. No additional properties are allowed at the root level.
