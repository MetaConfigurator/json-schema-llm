The schema describes an individual version. It is an object type. The main properties of the schema are: 'schema' is a string that represents the schema associated with this document, it is required and must be one of the allowed values: 'avalon-core:version-3.0'. 'type' is a string that describes the type of document, it is required and must be one of the allowed values: 'version'. 'parent' is a string that serves as a unique identifier to the parent document, it is optional. 'name' is a number that indicates the number of the version, it is required. 'locations' is an array of strings that specifies where on the planet this version can be found, it is optional. 'data' is an object that contains document metadata, it is required and must include the following properties: 'author' (a string representing the user logged on to the machine at the time of publish), 'source' (a string representing the original file from which this version was made), and 'time' (a string representing the ISO formatted, file-system compatible time). The 'data' object may also contain optional properties: 'timeFormat' (a string for the ISO format of time), 'version' (a number for the number of this version), and 'path' (a string for the unformatted path). No additional properties are allowed at the root level.
