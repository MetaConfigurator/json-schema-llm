The schema describes a lockable node. It is an object type. The main properties of the schema are: '_id' is a string the "_id" value: - MUST NOT change for the life of the node. - SHOULD be globally unique - SHOULD be generated by the app (ideally in default value closure... e.g. UuidIdentifier::generate()), required. 'status' is a string with a default value of 'draft', it can be one of the following values: 'unknown', 'published', 'scheduled', 'pending', 'draft', 'expired', 'archived', 'deleted', optional. 'etag' is a string that matches the pattern '^[\w\.:-]+$', optional. 'created_at' is a string that matches the pattern '^[1-9]{1}[0-9]{12,15}$', optional. 'creator_ref' is an object A fully qualified reference to what created this node. This is intentionally a message-ref and not a user id because it is often a program that creates nodes, not a user., required, with required fields: 'curie' and 'id', additionalProperties not allowed. 'updated_at' is a string that matches the pattern '^[1-9]{1}[0-9]{12,15}$', optional. 'updater_ref' is an object A fully qualified reference to what updated this node. This is intentionally a message-ref and not a user id because it is often a program that updates nodes, not a user. E.g. "acme:iam:node:app:cli-scheduler" or "acme:iam:node:user:60c71df0-fda8-11e5-bfb9-30342d363854", required, with required fields: 'curie' and 'id', additionalProperties not allowed. 'last_event_ref' is an object A reference to the last event that changed the state of this node. E.g. "acme:blog:event:article-published:60c71df0-fda8-11e5-bfb9-30342d363854", required, with required fields: 'curie' and 'id', additionalProperties not allowed. 'title' is a string with a minLength of 0 and a maxLength of 255, optional. 'is_locked' is a boolean A node being locked will vary in how it's implemented but the general idea is that only the user who locked it can modify it., optional. 'locked_by_ref' is a string that matches the pattern '^[\w\/\.:-]+$', representing the user (or app) that has "locked" the node., optional. No additional properties are allowed at the root level.
