The schema describes an object representing an application. It is an object type. The main properties of the schema are: 'id' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'creationDate' is a string formatted as date-time, optional. 'lastUpdated' is a string formatted as date-time, optional. 'ownerId' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'ownerType' is a string, with allowed values 'user' and 'organization', optional. 'name' is a string with a minimum length of 1 and a maximum length of 255, optional. 'description' is a string with a maximum length of 32767, optional. 'summary' is an object that contains properties: 'deviceCount' is a number, optional; 'flowCount' is a number, optional; 'webhookCount' is a number, optional; 'eventCount' is a number, optional; 'keyCount' is a number, optional. No additional properties are allowed at the root level.
