The schema describes a data source. It is an object type. It defines several reusable components under 'definitions': 'text' is an anyOf type, allowing either a number or a non-empty string. 'text_list' is an array type, requiring at least one unique item from the 'text' definition. 'source_type_type' is a string type, constrained by a specific pattern corresponding to acceptable source types and a maximum length of 255. 'foreignid_type' is an object type that requires 'domain' and 'id' properties; 'domain' must match the pattern '^database\.wfcc\.ch$' and 'id' must be an integer greater than or equal to 1, with no additional properties allowed. The main properties of the schema are: 'name' is a text which is required. 'type' is a source_type_type which is required. 'foreignids' is an array of foreignid_type, required to have at least one item. 'comments' is an optional text_list. No additional properties are allowed at the root level.
