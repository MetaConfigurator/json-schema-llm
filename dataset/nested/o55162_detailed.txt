The schema describes member information for a system. It is an object type. It defines several reusable components under 'definitions': 'member' is an object, which does not allow additional properties and contains the following properties: 'name' is a string with a maximum length of 191 and must match the pattern '^([^,]|$)'; 'email' is a string with a minimum length of 1 and a maximum length of 191, also matching the same pattern; 'avatar_image' strips to true; 'note' is a string with a minimum length of 0 and a maximum length of 2000; 'subscribed' is a boolean; 'comped' strips to true; 'id' strips to true; 'labels' is a reference to the 'member-labels' definition; 'created_at' strips to true; 'created_by' strips to true; 'updated_at' strips to true; 'updated_by' strips to true. 'member-labels' is an array, where each item can be either an object or a string with a maximum length of 191. If it is an object, it has properties: 'id', which is a string with a maximum length of 24; 'name', which is a string with a maximum length of 191; 'slug', which can be a string or null with a maximum length of 191; and 'members', which strips to true. At least one of 'id', 'name', or 'slug' is required for the object. The main properties of the schema are: 'member' is an object that references the 'member' definition and is required; 'member-labels' is an array that references the 'member-labels' definition and is required. No additional properties are allowed at the root level.
