The schema describes a structure for managing targets and actions within an application context. It is an object type. The main properties of the schema are: 'primaryTarget' is a array of objects each containing an 'id' (a string with a pattern of '^[A-Fa-f\d]{24}$'), a 'type' (a string with enum values 'Application', 'Dashboard', 'OrgInvite'), and a 'name' (a string with a maxLength of 1024). 'secondaryTarget' is a array of objects each containing an 'id' (a string with a pattern of '^[A-Fa-f\d]{24}$'), a 'type' (a string with enum values 'ApiToken', 'ApplicationKey', 'DataTable', 'Device', 'DeviceRecipe', 'Event', 'ExperienceDomain', 'ExperienceEndpoint', 'ExperienceGroup', 'ExperienceSlug', 'ExperienceUser', 'ExperienceVersion', 'ExperienceView', 'File', 'Flow', 'Integration', 'Webhook'), and a 'name' (a string with a maxLength of 1024). 'actor' is a array of objects each containing an 'id' (a string with a pattern of '^[A-Fa-f\d]{24}$'), a 'type' (a string with enum values 'Application', 'Device', 'Flow', 'User', 'ApiToken'), and a 'name' (a string with a maxLength of 1024). 'request' is a array of objects each containing a 'resource' (a string with a maxLength of 1024) and an 'action' (a string with a maxLength of 1024). 'responseCode' is a array of integers where each must be between 100 and 599. No additional properties are allowed at the root level.
