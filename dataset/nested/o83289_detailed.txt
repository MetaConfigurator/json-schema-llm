The schema describes the configuration for generating code in a Thrift-based application. It is an object type. The main properties of the schema are: 'annotationPrefix' is a string the prefix for HTTP annotations in Thrift files. It is required. 'clientConfig' is a string the directory for client module configs, relative to application root directory. It is optional. 'copyrightHeader' is a string the path of copyright header file, relative to application root directory. It is required. 'endpointConfig' is a string the directory for endpoint module configs, relative to application root directory. It is optional. 'genCodePackage' is a string the package path for thriftrw and easyjson generated structs and (de)serializers. It is required. 'genMock' is a boolean whether to generate mocks during code generation. It is required. 'middlewareConfig' is a string the directory for middleware module configs, relative to application root directory. It is required. 'packageRoot' is a string the application package root. It is required. 'targetGenDir' is a string an explanation about the purpose of this instance. It is optional. 'thriftRootDir' is a string the path of idl directory, relative to application root directory. It is required. 'traceKey' is a string the name of the generated trace header. It is required. No additional properties are allowed at the root level.
