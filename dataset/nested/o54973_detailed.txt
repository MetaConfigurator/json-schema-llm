The schema describes a data origin structure. It is an object type. It defines several reusable components under 'definitions': 'source' is an object, representing the main data source/repository data is retrieved from, and it has a required field 'name' with an enum value of 'UniProt', and optional fields 'url' and 'description'. 'citation' is an object that contains optional fields 'type' with enum values of 'DOI', 'Pubmed', and 'URL', and 'id' which can match specific patterns. The main properties of the schema are: 'scope' is an array, optional, with enum values of 'runtime' and 'data build'; 'name' is a string, required; 'version' is an object, required, with optional properties 'external' (a string) and 'internal' (a string with a pattern of '^\d+(\.\d+(\.\d+)?)?$'); 'download_date' is a string, optional; 'creation_date' is a string, optional; 'origin_url' is a string, optional; 'format' is a string, required, with enum values of 'FASTA', 'FASTQ', 'GENBBANK', 'BIOM', 'TEXT', 'BOWTIE INDEX', 'UNIPROT', and 'TAB'; 'type' is a string, required, with enum values of 'PROTEIN', 'NUCLEOTIDE', 'RNA', and 'EXPRESSION'; 'content' is an array, optional, with enum values of 'ORGANISM', 'HUMAN', and an empty string; 'source' is an object, required, with a reference to the 'source' definition; 'description' is a string, optional; 'citation' is a string, optional. No additional properties are allowed at the root level.
