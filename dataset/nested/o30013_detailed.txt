The schema describes a JSON application firewall profile binding. It is an object type. The main properties of the schema are: 'alertonly' is a string that indicates the alert status, with allowed values 'ON' and 'OFF'. 'comment' is a string that provides additional information. 'isautodeployed' is a string indicating deployment status, with allowed values 'AUTODEPLOYED' and 'NOTAUTODEPLOYED'. 'jsondosurl' is a string that represents the URL. 'jsonmaxarraylength' is an integer defining the maximum array length. 'jsonmaxarraylengthcheck' is a string indicating whether to check the maximum array length, with allowed values 'ON' and 'OFF'. 'jsonmaxcontainerdepth' is an integer specifying the maximum container depth. 'jsonmaxcontainerdepthcheck' is a string indicating whether to check the maximum container depth, with allowed values 'ON' and 'OFF'. 'jsonmaxdocumentlength' is an integer denoting the maximum document length. 'jsonmaxdocumentlengthcheck' is a string indicating whether to check the maximum document length, with allowed values 'ON' and 'OFF'. 'jsonmaxobjectkeycount' is an integer defining the maximum number of object keys. 'jsonmaxobjectkeycountcheck' is a string indicating whether to check the maximum object key count, with allowed values 'ON' and 'OFF'. 'jsonmaxobjectkeylength' is an integer specifying the maximum object key length. 'jsonmaxobjectkeylengthcheck' is a string indicating whether to check the maximum object key length, with allowed values 'ON' and 'OFF'. 'jsonmaxstringlength' is an integer denoting the maximum string length. 'jsonmaxstringlengthcheck' is a string indicating whether to check the maximum string length, with allowed values 'ON' and 'OFF'. 'name' is a string identifying the profile. 'state' is a string indicating the state of the profile, with allowed values 'ENABLED' and 'DISABLED'. No additional properties are allowed at the root level.
