The schema describes geocoding. It is an object type. The main properties of the schema are: 'key' is a string A unique identifier for the geocoding, optional. 'name' is a string Specific description of the geocoding result, optional. 'bbox' is an array Boundary box, required, item objects must contain 'lat' (a number Latitude) and 'lng' (a number Longitude). 'lat' is a number Latitude, required. 'lng' is a number Longitude, required, with a minimum of -180 and a maximum of 180. 'confidence' is a string Confidence ("high", "medium", "low"), optional. 'type' is a string Non-standardized. Is used for tooltip ("Street", "City" etc), optional. 'postalCode' is a string If the result has a postal code, it's returned here, optional. 'countryRegion' is a string If the region is known, it's returned here, optional. 'curbside_coordinates' is an array Curbside Coordinates, optional, item objects must contain 'lat' (a number Latitude) and 'lng' (a number Longitude). No additional properties are allowed at the root level.
