The schema describes a VCMI bonus system format. It is an object type. It defines several reusable components under 'definitions': 'nestedLimiter' is an anyOf type, which can be a string as a parameterless limiter or boolean operator at the start of an array, an object with 'type' as a string and 'parameters' as an array with additional items allowed, or an array of nested limiters optionally prefixed with a boolean operator. The main properties of the schema are: 'addInfo' is an anyOf type describing addInfo that can be a string, a number, or an array of items that are either strings or numbers. 'description' is a string describing the description. 'duration' is an anyOf type describing duration that can be a string or an array of strings. 'effectRange' is a string describing effectRange. 'limiters' is a reference to the 'nestedLimiter' definition describing a limiter. 'propagator' is an anyOf type describing propagator that can be a string or an array of strings. 'updater' is an anyOf type describing updater that can be a string or an object requiring 'type' and 'parameters' fields, where 'type' is a string and 'parameters' is an array with additional items allowed. 'sourceID' is a number describing sourceID. 'sourceType' is a string describing sourceType. 'stacking' is a string describing stacking. 'subtype' is an anyOf type describing subtype that can be a string or a number. 'turns' is a number describing turns. 'type' is a string describing type. 'val' is a number describing val. 'valueType' is a string describing valueType. No additional properties are allowed at the root level.
