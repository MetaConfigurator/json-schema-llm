The schema describes configuration settings for a marathon-lb instance. It is an object type. The main properties of the schema are: 'marathon-lb' is an object that contains configuration options. It has the following required fields: 'cpus', 'mem', 'haproxy-group', 'instances', and 'name', which define the runtime environment for the load balancer instance. The 'cpus' property is a number with a default value of 2, minimum of 1. The 'mem' property is a number with a default value of 1024.0 MB, minimum of 256.0. The 'haproxy-group' property is a string with a default value of 'external'. The 'instances' property is an integer with a default value of 1, minimum of 1. The 'name' property is a string with a default value of 'marathon-lb'. The 'auto-assign-service-ports' property is a boolean with a default of false, describing if service ports should be auto-assigned. The 'bind-http-https' property is a boolean with a default of true, used to reserve ports 80 and 443. The 'haproxy-map' property is a boolean with a default of true, enabling HAProxy VHost maps. The 'haproxy_global_default_options' property is a string with a default value of 'redispatch,http-server-close,dontlognull', which sets global HAProxy options. The 'maximumOverCapacity' property is a number with a default value of 0.2, minimum of 0. The 'minimumHealthCapacity' property is a number with a default value of 0.5, minimum of 0. The 'role' property is a string with a default value of 'slave_public'. The 'secret_name' property is a string with a default value of an empty string. The 'ssl-cert' property is a string for providing TLS Cert and private key for HTTPS. The 'sysctl-params' property is a string with a default value containing various sysctl parameters. The 'template-url' property is a string with a default value of an empty string. No additional properties are allowed at the root level.
