The schema describes a filter offering configuration. It is an object type. It defines several reusable components under 'definitions': 'ComputeOfferings' is an array, containing string items. 'DiskOfferings' is an array, containing string items. 'FilterOfferingDisabled' is an object, requiring 'filterOfferings' (which must be false) and an optional 'zones' property that can have additional properties of type 'OfferingsForZone'. 'FilterOfferingEnabled' is an object, requiring 'filterOfferings' (which must be true) and 'zones', which again can have additional properties of type 'OfferingsForZone'. 'OfferingsForZone' is an object that can have 'computeOfferings' (using 'ComputeOfferings') and 'diskOfferings' (using 'DiskOfferings'), and does not allow additional properties. The schema requires one of the definitions to be valid: either 'FilterOfferingEnabled' or 'FilterOfferingDisabled'. No additional properties are allowed at the root level.
