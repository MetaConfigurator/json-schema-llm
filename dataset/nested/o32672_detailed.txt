The schema describes Angular Universal App Options. It is an object type. The main properties of the schema are: 'appDir' is a string that describes the name of the application directory. It is optional and has a default value of 'app'. 'appId' is a string that describes the appId to use with ServerTransition. It is optional and has a default value of 'serverApp'. 'clientApp' is a string that describes the name or index of the related client app. It is optional and has a default value of '0'. 'index' is a string that describes the name of the index file. It is optional and has a default value of 'index.html'. 'main' is a string that describes the name of the main entry-point file. It is optional and has a default value of 'main.server.ts'. 'name' is a string that describes the name of the universal app. It is optional. 'outDir' is a string that describes the output directory for build results. It is optional and has a default value of 'dist-server'. 'root' is a string that describes the root directory of the app. It is optional and has a default value of 'src'. 'rootModuleClassName' is a string that describes the name of the root module class. It is optional and has a default value of 'AppServerModule'. 'rootModuleFileName' is a string that describes the name of the root module file. It is optional and has a default value of 'app.server.module.ts'. 'sourceDir' is a string that describes the path of the source directory. It is optional, has an alias of 'sd', and a default value of 'src'. 'test' is a string that describes the name of the test entry-point file. It is optional. 'testTsconfigFileName' is a string that describes the name of the TypeScript configuration file for tests. It is optional and has a default value of 'tsconfig.spec'. 'tsconfigFileName' is a string that describes the name of the TypeScript configuration file. It is optional and has a default value of 'tsconfig.server'. No additional properties are allowed at the root level.
