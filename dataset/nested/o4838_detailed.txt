The schema describes a submittable base schema. It is an object type. It defines several reusable components under 'definitions': 'attributes' is an object, attributes for describing a submittable. It has no required fields and allows additional properties that match the pattern '^.*$', each being an array containing at least one item. Each item must have a required field 'value' as a string with a minimum length of 1, and may also have 'units' as a string with a minimum length of 1, and 'terms' as an array of objects that require a 'url' of string format 'uri'. 'submittableRef' is an object, representing a reference structure. It has properties 'alias', 'accession', and 'team', each being a string with a minimum length of 1. It requires either both 'alias' and 'team', or just 'accession'. The main properties of the schema are: 'alias' is a string, required with a minimum length of 1. 'title' is a string, optional. 'description' is a string, optional. 'attributes' is a reference to 'attributes'. 'studyRef' is a reference to 'submittableRef', optional. 'protocolUses' is an array of objects, each containing 'protocolRef' as a reference to 'submittableRef', and 'attributes' as a reference to 'attributes'. 'sampleUses' is an array of objects, each containing 'sampleRef' as a reference to 'submittableRef', and 'attributes' as a reference to 'attributes'. No additional properties are allowed at the root level.
