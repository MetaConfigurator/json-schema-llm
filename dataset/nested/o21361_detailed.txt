The schema describes a content item structure. It is an object type. It defines several reusable components under 'definitions': 'guid' is a string, conforming to the pattern '^[a-f0-9]{8}-[a-f0-9]{4}-[1-5][a-f0-9]{3}-[89ab][a-f0-9]{3}-[a-f0-9]{12}$'. 'guid_list' is an array of unique items, where each item is a reference to 'guid'. The main properties of the schema are: 'bulk_publishing' is an optional boolean. 'links' is an optional object that restricts additional properties and contains multiple properties, each referencing 'guid_list': 'content_owners' is for maintaining the guide's community; 'email_alert_signup' is for the email alert signup page; 'facet_groups' and 'facet_values' are for metadata tagging; 'finder' links back to finders; 'lead_organisations' is deprecated and emphasises related organisations; 'linked_items' includes content ids referenced in details; 'mainstream_browse_pages' powers the site’s browse section; 'meets_user_needs' states user needs met by the content; 'ordered_related_items' includes related items based on editor determination; 'ordered_related_items_overrides' overrides auto-generated links; 'organisations' are linked to the content item; 'original_primary_publishing_organisation' indicates the original publishing organisation; 'parent' links to the parent content item with maxItems of 1; 'policy_areas' is a largely deprecated tag for email alerts; 'primary_publishing_organisation' is the organisation publishing the page, with maxItems of 1; 'suggested_ordered_related_items' are A/B test suggestions for related items; 'taxons' is a taxonomy label; and 'topics' powers the topic section of the site. No additional properties are allowed at the root level.
