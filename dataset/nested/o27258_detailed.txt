The schema describes options for configuring an NgRx container. It is an object type. The main properties of the schema are: 'changeDetection' is a string specifying the change detection strategy, defaulting to 'Default', with allowed values 'Default' and 'OnPush'. 'export' is a boolean specifying if the declaring module exports the component, defaulting to false. 'flat' is a boolean flag indicating if a directory is created, defaulting to false. 'inlineStyle' is a boolean specifying if the style will be in the TypeScript file, defaulting to false. 'inlineTemplate' is a boolean specifying if the template will be in the TypeScript file, defaulting to false. 'module' is a string allowing specification of the declaring module. 'name' is a string representing the name of the component, with a default value sourced from the argv at index 0. 'path' is a string specifying the path to create the component, formatted as a path and marked as not visible. 'prefix' is a string specifying the prefix to apply to generated selectors, formatted as an HTML selector. 'project' is a string representing the name of the project, with an alias of 'p'. 'selector' is a string defining the selector to use for the component, formatted as an HTML selector. 'skipImport' is a boolean flag to skip the module import, defaulting to false. 'skipTest' is a boolean indicating if test files should not be created when true, defaulting to false. 'state' is a string specifying the path to the state exports. 'stateInterface' is a string specifying the interface for the state, defaulting to 'State'. 'style' is a string defining the file extension or preprocessor to use for style files, defaulting to 'css'. 'testDepth' is a string specifying whether to create a unit test or an integration test, defaulting to 'integration', with allowed values 'unit' and 'integration'. 'viewEncapsulation' is a string specifying the view encapsulation strategy, with allowed values 'Emulated', 'Native', and 'None'. No additional properties are allowed at the root level.
