The schema describes an event structure for issues in the Eiffel framework. It is an object type. The main properties of the schema are: 'meta' is an object that contains required fields: 'id' (a string that must match a UUID pattern), 'type' (a string that must be 'EiffelIssueDefinedEvent'), 'version' (a string that must be '1.0.0' with a default of '1.0.0'), and 'time' (an integer). It also includes optional properties such as 'tags' (an array of strings), 'source' (an object with properties 'domainId', 'host', 'name', and 'uri', where 'serializer' is a required object with properties 'groupId', 'artifactId', and 'version'), and 'security' (an object with 'sdm' that has required properties 'authorIdentity' and 'encryptedDigest'). 'data' is an object that contains required fields: 'type' (a string that must be one of the values 'BUG', 'IMPROVEMENT', 'FEATURE', 'WORK_ITEM', 'REQUIREMENT', or 'OTHER'), 'tracker' (a string), 'id' (a string), and 'uri' (a string). It also includes an optional property 'title' (a string) and 'customData' (an array of objects each containing required fields 'key' and 'value'). 'links' is an array of objects with required fields 'type' (a string) and 'target' (a string that must match a UUID pattern). No additional properties are allowed at the root level.
