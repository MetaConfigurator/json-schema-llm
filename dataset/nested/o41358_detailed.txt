The schema describes an invocation manifest for a task converter. It is an object type. The main properties of the schema are: 'inputs' is an object that is required and contains required properties 'LogConfig' and 'file'. 'LogConfig' is an object that is required and contains required properties 'base', 'found', and 'value'. 'found' is a boolean. 'base' is a string. 'value' can be of any type. 'file' is an object that contains the property 'type', which is an enum with allowed values 'text' and 'tabular data'. 'config' is an object that is required but contains no properties. No additional properties are allowed at the root level.
