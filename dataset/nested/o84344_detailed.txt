The schema describes a Visual Pinball build. It is an object type. The main properties of the schema are: '_created_by' is a string Reference to the user who created this build. Only visible by contributors and above. It is readOnly. '_id' is a string Unique identifier for a build. Gets generated automatically on creation. It must match the pattern '^[a-z0-9._-]{2,}$' and is readOnly. 'built_at' is a string Date when the build was, well, built. It is optional. 'created_at' is a string Timestamp when the build was added to the database. It is readOnly. 'description' is a string A description of the build. Contains changelog for official releases. Is Markdown formatted. It is optional. 'is_active' is a boolean When creating a new build, it first becomes inactive. When linking a release to the build, it becomes active. It is readOnly. 'is_range' is a boolean Instead of covering one single release, it's also possible to create builds that cover a range of releases (e.g. 'v8.x.x'). It is optional. 'label' is a string Label of the build. Typically contains the version of the release, the revision number of daily builds and a short name for experimental builds. It must have a minimum length of 3 characters and is required. 'major_version' is a string The major version branch of the version for the given platform. For VP9, that would be '9'. It is optional. 'platform' is a string Platform of the build. Only Visual Pinball supported so far. It is required and can only be 'vp'. 'type' is a string Type of the build. Values: 'release', 'nightly', 'experimental'. It is required. No additional properties are allowed at the root level.
