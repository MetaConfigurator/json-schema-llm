The schema describes the configuration of etcd. It is an object type. The main properties of the schema are: 'name' is a string Name of etcd configuration. It is optional. 'type' is a string Type of kvStore. It is required and must be one of the following values: 'etcd'. 'kind' is a string Type of etcdConfig. It is optional and must be one of the following values: 'kvStore'. 'clientPorts' is an array of numbers Ports used to listen for client requests. It is required, has a minimum of 1 item, and defaults to [2379, 4001]. 'clusterToken' is a string Token used during etcd cluster formation. It is optional and defaults to 'espouse-monger-rarely'. 'uuidToken' is a boolean If true, uuidgen will be used to generate a unique token for each etcd cluster. It is optional and defaults to true. 'peerPorts' is an array of numbers Ports used for peer to peer communication. It is required, has a minimum of 1 item, and defaults to [2380]. 'ssl' is a boolean Whether or not SSL is used for etcd traffic. It is required. 'version' is a string Version of etcd container which will be run. It is required. No additional properties are allowed at the root level.
