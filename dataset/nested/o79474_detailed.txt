The schema describes a list of conditions that either allow login or inject roles. It is an array type. It defines several reusable components under 'definitions': 'wildCardCondition' is an object, representing a condition based on a SAML attribute and a wildcard, requiring 'attribute' and 'wildCard' fields. 'alwaysCondition' is a boolean, indicating a condition that is always true. 'notCondition' is an object, requiring a 'not' field that references 'condition'. 'condition' is an object, which must be one of the defined conditions. 'injectRoles' is an object, requiring 'injectRoles', which itself requires 'condition' and 'roles' fields as an array of strings.
