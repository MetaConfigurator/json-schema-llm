The schema describes an address representation. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'id' is a string A unique identifier for the address.json, required. 'type' is a string The json_api type for any address.json, required, with a pattern that must match '^addresses$'. 'attributes' is an object Contains properties such as 'meta_attributes', 'customer_account_id', 'first_name', 'last_name', 'middle_names', 'address_line_1', 'address_line_2', 'address_line_3', 'city', 'state', 'postcode', 'country', 'preferred_shipping', and 'preferred_billing', all of which are optional. 'relationships' is an object Contains a 'customer_account' property which is also an object. This includes 'links' (object with properties 'self' and 'related' as strings, no additional properties allowed), 'data' (object with 'id' and 'type' as strings, no additional properties allowed), and 'meta' (object that allows additional properties). 'links' is an object Contains a property 'self' which is a string, no additional properties allowed. 'meta' is an object that does not allow additional properties. No additional properties are allowed at the root level.
