The schema describes the configuration for managing visibility of a content area. It is an object type. The main properties of the schema are: 'area' is an object that defines the name of the exported area used to show or hide content, and it has an optional 'name' field with a default value of 'content'. 'show' is an object that controls the visibility of the content area upon action request, and it includes an optional 'onActions' array which specifies the actions on which the content should become visible. 'hide' is an object that handles the hiding of the content area upon action request, with an optional 'onActions' array specifying the actions for invisibility. 'visibility' is an object that provides features regarding the visibility of the controlled widget area, containing an optional 'initially' boolean field (defaulting to false) that determines if the content area is visible initially, an optional 'flag' string field which is published on changes to the visibility, and an optional 'toggleOn' array that includes flags toggling the visibility of the content. 'animation' is an object that offers to animate changes to the visibility state, containing an optional 'enabled' boolean field that indicates if changes to visibility are animated, defaulting to true. No additional properties are allowed at the root level.
