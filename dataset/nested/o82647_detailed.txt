The schema describes a document block. It is an object type. The main properties of the schema are: 'etag' is a string that must match the pattern '^[\w\.:-]+$'. It is optional. 'css_class' is a string that must match the pattern '^[\w\s-]+$'. It is optional and is used in rendering environments for the DOM elements' class attribute. 'updated_date' is a string formatted as date-time. It is optional and represents an update to the node this block is attached to. 'aside' is a boolean indicating if the block content is indirectly related to the main content. It is optional. 'node_ref' is a string that must match the pattern '^[\w\/\.:-]+$'. It is required. 'title' is a string with a minimum length of 0 and a maximum length of 255. It is optional and serves as an override for the node's title. 'launch_text' is a string with a minimum length of 0 and a maximum length of 255. It is optional. 'image_ref' is a string that must match the pattern '^[\w\/\.:-]+$'. It is optional and refers to an image asset. 'fallback_src_url' is a string that must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$'. It is optional, used for storing an old URL for imported document blocks. No additional properties are allowed at the root level.
