The schema describes OpenQuantum backend properties. It is an object type. It defines several reusable components under 'definitions': 'gate' is an object, which requires fields: 'qubits', 'gate', and 'parameters'. 'qubits' is an array of integers with a minimum of 1 item. 'gate' is a string. 'parameters' is an array of items that reference the 'nduv' definition with a minimum of 1 item. 'general_parameters' is an array, which is a collection of items that reference the 'nduv' definition with a minimum of 0 items. 'nduv' is an object that requires fields: 'name', 'date', 'unit', and 'value'. 'date' is a string formatted as date-time. 'name', 'unit' are strings, and 'value' is a number. The main properties of the schema are: 'backend_name' is a string that is required and describes the backend name. 'backend_version' is a string that must match the pattern '[0-9]+.[0-9]+.[0-9]+$' and is required. 'gates' is an array that requires at least 1 item and each item references the 'gate' definition. 'general' is a reference to 'general_parameters' and is required. 'last_update_date' is a string formatted as date-time and is required. 'qubits' is an array that requires at least 1 item, where each item is an array containing items that reference the 'nduv' definition. No additional properties are allowed at the root level.
