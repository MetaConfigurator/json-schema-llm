The schema describes QISKit device backend parameters. It is an object type. It defines several reusable components under 'definitions': 'measuredParameter' is an object, recorded parameter value. It requires fields 'date', 'unit', and 'value' where 'date' must be a string in date-time format, 'unit' must be a string, and 'value' must be a number. The main properties of the schema are: 'fridgeParameters' is an object, it requires fields 'Temperature' (a reference to 'measuredParameter') and 'cooldownDate' (must be a string in date-time format). 'lastUpdateData' is an optional string in date-time format. 'qubits' is an array of objects, each object requires fields 'T1' (a reference to 'measuredParameter'), 'T2' (a reference to 'measuredParameter'), 'buffer' (a reference to 'measuredParameter'), 'frequency' (a reference to 'measuredParameter'), 'gateTime' (a reference to 'measuredParameter'), and 'name' (must be a string). The array must contain at least one item. No additional properties are allowed at the root level.
