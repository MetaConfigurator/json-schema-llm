The schema describes a structured event with metadata, data, and links. It is an object type. The main properties of the schema are: 'meta' is an object containing required fields: 'id', 'type', 'version', and 'time'. The 'id' must match the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'. The 'type' must be 'EiffelEnvironmentDefinedEvent'. The 'version' must be '2.0.0' with a default value of '2.0.0'. The 'time' is an integer. The 'tags' is an optional array of strings. The 'source' is an object with optional properties: 'domainId', 'host', 'name', 'serializer' (which must match the pattern '^pkg:'), and 'uri', with additional properties not allowed. The 'security' is an optional object that must contain 'sdm', which has required fields: 'authorIdentity' and 'encryptedDigest', both strings, with additional properties not allowed. 'data' is an object with a required field 'name', and optional fields: 'version', 'image', 'host' (which is an object with required fields 'name' and 'user'), 'uri', and 'customData' (which is an array of objects with required fields 'key' and 'value'). 'links' is an array of objects, with required fields 'type' and 'target', where 'target' must match the same UUID pattern as 'id'. No additional properties are allowed at the root level.
