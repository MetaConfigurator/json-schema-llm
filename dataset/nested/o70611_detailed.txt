The schema describes a record serialization. It is an object type. The main properties of the schema are: '$meta' is an object that holds metadata about the record. 'coord' is an object that contains required properties 'lat' and 'lon', which are numbers representing the latitude and longitude of the device, respectively. 'data' is an object that includes pattern properties where the keys are numeric strings; each of these objects must contain required properties 'name' (a string), 'unit' (a string, defaulting to 'dimensionless'), and 'value' (either a number or null). 'device_name' is a string that holds the human-readable name of the device. 'name' is a string representing a unique reference name for the device. 'parsed' is an object that includes pattern properties with keys matching lowercase letters, underscores, or hyphens; each of these objects must contain required properties 'unit' (a string) and 'value' (either a number or null). 'timestamp' is a number indicating when the record was created. 'ttl' is a number representing the lifetime of the record. 'type' is a string that must be one of the supported device types, with a default value of 'UNKNOWN'. No additional properties are allowed at the root level.
