The schema describes configuration options for an Angular AppShell. It is an object type. The main properties of the schema are: 'appDir' is a string The name of the application directory. It is optional with a default value of 'app'. 'appId' is a string The appId to use withServerTransition. It is optional with a default value of 'serverApp'. 'clientProject' is a string Name of related client app. It is required. 'index' is a string Name of the index file. It is optional with a default value of 'index.html'. 'main' is a string The name of the main entry-point file. It is optional with a default value of 'main.server.ts'. 'name' is a string Name of the universal app. It is optional. 'outDir' is a string The output directory for build results. It is optional with a default value of 'dist-server'. 'root' is a string The root directory of the app. It is optional with a default value of 'src'. 'rootModuleClassName' is a string The name of the root module class. It is optional with a default value of 'AppServerModule'. 'rootModuleFileName' is a string The name of the root module file. It is optional with a default value of 'app.server.module.ts'. 'route' is a string Route path used to produce the app shell. It is optional with a default value of 'shell'. 'sourceDir' is a string The path of the source directory. It is optional with a default value of 'src' and has an alias 'D'. 'test' is a string The name of the test entry-point file. It is optional. 'testTsconfigFileName' is a string The name of the TypeScript configuration file for tests. It is optional with a default value of 'tsconfig.spec'. 'tsconfigFileName' is a string The name of the TypeScript configuration file. It is optional with a default value of 'tsconfig.server'. 'universalProject' is a string Name of related universal app. It is required. No additional properties are allowed at the root level.
