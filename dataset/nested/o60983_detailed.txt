The schema describes the Result of KeywordsTagging worker. It is an object type. It defines several reusable components under 'definitions': 'keywords_tagging_details' is an object, Details computed by KeywordsTagging worker. It has properties: 'description' which is an object with additionalProperties set to true, and 'keywords' which is an array of strings. No additional properties are allowed at the root level. The main properties of the schema are: 'schema' is an object containing properties: 'name' which is a string with a required status, described as Name of the schema and matching the pattern '^[a-zA-Z0-9_]+$', 'version' which is a string with a required status, described as Version of the schema and matching the pattern '^[0-9]+-[0-9]+-[0-9]+$', and 'url' which is an optional string described as Full URL of the schema in format uri. '_release' is a string and is optional, described as Unique release id in form of "ecosystem:package:version". 'status' is a string with required status and enumerated values of 'success' and 'error'. 'details' is an object reference to the 'keywords_tagging_details' definition. 'summary' is an array of strings with required status. No additional properties are allowed at the root level.
