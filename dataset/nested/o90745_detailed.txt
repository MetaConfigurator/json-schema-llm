The schema describes configuration settings for a Marathon load balancer. It is an object type. The main properties of the schema are: ["marathon-lb"] is an object containing configuration parameters for the load balancer. It is required and defines several properties, including: ["bind-http-https"] is a boolean that reserves ports 80 and 443 for the LB, with a default value of true. ["cpus"] is a number that specifies CPU shares allocated to each marathon-lb instance, required with a default of 2 and a minimum of 1. ["haproxy-group"] is a string providing the HAProxy group parameter, required with a default of "external". ["instances"] is an integer indicating the number of instances to run, required with a default of 1 and a minimum of 1. ["marathon-uri"] is a string for the URI of the Marathon instance, with a default of "http://master.mesos:8080". ["maximumOverCapacity"] is a number for maximum over capacity, with a default of 0.2 and a minimum of 0. ["mem"] is a number for memory (MB) allocated to each marathon-lb task, required with a default of 1024.0 and a minimum of 256.0. ["minimumHealthCapacity"] is a number for minimum health capacity, with a default of 0.5 and a minimum of 0. ["name"] is a string for the name of this LB instance, required with a default of "marathon-lb". ["role"] is a string specifying the role to deploy marathon-lb on, with a default of "slave_public". ["ssl-cert"] is a string for TLS Cert and private key for HTTPS. ["sysctl-params"] is a string for sysctl params to set at startup for HAProxy, with a default of "net.ipv4.tcp_tw_reuse=1 net.ipv4.tcp_fin_timeout=30 net.ipv4.tcp_max_syn_backlog=10240 net.ipv4.tcp_max_tw_buckets=400000 net.ipv4.tcp_max_orphans=60000 net.core.somaxconn=10000". ["template-url"] is a string for the URL to the tarball containing templates for customizing haproxy config, with a default value of "". No additional properties are allowed at the root level.
