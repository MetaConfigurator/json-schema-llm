The schema describes a document of some sort (PDF, TIFF, HTML) associated with a filing. It is an object type. The main properties of the schema are: 'uid' is a string unique id/reference for the filing if it has one. 'file_format' is a string format of the document, if known. Allowed values are 'xml', 'pdf', 'html', 'doc', null. 'original_source_url' is a string persistent URL for the document, if there is one. 'source_url' is a string source url of the web page for the filing or if there is not persistent URL the page from which it can be found (e.g. search page), with a minimum length of 11 characters. 'source_location' is a string if the accounts filing has been put on sources.opencorporates.com the path to the filing. 'language' is a string should be represented as two-letter ISO-639 code, with a minimum and maximum length of 2 characters. 'classification' is a string type of document (at the moment only XBRL is the only option here). Allowed values are 'xbrl', null. 'pages' is an integer number of pages, if known. The effective root schema requires at least one of the properties specified in the 'anyOf' condition: either 'source_location' or 'source_url'. No additional properties are allowed at the root level.
