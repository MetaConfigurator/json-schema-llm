The schema describes the chunk processor description. It is an object type. It defines several reusable components under 'definitions': 'severity' is a string, representing the severity of the processing action, with allowed values: 'abort_file', 'abort_scope', 'skip_record', 'skip_field'. The main properties of the schema are: 'keyHashFields' is an array, required, with a minimum of 1 string item, and all items must be unique. 'keyHashName' is a string, optional. 'contentHashFields' is an array, optional, with a minimum of 1 string item, and all items must be unique. 'contentHashName' is a string, optional. 'scopeHashFields' is an array, optional, with a minimum of 1 string item, and all items must be unique. 'scopeHashName' is a string, optional. 'multiRowFields' is an array, optional, with unique string items. 'checks' contains 'double_rows_same', which resolves to the referenced definition 'severity', and 'double_rows_different', which also resolves to the referenced definition 'severity'. No additional properties are allowed at the root level.
