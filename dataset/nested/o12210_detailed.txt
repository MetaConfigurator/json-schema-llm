The schema describes feedback submission for surveys. It is an object type. The main properties of the schema are: 'collection' is an object containing properties 'exercise_sid', 'instrument_id', and 'period', all of which are optional strings. 'data' is an object containing properties 'email', 'message', 'name', and 'url', all of which are optional strings. 'metadata' is an object containing properties 'ru_ref' and 'user_id', both of which are optional strings. 'origin' is a string with a pattern constraint that must match '^uk.gov.ons.edc.eq$'. 'submitted_at' is a string formatted as date-time, providing the time at which survey feedback was submitted. 'survey_id' is a string with a description stating it is the ONS id of the survey, with a pattern constraint that must match '^[0-9]{3}$'. 'tx_id' is a string with a description stating it is the survey transaction identifier, with a pattern constraint that must match '^[a-z0-9]{8}-[a-z0-9]{4}-[a-z0-9]{4}-[a-z0-9]{4}-[a-z0-9]{12}$'. 'type' is a string with a pattern constraint that must match '^uk.gov.ons.edc.eq:feedback$'. 'version' is a string with a pattern constraint that must match '^0.0.1$'. No additional properties are allowed at the root level.
