The schema describes a calendar application configuration. It is an object type. The main properties of the schema are: 'calendar' is an object providing configuration for the calendar with an optional property 'parameter' which is a string describing the place parameter to use for selecting dates, with a default value of 'date'. 'events' is an object for which events to display in the calendar, requiring the property 'resource', which is a string describing the events resource to pull events from, with a default value of 'events'. It also has an optional property 'types', which is an array of strings denoting the types of events that should be listed, with a default value of an empty array. 'search' is an object for subscribing to search queries, requiring the property 'resource', which is a string describing the resource to subscribe to. It has an optional property 'fields', which is an array of items that can be strings or arrays representing JSON pointers to match object members. Additionally, it has an optional 'match' property, which is an object allowing a mapping from regular expressions to field lists with 'additionalProperties' being an array of similar JSON pointers. 'details' is an object for displaying details, containing optional properties 'resource', a string for the resource to publish details with, and 'action', a string describing which action to trigger when details are displayed. 'navigation' is an object for navigation targets, containing optional properties 'resource', a string with topic formatting, and 'interval', a string restricted to the valid values: 'years', 'months', 'weeks', 'days'. No additional properties are allowed at the root level.
