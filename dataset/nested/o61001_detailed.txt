The schema describes CVEcheck worker results. It is an object type. It defines several reusable components under 'definitions': 'cvecheck_details' is an object, detailing one CVE with required fields: 'cvss', 'description', 'severity', 'id', and 'references'. The 'cvss' field must reference the 'cvss' definition. The 'references' field is an array of strings, each formatted as a URI. The 'cvss' is an object with required fields: 'score' and 'vector'. The main properties of the schema are: 'schema' is an object describing the schema with required fields: 'name' and 'version', where 'name' must match a specific pattern and 'version' must follow a defined format. '_release' is an optional string describing a unique release id. 'status' is a string with an enum of allowed values: 'success' and 'error'. 'details' is an array of 'cvecheck_details'. 'summary' is an array of strings. There are schema-level constraints requiring 'status', 'details', and 'summary' to be present. No additional properties are allowed at the root level.
