The schema describes a response for a composite schedule. It is an object type. The main properties of the schema are: 'status' is a string that is required and must be one of the following values: 'Accepted', 'Rejected'. 'connectorId' is an integer that is optional. 'scheduleStart' is a string in date-time format that is optional. 'chargingSchedule' is an object that is optional. It contains the following properties: 'duration' is an integer that is optional. 'startSchedule' is a string in date-time format that is optional. 'chargingRateUnit' is a string that is required and must be one of the following values: 'A', 'W'. 'chargingSchedulePeriod' is an array where each item is an object that is required. Each object has the properties: 'startPeriod' is an integer that is required, 'limit' is a number that is required and must be a multiple of 0.1, and 'numberPhases' is an integer that is optional. 'minChargingRate' is a number that is optional and must be a multiple of 0.1. No additional properties are allowed at the root level.
