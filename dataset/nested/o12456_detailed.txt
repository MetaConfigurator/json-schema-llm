The schema describes module specification. It is an object type. It defines several reusable components under 'definitions': 'coordinates' is an object, requiring fields 'x' and 'y', with additional properties disallowed, and properties 'x', 'y', and 'z' as optional numbers. 'affineTransform' is an array, describing a 3x3 row-major matrix with a minimum and maximum of 3 items, each an array of 3 numbers. The main properties of the schema are: '$otSharedSchema' is a string, required, describing the path to a valid Opentrons shared schema relative to the shared-data directory; 'moduleType' is a string, required, with allowed values 'temperatureModuleType', 'magneticModuleType', 'thermocyclerModuleType'; 'model' is a string, required, matching the pattern '^(temperatureModule|magneticModule|thermocyclerModule)V[0-9]+$'; 'labwareOffset' is an object, required, referenced from the 'coordinates' definition; 'dimensions' is an object, required, with properties 'bareOverallHeight' and 'overLabwareHeight' required, among others; 'calibrationPoint' is an object, required, referenced from the 'coordinates' definition; 'displayName' is a string, required; 'quirks' is an array, optional, of strings; 'slotTransforms' is an object, required, with flexible properties adhering to a specific pattern and containing nested transforms potentially referencing 'affineTransform'; 'compatibleWith' is an array, optional, containing strings describing compatible module models. No additional properties are allowed at the root level.
