The schema describes a Network Partition Policy. It is an object type. The main properties of the schema are: '_id' is a string representing the Network Partition Id, required. 'partitions' is an array of objects, representing partitions, required. Each object contains '_id', a string representing the Partition Id, required, and 'property', an array of objects representing properties, optional. Each property object includes 'name', a string representing the name, required, and 'value', a string representing the value, required. 'properties' is an array of objects, representing properties, required. Each object contains 'name', a string representing the name, required, and 'value', a string representing the value, required. 'provider' is a string representing the Network Partition provider, required. Allowed values for 'provider' are: 'mock', 'ec2', 'openstack', 'kubernetes', 'gce'. No additional properties are allowed at the root level.
