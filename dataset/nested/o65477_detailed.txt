The schema describes the structure for subsidiary-related data. It is an object type. The main properties of the schema are: 'confidence' is a string that indicates confidence in accuracy of data and is optional, with allowed values 'HIGH', 'MEDIUM', 'LOW'. 'direct' is a boolean indicating whether the control relationship is direct or indirect, and is optional. 'end_date' is a string formatted as a date that represents the latest known date this was known to be a subsidiary, and is optional. 'parent_identifier' is a string that represents any unique ID for the controlling company, and is optional. 'parent_jurisdiction' is a string with a minimum length of 1 that specifies the jurisdiction where the controlling company is based, and is required. 'parent_name' is a string with a minimum length of 1 that denotes the name of the controlling company, and is required. 'percentage_controlled' is a string that indicates the percentage of the subsidiary that is controlled, following a specific pattern, and is optional. 'retrieved_at' is a string formatted as a date, representing when the data was retrieved, and is optional. 'sample_date' is a string formatted as a date that indicates the date on which a fact was known to be true, and is required. 'source_url' is a string with a minimum length of 1 indicating the place where this fact can be verified, and is required. 'start_date' is a string formatted as a date that identifies the earliest known date this was known to be a subsidiary, and is optional. 'subsidiary_identifier' is a string representing any unique ID for the subsidiary, and is optional. 'subsidiary_jurisdiction' is a string with a minimum length of 1 that indicates the jurisdiction of the subsidiary, and is required. 'subsidiary_name' is a string with a minimum length of 1 that denotes the name of the subsidiary, and is required. No additional properties are allowed at the root level.
