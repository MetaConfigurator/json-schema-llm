The schema describes a state machine. It is an object type. It defines several reusable components under 'definitions': 'state' is an object, which has required fields 'name' and 'type'. 'sub' is an object, which has required fields 'name', 'type', and 'states' (an array containing items that can be 'state', 'sub', or 'parallel'). 'parallel' is an object, which has required fields 'name', 'type', and 'regions' (an array of arrays containing items that can be 'state', 'sub', or 'parallel'). 'transition' is an object, which has required fields 'name', 'kind', 'type', 'source', and 'target'. 'compound_transition' is an object, which has required fields 'name', 'type', 'vertices' (an object with required fields 'incoming' and 'outgoing', both being arrays of objects with required properties), and 'pseudo_state' (an object with required fields 'name', 'type', and 'region'). The main properties of the schema are: 'name' is a string, optional; 'states' is an array, optional, containing items that can be 'state', 'sub', or 'parallel'; 'transitions' is an array, optional, containing items that can be 'transition' or 'compound_transition'. No additional properties are allowed at the root level.
