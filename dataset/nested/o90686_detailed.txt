The schema describes configuration settings for a schema registry application. It is an object type. The main properties of the schema are: 'registry' is an object containing configuration details. It is required and contains the following properties: 'cpus' is a number representing CPU shares to allocate to each schema-registry instance, with a default value of 1 and a minimum value of 1. 'heap' is a number for JVM heap allocation (in MB) for the connect worker task, with a default value of 256 and a minimum value of 256. 'instances' is an integer indicating the number of instances to run, with a default value of 1 and a maximum value of 1. 'kafkastore-connection-url' is a string for the Zookeeper Connect string for the service cluster, with a default value of 'master.mesos:2181/dcos-service-confluent-kafka'. 'mem' is a number for memory (MB) allocated to each schema-registry instance, with a default value of 512 and a minimum value of 320. 'name' is a string for identifying the schema-registry application, with a default value of 'schema-registry'. 'role' is a string to deploy the schema-registry only on nodes with this role, with a default value of '*'. 'vip-label' is a string for the label for external access to the schema-registry service, with a default value of '8.0.8.1:8081'. No additional properties are allowed at the root level.
