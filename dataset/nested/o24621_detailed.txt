The schema describes a yotta shrinkwrap file format. It is an object type. It defines several reusable components under 'definitions': 'module' is an object, representing a module with required fields 'name' and 'version'. 'name' is a string that must match the pattern '^[a-z]+[a-z0-9-]*$', with a minimum length of 2 and a maximum length of 64, and it cannot be one of the values: 'test', 'source', 'yotta_modules', 'yotta_targets', 'include'. 'version' is a string that must follow the 'semverSpecification' format. 'target' is a reference to 'module', effectively being the same structure. 'modulesArray' is an array of module references, and 'targetsArray' is an array of target references. The main properties of the schema are: 'modules' is an array of module references, which is required. 'targets' is an array of target references, which is required. No additional properties are allowed at the root level.
