The schema describes a Transitland Datastore Feed. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'onestopId' is a string required property formatted as 'feed-onestop-id'. 'newOnestopId' is a string optional property formatted as 'feed-onestop-id'. 'onestopIdsToMerge' is an array required property with a minimum of 1 item, consisting of unique strings formatted as 'feed-onestop-id'. 'geometry' is an object optional property. 'name' is a string optional property. 'tags' is an object optional property. 'url' is a string optional property. 'feedFormat' is a string optional property. 'licenseName' is a string optional property. 'licenseUrl' is a string optional property. 'licenseUseWithoutAttribution' is a string optional property. 'licenseAttributionText' is a string optional property. 'licenseCreateDerivedProduct' is a string optional property. 'licenseRedistribute' is a string optional property. 'includesOperators' is an array required property with a minimum of 1 item, consisting of objects that require 'operatorOnestopId' formatted as 'operator-onestop-id' and 'gtfsAgencyId' as string. 'doesNotIncludeOperators' is an array required property with a minimum of 1 item, consisting of objects that require 'operatorOnestopId' formatted as 'operator-onestop-id' and 'gtfsAgencyId' as string. No additional properties are allowed at the root level.
