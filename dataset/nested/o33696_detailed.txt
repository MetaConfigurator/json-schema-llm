The schema describes a PACTE Tokenizer example. It is an object type. The main properties of the schema are: '@context' is a string JSON-LD context. It is required. '@type' is a string JSON-LD type. It is required. '_corpusID' is a string Internal Corpus GUID. It is required. '_documentID' is a string Internal document GUID. It is required. 'category' is a string representing the tagset, with allowed values: 'CC', 'CD', 'DT', 'EX', 'FW', 'IN', 'JJ', 'JJR', 'JJS', 'LS', 'MD', 'NN', 'NNS', 'NNP', 'NNPS', 'PDT', 'POS', 'PRP', 'PRP$', 'RB', 'RBR', 'RBS', 'RP', 'SYM', 'TO', 'UH', 'VB', 'VBD', 'VBG', 'VBN', 'VBP', 'VBZ', 'WDT', 'WP', 'WP$', 'WRB'. It is required. 'gender' is a string. It is optional. 'kind' is a string. It is optional. 'lemma' is a string. It is optional. 'length' is an integer with a minimum value of 0. It is required. 'number' is a string. It is optional. 'offsets' is an array of objects, with each object containing 'begin' and 'end' properties which are integers with minimum value of 0. It has a minimum of 1 item and is required. 'orth' is a string with allowed values: 'upperInitial', 'allCaps', 'lowercase'. It is optional. 'schemaType' is a string Schema type. It is required. 'string' is a string. It is optional. No additional properties are allowed at the root level.
