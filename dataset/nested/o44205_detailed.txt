The schema describes a comment structure. It is an object type. The main properties of the schema are: 'timestamp' is a string the number of seconds since the Unix epoch. It is required, must be 10 characters long and match the pattern '[0-9]{10,10}'. 'author' is a string. It is required. 'original' is a string the SHA1 hash of another comment on the same revision, and it means this comment is an updated version of that comment. It is optional. 'parent' is a string the SHA1 hash of another comment on the same revision, and it means this comment is a reply to that comment. It is optional. 'location' is an object that contains: 'commit' as a string, 'path' as a string, and 'range' as an object that includes 'startLine', 'startColumn', 'endLine', and 'endColumn', all of which are integers. It is optional. 'description' is a string. It is optional. 'resolved' is a boolean. It is optional. 'v' is an integer with an enum value of 0. It is optional. No additional properties are allowed at the root level.
