The schema describes metadata for wed modes. It is an object type. It defines several reusable components under 'definitions': 'date' is a string, the date at which the file was generated, formatted as date-time. 'dochtml' is an object, configuration of the online HTML documentation for the XML schema, requiring 'method' and 'pattern' properties. 'elements' is an array of objects, representing all elements in the schema, each requiring 'name' and 'desc' properties. 'generator' is a string, the name of the software that generated the JSON. 'inline' is an object, determining which elements are inline, requiring 'method' and 'rules' properties. 'namespaces' is an object, mapping prefixes to absolute namespaces. 'version' is a string, recording the version number of the format of the metadata file. The main properties of the schema are: 'date' is a string, representing the date at which the file was generated; 'dochtml' is a string, a URL to where the schema's documentation is stored; 'elements' is an array of all elements in the schema; 'generator' is a string, the name of the software that generated the JSON; 'namespaces' is an object, prefix to uri mapping of absolute namespaces; 'version' is a string, allowed value is '1' or '2'. The schema requires either 'version' and 'namespaces' when version is '2', or just 'version' when it is '1'. No additional properties are allowed at the root level.
