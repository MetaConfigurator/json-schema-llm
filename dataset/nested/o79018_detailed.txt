The schema describes an AMQP 0-9-1 Operation Binding Object. It is an object type. The main properties of the schema are: 'expiration' is an integer that represents the TTL (Time-To-Live) for the message. It is optional and must be greater than or equal to zero. 'userId' is a string that identifies the user who has sent the message. It is optional. 'cc' is an array of strings that represent the routing keys the message should be routed to at the time of publishing. It is optional. 'priority' is an integer that serves as a priority for the message. It is optional. 'deliveryMode' is an integer that indicates the delivery mode of the message. Its value must be either 1 (transient) or 2 (persistent). It is optional. 'mandatory' is a boolean that indicates whether the message is mandatory or not. It is optional. 'bcc' is an array of strings, like cc, but consumers will not receive this information. It is optional. 'replyTo' is a string that represents the name of the queue where the consumer should send the response. It is optional. 'timestamp' is a boolean indicating whether the message should include a timestamp. It is optional. 'ack' is a boolean indicating whether the consumer should ack the message or not. It is optional. 'bindingVersion' is a string that indicates the version of this binding. It is optional, and if omitted, 'latest' must be assumed. Allowed values are '0.1.0', 'latest'. No additional properties are allowed at the root level.
