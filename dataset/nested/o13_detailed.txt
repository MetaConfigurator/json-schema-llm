The schema describes an array of station objects. It is an array type. The main properties of the schema are: Each object in the array contains the following properties: 'address' is a string representing the address of the station, required. 'addressNumber' is a string representing the address number, optional. 'bikes' is a string adhering to the pattern '^\d{1,2}$', representing the number of bikes available, required. 'id' is a string adhering to the pattern '^\d{1,3}$', serving as the unique identifier for the station, required. 'lat' is a string adhering to the pattern '^\d{1,3}\.\d{2,6}$', representing the latitude of the station, required. 'lon' is a string adhering to the pattern '^\d{1,3}\.\d{2,6}$', representing the longitude of the station, required. 'slots' is a string adhering to the pattern '^\d{1,2}$', representing the number of slots available, required. 'stationType' is a string with allowed values 'BIKE' and 'ELECTRIC_BIKE', representing the type of the station, required. 'status' is a string with allowed values 'OPN' and 'CLS', representing the status of the station, required. 'district' is a string representing the district of the station, required. 'zip' is a string representing the zip code of the station, required. 'name' is a string representing the name of the station, required. 'nearbyStations' is a string representing nearby stations, required. No additional properties are allowed at the root level.
