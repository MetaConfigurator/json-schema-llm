The schema describes a configuration object. It is an object type. The main properties of the schema are: 'opt' is an object that contains the properties 'id' (a string, required), 'name' (a string, required), and 'cmd' (a string, optional). Additional properties are not allowed for 'opt'. 'resource' is an object that contains the property 'assets' (another object, required). The 'assets' object contains the property 'uris' (an object, required), which in turn contains the property 'blob' (a string, required). Additional properties are not allowed for 'resource' or its nested properties. The overall schema requires both 'opt' and 'resource'. No additional properties are allowed at the root level.
