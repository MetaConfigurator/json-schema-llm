The schema describes a configuration object for hierarchical data structures. It is an object type. It defines several reusable components under 'definitions': 'hierarchy' is an object, representing a hierarchical structure with required fields 'name' (a string), 'path' (a string), and 'asPostfix' (a boolean). 'subType' is an object, representing a subtype with required fields 'name' (a string), 'superType' (can be a string or null), and 'properties' (a reference to a property definition). 'property' is an object with a required field 'name' (a string). The main properties of the schema are: '$schema' is a string, required; 'hierarchies' is an array of hierarchy objects, required. No additional properties are allowed at the root level.
