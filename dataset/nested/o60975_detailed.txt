The schema describes the result of Digester worker. It is an object type. It defines several reusable components under 'definitions': 'digester_details' is an object, details of Digester run on one file. It requires the fields 'path', 'ssdeep', 'md5', 'sha1', and 'sha256'. No additional properties are allowed in 'digester_details'. The main properties of the schema are: 'schema' is an object defining the schema's details. It requires 'name' and 'version', with 'name' matching the pattern '^[a-zA-Z0-9_]+$' and 'version' matching the pattern '^[0-9]+-[0-9]+-[0-9]+$'. Additional properties are not allowed in 'schema'. '_release' is an optional string, description is a unique release id in the form of 'ecosystem:package:version'. 'status' is a string, it is required and can be either 'success' or 'error'. 'details' is an array of 'digester_details', which specifies the digester run details. 'summary' is an array of strings. 'status', 'details', and 'summary' are required fields. No additional properties are allowed at the root level.
