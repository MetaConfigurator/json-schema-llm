The schema describes configuration settings for a Linkerd instance. It is an object type. The main properties of the schema are: 'linkerd' is an object that specifies various settings for the Linkerd instance. It is required and consists of the following properties: 'admin-port' is an integer that provides an administrative UI for this instance, with a default value of 9990. 'cpus' is a number that specifies CPU shares to allocate to each Linkerd instance, with a minimum value of 0.25 and a default value of 0.25. 'instances' is an integer that indicates the number of instances to run, with a minimum value of 1 and a default value of 1. 'internal-port' is an integer that represents the internal routing port, with a default value of 4141. 'marathon-host' is a string that denotes the DC/OS leader hostname for Marathon API access, with a default value of 'leader.mesos'. 'marathon-port' is an integer for the DC/OS leader port for Marathon API access, with a default value of 8080. 'marathon-uri-prefix' is a string indicating the Marathon API prefix, typically left blank or set to '/marathon', with a default value of an empty string. 'mem' is a number that indicates the memory (MB) to allocate to each Linkerd task, with a minimum value of 256.0 and a default value of 256.0. 'resource-roles' is a string that specifies the accepted resource roles, with a default value of '"*", "slave_public"'. 'routing-port' is an integer representing the routing port, with a default value of 4140. 'secret_name' is a string that indicates the name of the Secret Store credentials for DC/OS service authentication, with a default value of an empty string. No additional properties are allowed at the root level.
