The schema describes an object structure for a configuration. It is an object type. It defines several reusable components under 'definitions': 'nonEmptyString' is a string, it must have a minimum length of 1. 'multiString' is an anyOf type, it can be either a non-empty string or an array of unique non-empty strings. The main properties of the schema are: 'name' is a non-empty string and required. 'description' is a string and optional. 'namespace' is a non-empty string and required. 'version' is a non-empty string and required. 'downloadURL' is a non-empty string and optional. 'updateURL' is a non-empty string and optional. 'icon' is a non-empty string and optional. 'include' is a multiString and optional. 'exclude' is a multiString and optional. 'match' is a multiString and optional. 'require' is a multiString and optional. 'resource' is a multiString and optional. 'grant' is an anyOf type, it can be an array of unique strings from the enumerated values ['unsafeWindow', 'GM_getValue', 'GM_setValue', 'GM_listValues', 'GM_deleteValue', 'GM_getResourceText', 'GM_getResourceURL', 'GM_addStyle', 'GM_log', 'GM_openInTab', 'GM_registerMenuCommand', 'GM_setClipboard', 'GM_xmlhttpRequest'] or the string 'none'. It has a default value of 'none'. 'run-at' is a string, it can be one of 'document-end', 'document-start', 'document-idle' with a default value of 'document-end'. 'noframes' is a boolean with a default value of false. No additional properties are allowed at the root level.
