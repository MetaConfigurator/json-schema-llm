The schema describes a configuration for an organization including its blocks. It is an object type. The main properties of the schema are: 'orgId' is a string that must match the pattern '^[A-Fa-f\d]{24}$', optional. 'name' is a string with a minimum length of 1 and a maximum length of 255, required. 'blocks' is an array containing objects, where each object has required properties: 'id' is a string, 'blockType' is a string, 'startX' is an integer, 'startY' is an integer, 'width' is an integer, and 'height' is an integer. Each block object may also have optional properties: 'title' is a string with a maximum length of 255, and 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$'. No additional properties are allowed at the root level.
