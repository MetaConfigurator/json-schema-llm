The schema describes configuration options for NgRx Data. It is an object type. The main properties of the schema are: 'effect' is a boolean indicating when false use the EntityDataModuleWithoutEffects module instead of EntityDataModule, with a default value of true. 'entityConfig' is a boolean indicating whether to create the Entity config file, with a default value of true. 'migrateNgrxData' is a boolean for migrating from ngrx-data, which will rename modules, with a default value of false and an alias of 'migrate'. 'module' is a string allowing specification of the declaring module, with a default value of 'app' and an alias of 'm'. 'path' is a string indicating the path to the module, formatted as a path, and it is not visible. 'project' is a string representing the name of the project, with an alias of 'p'. 'skipPackageJson' is a boolean indicating not to add @ngrx/data as a dependency to package.json, with a default value of false. No additional properties are allowed at the root level.
