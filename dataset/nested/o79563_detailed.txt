The schema describes details for a previously uploaded file. It is an object type. It defines several reusable components under 'definitions': 'data' is an object, it contains properties that specify the media encoding as base64 and has a type of string. 'fileName' is a string, it defines the expected format for file names. 'mediaType' is a string, it describes allowed values for media types, including 'application/pdf', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document', 'application/msword', 'application/rtf', 'text/plain', 'image/png', 'image/jpeg', 'image/gif', 'text/html', 'audio/x-mpeg-3', 'video/x-msvideo', and 'video/mp4'. 'via' is a string, it specifies the mechanism by which the file was uploaded, which can either be 'device' or a fully qualified domain name of a cloud storage platform (e.g. dropbox.com). The main properties of the schema are: 'dateUploaded' is a string, it describes the date and time the file was uploaded, and it is optional. 'fileName' is a string that resolves to the 'fileName' definition. It is required. 'md5' is a string, it describes the hash of the file's binary content, and it is optional. 'mediaType' is a string that resolves to the 'mediaType' definition. It is required. 'preSignedUri' is a string, it describes a pre-signed link for accessing the raw data; it is required. 'size' is an integer, it specifies the size of the file, and it is required. No additional properties are allowed at the root level.
