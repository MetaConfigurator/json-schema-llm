The schema describes the result of the PackageKeywordsTagging worker. It is an object type. It defines several reusable components under 'definitions': 'package_keywords_tagging_details' is an object, details computed by PackageKeywordsTagging worker. It contains properties: 'README' is an object, 'package_name' is an object with additional properties allowed, 'repository_description' is an object with additional properties allowed, and 'gh_topics' is an array of strings. No additional properties are allowed at the root level. The main properties of the schema are: 'schema' is an object with properties 'name', 'version', and 'url', where 'name' is a string that must match the pattern '^[a-zA-Z0-9_]+$', 'version' is a string that must match the pattern '^[0-9]+-[0-9]+-[0-9]+$', and 'url' is a string in URI format. 'schema' requires 'name' and 'version'. '_release' is an optional string, 'status' is a required string with an enum of allowed values: 'success', 'error'. 'details' references the 'package_keywords_tagging_details' definition. 'summary' is an array of strings. Additional properties are not allowed at the root level.
