The schema describes an unsubscribe operation. It is an object type. The main properties of the schema are: 'email' is a string representing the user's email, required. It must follow an email format. 'list_code' is a string that is optional. 'list_id' is an integer that is optional. 'user_id' is an integer representing the user's ID, required. 'utm_params' is an object that is optional and may contain properties 'utm_campaign', 'utm_content', 'utm_medium', and 'utm_source', each being a string. 'variant_id' is either an integer or null and is optional. There are compositional constraints stating that the schema allows any of the configurations where either 'list_id' or 'list_code' is required. No additional properties are allowed at the root level.
