The schema describes configuration settings for a schema registry. It is an object type. The main properties of the schema are: 'registry' is an object containing configuration options for the schema registry service. It is required and has the following properties: 'access-control-allow-methods' is a string that sets the value to Jetty Access-Control-Allow-Origin header for specified methods, with a default value of an empty string. 'access-control-allow-origin' is a string that sets the value for Jetty Access-Control-Allow-Origin header, with a default value of an empty string. 'cpus' is a number that specifies CPU shares to allocate to each schema-registry instance, with a default value of 1 and a minimum of 1. 'heap' is a number that indicates JVM heap allocation (in MB) for connect worker task; it should be no greater than ~64MB less than total memory for the instance, with a default value of 256 and a minimum of 256. 'instances' is an integer that denotes the number of instances to run (currently limited to 1), with a default value of 1 and a maximum of 1. 'kafkastore' is a string that names the Kafka service hosting the storage for this Schema Registry edition, with a default value of 'dcos-service-confluent-kafka'. 'mem' is a number that represents memory (MB) to allocate to each schema-registry instance, with a default value of 512 and a minimum of 320. 'name' is a string that specifies the service name for the schema-registry instance(s), with a default value of 'schema-registry'. 'role' is a string that allows deployment of schema-registry only on nodes with this role, with a default value of '*'. 'zookeeper-master' is a string that indicates the Zookeeper Connect string for the service cluster, with a default value of 'master.mesos:2181'. No additional properties are allowed at the root level.
