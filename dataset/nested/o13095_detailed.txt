The schema describes a configuration file for qcodes. It is an object type. The main properties of the schema are: 'user' is an object that controls user settings of qcodes. It is required. It includes properties: 'analysis_location' is a string with a default value of 'A:/TransmonExperiments/GoogleDrive/T2acquisition/Data/', 'data_location' is a string with a default value of 'A:/TransmonExperiments/GoogleDrive/T2acquisition/Data/', 'log_location' is a string with a default value of 'A:/TransmonExperiments/GoogleDrive/T2acquisition/Data/', 'temp_dict_location' is a string with a default value of '/', 'data_format' is a string with a default value of '{counter}', 'pulse_location' is a string with a default value of 'A:/TransmonExperiments/GoogleDrive/T2acquisition/Data/'. 'gui' is an object that controls the GUI of qcodes. It is required. It includes properties: 'notebook' is a boolean with a default value of true, 'plotlib' is a string with an enum of allowed values: 'QT', 'matplotlib', and a default value of 'matplotlib'. 'core' is an object that controls core settings of qcodes. It is required. It includes properties: 'loglevel' is a string with an enum of allowed values: 'CRITICAL', 'ERROR', 'WARNING', 'INFO', 'DEBUG', and a default value of 'DEBUG'; 'default_fmt' is a string with a default value of 'data/{date}/#{counter}_{name}_{time}'. Compositional constraints require all properties 'gui', 'core', and 'user' to be present. No additional properties are allowed at the root level.
