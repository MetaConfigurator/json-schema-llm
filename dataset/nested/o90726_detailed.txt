The schema describes a configuration for linkerd instances. It is an object type. The main properties of the schema are: 'linkerd' is an object that contains various properties for configuring linkerd. It is required. The properties within 'linkerd' are: 'admin-port' is an integer that provides an administrative UI for this instance. It is required and has a default value of 9990. 'config-filename' is a string that specifies the URI of the linkerd config file and has a default value of 'linkerd-dcos-060.yaml'. It is required. 'config-uri' is a string that indicates the URI of the linkerd config file and has a default value of 'https://s3.amazonaws.com/buoyant-dcos'. It is required. 'cpus' is a number that specifies the CPU shares to allocate to each linkerd instance. It has a minimum value of 0.25 and a default value of 0.25. It is required. 'instances' is an integer that denotes the number of instances to run, with a minimum value of 1 and a default value of 1. It is required. 'mem' is a number that indicates the memory (MB) to allocate to each linkerd task, with a minimum value of 256.0 and a default value of 256.0. It is required. 'name' is a string that provides a name for the linkerd instance(s), with a default value of 'linkerd'. It is required. 'routing-port' is an integer that specifies the routing port, with a default value of 4140. It is required. No additional properties are allowed at the root level.
