The schema describes a structured logging event. It is an object type. The main properties of the schema are: 'level' is a string that is required and must be one of the following values: 'info', 'warning', 'error', 'critical'. 'state' is a string that is optional and must be one of the following values: 'new', 'acknowledged', 'resolved'. 'subject' is a string that is required and must have a minimum length of 1 and a maximum length of 255. 'message' is a string that is optional and must have a maximum length of 32767. 'data' is an object that is optional. 'deviceId' is a string that is optional and must match the pattern '^[A-Fa-f\d]{24}$'. 'eventTags' is an object that is optional and must have property names that match the pattern '^[0-9a-zA-Z_-]{1,255}$', where each value must be a string with a minimum length of 1 and a maximum length of 255. No additional properties are allowed at the root level.
