The schema describes the inventory of tax notes. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'estoqueNotasFiscais' is an array containing objects, each with the properties: 'numero' is a string, 'urf' is a string, 'recinto' is a string, 'latitude' is a string, 'longitude' is a string, 'registro' is a string, 'responsavel' is a string, and 'itens' is an array containing objects with properties: 'item' is a number and 'saldo' is a number; 'mensagens' is an array containing objects with properties: 'codigo' is a string and 'mensagem' is a string. No additional properties are allowed at the root level.
