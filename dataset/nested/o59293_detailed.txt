The schema describes configuration settings for synchronization. It is an object type. It defines several reusable components under 'definitions': 'localTarget' is an object, which requires a 'path' property and can include optional properties: 'files' (default '**/**'), 'exclude' (array of strings, default empty), 'defaultTarget' (string), and 'deleteRemoteFiles' (boolean, default false). 'remoteTarget' is an object, which requires 'host', 'path', and 'user' properties and can include optional properties: 'pass' (string), 'port' (number, default 21), 'cache' (string, default '.synco-filelist'), 'prompt' (string or boolean, default true), 'enabled' (boolean, default true), and 'type' (string). The main properties of the schema are: 'local' is a object referencing the 'localTarget' definition (required), 'targets' is an object with additional properties referencing the 'remoteTarget' definition (required). No additional properties are allowed at the root level.
