The schema describes a configuration for specifying geographical points and a radius. It is an object type. It defines several reusable components under 'definitions': '_gpsCheck' is a string, with a minimum length of 1. '_latCheck' is a string, with a minimum length of 1. '_lonCheck' is a string, with a minimum length of 1. '_gpsCenter' is a string, with a minimum length of 1. '_latCenter' is a string, with a minimum length of 1. '_lonCenter' is a string, with a minimum length of 1. 'radius' is a string. 'resultPath' is a string, with a description stating it's a JSON Path for where to put the distance between the two coordinates. The main properties of the schema are: 'gpsCheck' is a string which is required. 'gpsCenter' is a string which is required. 'radius' is a string which is required. 'resultPath' is a string which is optional. 'latCheck' is a string which is required. 'lonCheck' is a string which is required. 'latCenter' is a string which is required. 'lonCenter' is a string which is required. Compositional constraints specify that any valid configuration must meet one of the four defined schemas, each of which requires 'radius' along with either 'gpsCheck', 'gpsCenter', or both 'latCenter' and 'lonCenter' along with 'latCheck' and 'lonCheck'.
