The schema describes supplies the meta data necessary to interpret N-dimensional image data returned by DVID. It is an object type. It defines several reusable components under 'definitions': 'axis' is an object, defines an axis of the image with required fields: 'Label', 'Resolution', 'Units', 'Size'. 'DataType' is an enum with allowed values: 'uint8', 'int8', 'uint16', 'int16', 'uint32', 'int32', 'uint64', 'int64', 'float32', 'float64'. 'Label' is an optional string. 'Units' is an enum with allowed values: 'nanometers', 'micrometers', 'millimeters', ''. 'Resolution' is a number with a minimum of 0. 'Size' is an integer with a minimum of 0. The main properties of the schema are: 'Axes' is an array, required, with a minimum of 1 item that references the 'axis' definition and must have unique items. 'Properties' is an object, required, containing the property 'Values', which is an array, required, with a minimum of 1 item that references the 'channel' definition. 'version' is an optional string. No additional properties are allowed at the root level.
