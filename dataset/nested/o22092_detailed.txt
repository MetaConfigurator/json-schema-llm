The schema describes the configuration options for a prerender target. It is an object type. The main properties of the schema are: 'browserTarget' is a string that specifies the target to build. It is required and must match the pattern '^[^:\s]+:[^:\s]+(:[^\s]+)?$'. 'serverTarget' is a string that specifies the server target to use for prerendering the app. It is required and must also match the pattern '^[^:\s]+:[^:\s]+(:[^\s]+)?$'. 'routesFile' is a string that indicates the path to a file containing routes separated by newlines. It is optional. 'routes' is an array that represents the routes to render. It is optional, requires at least one unique string, and has a default value of an empty array. 'guessRoutes' is a boolean that determines whether the builder should extract routes and guess which paths to render. It has a default value of true and is optional. 'numProcesses' is a number that specifies the number of CPUs to use, with a minimum value of 1. It is optional. The schema has two valid configurations: either 'routes' is required or 'routesFile' is required. No additional properties are allowed at the root level.
