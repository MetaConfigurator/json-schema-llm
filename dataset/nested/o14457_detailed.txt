The schema describes an action target template. It is an object type. The main properties of the schema are: 'name' is a string that identifies the action target template and is required. 'organizationId' is an integer that links the event source template and is required. 'public' is a boolean that indicates if the action target template is available to everybody; it is optional. 'configuration' is an object providing a schema to assist the WUI for configuration and is optional. It contains the properties: 'schema', which is an object and required, and 'url', which is a string and required. 'configurationUi' is an object that provides the configuration for the dynamic UI; it is optional and contains the property 'schemaForm', which is an array. 'target' is an object that represents the target configuration and is required. It contains the property 'url', which is a string and required, and 'token', which is a string for authentication; it is optional. Compositional constraints include that 'configuration' must have both 'schema' and 'url' as required fields, and 'target' must have 'url' as a required field. No additional properties are allowed at the root level.
