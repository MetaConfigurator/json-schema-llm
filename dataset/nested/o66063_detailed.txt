The schema describes a shipping address. It is an object type. The main properties of the schema are: 'recipient_name' is a string that represents the name of the recipient at this address, with a maximum of 50 characters. It is required. 'type' is a string that indicates the address type, which must be one of the following: 'residential', 'business', or 'mailbox'. It is optional. 'line1' is a string that represents Line 1 of the address (e.g., Number, street, etc), with a maximum of 100 characters. It is required. 'line2' is a string that represents Line 2 of the address (e.g., Suite, apt #, etc), with a maximum of 100 characters. It is optional. 'city' is a string that represents the city name, with a maximum of 50 characters. It is required. 'country_code' is a string that represents the 2-letter country code, with a maximum and minimum of 2 characters. It is required. 'postal_code' is a string that represents the zip code or equivalent, usually required for countries that have them, with a maximum of 20 characters. It is optional. 'state' is a string that represents the 2-letter code for US states and the equivalent for other countries, with a maximum of 100 characters. It is optional. 'phone' is a string that represents the phone number in E.123 format, with a maximum of 50 characters. It is optional. No additional properties are allowed at the root level.
