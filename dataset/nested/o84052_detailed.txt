The schema describes the configuration for deploying an application. It is an object type. The main properties of the schema are: 'metrics' is an object that contains details about Prometheus metrics, 'persistence' is an object that configures persistence for the application, 'rabbitmq' is an object describing RabbitMQ connection settings, 'replicas' is an integer representing the number of replicas to deploy, and 'volumePermissions' is an object that manages volume permissions. 'metrics.enabled' is a boolean that creates a side-car container to expose Prometheus metrics. 'metrics.serviceMonitor' is an object with the property 'enabled', a boolean that creates a ServiceMonitor to track metrics using Prometheus Operator. 'persistence.enabled' is a boolean that enables persistence using Persistent Volume Claims. 'persistence.size' is a string, which has a slider range from 1 to 100 Gi. 'rabbitmq.extraConfiguration' is a string for extra RabbitMQ configurations. 'rabbitmq.password' is a string that defaults to a random 10-character alphanumeric string if not set. 'rabbitmq.username' is a string for the RabbitMQ user. 'replicas' is a required integer for the number of replicas. 'volumePermissions.enabled' is a boolean to use an init container to set required folder permissions. No additional properties are allowed at the root level.
