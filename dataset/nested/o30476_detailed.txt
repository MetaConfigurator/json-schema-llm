The schema describes a binding for a load balancer monitor service. It is an object type. The main properties of the schema are: 'failedprobes' is an integer, readonly. 'lastresponse' is a string, readonly. 'monitor_name' is a string. 'monitor_state' is a string, readonly, with allowed values: 'UP', 'DOWN', 'UNKNOWN', 'BUSY', 'OUT OF SERVICE', 'GOING OUT OF SERVICE', 'DOWN WHEN GOING OUT OF SERVICE', 'NS_EMPTY_STR', 'Unknown', 'DISABLED'. 'monitorcurrentfailedprobes' is an integer, readonly. 'monitortotalfailedprobes' is an integer, readonly. 'monitortotalprobes' is an integer, readonly. 'monstatcode' is an integer, readonly. 'monstate' is a string with allowed values: 'ENABLED', 'DISABLED'. 'monstatparam1' is an integer, readonly. 'monstatparam2' is an integer, readonly. 'monstatparam3' is an integer, readonly. 'responsetime' is an integer, readonly. 'servicename' is a string. 'totalfailedprobes' is an integer, readonly. 'weight' is an integer. No additional properties are allowed at the root level.
