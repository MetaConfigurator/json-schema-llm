The schema describes the structure of a configuration object for a page layout. It is an object type. The main properties of the schema are: 'layout' is a string that describes the layout to use. It is optional and may be omitted if another page in the extension hierarchy defines one. 'extends' is a string that describes the name of the page to extend. It is optional. 'features' is an object that describes the available configuration features for compositions. It is optional and allows additional properties. 'areas' is an object that provides a map from area name to a list of widgets to display within that area. It is optional and does not allow additional properties. The properties of the items in the 'areas' array are: 'widget' is a string that describes the path to the widget that should be rendered. 'composition' is a string that describes the path to the composition that should be included. 'layout' is a string that describes the path to the layout that should be inserted. 'id' is a string that must match the pattern '^[a-z][a-zA-Z0-9_-]*$' and describes the ID of the widget or composition. It will be generated if missing. 'insertBeforeId' is a string that describes the ID of the widget this widget or composition should be inserted before. 'features' is an object that describes the configuration of the features defined by the widget or composition. 'enabled' is a boolean that has a default value of true, indicating whether to omit widgets for debugging purposes. 'comment' is a string that allows the page author to comment on this widget instance. No additional properties are allowed at the root level.
