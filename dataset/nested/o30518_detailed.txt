The schema describes an IPsec profile. It is an object type. The main properties of the schema are: 'builtin' is an object that defines the status of the profile, with values restricted to 'MODIFIABLE', 'DELETABLE', 'IMMUTABLE', 'PARTITION_ALL' and is read-only. 'encalgo' is an object representing the encryption algorithm with allowed values 'AES', '3DES', 'AES192', 'AES256'. 'feature' is a string that specifies the features available in the profile, with a long list of allowed values such as 'WL', 'WebLogging', 'LB', and many more, and is read-only. 'hashalgo' is an object that defines the hashing algorithm with values restricted to 'HMAC_SHA1', 'HMAC_SHA256', 'HMAC_SHA384', 'HMAC_SHA512', 'HMAC_MD5'. 'ikeretryinterval' is an integer representing the retry interval for IKE. 'ikeversion' is a string that can take the values 'V1' or 'V2'. 'lifetime' is an integer that specifies the lifetime of the profile. 'livenesscheckinterval' is an integer indicating the interval for liveness checks. 'name' is a string that represents the name of the profile. 'peerpublickey' is a string that holds the public key of the peer. 'perfectforwardsecrecy' is a string that can either be 'ENABLE' or 'DISABLE'. 'privatekey' is a string representing the private key. 'psk' is a string for the pre-shared key. 'publickey' is a string for the public key. 'replaywindowsize' is an integer that specifies the size of the replay window. 'responderonly' is a string that can be 'YES' or 'NO' and is read-only. 'retransmissiontime' is an integer that indicates the time for retransmission. No additional properties are allowed at the root level.
