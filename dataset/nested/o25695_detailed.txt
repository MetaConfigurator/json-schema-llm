The schema describes a structure for capturing changes to files and links. It is an object type. It defines several reusable components under 'definitions': 'file_changed' is an object type, requiring the field 'status' to be equal to 'changed' and validating against either 'file_changed_modified' or 'link_changed_modified'. 'file_changed_modified' is an object type, requiring the field 'changes' (an array with at least one item from the enum: 'size', 'mode', 'md5', 'device_number', 'link_path', 'user', 'group', 'time', 'capabilities', 'replaced', 'other_rpm_changes', 'deleted'), along with the optional fields 'mode' (a string matching the pattern '^[0-7]{3,4}$'), 'user' (a non-empty string), 'group' (a non-empty string), and 'type' (either 'file' or 'dir'). 'link_changed_modified' is an object type that requires 'target', 'changes' (an array with at least one item from the enum: 'size', 'mode', 'md5', 'device_number', 'link_path', 'user', 'group', 'time', 'capabilities', 'replaced', 'other_rpm_changes'), 'mode' (a string matching the pattern '^[0-7]{3,4}$'), 'user' (a non-empty string), 'group' (a non-empty string), and 'type' (equal to 'link'). 'file_error' is an object type, requiring both 'status' (equal to 'error') and 'error_message' (a string). The main properties of the schema are: '_attributes' is an object type, required, with a required field 'extracted' (a boolean). '_elements' is an array type, required, containing objects that must have 'name', 'package_name' (a non-empty string), and 'package_version' (a non-empty string), and validate against oneOf with either 'file_changed' or 'file_error' references. No additional properties are allowed at the root level.
