The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'engine' is an object that represents the engine configuration. It is required and must contain 'timeout', 'dsps', and 'protocol'. 'timeout' is an integer representing the timeout duration and is required. 'dsps' is an array of objects, each containing 'bid_host', 'bid_port', 'bid_path', and 'id', all of which are required string and integer fields. 'protocol' is a string that specifies the protocol and is required. 'log_level' is a string that indicates the logging level, and it must be one of the following values: 'info', 'error', 'verbose', 'debug'. 'port' is an integer representing the port number and is optional. 'mongolog' is an object that represents the MongoDB logging configuration and is optional. No additional properties are allowed at the root level.
