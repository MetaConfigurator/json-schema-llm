The schema describes the parameters of the add_disk command. It is an object type. The main properties of the schema are: 'address' is a string that must match the pattern '^\d+(:\d+){1,3}$', optional. 'boot' is a boolean, optional. 'bus_address' is a string, optional. 'comments' is a string, optional. 'controller' is a string with an enum of allowed values: 'sata', 'sas', 'scsi', 'cciss', 'flash', 'fibrechannel', 'ide', 'nvme', required. 'filesystem' is a string that describes the filesystem name where the virtual disk image is stored, optional. 'iops_limit' is an integer with a minimum value of 0, optional. 'resourcegroup' is a string that describes the name of the resourcegroup which holds the share or filesystem, optional. 'share' is a string that describes the NAS share where the virtual disk image is stored, optional. 'size' is an integer that describes the disk size, in GiB, with a minimum value of 0, required. 'snapshot' is a boolean, optional. 'wwn' is a string, optional. No additional properties are allowed at the root level.
