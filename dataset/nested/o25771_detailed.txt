The schema describes a file structure. It is an object type. It defines several reusable components under 'definitions': 'file_common' is an object, it requires 'name' and 'type'. 'file' is an object, it requires 'name' and 'type', and has additional optional properties 'user' (a string with minLength 1) and 'group' (a string with minLength 1), and must conform to one of the 'file_file', 'file_dir', or 'file_link' definitions. 'file_file' is an object, it requires 'type' with the enum value 'file', an optional 'size' (an integer, minimum 0), and an optional 'mode' (a string matching the pattern '^[0-7]{3,4}$'). 'file_dir' is an object, it requires 'type' with the enum value 'dir', an optional 'size' (an integer, minimum 0), an optional 'mode' (a string matching the pattern '^[0-4]?[0-7]{3}$'), and an optional 'files' (an integer, minimum 0). 'file_link' is an object, it requires 'type' with the enum value 'link'. 'file_remote_dir' is an object, it requires 'name' and 'type' and must have 'type' with the enum value 'remote_dir'. The main properties of the schema are: '_attributes' is an object, it is required, and it requires 'extracted' (a boolean) and 'has_metadata' (a boolean). '_elements' is an array that is required, it can contain items that conform to either the 'file' or 'file_remote_dir' definitions. No additional properties are allowed at the root level.
