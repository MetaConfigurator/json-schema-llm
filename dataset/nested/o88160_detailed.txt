The schema describes a submission object structure. It is an object type. The main properties of the schema are: '_schema' is a string that must match the pattern '^pbj:([a-z0-9-]+):([a-z0-9\.-]+):([a-z0-9-]+)?:([a-z0-9-]+):([0-9]+-[0-9]+-[0-9]+)$' and defaults to 'pbj:eme:solicits:edge:submissions:1-0-0'. 'from_ref' is a string that must match the pattern '^[\w\/\.:-]+$', and it is required. 'to_ref' is a string that must match the pattern '^[\w\/\.:-]+$', and it is required. 'multiplicity' is a string that has a default value of 'one2many' and is limited to the enum values: 'unknown', 'multi', 'simple', 'many2one', 'one2many', 'one2one'. 'created_at' is a string that must match the pattern '^[1-9]{1}[0-9]{12,15}$'. No additional properties are allowed at the root level.
