The schema describes an invocation manifest for Nobrainer. It is an object type. The main properties of the schema are: 'config' is an object that defines configuration settings, required fields are 'largest-label', 'threshold', and 'rotate-and-predict'. 'largest-label' is a boolean with a default value of false. 'threshold' is a number with a default value of 0.3, a minimum of 0, and a maximum of 1. 'rotate-and-predict' is a boolean with a default value of false. 'inputs' is an object that defines input settings, with 'T1W' being a required property. 'T1W' is an object that contains a property 'type', which is an enum with the allowed value 'nifti'. 'model' is an optional object with no defined properties. The root schema requires 'config' and 'inputs'. No additional properties are allowed at the root level.
