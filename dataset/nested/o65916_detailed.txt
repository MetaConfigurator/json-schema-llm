The schema describes a file extraction report. It is an object type. It defines several reusable components under 'definitions': 'file_changed' is an object, specifies a required field 'status' with an enum that allows only the value 'changed'. It has a oneOf constraint for definitions 'file_changed_modified' and 'file_changed_deleted'. 'file_changed_modified' is an object, requires fields 'changes', 'mode', 'user', and 'group'. The 'changes' field is an array with a minimum of 1 item, each item being an enum that allows specific change types. The 'mode' field is a string that must match the regex pattern '^[0-7]{3,4}$'. The 'user' and 'group' fields are strings with a minimum length of 1. 'file_changed_deleted' is an object, requires a field 'changes' with an enum that strictly allows [['deleted']]. 'file_error' is an object, requires fields 'status' and 'error_message'. The 'status' field is an enum that allows only the value 'error', while 'error_message' is a string. The main properties of the schema are: 'extracted' is a boolean, required. 'files' is an array of objects, each required to have 'name', 'package_name', and 'package_version'. The 'package_name' and 'package_version' are strings with a minimum length of 1. Each object in the 'files' array must conform to oneOf the definitions 'file_changed' or 'file_error'. No additional properties are allowed at the root level.
