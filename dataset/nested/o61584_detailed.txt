The schema describes configuration settings for FileVault. It is an object type. The main properties of the schema are: 'Certificate' is a string that holds DER-encoded certificate data if an institutional recovery key will be added. It is optional. 'Defer' is a boolean that indicates whether to defer enabling FileVault until the designated user logs out. It is optional. 'DeferDontAskAtUserLogout' is a number that specifies whether to not request enabling FileVault at user logout time when using the Defer option. It is optional. 'DeferForceAtUserLoginMaxBypassAttempts' is a number that sets the maximum number of times a user can bypass enabling FileVault when using the Defer option. It is optional. 'Enable' is a boolean that is required to enable (set to 'On') or disable (set to 'Off') FileVault. 'OutputPath' is a string that specifies the location where the recovery key and computer information plist will be stored. It is optional. 'Password' is a string representing the user password for the Open Directory user that will be added to FileVault. It is optional. 'PayloadCertificateUUID' is a string that holds the UUID of the payload containing the asymmetric recovery key certificate payload. It is optional. 'PayloadDescription' is a string describing the payload, with a default value of 'FileVault 2'. 'PayloadDisplayName' is a string representing the name of the payload, with a default value of 'FileVault 2'. 'PayloadIdentifier' is a string that serves as a unique identifier for the payload, with a default value of 'com.apple.MCX.FileVault2'. 'PayloadOrganization' is a string that describes the issuing organization of the profile as displayed to the user. It is optional. 'PayloadType' is a string representing the type of the payload, with a default value of 'com.apple.MCX.FileVault2'. 'PayloadUUID' is a string that holds a unique identifier for the payload, following a specific pattern, with a default value of an empty string. 'PayloadVersion' is a number representing the version of the configuration profile, with a default value of 1. 'ShowRecoveryKey' is a boolean that controls whether to display the personal recovery key after enabling FileVault, with a default value of true. 'UseKeychain' is a boolean that indicates whether to use the existing keychain if no certificate information is provided, and it is optional. 'UseRecoveryKey' is a boolean that is set to true to create a personal recovery key, with a default value of true. 'UserEntersMissingInfo' is a boolean that, when set to true, prompts for missing user name or password fields during manual profile installs. It is optional. 'Username' is a string indicating the user name of the Open Directory user that will be added to FileVault. It is optional. No additional properties are allowed at the root level.
