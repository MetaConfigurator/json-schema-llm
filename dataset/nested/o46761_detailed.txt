The schema describes configuration settings for a system. It is an object type. It defines several reusable components under 'definitions': 'crawlers' is an object, which requires the properties 'enabled' for both 'twitter' and 'facebook'. 'db' is an object, which requires the properties 'db_name' (a string) and 'db_tables' (an object containing at least one property with keys of string type and values from a predefined enum). 'dbs' is an object, which requires the properties: 'dbs_addr' (a string with a format of either hostname, ipv4, or ipv6), 'dbs_port' (an integer between 0 and 65535), 'dbs_login' (a string with a minimum length of 1 and a maximum length of 32), 'dbs_pass' (a string with no minimum length), 'dbs_unicode' (a boolean), and 'dbs_charset' (a string that must be either 'utf8' or 'utf16'). The main properties of the schema are: 'ntm-config' is an object that requires the properties 'crawlers', 'db', and 'dbs', each referring to their respective definitions. No additional properties are allowed at the root level.
