The schema describes a configuration for a task execution. It is an object type. It defines several reusable components under 'definitions': 'pathpair' is an object, representing a pair of paths with required fields 'cloud' and 'local'. The 'cloud' field must match a specific pattern that includes 'gs://', 'http://', or 'https://'. The 'local' field must match a pattern starting with '/'. 'bytesunit' is a string, representing a unit of bytes with a default value of 'MB'. It must match one of the following values: 'BYTES', 'KB', 'MB', 'GB', or 'TB'. The main properties of the schema are: 'name' is a string with a default value of an empty string. 'inputs' is an array of items, each referencing the 'pathpair' definition. 'commandline' is a string. 'outputs' is an array of items, each referencing the 'pathpair' definition. 'stdout' is a string, required to match the pattern '^gs://(.+)', with a default value of an empty string. 'stderr' is a string, required to match the pattern '^gs://(.+)', with a default value of an empty string. 'resources' is an object with required fields: 'cpus', which is a number with a minimum value of 0.1; 'mem', which is an integer with a minimum value of 1; 'memunit', which references the 'bytesunit' definition; 'disk', which is an integer with a minimum value of 1; 'diskunit', which references the 'bytesunit' definition. 'docker' is a string. The root schema requires the properties: 'inputs', 'commandline', 'outputs', 'resources', and 'docker'. No additional properties are allowed at the root level.
