The schema describes O3D3xx Logic Layer format. It is an object type. It defines several reusable components under 'definitions': 'block' is an object, which contains properties 'type' (a string) and 'properties' (an object). 'connector' is an object, which contains properties 'src' (a string), 'srcEP' (a number), 'dst' (a string), and 'dstEP' (a number) and requires all of these fields. The main properties of the schema are: 'blocks' is an object that is required, with a minimum of 1 property, no additional properties allowed, and pattern properties matching '^B[0-9]+$', which refer to the 'block' definition. 'connectors' is an object that is optional, with a minimum of 1 property, no additional properties allowed, and pattern properties matching '^C[0-9]+$', which refer to the 'connector' definition. 'IOMap' is an object that is optional, with pattern properties matching '^(OUT1|OUT2|OUT3|IO1|IO2)$', where each value must be a string matching the pattern '^(NONE|ERRPIN|RFT|AQUFIN|EVAFIN|USEROUT|ANALOG|PCIC|USERIN)$'. No additional properties are allowed at the root level.
