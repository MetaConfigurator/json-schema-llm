The schema describes a widget configuration. It is an object type. The main properties of the schema are: 'name' is a string that represents the name of the widget or activity and is required. 'description' is a string that provides a short description of the purpose of this widget and is required. 'integration' is an object that details how this widget should be loaded by the LaxarJS runtime. It has properties: 'type', which is a string with a default value of 'widget'; and 'technology', which is a string with a default value of 'angular'. 'controls' is an array of strings representing each control as a path that can be resolved using the module system, with a default value of an empty array. 'features' is an object that serves as a JSON schema for the widget feature configuration, with a default value of an empty object. 'compatibility' is an array of strings for compatibility flags, with a default value of an empty array, and it accepts the enum value 'json-patch'. 'templateSource' is a string that indicates the location of the widget's HTML template. 'styleSource' is a string that indicates the location of the widget's stylesheet. 'assets' is an array of strings listing files to make available as raw content via the artifact provider API. 'assetUrls' is an array of strings listing files to make available as URLs via the artifact provider API. 'assetsForTheme' is an array of strings for themed files as raw content via the artifact provider API. 'assetUrlsForTheme' is an array of strings for themed files as URLs via the artifact provider API. No additional properties are allowed at the root level.
