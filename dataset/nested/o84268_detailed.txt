The schema describes a generic log container format. It is an object type. The main properties of the schema are: 'context' is an object that contains several optional string properties: 'class', 'file', 'host', 'line', 'method', 'namespace', 'processId', and 'threadId'. 'data' is an object that can contain any additional properties. 'exception' is an object that requires the properties: 'type' (a string), 'message' (a string), and 'backtrace' (an array of strings). 'id' is an optional string. 'level' is a string that must match the pattern '^(debug|info|warn|crit|fatal|unknown)$'. 'name' is an optional string. 'time' is a required string formatted as date-time. 'version' is an optional string that must match the pattern '^0$'. The schema requires 'time', 'name', and 'level' to be present. No additional properties are allowed at the root level.
