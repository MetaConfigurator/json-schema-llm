The schema describes a server configuration. It is an object type. The main properties of the schema are: 'base_image_id' is a string that signifies the Id of the base image OS that the server will be reimaged with. 'cluster_id' is a string that indicates the Id of the cluster that the server belongs to. 'contrail' is an object providing properties related to Contrail configurations. 'domain' is a string that represents the domain that the server belongs to. 'email' is a string representing the email associated with the server. 'id' is a string that represents the id of the server and is required. 'ipmi_address' is a string that specifies the address of the IPMI Interface for this server (if available) and must adhere to a specified pattern. 'ipmi_interface' is a string associated with the IPMI interface. 'ipmi_password' is a string that represents the password to log in to the IPMI interface. 'ipmi_username' is a string indicating the username to log in to the IPMI interface. 'network' is an object describing network settings, with 'management_interface' being a required string for the management interface name. 'package_image_id' is a string that denotes the Id of the Contrail package the server will be provisioned with. 'parameters' is an object that holds additional parameters, with nested properties including 'partition' and 'provision'. 'password' is a string indicating the root password for this server and is required. 'roles' is an array of strings representing the roles assigned to the server. 'tag' is an object with no defined properties. 'top_of_rack' is an object that includes a property 'switches', which is an array. The schema requires the properties 'id', 'ipmi_address', and 'password'. No additional properties are allowed at the root level.
