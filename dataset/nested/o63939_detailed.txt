The schema describes the configuration for activities items in an activities array of a main config file. It is an object type. The main properties of the schema are: 'activityTaskList' is a string that represents the tasklist (queue) where tasks are stacked and polled from. It is required and must have a length between 3 and 64 characters, matching the pattern '^[\w\.\- ]+$'. 'class' is a string that specifies the class name implementing this activity. It is required and must have a length between 8 and 256 characters, matching the pattern '^[\w\-]+$'. 'description' is a string that provides an activity description. It is required and must have a length between 1 and 256 characters. 'file' is a string representing the file implementing this activity. It is required and must have a length between 16 and 1024 characters, matching the pattern '^[\w\-\./]+\.php+$'. 'heartbeatTimeout' is an integer indicating the maximum time between heartbeats. It is required and must be between 1 and 99999999. 'name' is a string that denotes the activity name. It is required and must have a length between 3 and 64 characters. 'scheduleToCloseTimeout' is an integer representing the maximum time from schedule to close for a task. It is required and must be between 1 and 31536000. 'scheduleToStartTimeout' is an integer indicating the maximum time from schedule to start for a task. It is required and must be between 1 and 31536000. 'startToCloseTimeout' is an integer that represents the maximum time from start to close for a task. It is required and must be between 1 and 31536000. 'version' is a string that indicates the activity version. It is required and must have a length between 1 and 32 characters, matching the pattern '^[\w\.\- ]+$'. No additional properties are allowed at the root level.
