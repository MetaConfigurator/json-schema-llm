The schema describes configuration properties for a Ceph service. It is an object type. The main properties of the schema are: 'ceph' is an object representing ceph service configuration properties, required fields are 'cpus' and 'mem'. 'cpus' is a number that specifies CPU shares to allocate to each service instance, with a minimum value of 0.5 and a default value of 1. 'mem' is a number that specifies memory to allocate to each service instance, with a minimum value of 512.0 and a default value of 1024.0. 'networking' is an object representing Ceph networking configuration properties, required fields are 'cluster_network' and 'public_network'. 'cluster_network' is a string indicating the network where the Ceph nodes live. 'public_network' is a string indicating the network that the Ceph services are exposed on. 'external_access' is an optional object providing properties for access from outside the cluster through Marathon-LB. 'enable' is a boolean, defaulting to true, determining if a VIP for external access should be created. 'external_access_port' is a number, defaulting to 5000, used for clear communication in the external Marathon-LB load balancer. 'virtual_host' is a string, defaulting to 'ceph.example.org', representing the Virtual Host URL for external access in the load balancer. 'service' is an object for DC/OS service configuration properties. The 'name' is a string indicating the service instance name, defaulting to 'ceph'. No additional properties are allowed at the root level.
