The schema describes a Data Table collection. It is an object type. The main properties of the schema are: 'items' is an array containing objects that represent individual Data Tables, which are required to adhere to specific properties. 'count' is an integer representing the number of items. 'totalCount' is an integer representing the overall count of items available. 'perPage' is an integer indicating the items per page. 'page' is an integer indicating the current page. 'filter' is a string for filtering items. 'filterField' is a string specifying the field to filter by. 'sortField' is a string representing the field to sort by. 'sortDirection' is a string with allowed values: 'asc', 'desc', 'ASC', 'DESC', and ''. 'applicationId' is a string with a pattern of '^[A-Fa-f\d]{24}$'. Each Data Table must include 'id', 'dataTableId', 'applicationId', 'creationDate', 'lastUpdated', 'name', 'description', and 'columns'. 'columns' is an array that must contain up to 50 Data Table columns, each defined as an object that includes 'name', 'dataType', and 'constraint' as required fields. 'dataType' can be one of: 'string', 'number', 'boolean', and 'constraint' can be one of: 'unique', 'required', 'optional'. The 'defaultValue' may be of type string, number, or boolean. No additional properties are allowed at the root level.
