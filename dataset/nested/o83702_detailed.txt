The schema describes postal addresses for the country of Canada. It is an object type. It defines several reusable components under 'definitions': 'CAN_province' is a string, it must be one of the following values: 'AB', 'BC', 'MB', 'NB', 'NL', 'NS', 'NT', 'NU', 'ON', 'PE', 'QC', 'SK', 'YT'. 'CAN_postalCode' is a string, it must match the pattern '^[A-Z][0-9][A-Z]( )?[0-9][A-Z][0-9]$'. The main properties of the schema are: 'address' is an object, it is required and must contain the following properties: 'name' is a string, it is required; 'number' is a string, it is required; 'street' is a string, it is required; 'street2' is a string, it is optional; 'city' is a string, it is required; 'province' is a string, it is required and must resolve to the 'CAN_province' definition; 'postalCode' is a string, it is required and must resolve to the 'CAN_postalCode' definition. No additional properties are allowed at the root level.
