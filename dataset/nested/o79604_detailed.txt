The schema describes the status of a previously launched upload. It is an object type. It defines several reusable components under 'definitions': 'crudOutcome' is an object, which describes the result of a CRUD operation with required fields: 'operation' (an enum with allowed values 'Created', 'Updated', 'Closed'), 'succeeded' (an integer), and 'failed' (an integer), along with an optional 'failureDetails' (an array of 'failureDetail'). 'failureDetail' is an object, which describes the details of why a single object failed the CRUD operation; it contains required fields: 'code' (a string) and 'message' (a string). 'crudOutcomes' is an array of 'crudOutcome' definitions. The main properties of the schema are: 'id' is an integer, required; 'results' is an array of 'crudOutcome' definitions, required; 'started' is a string in 'date-time' format, required; 'status' is a string with enum values 'Queued', 'Underway', 'Complete', 'Error', required. No additional properties are allowed at the root level.
