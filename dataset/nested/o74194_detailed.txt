The schema describes a rule configuration. It is an object type. It defines several reusable components under 'definitions': 'rule-items' is an array, containing objects that require 'type' and 'value' fields. The 'type' field is a string that must be one of the following values: 'file.type', 'file.name', 'file.measurements', 'container.has-type', 'container.has-measurement'. The 'value' field is a string. The 'regex' field is a boolean. No additional properties are allowed in the items. The main properties of the schema are: 'rule-items' is a rule-items array that fulfills the defined constraints. 'rule-input' is a rule-input object containing optional fields '_id', 'project_id', 'alg', 'name', and 'disabled', as well as 'any' and 'all', which reference the rule-items definition. 'rule-output' is a rule-output object similar to rule-input, containing the same optional fields and references to rule-items. The schema requires the presence of 'rule-items', 'rule-input', and 'rule-output'. No additional properties are allowed at the root level.
