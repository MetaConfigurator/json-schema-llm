The schema describes a content item structure. It is an object type. It defines several reusable components under 'definitions': 'guid' is a string, matching the pattern '^[a-f0-9]{8}-[a-f0-9]{4}-[1-5][a-f0-9]{3}-[89ab][a-f0-9]{3}-[a-f0-9]{12}$'. 'guid_list' is an array, containing unique items that each reference 'guid'. The main properties of the schema are: 'bulk_publishing' is an optional boolean indicating whether the item is bulk published. 'links' is an optional object with multiple properties: 'facet_groups', 'facet_values', 'finder', 'government', 'lead_organisations', 'mainstream_browse_pages', 'meets_user_needs', 'ordered_related_items', 'ordered_related_items_overrides', 'organisations', 'original_primary_publishing_organisation', 'parent', 'people', 'policy_areas', 'primary_publishing_organisation', 'related_policies', 'related_statistical_data_sets', 'roles', 'suggested_ordered_related_items', 'taxons', 'topical_events', 'topics', and 'world_locations', all of which are arrays referencing 'guid_list'. 'government', 'primary_publishing_organisation', and 'parent' can each have a maximum of one item. 'previous_version' is an optional string. No additional properties are allowed at the root level.
