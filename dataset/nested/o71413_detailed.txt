The schema describes tracking history. It is an object type. The main properties of the schema are: 's' is a number or null, Speed at the time of the location transaction event and is required. 'lt' is a number or null, Latitude at the time of the location transaction event and is required. 'lg' is a number or null, Longitude at the time of the location transaction event and is required. 'd' is a string or null, Direction/Heading at the time of the location transaction event and is optional. 'ts' is a string or null, The original timestamp in unix timestamp format at the moment location transaction event and is required. 'ts_friendly' is a string or null, The original timestamp in a human readable timestamp format at the moment location transaction event and is required. No additional properties are allowed at the root level.
