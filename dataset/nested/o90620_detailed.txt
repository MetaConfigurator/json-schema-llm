The schema describes the configuration properties for a Bookkeeper instance. It is an object type. The main properties of the schema are: 'bookkeeper' is an object bookkeeper specific configuration properties. It is required and has several required fields: 'cpus', 'mem', 'instances', 'BK_BOOKIE_EXTRA_OPTS', 'BK_zkServers', 'BK_bookiePort', 'BK_journalDirectory', 'BK_ledgerDirectories', and 'BK_indexDirectories'. The 'bookkeeper' object contains the following properties: 'BK_BOOKIE_EXTRA_OPTS' is a string representing JVM runtime options for Bookkeeper, with a default value of "-Xms1g -Xmx1g -XX:MaxDirectMemorySize=1g -XX:+UseG1GC -XX:MaxGCPauseMillis=10 -XX:+ParallelRefProcEnabled -XX:+UnlockExperimentalVMOptions -XX:+AggressiveOpts -XX:+DoEscapeAnalysis -XX:ParallelGCThreads=32 -XX:ConcGCThreads=32 -XX:G1NewSizePercent=50 -XX:+DisableExplicitGC -XX:-ResizePLAB". 'BK_bookiePort' is an integer representing the export service port, with a default value of 3181. 'BK_indexDirectories' is a string representing the data ledgers storage directory, with a default value of '/bookkeeper/data/index'. 'BK_journalDirectory' is a string representing the journal storage directory, with a default value of '/bookkeeper/data/journal'. 'BK_ledgerDirectories' is a string representing the ledgers storage directory, with a default value of '/bookkeeper/data/ledgers'. 'BK_zkServers' is a string representing the Zookeeper server instance, with a default value of 'master.mesos:2181'. 'bookkeeper_cmd' is an array of strings for the command passed to entrypoint.sh, with a default value of []. It requires items to match the pattern '^[\s]+'. 'cpus' is a number for CPU shares allocation, required with a minimum of 0.5 and default value of 1. 'index_volume_size' is a number for persistent volume size (MB) for index data, with a minimum of 2000 and a default value of 7000. 'instances' is an integer representing the number of Bookkeeper instances to run, required with a minimum of 1 and a default value of 1. 'journal_volume_size' is a number for persistent volume size (MB) for journal data, with a minimum of 2000 and a default value of 4000. 'ledgers_volume_size' is a number for persistent volume size (MB) for ledgers data, with a minimum of 10000 and a default value of 20000. 'mem' is a number for memory allocation (MB), with a minimum of 512.0 and a default value of 1024.0. 'name' is a string representing the service name, with a default value of 'bookkeeper'. No additional properties are allowed at the root level.
