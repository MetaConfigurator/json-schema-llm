The schema describes a test case definition that is understood by https://github.com/FaKeller/sireg. It is an object type. It defines several reusable components under 'definitions': 'FileLoader' is an object, it includes required fields 'loader' (with an enum value of 'file') and 'options' (which must have a required field 'path' of type string). 'CsvLoader' is an object, it includes required fields 'loader' (with an enum value of 'csv') and 'options' (which must have a required field 'path' of type string). 'SitemapLoader' is an object, it includes required fields 'loader' (with an enum value of 'sitemap') and 'options' (which must have a required field 'sitemap' of type string). 'StaticReplacer' is an object, it includes required fields 'replacer' (with an enum value of 'static') and 'options' (which must have required fields 'replace' and 'with' of type string). The main properties of the schema are: 'testSuite' is a string, it is required. 'loaders' is an array, it is required and must have at least one item, each of which must be one of the referenced definitions. 'replacers' is an array, it is required and must have at least one item, each of which must be one of the referenced definitions. 'settings' is an object, it is optional and cannot have additional properties. No additional properties are allowed at the root level.
