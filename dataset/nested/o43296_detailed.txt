The schema describes the Git LFS HTTPS Lock Verify API Response. It is an object type. It defines several reusable components under 'definitions': 'lock' is an object, representing a lock with required fields 'id' and 'path'. It also has optional fields 'locked_at' (a string) and 'owner' (an object with an optional 'name' field). The main properties of the schema are: 'ours' is an array of lock objects, required; 'theirs' is an array of lock objects, required; 'next_cursor' is an optional string. No additional properties are allowed at the root level.
