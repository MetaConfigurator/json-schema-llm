The schema describes options for a Material Dashboard component. It is an object type. The main properties of the schema are: 'changeDetection' is a string that specifies the change detection strategy, optional, with a default value of 'Default', and allowed values are 'Default', 'OnPush'. 'export' is a boolean that specifies if declaring module exports the component, optional, with a default value of false. 'flat' is a boolean that indicates if a directory is created, optional, with a default value of false. 'inlineStyle' is a boolean that specifies if the style will be in the TypeScript file, optional, with a default value of false. 'inlineTemplate' is a boolean that specifies if the template will be in the TypeScript file, optional, with a default value of false. 'module' is a string that allows specification of the declaring module, optional. 'name' is a string that is the name of the component, optional, with a default value sourced from 'argv' at index 0. 'path' is a string that provides the path to create the component, optional, with a format specified as 'path' and is not visible. 'prefix' is a string that denotes the prefix to apply to generated selectors, optional, with a format specified as 'html-selector'. 'project' is a string that is the name of the project, optional, with a default value sourced from 'projectName' and is not visible. 'selector' is a string that specifies the selector to use for the component, optional, with a format specified as 'html-selector'. 'skipImport' is a boolean flag to skip the module import, optional, with a default value of false. 'spec' is a boolean that specifies if a spec file is generated, optional, with a default value of true. 'styleext' is a string that indicates the file extension to be used for style files, optional, with a default value of 'css'. 'viewEncapsulation' is a string that specifies the view encapsulation strategy, optional, with allowed values 'Emulated', 'Native', 'None'. No additional properties are allowed at the root level.
