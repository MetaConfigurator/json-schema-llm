The schema describes a revision document. It is an object type. The main properties of the schema are: 'id' is a number that serves as the revision identifier, required. 'timestamp' is a string representing the time that the revision was saved in YYYY-MM-DDTHH:MM:SSZ format, required. 'page' is an object containing metadata about the page the revision was saved to, required. 'contributor' is an object that contains metadata about the user who saved the edit, optional, and might be null. 'minor' is a boolean indicating if this is a minor edit, required. 'comment' is a string for the revision summary, optional, and might be null. 'text' is a string representing the content of the revision, optional, and might be null. 'diff' is an array of operations that represent the changes made in this revision, required, with each item being an object that includes 'name' (an enum with values 'insert', 'delete', 'replace', 'equals'), 'a1' (a number), 'a2' (a number), 'b1' (a number), and 'b2' (a number). 'bytes' is a number indicating the size of revision content in bytes, optional, and might be null. 'sha1' is a string representing a sha1 hash of the revision content, required. 'parent_id' is a number for the revision ID of the preceding revision, optional, and might be null. 'model' is a string, required. 'format' is a string, required. No additional properties are allowed at the root level.
