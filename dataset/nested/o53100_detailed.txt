The schema describes an array of test sets. It is an array type. It defines several reusable components under 'definitions': 'outputItem' is an object, which includes properties: 'valid' as a boolean, 'keywordLocation' as a string, 'absoluteKeywordLocation' as a string formatted as a URI, 'instanceLocation' as a string, 'annotations' as an array of outputItems, and 'errors' as an array of outputItems. 'test' is an object, which includes required properties: 'description' as a string, 'data' as a description of the instance to be validated against the schema in 'schema', and 'valid' as a boolean. It also has optional properties: 'id' as a string formatted as a URI, 'comment' as a string, and 'output' as an object with required properties: 'basic', 'detailed', and 'verbose', each referencing outputItem. The main properties of the schema are: 'id' is a string that optionally identifies a set of tests with a format of URI; 'description' is a string that provides the test set description; 'comment' is a string that offers any additional comments about the test set; 'schema' is a description indicating it should be a valid schema and a reference to a meta-schema if schema keywords need testing; 'tests' is an array that is required, containing a minimum of one related test that uses the same schema, each referencing the 'test' definition. No additional properties are allowed at the root level.
