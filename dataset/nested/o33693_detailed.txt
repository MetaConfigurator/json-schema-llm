The schema describes a representation of a face detection in a video media file. It is an object type. The main properties of the schema are: '@context' is a string that provides the JSON-LD context, required and locked, not searchable. '@type' is a string that indicates the JSON-LD type, required and locked, not searchable. 'schemaType' is a string representing the schema type, required and locked, searchable with the search mode 'noop'. '_documentID' is a string for the internal document GUID, required and locked, searchable with the search mode 'noop'. '_corpusID' is a string for the internal Corpus GUID, required and locked, searchable with the search mode 'noop'. 'begin' is a number representing the annotation starting time in milliseconds, required and locked, searchable with the search mode 'basic'. 'end' is a number for the annotation ending time in milliseconds, required and locked, searchable with the search mode 'basic'. 'confidence' is a number representing the confidence degree, required and locked, searchable with the search mode 'basic', with a minimum value of 0 and a maximum of 100. 'faceId' is a string for face detection faceId, optional, locked, not searchable. 'poseType' is a number indicating the face detection poseType with allowed values: -2, -1, 0, 1, 2, optional, and defaults to 0, not searchable. No additional properties are allowed at the root level.
