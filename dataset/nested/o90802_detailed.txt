The schema describes configuration properties for an OpenLDAP Admin service. It is an object type. The main properties of the schema are: 'openldap-admin' is an object OpenLDAP Admin service configuration properties, required fields are 'cpus' and 'mem'. It defines several properties: 'cpus' is a number CPU shares to allocate to each OpenLDAP node, with a default of 0.1 and a minimum of 0.1; 'ldap_host' is a string, the name of the OpenLDAP instance to connect to, with a default of 'openldap'; 'ldap_host_port' is a number, the port where the OpenLDAP server is listening on, defaulting to 389; 'mem' is a number, memory to allocate to each OpenLDAP node, with a default of 128.0 and a minimum of 128.0; 'name' is a string, the package name, defaulting to 'openldap-admin'; 'port' is a number, port number to be used for external clear web traffic, defaulting to 11389; 'version' is a string, the package version, with a default of '1.2.2'; 'virtual_host' is a string, the virtual host to be used to publish the UI, with a default of 'phpldapadmin.example.org'. Additionally, 'service' is an object DC/OS service configuration properties, with the property 'name' which is a string, representing the name of the OpenLDAP Admin service instance, defaulting to 'openldap-admin'. No additional properties are allowed at the root level.
