The schema describes a financial transaction record. It is an object type. It defines several reusable components under 'definitions': 'IDType' is a string, representing a 10-digit numerical identifier. 'ISO8601-Date' is a string, following the pattern of YYYY-MM-DD. 'ISO8601-Timestamp' is a string, following the pattern of a timestamp formatted as YYYY-MM-DDThh:mm:ss or similar variations. 'LISAManagerReferenceNumberType' is a string, matching the pattern of a letter followed by six digits, starting with 'Z'. 'financialPeriodType' is an object that contains 'startDate' and 'endDate', both of which reference 'ISO8601-Date'. 'transactionTypeType' is a string, with allowed values being 'bonus' and 'charge'. The main properties of the schema are: 'LISAManagerReferenceNumber' is a string, required, referencing 'LISAManagerReferenceNumberType'. '_links' is an optional object. 'accountID' is a string, required, referencing 'IDType'. 'amount' is a number, required. 'applicablePeriod' is an object, required, containing 'startDate' and 'endDate', both referencing 'ISO8601-Date'. 'hmrcTransactionID' is a string, required, referencing 'IDType'. 'paymentForTransaction' is a string, required, referencing 'IDType'. 'paymentReference' is a string, required, referencing 'IDType'. 'transactionTimestamp' is a string, required, referencing 'ISO8601-Timestamp'. 'transactionType' is a string, required, referencing 'transactionTypeType'.
