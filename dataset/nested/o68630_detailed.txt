The schema describes an array of items. It is an object type. The main properties of the schema are: 'canUninstall' is a boolean optional property. 'componentClasses' is an array of strings optional property. 'description' is a string required property. 'helpUrl' is a string required property. 'id' is a string required property. 'installerClass' is a string optional property. 'markerClass' is a nullable string optional property. 'name' is a string required property. 'screenshotUrl' is a string required property. 'vendor' is a string required property. 'versions' is an object required property. The 'versions' property follows a pattern where keys match the regex '^\d+(\.\d+)*(-\w+)?$' and the associated values are objects that contain the properties: 'changes' is a string optional property, 'depends' is an array of strings optional property, 'downloadUrl' is a nullable string required property, and 'libs' is an object with pattern properties where keys are strings and values are strings optional property. No additional properties are allowed at the root level.
