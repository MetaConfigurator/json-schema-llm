The schema describes a resource JSON format. It is an object type. The main properties of the schema are: 'street' is a string representing the street name. 'number' is an integer representing the construction street number. 'dir' is a string formatted as a URI reference, representing the root directory for all resources for this construction. 'location' is an object representing the geographical coordinates of construction, with properties: 'type' which is a string describing the GeoJSON type Point, and 'coordinates' which is an array of numbers with a minimum of 2 items representing longitude and latitude. 'resources' is an array of models related to this construction, where each item is an object with properties: 'format' which is a string describing the format type of the model, 'filename' which is a string describing the filename of the model, 'url' which is a string formatted as a URI reference representing the full URL of the model, 'uploaded' which is a string formatted as a date-time representing the timestamp of upload, and 'textures' which is an array of texture documents where each texture is an object with properties: 'type' which is an integer describing the texture type, 'format' which is a string describing the format type of the texture, 'filename' which is a string describing the filename of the texture, 'url' which is a string formatted as a URI reference representing the full URL of the texture, and 'uploaded' which is a string formatted as a date-time representing the timestamp of upload. No additional properties are allowed at the root level.
