The schema describes a request to send a local authorization list. It is an object type. The main properties of the schema are: 'updateType' is a string representing the type of update, which is required and must be either 'Differential' or 'Full'. 'listVersion' is an integer representing the version of the list, which is required. 'localAuthorisationList' is an array containing items that are objects; each object must have an 'idTag', which is a required string with a maximum length of 20, and an 'idTagInfo', which is an object that must include a 'status' (required string, must be one of 'Accepted', 'Blocked', 'Expired', 'Invalid', 'ConcurrentTx') and may include 'expiryDate' (string formatted as date-time) and 'parentIdTag' (optional string with max length 20). 'hash' is an optional string. No additional properties are allowed at the root level.
