The schema describes placing an order for a platform user. It is an object type. The main properties of the schema are: 'customer' is a string representing the platform's customer, required, with a minimum length of 3 and a maximum length of 96. 'account_identifier' is a string representing the identifier of the account for whom to place the order, required, with a minimum length of 5, a maximum length of 96, and a pattern of '^[\+\w-]{5,96}$'. 'recipient' is an object containing information for the person who is to receive this reward, required. The object has required properties: 'name', which is a string with a minimum length of 2 and a maximum length of 255, and 'email', which is a string matching the pattern '^\S+@\S+$'. 'sku' is a string identifying the reward to be purchased, required. 'amount' is a number representing the desired amount of the reward, optional, with a minimum of 1 and a maximum of 100000. 'reward_subject' is a string for an optional subject line for the reward email, with a minimum length of 1 and a maximum length of 255. 'reward_message' is a string for an optional message to the recipient, with a minimum length of 0 and a maximum length of 255. 'reward_from' is a string for an optional 'from name' to display to the recipient, with a minimum length of 0 and a maximum length of 255. 'send_reward' is a boolean indicating whether Tango Card should send the reward, optional, with a default value of true. No additional properties are allowed at the root level.
