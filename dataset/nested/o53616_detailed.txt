The schema describes pantry and role access level that user has. It is an object type. The main properties of the schema are: 'pantryId' is a string unique ID of food pantry, required, with a minimum length of 1. 'userId' is a string unique ID of user, optional, with a minimum length of 1. 'role' is a string role of the user, required, with allowed values: 'administrator', 'contributor', 'viewer'. 'createdAt' is a string [Output only] creation time of the user role, optional, with a format of date-time and a minimum length of 1. 'createdBy' is a string [Output only] creator of the user role, optional, with a minimum length of 1. 'updatedAt' is a string [Output only] last modification time of the user role, optional, with a format of date-time and a minimum length of 1. 'updatedBy' is a string [Output only] user to last modify the user role, optional, with a minimum length of 1. No additional properties are allowed at the root level.
