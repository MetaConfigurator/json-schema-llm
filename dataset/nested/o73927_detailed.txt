The schema describes JSON schema for host-meta files. It is an object type. It defines several reusable components under 'definitions': 'link' is an object, which has optional properties: 'href' as a string, 'properties' as a reference to the 'property' definition, 'rel' as a string, 'template' as a string with a URI format, 'titles' as an object with additional properties being strings and having a 'default' property as a string, and 'type' as a string; 'property' is an object, allowing any additional properties that are either null or strings. The main properties of the schema are: 'aliases' is an array of strings, which is optional; 'expires' is a string with a date-time format, which is optional; 'links' is an array of items that reference the 'link' definition, which is optional; 'properties' is a reference to the 'property' definition, which is optional; 'subject' is a string with a URI format and is required. No additional properties are allowed at the root level.
