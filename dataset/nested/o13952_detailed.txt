The schema describes a set of services and their relationships. It is an object type. The main properties of the schema are: 'relations' is an array of objects, where each object contains 'serviceA' and 'serviceB' as string properties and 'sharedEntities' as an array of strings. 'services' is an array of objects, where each object contains 'id' and 'name' as string properties and 'nanoentities' as an array of strings. 'useCaseResponsibility' is an object type. No additional properties are allowed at the root level.
