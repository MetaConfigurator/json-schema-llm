The schema describes a user authentication structure. It is an object type. The main properties of the schema are: 'token' is a string representing a JWT token, which is required. 'user' is an object containing user details, which is also required. It has several properties: 'auth_providers' is an array of objects, where each object must have required properties 'authId' (a string) and 'name' (a string). 'createdAt' is a string formatted as a past date and is required. 'email' is a string formatted as an email and is required. 'firstName' is a string formatted as a first name and is required. 'id' is an integer and is required. 'lastName' is a string and is optional. 'updatedAt' is a string formatted as a past date and is required. 'username' is a string formatted as a username and is required. No additional properties are allowed at the root level.
