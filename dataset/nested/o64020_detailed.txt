The schema describes configuration for running an update of the search index. It is an object type. The main properties of the schema are: 'name' is a string indicating the name of the bucket that will be operating on, required; 'accessKey' is a string representing the public part of the key pair to access the cloud, required; 'secretKey' is a string representing the private part of the key pair to access the cloud, required; 'elasticsearch' is an object containing connection details with the property 'connectionString', which is a string that is required; 'logLevel' is a string specifying what we should set the log level to, which must either be 'DEBUG' or 'INFO', required; 'chunkSize' is an integer defining how many records to process at once, required; 'bulkUpdateLogDirectory' is a string indicating the location of a writable directory for storing logs per bucket, required. No additional properties are allowed at the root level.
