The schema describes an Application Policy. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: '_id' is a string and is required. 'algorithm' is a string and is required, with allowed values: 'one-after-another', 'all-at-once'. 'networkPartitions' is an array of strings and is required, with items being string type. 'properties' is an array of objects and is required, with each object containing 'name' as a string and 'value' as a string. No additional properties are allowed at the root level.
