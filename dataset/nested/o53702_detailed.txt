The schema describes the conda configuration file. It is an object type. It defines several reusable components under 'definitions': 'channel' is a string, 'feature' is a string, 'package' is a string, 'path' is a string. The main properties of the schema are: 'channels' is an array of strings, where each item references the 'channel' definition; 'allow_other_channels' is an optional boolean; 'default_channels' is an array of strings, where each item references the 'channel' definition; 'auto_update_conda' is an optional boolean; 'always_yes' is an optional boolean; 'show_channel_urls' is an optional boolean; 'changeps1' is an optional boolean; 'add_pip_as_python_dependency' is an optional boolean; 'use_pip' is an optional boolean; 'proxy_servers' is an object with additional properties defined as strings in hostname format; 'ssl_verify' is an optional boolean; 'offline' is an optional boolean; 'allow_softlinks' is an optional boolean; 'channel_alias' is a string formatted as a uri; 'create_default_packages' is an array of strings, where each item references the 'package' definition; 'track_features' is an array of strings, where each item references the 'feature' definition; 'update_dependencies' is an optional boolean; 'disallow' is an array of strings, where each item references the 'package' definition; 'add_anaconda_token' is an optional boolean; 'envs_dirs' is an array of strings, where each item references the 'path' definition; 'anaconda_upload' is an optional boolean; 'conda-build' is an object with properties: 'root-dir' referencing the 'path' definition, 'quiet' is an optional boolean, 'filename_hashing' is an optional boolean, 'no_verify' is an optional boolean, 'set_build_id' is an optional boolean, 'skip_existing' is an optional boolean, 'include_recipe' is an optional boolean, 'activate' is an optional boolean, 'pypirc' referencing the 'path' definition, and 'pypi_repository' is a string. No additional properties are allowed at the root level.
