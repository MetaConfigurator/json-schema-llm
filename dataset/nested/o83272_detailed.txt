The schema describes resource specifications for a given date range. It is an object type. The main properties of the schema are: 'resource' is a string that represents the resource name/code and is required with a maximum length of 64. 'nodes' is an integer that indicates the number of nodes in the resource and is required with a minimum value of 0. 'processors' is an integer that specifies the number of processors/cores in the resource and is required with a minimum value of 0. 'ppn' is a number that represents the number of processors per node in the resource with a minimum value of 0. 'start_date' is a string formatted as a date, representing the resource specifications start date (YYYY-MM-DD). 'end_date' is a string formatted as a date, representing the resource specifications end date (YYYY-MM-DD). 'percent_allocated' is a number that indicates the percentage of the resource to include in the utilization metric calculation, with a minimum value of 0 and a maximum value of 100. 'comments' is a string that provides comments explaining the specifications, with a maximum length of 500. No additional properties are allowed at the root level.
