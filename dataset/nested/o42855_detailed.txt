The schema describes a publishable entity. It is an object type. The main properties of the schema are: "_id" is a string that represents the entity's unique identifier and is required. It must match the pattern "^[\w\/\.:-]+$". "status" is a string that indicates the publication status with a default value of "draft" and can be one of the following values: "unknown", "published", "scheduled", "pending", "draft", "expired", "archived", "deleted". "etag" is a string that allows matching a specific version of the entity, following the pattern "^[\w\.:-]+$". "created_at" is a string formatted to match the pattern "^[1-9]{1}[0-9]{12,15}$". "creator_ref" is an object that includes properties "curie" (a required string matching the pattern "^([a-z0-9-]+):([a-z0-9\.-]+):([a-z0-9-]+)?:([a-z0-9-]+)$"), and "id" (a required string matching the pattern "^[A-Za-z0-9:_\-]+$"). It also allows an optional property "tag" (a string matching the pattern "^([\w\/-]|[\w-][\w\/-]*[\w-])$"). "updated_at" is a string matching the same pattern as "created_at". "updater_ref" is similar to "creator_ref" with the same required properties and constraints. "last_event_ref" follows the same structure as "updater_ref". Lastly, "title" is an optional string with a minimum length of 0 and a maximum length of 255. "published_at" is a string formatted as a date-time. No additional properties are allowed at the root level.
