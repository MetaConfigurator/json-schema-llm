The schema describes a collection of experience versions. It is an object type. The main properties of the schema are: 'items' is an array containing objects that each represent a single Experience Version. It requires that each item has properties such as 'id' (a string matching the pattern '^[A-Fa-f\d]{24}$'), 'experienceVersionId' (a string matching the pattern '^[A-Fa-f\d]{24}$'), 'applicationId' (a string matching the pattern '^[A-Fa-f\d]{24}$'), 'creationDate' (a string in date-time format), 'createdById' (a string matching the pattern '^[A-Fa-f\d]{24}$'), 'createdByType' (a string that must be one of 'flow', 'user', or 'apiToken'), 'lastUpdated' (a string in date-time format), 'endpointDefaultCors' (a boolean), 'version' (a string with a minimum length of 1 and a maximum length of 255), 'description' (a string with a maximum length of 32767), 'attachedDomains' (an array of objects where each has 'domainName' (a string with a minimum length of 3 and a maximum length of 1024) and 'id' (a string matching the pattern '^[A-Fa-f\d]{24}$')), and 'attachedSlugs' (an array of objects where each has 'slug' (a string) and 'id' (a string matching the pattern '^[A-Fa-f\d]{24}$')). 'count' is an integer, 'totalCount' is an integer, and 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$'. No additional properties are allowed at the root level.
