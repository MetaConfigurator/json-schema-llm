The schema describes a network configuration. It is an object type. It defines several reusable components under 'definitions': 'network' is an object, representing a network with required properties 'name', 'meta', 'transactions', and 'ussd_codes'. 'metadata' is an object, which requires 'spec' (a string that matches the pattern '^[0-9].[0-9]$'), 'date_updated' (a reference to a date), and 'url' (a string formatted as a URI). 'transaction' is an object, which requires 'name' (a reference to a name) and 'classes' (an array of class objects). 'class' is an object that requires 'name' and can have an array of 'ranges' (each a reference to a range object) and an optional 'amount' (a reference to a cost) and 'message' (a string). 'range' is an object that requires 'low', 'high', and 'amount' (all references to costs). 'cost' can be a number or a string, with a minimum value of -2 and a pattern match for '^[+-]Infinity$'. 'ussd-code' is an object that requires 'code' and 'description', both strings. 'name' is a string. 'date' is a string formatted as a date. 'error' is an object with required properties 'message' (a string), 'name' (a string), and 'statusCode' (an integer). The root schema resolves to the referenced definition 'network'. The main properties of the schema are: 'name' is a reference to a name, required; 'meta' is a reference to metadata, required; 'transactions' is an array of transaction objects, required; 'ussd_codes' is an array of USSD code objects, required. No additional properties are allowed at the root level.
