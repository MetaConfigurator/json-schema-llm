The schema describes configuration properties for a bookkeeper service. It is an object type. The main properties of the schema are: 'bookkeeper' is an object containing specific configuration properties for the bookkeeper service. It has required fields: 'cpus', 'mem', and 'instances'. The properties within 'bookkeeper' are: 'cpus' is a number representing CPU shares to allocate to each bookkeeper instance, with a default value of 1 and a minimum value of 1. 'instances' is an integer indicating the number of bookkeeper instances to run, with a default value of 1 and a minimum value of 1. 'mem' is a number representing the memory (MB) allocated to each bookkeeper task, with a default value of 1024.0 and a minimum value of 512.0. 'name' is a string representing the name of the DC/OS service, with a default value of 'bookkeeper'. 'service_port' is an integer defining the bookkeeper export service port, with a default value of 3181. 'volume_size' is a number for the persistent volume size (MB), with a default value of 70 and a minimum value of 10. 'zk_client' is a string for the zookeeper client instance, with a default value of 'master.mesos:2181'. No additional properties are allowed at the root level.
