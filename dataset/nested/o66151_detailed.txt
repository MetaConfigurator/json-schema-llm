The schema describes a process for coding and executing steps. It is an object type. The main properties of the schema are: 'code_base' is a string that represents the user's code and is required. 'processing_steps' is an array of objects that each represents a processing step and is required. Each processing step must include 'configuration_file', 'run_instructions', and 'output_data', all of which are strings. The 'configuration_file' property has a placeholder 'E.g. my_job_config.py'. The 'output_data' property has a placeholder 'E.g. myfile-data.root'. The 'run_instructions' property has a placeholder 'E.g. README.md' and provides run instructions. No additional properties are allowed at the root level.
