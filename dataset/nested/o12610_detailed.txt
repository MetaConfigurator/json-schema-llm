The schema describes a physical Wi-Fi device and its configuration. It is an object type. The main properties of the schema are: 'channel' is an integer Wi-Fi channel number, optional, with a minimum value of 1 and a maximum value of 165. 'htmode' is a string that enables 802.11n or 802.11ac modes, optional, with allowed values of null, 'HT20', 'HT40+', 'HT40-', 'VHT20', 'VHT40', 'VHT80'. 'hwmode' is a string that defines the basic operating mode (11b for old hardware, 11g for 2.4 GHz, 11a for 5 GHz), optional, with allowed values of '11b', '11g', '11a'. 'id' is a string representing the physical identifier (e.g. pci-wifi-1 or usb-wifi-0), optional. 'rx_stbc' is an integer that indicates support for receiving frames using STBC, optional, with a minimum value of 0 and a maximum value of 1. 'short_gi_20' is a boolean that enables short guard interval (higher data rates) in 20 MHz channels, must be supported by device, optional. 'short_gi_40' is a boolean that enables short guard interval (higher data rates) in 40 MHz channel, must be supported by device, optional. 'short_gi_80' is a boolean that enables short guard interval (higher data rates) in 80 MHz channel, must be supported by device, optional. 'tx_stbc' is an integer that indicates support for transmitting frames using STBC, optional, with a minimum value of 0 and a maximum value of 1. No additional properties are allowed at the root level.
