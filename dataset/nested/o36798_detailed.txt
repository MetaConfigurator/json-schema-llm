The schema describes a configuration for a device. It is an object type. It defines several reusable components under 'definitions': 'knx-item' is an object, containing required properties 'read' and 'write', both of which must match the pattern '^\\d+/\\d+/\\d+$'. The main properties of the schema are: 'uuid' is a string matching the pattern '^[A-F0-9]{8}-[A-F0-9]{4}-[A-F0-9]{4}-[A-F0-9]{12}$' and is required; 'knx' is an object that is required and contains required properties 'up-down', 'stop', 'position', 'fully-down', 'fully-up', and 'direction', each of which references the 'knx-item' definition. No additional properties are allowed at the root level.
