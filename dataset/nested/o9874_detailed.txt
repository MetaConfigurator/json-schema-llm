The schema describes an API endpoint configuration. It is an object type. The main properties of the schema are: 'description' is a string that has a maximum length of 32767. 'route' is a string that must have a minimum length of 1 and a maximum length of 1024. 'enabled' is a boolean that indicates whether the endpoint is active. 'method' is a string that must be one of the following values: 'delete', 'get', 'options', 'patch', 'post', 'put'. 'access' is a string that must be one of the following values: 'public', 'authenticated', 'group'. 'endpointTags' is an object with properties that can only match the pattern '^[0-9a-zA-Z_-]{1,255}$' and contain strings with a minimum length of 1 and a maximum length of 255; no additional properties are allowed. 'experienceGroupIds' is an array of strings, each matching the pattern '^[A-Fa-f\d]{24}$', with a maximum of 1000 items. 'staticReply' is either an object with required properties 'value' (a string) and 'type' (which must be either 'page' or 'redirect', with an integer 'statusCode' between 100 and 599), or it can be null. 'unauthorizedReply' has the same structure and requirements as 'staticReply', being either an object or null. No additional properties are allowed at the root level.
