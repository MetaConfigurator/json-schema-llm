The schema describes the semantics of specific data elements or state transitions that MAY exist in an associated representation. It is an object type. The main properties of the schema are: 'links' is an array of objects that identifies a link between an ALPS element and some other (possibly external) resource. This property is optional and requires each object to have 'rel' and 'href' as required strings. 'descriptors' is an array that defines the semantics of specific data elements or state transitions. This property is optional. 'doc' is an object that provides a description of the element, containing optional properties 'format' (an enum with allowed values 'text', 'html', 'asciidoc'), 'href' (a string), and 'content' (a string). 'ext' is an object representing an author-specific extension to an element, with a required 'id' string and optional 'href' (a string) and 'value' (a string). 'id' is a string that serves as a document-wide unique identifier for the descriptor element. 'name' is a string that names a descriptor element. 'type' is an enum specifying the type of hypermedia control, with allowed values 'semantic', 'safe', 'idempotent', 'unsafe'. 'rt' is a string representing the ID of another descriptor in the document, valid for 'safe', 'idempotent', and 'unsafe' types. No additional properties are allowed at the root level.
