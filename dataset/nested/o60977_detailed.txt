The schema describes the result of a Github worker. It is an object type. It defines several reusable components under 'definitions': 'github_extracted_details' is an object, details of Github inspection. It has optional integer properties: 'forks_count', 'open_issues_count', 'stargazers_count', 'subscribers_count', and references to 'github_last_year_commits_details' for 'last_year_commits', 'github_issues_details' for 'updated_issues', and 'github_pull_requests_details' for 'updated_pull_requests'. No additional properties are allowed. 'github_issue&prs_with_time_duration' is an object, details of Github issues + prs yearly or monthly or any given date-range. It has required integer properties: 'opened' and 'closed'. No additional properties are allowed. 'github_issues_details' is an object, details of updated Github issues. It references 'github_issue&prs_with_time_duration' for 'year' and 'month'. No additional properties are allowed. 'github_last_year_commits_details' is an object, details of last year Github commits. It has required integer property 'sum' and an array property 'weekly' with integer items. No additional properties are allowed. 'github_pull_requests_details' is an object, details of updated Github pull requests. It references 'github_issue&prs_with_time_duration' for 'year' and 'month'. No additional properties are allowed. The main properties of the schema are: 'schema' is an object that has required properties: 'name' (a string with pattern '^[a-zA-Z0-9_]+$') and 'version' (a string with pattern '^[0-9]+-[0-9]+-[0-9]+$'). It also contains an optional 'url' (a string with format 'uri'). '_release' is an optional string, description of unique release id in form of "ecosystem:package:version". 'status' is a string that must be one of the following values: 'success', 'error', 'unknown'. 'details' references 'github_extracted_details'. 'summary' is an array of strings. No additional properties are allowed at the root level.
