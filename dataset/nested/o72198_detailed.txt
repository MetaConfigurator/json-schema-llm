The schema describes a context for rendering custom unit prettyValue. It is an object type. The main properties of the schema are: 'unit' is a string required field that represents the unit of the reward with a minimum length of 1. 'value' is an integer required field that represents the value of the reward, where if the reward is currency based, then this is the value in the currency's fractional unit. 'name' is a string optional field that represents the name of the reward. 'currencySymbol' is a string optional field that represents the currency symbol in the rendering locale with a minimum length of 1. 'currencyLocalizedSymbol' is a string optional field that represents the currency symbol in the currency's native locale with a minimum length of 1. 'currencyCode' is a string optional field that represents the ISO 4217 currency code, following the pattern '^[A-Z]{3}$'. 'currencyNumericCode' is an integer optional field that represents the ISO 4217 numeric currency code, which must be between 0 and 999. 'currencyDisplayName' is a string optional field that represents the display name of the currency in the rendering locale, with a minimum length of 1. 'currencyLocalizedDisplayName' is a string optional field that represents the display name of the currency in the currency's native locale with a minimum length of 1. 'currencyFractionalUnit' is a string optional field that represents the fractional unit of the currency, e.g. 'Cent', with a minimum length of 1. 'currencyNumberToBasic' is an integer optional field that represents how many the currency subdivision go into the currency, e.g. 100 pence to the pound, which must be at least 1. 'valueInCurrencyBaseUnit' is a number optional field that represents the value of the reward in the currency's base unit. No additional properties are allowed at the root level.
