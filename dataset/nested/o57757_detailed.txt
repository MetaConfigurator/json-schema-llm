The schema describes job postings. It is an object type. The main properties of the schema are: 'company' is a string representing the company name, required. 'jobs' is an array containing job objects, required. Each job object has the following properties: 'apply' is a string describing how to apply, optional. 'description' is a string describing the job role, optional. 'equity' is an object representing stock/options grant range, optional. 'location' is a string describing the job's location, optional. 'perks' is an array of strings listing job perks, optional. 'position' is a string indicating the job position, optional. 'posted' is a string formatted in ISO 8601 date standard representing the posting date, optional. 'salaryRange' is an object representing the salary range, optional. 'skills' is an array of strings listing required skills, optional. 'title' is a string representing the job title, optional. 'type' is a string indicating the job type, optional. 'url' is a string formatted as a URI representing the job application URL, optional. 'market' is a string describing the target market of the company, required. 'remoteFriendly' is a boolean indicating if the company supports remote work, required. 'size' is a string indicating the company's size, required. No additional properties are allowed at the root level.
