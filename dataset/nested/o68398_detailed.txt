The schema describes a user registration process. It is an object type. The main properties of the schema are: 'completed' is a boolean indicating whether the user has validated his e-mail address and completed the registration process, and it is read-only. 'completedAt' is a string in date-time format indicating the date at which the registration was completed, and it is read-only. 'createdAt' is a string in date-time format indicating the date at which the registration was created, and it is read-only. 'expiresAt' is a string in date-time format indicating the date at which the validity of the link sent in the registration e-mail will expire, and it is read-only. 'id' is a string indicating the unique identifier of the registration, and it is read-only. 'organization' is an object representing the organization details, and it is required. It has properties: 'displayName' is a string that defaults to the 'name' property if not set, with a maximum length of 50. 'name' is a string with a maximum length of 50, must match the pattern '^[A-Za-z0-9]+(-[A-Za-z0-9]+)*$', and must be unique. 'public' is a boolean indicating whether the organization should be public. 'user' is an object representing user account details, and it is required. It has properties: 'name' is a string with a maximum length of 25, must match the pattern '^[A-Za-z0-9]+(-[A-Za-z0-9]+)*$', and must be unique. 'primaryEmail' is a string in email format with a maximum length of 255 and must be unique. No additional properties are allowed at the root level.
