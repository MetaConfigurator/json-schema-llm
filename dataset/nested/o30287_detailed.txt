The schema describes a cluster node configuration. It is an object type. The main properties of the schema are: 'backplane' is a string optional description of the backplane. 'cfgflags' is an integer read-only configuration flags. 'clearnodegroupconfig' is a string optional field that can be either 'YES' or 'NO'. 'clusterhealth' is a string read-only property indicating the health status, with possible values such as 'UP', 'UNKNOWN', and various failure messages. 'delay' is an integer optional delay value. 'disabledifaces' is a string read-only field indicating disabled interfaces. 'effectivestate' is a string read-only property describing the effective state, with values like 'UP' and 'NOT UP'. 'enabledifaces' is a string read-only field indicating enabled interfaces. 'hamonifaces' is a string read-only field related to HAMON interfaces. 'health' is a string read-only property that outlines the health status with several possible values including 'UP', 'DOWN', and error states. 'ifaceslist' is an object read-only property listing interfaces. 'ipaddress' is a string specifying the IP address. 'isconfigurationcoordinator' is a boolean read-only field indicating if it's the configuration coordinator. 'islocalnode' is a boolean read-only field marking if it's the local node. 'masterstate' is a string read-only property indicating the master state, with values like 'ACTIVE' and 'INACTIVE'. 'name' is a string read-only property for the node's name. 'netmask' is a string read-only property for the netmask. 'nodegroup' is a string optional field specifying the node group. 'nodeid' is an integer optional identifier for the node. 'nodejumbonotsupported' is a boolean read-only property indicating if jumbo frames are not supported. 'nodelicensemismatch' is a boolean read-only field marking license mismatch status. 'nodelist' is an object read-only property listing nodes. 'nodersskeymismatch' is a boolean read-only property to indicate a key mismatch. 'operationalsyncstate' is a string read-only property with possible states like 'ENABLED' and 'FAILED'. 'partialfailifaces' is a string read-only field detailing partially failed interfaces. 'priority' is an integer optional priority setting. 'routemonitor' is a string read-only property for route monitoring details. 'state' is a string optional field describing the state with values 'ACTIVE', 'SPARE', and 'PASSIVE'. 'syncstate' is a string read-only property indicating sync status, with options 'ENABLED' and 'DISABLED'. 'tunnelmode' is a string optional field with options 'NONE', 'GRE', or 'UDP'. No additional properties are allowed at the root level.
