The schema describes the configuration for a middleware. It is an object type. The main properties of the schema are: 'config' is an object that contains properties 'path' and 'schema'. 'path' is a string describing the package path to import the middleware, with examples like 'github.com/uber/zanzibar/examples/example-gateway/middlewares/example'. 'schema' is a string indicating the path to the schema of the middleware, with examples like './middlewares/example/example_schema.json'. 'dependencies' is an object that includes a property 'client', which is an array of strings where each item represents the name of a client this middleware depends on, with examples like 'baz'. 'name' is a string required for the name of the middleware, with examples like 'example'. 'type' is a string specifying the type of the middleware, currently only valid type is 'default', with examples like 'default'. All properties are required. No additional properties are allowed at the root level.
