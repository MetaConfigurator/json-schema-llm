The schema describes a response containing API tokens. It is an object type. The main properties of the schema are: 'items' is an array of objects, each representing a single API Token. Each token object includes properties such as: 'id' is a string with a pattern that must match '^[A-Fa-f\d]{24}$', 'apiTokenId' is a string with the same pattern, 'ownerId' is a string with the same pattern, 'ownerType' is a string that must be 'application', 'creatorId' is a string with the same pattern, 'creatorType' is a string that can be 'apiToken', 'user', or 'flow', 'creatorName' is a string with a maximum length of 1024, 'name' is a string with a minimum length of 1 and a maximum length of 255, 'description' is a string with a maximum length of 32767, 'creationDate', 'lastUpdated', and 'expirationDate' are strings formatted as date-time, 'scope' is an array of strings, each of which must have a minimum length of 1 and a maximum length of 1024, 'status' is a string that can be 'active' or 'inactive', and 'token' is a string with a minimum length of 1. 'count' is an integer, 'totalCount' is an integer, 'perPage' is an integer, 'page' is an integer, 'filter' is a string, 'filterField' is a string, 'sortField' is a string, and 'sortDirection' is a string that can be 'asc', 'desc', 'ASC', 'DESC', or an empty string. No additional properties are allowed at the root level.
