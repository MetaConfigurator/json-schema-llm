The schema describes the result of Dependency Snapshot worker. It is an object type. It defines several reusable components under 'definitions': 'dependencies_counts' is an object, counts of various types of dependencies. It has a property 'runtime' which is an integer. 'dependency_object' is an object, Dependency Object. It has required properties: 'ecosystem' (a string), 'name' (a string), 'version' (either a string or null), 'declaration' (a string), and 'resolved_at' (a string). 'dependency_snapshot_detail' is an object, details of Dependency Snapshot run on a component. It has a property 'runtime' which is an array of items that reference 'dependency_object'. 'dependency_snapshot_summary' is an object, summary of Dependency Snapshot run on a component. It has required properties: 'errors' (an array of strings) and 'dependency_counts' (referencing 'dependencies_counts'). The main properties of the schema are: 'schema' is an object, with required properties: 'name' (a string matching pattern '^[a-zA-Z0-9_]+$') and 'version' (a string matching pattern '^[0-9]+-[0-9]+-[0-9]+$'), and additional properties are not allowed. '_release' is a string, describing a unique release id in the form of "ecosystem:package:version". 'status' is a string, and it must be one of the following values: 'success', 'error'. 'details' references 'dependency_snapshot_detail'. 'summary' references 'dependency_snapshot_summary'. No additional properties are allowed at the root level.
