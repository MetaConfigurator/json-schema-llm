The schema describes a structure for defining named blocks. It is an object type. The main properties of the schema are: 'name' is a string with a minimum length of 1 and a maximum length of 255, and is optional; 'blocks' is an array of objects, which is required. Each object in 'blocks' must contain the following required properties: 'id' is a string; 'blockType' is a string; 'startX' is an integer; 'startY' is an integer; 'width' is an integer; 'height' is an integer. The objects can also include optional properties: 'title' is a string with a maximum length of 255; 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$'; 'config' is an object. No additional properties are allowed at the root level.
