The schema describes a media file configuration. It is an object type. The schema resolves to the referenced definition for 'edit_info' and 'file' as effective root schemas. The main properties of the schema are: 'edit_info' is an object that contains required fields 'video_file' (a string in URI format), 'audio_file' (a string in URI format), and 'start_timecode' (a string). 'file' is an object that contains required fields 'title' (a string with a length of exactly 7 characters), 'author' (a string), 'licence' (a string), 'licence_uri' (a string in URI format), 'audio_stream' (an object with required fields 'title' (a string), 'author' (a string), 'author_uri' (a string in URI format), 'licence' (a string), 'licence_uri' (a string in URI format), and 'modifications' (a string)), and an optional 'video_stream' (an object that can contain fields 'title' (a string), 'author' (a string), 'author_uri' (a string in URI format), 'licence' (a string), 'licence_uri' (a string in URI format), and 'modifications' (a string)). No additional properties are allowed at the root level.
