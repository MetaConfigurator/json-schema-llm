The schema describes a fueling creation process. It is an object type. It defines several reusable components under 'definitions': 'numberWithUnit' is an object, representing a number associated with a unit. It requires 'value' of type number and 'unit' of type string. The main properties of the schema are: 'car' is a string required; 'comment' is a string optional; 'cost' is a reference to 'numberWithUnit' required; 'fuelType' is a string required; 'mileage' is a reference to 'numberWithUnit' required; 'missedFuelStop' is a boolean optional; 'time' is a string formatted as date-time required; 'volume' is a reference to 'numberWithUnit' required. No additional properties are allowed at the root level.
