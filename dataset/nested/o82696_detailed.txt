The schema describes a Spotify embed block. It is an object type. The main properties of the schema are: 'aside' is a boolean when true it means this block represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes. It is optional. 'css_class' is a string in rendering environments that support HTML the css_class can be appended to the dom elements' class attribute. It must match the pattern '^[\w\s-]+$'. It is optional. 'etag' is a string that must match the pattern '^[\w\.:-]+$'. It is optional. 'spotify_id' is a string that must match the pattern '^\w+$'. It is required. 'spotify_type' is a string representing the type portion of a Spotify URI. It must match the pattern '^(album|artist|episode|playlist|show|track)$'. It is required. 'updated_date' is a string representing an update that occurred on the node this block is attached to. It must be in date-time format. It is optional. No additional properties are allowed at the root level.
