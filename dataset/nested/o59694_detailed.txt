The schema describes a markup for a geek. It is an object type. It defines several reusable components under 'definitions': 'ranking' is an object, describes a geek's participation in geek phenomena - shows, games, etc., requiring 'name' and 'rank'. 'contact' is an object, describes a geek's contact information, requiring 'contactType' and 'contactValue'. 'media' is an object, describes a geek's social media information, requiring 'mediaType' and 'mediaLocator'. The main properties of the schema are: 'name' is a string, required; 'dateMade' is a string, required; 'summary' is a string, optional; 'contacts' is an array of contact objects, optional; 'socialMedia' is an array of media objects, optional; 'skills' is an array of unique ranking objects, optional; 'organizations' is an array of unique ranking objects, optional; 'games' is an array of unique ranking objects, optional; 'fandoms' is an array of unique ranking objects, optional. No additional properties are allowed at the root level.
