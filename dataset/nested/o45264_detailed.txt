The schema describes a structure for handling hooks in a database operation. It is an object type. It defines several reusable components under 'definitions': 'hook' is an object, describing async jobs for after callbacks. It must have properties matching the pattern 'after_[create|update|destroy]', which is an array containing objects. Each object must have a required property 'name', and may include optional properties 'run_at' and 'conditions'. The 'conditions' property is an array of strings. The array must contain at least 1 item and can have a maximum of 20 items. The main properties of the schema are: 'hook' is a reference to the 'hook' definition, required. 'table_name' is a string, required. 'record' is an object, optional. 'record_uuid' is a string, required. 'result' is an optional property of unspecified type. No additional properties are allowed at the root level.
