The schema describes logging captured sensor values. It is an object type. The main properties of the schema are: 'date' is a string or null, describing the date of the capture; 'interval' is a number or null, describing the capture interval configured for the source, in seconds; 'sensors' is an object or null, describing the sensor values captured on a Phidget board, with pattern properties that require at least one character key. Each sensor object must include 'description' (a string, describing the sensor), 'type' (a string, describing the sensor type), 'value' (a number, representing the captured value after applying the formula), and 'valueRaw' (a number, representing the captured value before applying the formula). No additional properties are allowed at the root level.
