The schema describes json responses for applications following the jsend format. It is an object type. It defines several reusable components under 'definitions': 'success' is an object, describing a successful response that requires 'status' and 'data'. 'fail' is an object, indicating a failure that also requires 'status' and 'data'. 'error' is an object, representing an error that requires 'status' and 'message'. 'data' is a generic container for various types of data, allowing strings, null, arrays, objects, integers, and booleans. 'code' is a numeric code representing an error, if applicable, and can be a string or integer. 'message' is a string providing a user-readable explanation of the error. The main properties of the schema are: The root schema resolves to the referenced definition. The 'success' definition has properties: 'status' is a string matching the pattern '^success$', and 'data' is a reference to 'data'. The 'fail' definition has properties: 'status' is a string matching the pattern '^fail$', and 'data' is a reference to 'data'. The 'error' definition has properties: 'status' is a string matching the pattern '^error$', 'message' is a reference to 'message', 'code' is a reference to 'code', and 'data' is a reference to 'data'. Compositional constraints require any instance to match one of the three definitions: 'success', 'fail', or 'error'. No additional properties are allowed at the root level.
