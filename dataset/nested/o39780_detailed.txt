The schema describes the result of a Github worker. It is an object type. It defines several reusable components under 'definitions': 'github_extracted_details' is an object, details of Github inspection. It has optional properties: 'forks_count' (integer), 'last_year_commits' (references 'github_last_year_commits_details'), 'open_issues_count' (integer), 'stargazers_count' (integer), 'subscribers_count' (integer), 'contributors_count' (integer), 'topics' (array of strings), 'license' (object), and 'updated_on' (string). It is required to include 'updated_on'. Additional properties are not allowed. 'github_last_year_commits_details' is an object, details of last year Github commits. It has optional properties: 'sum' (integer) and 'weekly' (array of integers). It requires 'sum' and 'weekly'. Additional properties are not allowed. The main properties of the schema are: 'schema' is an object, which has required properties: 'name' (string, must match pattern '^[a-zA-Z0-9_]+$') and 'version' (string, must match pattern '^[0-9]+-[0-9]+-[0-9]+$'); 'url' (string, must be a valid URI) is optional. '_release' is a string, unique release id in form of 'ecosystem:package:version'; 'status' is a string, required, and must be one of the values: 'success', 'error', 'unknown'; 'details' references 'github_extracted_details'; 'summary' is an array of strings. No additional properties are allowed at the root level.
