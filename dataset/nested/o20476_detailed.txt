The schema describes an authorization request. It is an object type. The main properties of the schema are: 'client_id' is a string that represents the OAuth 2.0 client identifier, obtained at registration. It is required. 'code_challenge' is a string that describes the code challenge, computed from the code verifier using a transform. It is optional. 'code_challenge_method' is a string that describes the code transform, defaults to plain if not specified. The supported methods are S256 (recommended) and plain. It is optional. 'nonce' is a string that serves as an opaque value, e.g. a random string, used to associate a client session with an ID Token and to mitigate replay attacks. Use of this parameter is required in the implicit flow. It is optional. 'redirect_uri' is a string and must exactly match one of the registered redirection URIs for the client. It is required. 'response_type' is a string that represents the OAuth 2.0 response type. For clients using the OAuth code flow, it should be set to code. For clients using the implicit flow, it should be set to id_token or token id_token. It is required and has an enum of allowed values: 'code', 'token', 'id_token'. 'scope' is an array that contains a space-separated list of the requested scope values. It must include at least the openid value. It is required and must contain items that are strings with the following allowed values: 'openid', 'profile', 'email', 'address', 'phone'. 'state' is a string that serves as an opaque value, e.g. a random string, used to maintain state between the request and the callback. Use of this parameter is not required but highly recommended. It is optional. No additional properties are allowed at the root level.
