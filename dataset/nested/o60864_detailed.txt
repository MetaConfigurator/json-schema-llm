The schema describes a project configuration. It is an object type. The main properties of the schema are: 'schema' is a string optional field that provides the schema identifier for payload. 'template' is an object required field that does not allow additional properties and contains pattern properties where keys can match any string and the values must be strings. 'tasks' is an array required field containing objects, each of which has a required field 'name' of type string and optional fields 'icon', 'group', and 'label', also of type string. 'apps' is an array required field with the same structure as 'tasks'. 'families' is an array required field containing objects, each of which has a required field 'name' of type string and optional fields 'icon', 'label', and 'hideFilter' of types string, string, and boolean respectively. 'groups' is an array required field containing objects, each of which has a required field 'name' of type string and optional fields 'icon' and 'color' of type string, along with an optional field 'order' that can be either an integer or a number. 'copy' is an optional object field. No additional properties are allowed at the root level.
