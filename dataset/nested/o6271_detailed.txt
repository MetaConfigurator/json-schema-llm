The schema describes a structured representation for a list of items along with pagination details. It is an object type. The main properties of the schema are: 'items' is an array containing objects, each with an optional 'id' (a string that must match the pattern '^[A-Fa-f\d]{24}$'), an optional 'webhookId' (a string that must match the pattern '^[A-Fa-f\d]{24}$'), an optional 'applicationId' (a string that must match the pattern '^[A-Fa-f\d]{24}$'), an optional 'creationDate' (a string formatted as date-time), an optional 'lastUpdated' (a string formatted as date-time), an optional 'name' (a string between 1 and 255 characters), and an optional 'token' (a string). 'count' is an integer representing the number of items in the current response. 'totalCount' is an integer representing the total number of items available. 'perPage' is an integer indicating the number of items per page. 'page' is an integer representing the current page number. 'filter' is a string used for filtering items. 'filterField' is a string specifying the field to filter by. 'sortField' is a string indicating the field to sort by. 'sortDirection' is a string that can be either 'asc' or 'desc' indicating the sorting order. No additional properties are allowed at the root level.
