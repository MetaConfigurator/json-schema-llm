The schema describes additional information concerning a whole submission rather than individual data tables. It is an object type. The main properties of the schema are: 'record_ids' is an array of objects, each containing required properties 'type' (a string) and 'id' (a string or number), with no additional properties allowed for each object. 'preprintyear' is an optional string indicating the year when the submission preprint was first available. 'publicationyear' is an optional string indicating the year when the submission publication was available. 'dateupdated' is an optional string indicating when the submission was last updated. 'modifications' is an array of objects, each containing required properties 'action' (a string) and 'who' (a string). 'additional_resources' is an array of objects, each containing required properties 'location' (a string) and 'description' (a string), as well as a required 'license' object with required properties 'name' (a string) and 'url' (a string), with no additional properties allowed for the license object. 'comment' is an optional string providing information that applies to all data tables. No additional properties are allowed at the root level.
