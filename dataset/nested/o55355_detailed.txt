The schema describes Zone Transfer Accept. It is an object type. The main properties of the schema are: 'transfer_accept' is an object that represents the details of a zone transfer request. It is required and has the following properties: 'created_at' is a string that indicates the date and time of request creation. It is read-only. 'id' is a string that serves as the zone transfer request identifier. It follows a specific pattern and is also read-only. 'key' is a string that indicates the password used to complete the transfer. It is required and may have a maximum length of 160 characters. 'links' is an object that contains additional links related to the transfer. Its property 'self' is a string formatted as a URL. 'status' is a string that reflects the zone status, with allowed values being 'ACTIVE', 'PENDING', 'DELETED', 'ERROR', 'COMPLETE'. It is read-only. 'updated_at' may be a string or null, indicating the date and time of the last request modification. It is read-only. 'zone_transfer_request_id' is a string that acts as a request identifier and follows a specific pattern. It is required. No additional properties are allowed at the root level.
