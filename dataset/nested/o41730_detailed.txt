The schema describes a routing entry for an Okapi proxy. It is an object type. The main properties of the schema are: 'methods' is an array that lists methods such as GET, POST, etc. It is optional and contains strings as items. 'pathPattern' is a string that defines a path pattern match, which is optional. 'path' is an optional string representing a path prefix match. 'rewritePath' is an optional string describing a rewrite path for filters, such as rewriting '/*' to '/events/*'. 'phase' is an optional string that indicates the phase, which may be 'auth', 'pre', or 'post'. 'level' is an optional string indicating the processing level, with a default value of 50. 'type' is an optional string for proxy handling, which can be one of: 'request-response', 'request-only', 'headers', 'redirect', 'system', 'internal', 'request-response-1.0', with a default of 'request-response'. 'redirectPath' is an optional string for an internal redirect path. 'unit' is an optional string representing the delay unit. 'delay' is an optional string for the delay between calls. 'permissionsRequired' is an array that may be null, containing strings representing required permissions for this entry, and is optional. 'permissionsDesired' is an array that may be null, containing strings for desired permissions and is also optional. 'modulePermissions' is an array that may be null, holding strings for module permissions for this entry, and is optional. 'delegateCORS' is a boolean indicating if CORS handling is delegated to the module, also optional. No additional properties are allowed at the root level.
