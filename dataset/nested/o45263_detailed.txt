The schema describes a database table definition. It is an object type. It defines several reusable components under 'definitions': 'hook' is an object, Async jobs for after callbacks. It has pattern properties for keys matching 'after_[create|update|destroy]' which are arrays of objects. Each object must have a 'name' property of type string and may include 'run_at' and 'conditions' properties of type string and array, respectively, with constraints on array item counts. The main properties of the schema are: 'table_schema' is a string required property, 'name' is a string required property, 'schema' is an object required property, and 'indexes' is an array with a minimum of 1 and a maximum of 10 items, each item being an object that must have a 'cols' property of type object with pattern properties for string values. Other properties 'unique' and 'concurrently' are optional booleans. No additional properties are allowed at the root level.
