The schema describes the structure of an asset folder JSON. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'id' is a string A unique identifier for the asset_folder.json, which is required. 'type' is a string The json_api type for any asset_folder.json, which is required and must match the pattern '^asset_folders$'. 'attributes' is an object This object contains various properties such as 'meta_attributes', 'updated_at', 'created_at', 'deleted_at', 'name', 'reference', 'children_count', 'depth', 'lft', and 'rgt', all of which are optional. 'relationships' is an object This object contains 'asset_files' and 'sub_folders', both of which are objects containing 'links' and 'data'. 'links' is an object that includes 'self' and 'related', both of which are strings and do not allow additional properties. 'data' must be either a single object with properties 'id' and 'type', or an array of such objects, and neither allows additional properties. 'meta' is an object for both 'asset_files' and 'sub_folders' which allows additional properties. 'links' is an object This links section contains 'self', which is a string and does not allow additional properties. 'meta' is an object that disallows additional properties. No additional properties are allowed at the root level.
