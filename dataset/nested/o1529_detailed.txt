The schema describes an Azure deployment parameter file. It is an object type. It defines several reusable components under 'definitions': 'parameter' is an object, representing a parameter with client-specific metadata and an input value. It requires the field 'value', which can be of type string, boolean, integer, array, object, or null. The main properties of the schema are: '$schema' is a string, required. 'contentVersion' is a string, required, with a pattern validating a 4 number format. 'parameters' is an object, required, with additional properties referencing the 'parameter' definition. No additional properties are allowed at the root level.
