The schema describes an asset file structure. It is an object type. The main properties of the schema are: 'id' is a string A unique identifier for the asset_file.json. 'type' is a string The json_api type for any asset_file.json, with a pattern that must match '^asset_files$'. 'attributes' is an object An object containing various attributes related to the asset, including 'meta_attributes', 'updated_at', 'created_at', 'deleted_at', 'name', 'reference', 'asset_file', 'file_content_filename', 'file_content_content_type', 'file_content_size', 'image_height', 'image_width', and 's3_url', each described specifically. 'relationships' is an object An object defining relationships, particularly with 'asset_folder', which contains links and data properties. 'links' is an object It contains a 'self' property, defined as a string. 'meta' is an object An empty object without additional properties. No additional properties are allowed at the root level.
