The schema describes the output of a generic job. It is an object type. The main properties of the schema are: 'status' is a string representing the result status of the job. It is required and can have one of the following values: 'Success', 'Failure', or 'Timeout'. 'message' is a string that should capture a summarized error message in case of Failures. It is optional. 'results' is an object that holds the job specific output details. It is optional. No additional properties are allowed at the root level.
