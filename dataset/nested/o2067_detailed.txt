The schema describes UCSC Genomics Core's schema for an analysis workflow's metadata record. It is an object type. It defines several reusable components under 'definitions': 'schema_version' is a string, describing the version of this schema with a pattern of '^0.0.3$'. 'time_stamp' is a string, describing ISO 8601 / RFC 3339 format, with a custom format '_format' set to 'date-time'. 'uuid' is a string, describing uuid with all lower case or all upper case characters, and must conform to one of two patterns: lower case using '^[0-9a-f]{8}-([0-9a-f]{4}-){3}[0-9a-f]{12}$' or upper case using '^[0-9A-F]{8}-([0-9A-F]{4}-){3}[0-9A-F]{12}$'. The main properties of the schema are: 'analysis_type' is a string describing the analysis type which needs to be tightly controlled for metadata merging, and it is required; it can be one of the following values: 'sequence_upload', 'alignment', 'rna_seq_quantification', 'germline_variant_calling', 'somatic_variant_calling', 'immuno_target_pipelines'. 'bundle_uuid' is a uuid, which is required and references the definition of 'uuid'. 'parent_uuids' is an array of uuids for this workflow and is required; it contains items that must conform to the 'uuid' definition. 'timestamp' is a string that references the 'time_stamp' definition, and it is required with the description indicating it is not required when uploading via the upload tool. 'workflow_name' is a string that is required. 'workflow_outputs' is an array of objects, each containing required properties 'file_path' and 'file_type', and an optional 'file_uuid' that references the 'uuid' definition. 'workflow_version' is a string that is required. No additional properties are allowed at the root level.
