The schema describes schema definitions for reusable properties. It is an object type. It defines several reusable components under 'definitions': 'url' is a string, a valid URL following the regex pattern '(https?|ftp):\/\/(-\.)?([^\s\/?\.#-]+\.?)+(\/[^\s]*)?$'. 'uri' is a string, the location for online access using a Uniform Resource Identifier (URI), with a regex pattern that validates various URI formats. 'date' is a string, an ISO 8601 date/timestamp validated by a specific regex pattern. 'dateObject' is an object, an ISO 8601 date/timestamp and type, which requires 'date' (a reference to the 'date' definition) and 'dateType' (a string with a minimum length of 1). It may also include an optional 'description' property for supplemental information. The main properties of the schema are: 'url' is a string required property that references the 'url' definition. 'uri' is a string required property that references the 'uri' definition. 'date' is a string required property that references the 'date' definition. 'dateObject' is an object required property that references the 'dateObject' definition. No additional properties are allowed at the root level.
