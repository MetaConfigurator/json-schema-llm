The schema describes configuration details for plugins. It is an object type. It defines several reusable components under 'definitions': 'orgName' is a string, Username may only contain alphanumeric characters or single hyphens, and cannot begin or end with a hyphen. 'pluginName' is a string, it must not match the pattern .{14,}(?:ghlint|plugin) and must match the pattern ^ghlint-plugin-(?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]$. 'pluginShortName' is a string, it must not match the pattern ghlint|plugin and must match the pattern ^(?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]$. 'repoName' is a string, it must match the pattern ^(?:(?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]/)?[A-Za-z0-9_-]+$. 'repoNamesList' is a string, it must match the pattern ^(?:(?:(?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]/)?[A-Za-z0-9_-]+\s*,\s*)*(?:(?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]/)?[A-Za-z0-9_-]+$. 'ruleName' is a string, it must match the pattern ^[A-Za-z](?:[A-Za-z0-9]+[-]?)+[A-Za-z0-9]$. 'teamName' is a string, no description available. The main properties of the schema are: 'orgName' is a string required, resolving to the definition 'orgName'. 'pluginName' is a string required, resolving to the definition 'pluginName'. 'pluginShortName' is a string required, resolving to the definition 'pluginShortName'. 'repoName' is a string required, resolving to the definition 'repoName'. 'repoNamesList' is a string required, resolving to the definition 'repoNamesList'. 'ruleName' is a string required, resolving to the definition 'ruleName'. 'teamName' is a string required, resolving to the definition 'teamName'. No additional properties are allowed at the root level.
