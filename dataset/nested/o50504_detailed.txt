The schema describes files within the filesystem. It is an object type. It defines several reusable components under 'definitions': 'stats' is an object, which includes fields: 'size' (an integer, required, minimum value 0), 'mode' (an integer, optional), 'write' (a boolean, optional), '_uid' (an integer, optional), and '_gid' (an integer, optional). 'name' is an object, which includes fields: 'oldname' (a string, optional) and 'moniker' (a string, optional). The main properties of the schema are: 'depth' is an integer, optional, with a minimum value of 0; 'stats' is a reference to the 'stats' definition; 'name' is a reference to the 'name' definition. No additional properties are allowed at the root level.
