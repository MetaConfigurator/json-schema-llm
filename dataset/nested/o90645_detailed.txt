The schema describes the configuration properties for a DC/OS service. It is an object type. The main properties of the schema are: 'service' is an object that contains several properties related to the service configuration. It is required and defines the following properties: 'ceph_client_admin_keyring_URI' is a string that specifies the URI location of the ceph.client.admin.keyring keyring file used to authenticate against the Ceph cluster. It is required. 'ceph_conf_URI' is a string that indicates the URI location of the ceph.conf configuration file holding the Ceph cluster configuration. It is required. 'cpus' is a number that represents the CPU shares to allocate to this service instance, with a default value of 0.5 and a minimum of 0.3. It is required. 'external_access' is an object that enables access from outside the cluster through Marathon-LB; it is optional and defines the properties: 'enable' is a boolean that indicates whether to create a VIP for external access, with a default value of true. 'external_access_port' is a number that specifies the port number for clear communication in the external load balancer, with a default value of 15000. 'virtual_host' is a string that denotes the Virtual Host URL to be used in the external load balancer, with a default value of 'ceph-dash.example.org'. 'mem' is a number that denotes the memory to allocate to this service instance, with a default value of 512.0 and a minimum of 512.0. It is required. 'name' is a string that represents the name of this service instance, with a default value of 'ceph-dash'. No additional properties are allowed at the root level.
