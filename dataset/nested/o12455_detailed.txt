The schema describes module specifications. It is an object type. It defines several reusable components under 'definitions': 'coordinates' is an object, which requires the fields 'x' and 'y'. It allows additional properties to be false and includes properties 'x', 'y', and 'z', each of type number. The main properties of the schema are: Each pattern property is an object that requires the fields 'labwareOffset', 'dimensions', 'calibrationPoint', 'displayName', 'loadName', and 'quirks'. 'labwareOffset' is a required property that references the 'coordinates' definition. 'dimensions' is a required object that requires 'bareOverallHeight' and 'overLabwareHeight'. It allows additional properties to be false and includes 'bareOverallHeight', 'overLabwareHeight', and optionally 'lidHeight', all of type number. 'calibrationPoint' is a required property that references the 'coordinates' definition. 'displayName' is a required string. 'loadName' is a required string. 'quirks' is a required array of strings, described as a list of quirks for a given module. No additional properties are allowed at the root level.
