The schema describes an event message structure. It is an object type. The main properties of the schema are: 'meta' is an object that contains system metadata for the event and is required. It includes the required properties: 'id' which is a string following a specific pattern, 'type' which is a string and can only be 'EiffelServiceDeployedEvent', 'version' which is a string defaulting to '1.0.0' and can only be '1.0.0', 'time' which is an integer, and 'source' which is an object that requires 'domainId', 'host', 'name', 'serializer' (where 'serializer' is an object requiring 'groupId', 'artifactId', and 'version' as strings), and 'uri', with no additional properties allowed. 'data' is an object that is required, containing the properties: 'servicename', 'instancename', and 'owner' as required strings, and optional properties 'deploymentType' and 'uri', and 'customData' which is an array of objects that are required to have 'key' and 'value' properties, with no additional properties allowed. 'links' is an array of objects that are required and must include 'type' as a string and 'target' as a string following a specific pattern, with no additional properties allowed. No additional properties are allowed at the root level.
