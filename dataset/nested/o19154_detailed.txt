The schema describes Infra Topology for openstack. It is an object type. It defines several reusable components under 'definitions': 'email' is a string type. The main properties of the schema are: 'topology_name' is a string which is required. 'date' is a string which is optional. 'infra_type' is a string that is required and must be one of the following values: 'aws', 'openstack', 'gce', 'beaker_bm'. 'site' is a string which is required. 'credentials' is an array that is required, containing a minimum of one string item. 'resources' is an array which is optional, containing a minimum of one object with the following properties: 'resource_name' is a string which is required. 'count' is a number which is required. 'res_def' is an object which is required and must match one of the specified definitions. It may require properties 'flavor', 'image', 'keypair', and 'networks' in one configuration, and 'name' in a different configuration. 'associated_creds' is a string which is optional. No additional properties are allowed at the root level.
