The schema describes a Vimeo video block. It is an object type. The main properties of the schema are: 'aside' is a boolean that indicates whether this block represents a portion of a document whose content is only indirectly related to the document's main content; it is optional. 'autoplay' is a boolean that indicates whether the video should autoplay; it is optional. 'css_class' is a string that can be appended to the DOM elements' class attribute, matching the pattern '^[\w\s-]+$'; it is optional. 'description' is a string with a maximum length of 5000 and a minimum length of 0; it is optional. 'etag' is a string that matches the pattern '^[\w\.:-]+$'; it is optional. 'id' is a string that matches the pattern '^\d+$'; it is required. 'loop' is a boolean that indicates whether the video should loop; it is optional. 'poster_image_ref' is a string that is a reference to an image asset, matching the pattern '^[\w\/\.:-]+$'; it is optional. 'show_byline' is a boolean that indicates whether to show the byline in the thumbnail; it is optional. 'show_portrait' is a boolean that indicates whether to show the portrait in the thumbnail; it is optional. 'show_title' is a boolean that indicates whether to show the video title in the thumbnail; it is optional. 'title' is a string with a maximum length of 255 and a minimum length of 0; it is optional. 'updated_date' is a string in 'date-time' format that represents an update that occurred on the node this block is attached to; it is optional. 'user_id' is a string that matches the pattern '^[\w\.-]+$'; it is optional. 'user_name' is a string that matches the pattern '^[\s\w\.-]+$'; it is optional. No additional properties are allowed at the root level.
