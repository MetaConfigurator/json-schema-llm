The schema describes a list of RNAcentral entries to be imported by Ensembl. It is an array type. The main properties of the schema are: Each entry is an object with several properties: 'description' is a string representing the human-readable name of the sequence, it is required and must have a minimum length of 1. 'md5' is a string representing the MD5 hash of the sequence in lowercase, it is required and must match the pattern /^[a-f\d]{32}$/. 'rna_type' is a string indicating the INSDC type of the sequence, it is required with an enum that includes values such as 'RNase_MRP_RNA', 'miRNA', 'tRNA', among others. 'rnacentral_id' is a string specifying the unique URS of the sequence in RNAcentral, it is required and must match the pattern /^URS[A-Z0-9]{10}_\d+$. 'sequence' is a string for the sequence in uppercase with U's, it is required, must have a minimum length of 10, and match the pattern /^[ACGTYRWSKMDVHBNXFI]+$/. 'taxon_id' is a number representing the NCBI taxon id of the organism, it is required. 'xrefs' is an array of objects, each representing a list of cross references to other databases, it is required and must contain unique items; each array item must have properties: 'database', a string with an enum of values including 'GtRNAdb', 'miRBase', and 'Rfam', and 'id', which is a required string with a minimum length of 1. The items in the list must have a minimum of one item and maintain uniqueness. No additional properties are allowed at the root level.
