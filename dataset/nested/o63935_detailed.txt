The schema describes a specification for video thumbnails output. It is an object type. The main properties of the schema are: 'output_type' is a string indicating the type of the file to output, which is optional and has a default value of 'THUMB'. 'mode' is a string specifying the type of thumbnails generation needed, which is optional and can be either 'snapshot' or 'intervals'. 'output_bucket' is a string representing the S3 bucket and path where the thumbnails will be uploaded, which is optional and must match the pattern '^[0-9a-zA-Z\.\-_]+$', with a minimum length of 3 and a maximum length of 63. 'output_file' is a string for the basename of the thumbnails to generate, which is optional and must follow the pattern '^[\w /-]+\.[0-9a-zA-Z]+$', having a minimum length of 1 and a maximum length of 1024. 'intervals' is an integer to override the default interval in seconds, which is optional with a minimum value of 1. 'snapshot_sec' is an integer indicating the time in seconds in the video to take the snapshot, which is optional with a minimum value of 0. 's3_rrs' is a boolean to activate reduced redundancy in S3 storage, which is optional. 's3_encrypt' is a boolean to activate backend storage encryption, which is optional. 'size' is a string representing the size of the thumbnails, which is required and must match the pattern '^[0-9]+x[0-9]+$'. No additional properties are allowed at the root level.
