The schema describes an HTTP request configuration. It is an object type. It defines several reusable components under 'definitions': '_methodNoBody' is an enum, allowing the values 'GET' and 'DELETE'. '_methodWithBody' is an enum, allowing the values 'POST', 'PATCH', and 'PUT'. 'responsePath' is a string, representing the JSON path for where to store the response and requires a minimum length of 1. 'uriTemplate' is a string, requiring a minimum length of 1, and has a custom error message indicating that it is required. 'bodyTemplate' is a string representing the request body. 'headerInfo' is an array, with each item being an object that requires 'key' (a string with a minimum length of 1) and 'valueTemplate' (a string with a minimum length of 1), and does not allow additional properties. The main properties of the schema are: 'method' is a string that references either '_methodNoBody' or '_methodWithBody' depending on the provided configuration, 'responsePath' is a string that references 'responsePath' and is optional, 'uriTemplate' is a string that references 'uriTemplate' and is required, 'bodyTemplate' is a string that references 'bodyTemplate' and is required only if using a method that requires a body, and 'headerInfo' is an array that references 'headerInfo'. The schema requires either the first configuration with 'method', 'uriTemplate' or the second configuration with 'method', 'uriTemplate', 'bodyTemplate'. No additional properties are allowed at the root level.
