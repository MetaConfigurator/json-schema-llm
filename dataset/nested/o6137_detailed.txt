The schema describes a configuration for a service deployment. It is an object type. The main properties of the schema are: 'metrics' is an object that contains details for Prometheus metrics, including an optional boolean property 'enabled' to create a side-car container for exposing metrics and an optional object property 'serviceMonitor', which includes an optional boolean property 'enabled' to create a ServiceMonitor. 'persistence' is an object that configures persistence with an optional boolean property 'enabled' to enable persistence using Persistent Volume Claims and an optional string property 'size' for persistent volume size with specific rendering rules. 'rabbitmq' is an object containing credentials with optional string properties 'username' for the RabbitMQ user and 'password' that defaults to a random 10-character alphanumeric string if not set. 'replicas' is an integer representing the number of replicas to deploy, and 'volumePermissions' is an object that includes an optional boolean property 'enabled' to use an init container for setting folder permissions. No additional properties are allowed at the root level.
