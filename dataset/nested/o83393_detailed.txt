The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': 'blockName' is a string, matching the pattern '^[a-z]+(?:-+[a-z]+)*-v\d+$'. 'blockGroup' is a string, matching the pattern '^[a-z0-9]+(?:-[a-z0-9]+)*$'. 'sectionConstraint' is an object, representing schema for documenting and validating section constraints. It has properties: 'allowedBlocks', which is an array of blockName and must contain at least 1 item; 'deprecatedBlocks', which is an array of blockName and must contain at least 1 item; 'defaultBlocks', which is an array of blockName; 'allowedBlockGroups', which is an array of blockGroup; 'min', which is a number between 1 and 1000; 'max', which is a number between 1 and 1000; 'size', which is a number between 1 and 1000; and 'optional', which is a boolean. The main properties of the schema are: 'name' is a string and required. 'label' is a string and required. 'version' is an integer with a default value of 1 and is required. 'sections' is an array of sectionConstraint and required. No additional properties are allowed at the root level.
