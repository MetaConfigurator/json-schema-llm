The schema describes a consulate appointments transaction record by channel. It is an object type. The main properties of the schema are: '_id' is a string unique url-friendly base64 encoded, utf8 encoded concatenation identifier. It is required. '_timestamp' is a string ISO8601 combined date and time in UTC e.g. '2014-12-01T00:00:00+00:00'. It is required. 'channel' is a string user contact channel. It is required. 'consulate' is a string The name or identity of the consulate providing the appointment. It is required. 'count' is an integer the number of completed transactions over the specific channel. It is required and must be at least 0. 'period' is a string The period covered by each data point from the _timestamp. It is required and allowed values are: 'hour', 'day', 'week', 'month', 'quarter'. 'service_type' is a string The name of the service needed by the user e.g. emergency documentation, register birth etc. It is required. 'transaction' is a string The user transaction e.g. book-appointment, cancel-appointment etc. It is required. No additional properties are allowed at the root level.
