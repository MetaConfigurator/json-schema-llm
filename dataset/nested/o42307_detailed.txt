The schema describes an employee record with associated address and phone information. It is an object type. It defines several reusable components under 'definitions': 'EmployeeSingle' is an object, containing properties 'id' (a string), 'name' (a string), 'address-id' (a string), and 'phone-id' (a string). No additional properties are allowed. 'AddressSingle' is an object, containing properties 'aid' (a string), 'street' (a string), 'city' (a string), 'country' (a string), and 'zip' (a string). No additional properties are allowed. 'PhoneSingle' is an object, containing properties 'id' (a string) and 'number' (a string). No additional properties are allowed. The main properties of the schema are: 'root' is an object containing properties 'employee' (a required reference to the 'EmployeeSingle' definition), 'address' (an optional array of 'AddressSingle' items), and 'phone-number' (an optional array of 'PhoneSingle' items). No additional properties are allowed at the root level.
