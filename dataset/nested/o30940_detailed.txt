The schema describes SSL Certificate Revocation List (CRL) configuration. It is an object type. The main properties of the schema are: 'basedn' is a string representing the base distinguished name. 'binary' is a string, which can either be 'YES' or 'NO'. 'binddn' is a string representing the bind distinguished name. 'cacert' is a string for the certificate authority certificate. 'cacertfile' is a string for the certificate authority certificate file. 'cakeyfile' is a string for the certificate authority key file. 'crlname' is a string for the CRL name. 'crlpath' is a string for the CRL path. 'day' is an integer representing the day. 'daystoexpiration' is a readonly integer indicating the days to expiration. 'flags' is a readonly integer for flags. 'gencrl' is a string for generating a CRL. 'indexfile' is a string for the index file. 'inform' is a string, which can either be 'DER' or 'PEM'. 'interval' is a string, which can be 'MONTHLY', 'WEEKLY', 'DAILY', 'NOW', or 'NONE'. 'issuer' is a readonly string for the issuer. 'lastupdate' is a readonly string for the last update. 'lastupdatetime' is a readonly integer for the last update time. 'method' is a string, which can either be 'HTTP' or 'LDAP'. 'nextupdate' is a readonly string for the next update. 'password' is a string for the password. 'port' is an integer for the port. 'refresh' is a string, which can be 'ENABLED' or 'DISABLED'. 'revoke' is a string for revocation. 'scope' is a string, which can either be 'Base' or 'One'. 'server' is a string for the server. 'signaturealgo' is a readonly string for the signature algorithm. 'time' is a string for the time. 'url' is a string for the URL. 'version' is a readonly integer for the version. No additional properties are allowed at the root level.
