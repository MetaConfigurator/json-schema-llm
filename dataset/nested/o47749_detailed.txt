The schema describes the configuration for a virtual machine image. It is an object type. It defines several reusable components under 'definitions': 'cpu' is an object, representing the virtual CPU. It contains an array of features, which must include items such as 'xsave', 'avx', and 'aes', and a model which can be one of 'host', 'pentium', 'qemu64', or 'kvm64'. The main properties of the schema are: 'bios' is a string that provides a 64k BIOS image, 'cpu' is a reference to the 'cpu' definition, 'description' is a string that offers a human-readable description of this config, 'drives' is an array of objects representing additional virtual hard drives, where each object must have required fields 'file', 'type', 'format', and 'media'; 'image' is a string with a default value of 'service.img', describing a bootable virtual machine image, 'mem' is a number with a default value of 128, indicating the amount of memory in megabytes, 'modules' is an array of objects containing multiboot 'modules', each having properties for 'args' and 'path', 'net' is an array of objects representing network devices, where each object requires the 'device' field; 'smp' is a number indicating the number of virtual CPUs, 'vfio' is a string for VFIO PCI-passthrough on device, and 'vga' is an enumeration with allowed values 'std', 'cirrus', 'vmware', 'qxl', 'xenfb', 'tcx', 'cg3', 'virtio', 'none'. No additional properties are allowed at the root level.
