The schema describes a structured object for managing plugin metadata. It is an object type. It defines several reusable components under 'definitions': 'htmlStringLines' is an array, which requires at least one string item, joined by \n. 'urlString' is a string, which must match the pattern '^(ftp|http|https)://[^ "]+$' and is formatted as a URI. 'fileLocations' is an object, which requires at least one property and does not allow additional properties; it has 'main' and 'user' as string properties. The main properties of the schema are: 'name' is a string and is required. 'previousVersions' is an object, which requires at least one property and allows additional string properties. 'description' is an array, which must satisfy 'htmlStringLines' definition. 'links' is an object, which requires at least one property, has 'Website', 'Fixture library download', and 'Fixture format documentation' as references to 'urlString', and allows additional properties that also reference 'urlString'. 'fixtureUsage' is an array, which must satisfy 'htmlStringLines' definition. 'fileLocations' is an object, which requires at least one property; it includes 'subDirectoriesAllowed' as an optional boolean, 'Windows', 'Mac OS', and 'Linux' as references to 'fileLocations', and allows additional properties that also reference 'fileLocations'. 'additionalInfo' is an array, which must satisfy 'htmlStringLines' definition. 'helpWanted' is a string and is optional. The schema enforces that additional properties are not allowed at the root level.
