The schema describes NodeInfo schema version 1.0. It is an object type. The main properties of the schema are: 'version' is a string the schema version, must be 1.0, and is required with an enum value of ['1.0']. 'software' is an object metadata about server software in use, and is required. It contains 'name', which is an enum with values ['diaspora', 'friendica', 'redmatrix'] and 'version', which is a string and required. 'protocols' is an object the protocols supported on this server, and is required. It contains 'inbound', which is an array of enums with a minimum of 1 item allowed, and the allowed values are ['buddycloud', 'diaspora', 'friendica', 'gnusocial', 'libertree', 'mediagoblin', 'pumpio', 'redmatrix', 'smtp', 'tent'], and 'outbound', which is an array of enums with a minimum of 1 item allowed and the same allowed values. 'services' is an object the third party sites this server can connect to via their application API, and is required. It contains 'inbound', which is an array of enums with a minimum of 0 items allowed, and the allowed values are ['appnet', 'gnusocial', 'pumpio'], and 'outbound', which is an array of enums with a minimum of 0 items allowed with the allowed values being ['appnet', 'blogger', 'buddycloud', 'diaspora', 'dreamwidth', 'drupal', 'facebook', 'friendica', 'gnusocial', 'google', 'insanejournal', 'libertree', 'linkedin', 'livejournal', 'mediagoblin', 'myspace', 'pinterest', 'posterous', 'pumpio', 'redmatrix', 'smtp', 'tent', 'tumblr', 'twitter', 'wordpress', 'xmpp']. 'openRegistrations' is a boolean whether this server allows open self-registration. 'usage' is an object usage statistics for this server, and is required. It contains 'users', which is an object with required 'total', 'activeHalfyear', and 'activeMonth', each being an integer with a minimum of 0. 'localPosts' is an integer, the amount of posts made by registered users with a minimum of 0. 'localComments' is an integer, the amount of comments made by registered users with a minimum of 0. 'metadata' is an object free form key value pairs for software specific values, with additionalProperties allowed. No additional properties are allowed at the root level.
