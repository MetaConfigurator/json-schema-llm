The schema describes QISKit device backend configuration. It is an object type. The main properties of the schema are: 'name' is a string required field describing the name of the backend. 'version' is a string optional field providing the version of the backend. 'url' is a string optional field with a format of uri for the backend's URL. 'simulator' is a boolean required field with a default value of false indicating if the backend is a simulator. 'local' is a boolean required field with a default value of false indicating if the backend is local. 'description' is a string optional field providing a description of the backend. 'n_qubits' is an integer optional field with a minimum value of 1, specifying the number of qubits. 'coupling_map' is an optional field that can be either a string with an enum of 'all-to-all' or an array of integer arrays, where each inner array has exactly 2 items and all items are unique. 'basis_gates' is a string optional field specifying the basis gates of the backend. 'online_date' is a string optional field with a format of date indicating the online date of the backend. 'chip_name' is a string optional field giving the name of the chip. No additional properties are allowed at the root level.
