The schema describes a collection of payment transactions. It is an array type. It defines several reusable components under 'definitions': 'ISO8601-Date' is a string, matching the pattern '^\d{4}-\d{2}-\d{2}$'. 'financialPeriodType' is an object, with properties 'startDate' and 'endDate', both referencing 'ISO8601-Date'. 'IDType' is a string, matching the pattern '^\d{10}$'. 'LISAManagerReferenceNumberType' is a string, matching the pattern '^Z\d{6}$'. 'paymentMethodType' is a string, with enum values 'directDebit' and 'BACS'. 'paymentDirectionType' is a string, with enum values 'credit' and 'debit'. 'paymentStatusType' is a string, with enum values 'pending', 'submitted', and 'completed'. The main properties of the schema are: 'LISAManagerReferenceNumber' is a LISAManagerReferenceNumberType required field. 'paymentReference' is an IDType required field. 'method' is a paymentMethodType required field. 'direction' is a paymentDirectionType required field. 'paymentDate' is an ISO8601-Date required field. 'amount' is a number required field. 'status' is a paymentStatusType optional field. 'financialPeriod' is a financialPeriodType required field. '_links' is an object optional field. No additional properties are allowed at the root level.
