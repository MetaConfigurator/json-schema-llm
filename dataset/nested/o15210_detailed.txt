The schema describes a collection of spell metadata. It is an object type. It defines several reusable components under 'definitions': 'spellMeta' is an object, representing the metadata of a spell that includes required fields: 'name', 'source', and 'data'. The 'data' property is an object that cannot have additional properties and includes various string fields such as 'Save', 'Damage', 'Damage Type', 'Target', among others, all of which are optional. The main properties of the schema are: 'spell' is an array of spellMeta objects. The root schema requires the 'spell' property to be present as an array of items conforming to the spellMeta definition. No additional properties are allowed at the root level.
