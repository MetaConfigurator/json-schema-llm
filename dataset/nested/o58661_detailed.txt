The schema describes the JSON representation of an executable unit. It is an object type. It defines several reusable components under 'definitions': 'variableName' is a string, a simple name for a variable. 'internalFunctionName' is a string, a name used internally for a function. 'constructorName' is a string, a name for a constructor. 'valueTypeName' is a string, a name denoting a value type. 'literalValue' is an object with no specified properties. 'typedVariableName' is an object, representing a variable with a type used for introducing variables into scope. It requires 'name' and 'type', both referencing definitions. 'functionDefinition' is an object, defining a function with properties 'name', 'params', and 'body'. It requires 'name', 'params', and 'body', with 'params' being an array of 'typedVariableName'. 'pattern' is an object schema that can take several forms: literal pattern (requires 'patType', 'value', and 'type'), variable pattern (requires 'patType', 'value', and 'type'), constructor pattern (requires 'patType', 'name', and 'args'), and tuple pattern (requires 'patType' and 'components'). 'expression' is an object schema with several valid configurations, including literal expression, variable expression, let expression (requiring 'expType', 'bindings', and 'main'), case expression (requiring 'expType', 'discriminant', and 'branches'), constructor application (requiring 'expType', 'name', and 'args'), function applications (both internal and external), and letfun expression (requiring 'expType', 'defs', and 'main'). The main properties of the schema are: 'name' is an optional object, 'params' is an array of 'typedVariableName', defaulting to an empty array, and 'body' is a reference to 'expression'. The schema is required to have 'name' and 'body'. No additional properties are allowed at the root level.
