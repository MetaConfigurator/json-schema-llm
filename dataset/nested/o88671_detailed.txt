The schema describes the configuration for a system. It is an object type. The main properties of the schema are: 'self' is an object that contains 'type', which is also an object with required properties 'id' (a string) and 'name' (a string), and an optional 'icon' (a string), and 'module' (a string, required) and 'extra' (an object, optional); 'zookeeper' is an object that has required properties 'host' (a string) and 'port' (an integer); 'meshblu_server' is an object that has required properties 'db_url' (a string), 'host' (a string), 'port' (an integer), and 'message_timeout' (an integer). The schema requires 'self' and 'zookeeper' at the root level. No additional properties are allowed at the root level.
