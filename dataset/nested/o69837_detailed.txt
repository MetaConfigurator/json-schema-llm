The schema describes a form structure. It is an object type. The main properties of the schema are: 'divs' is an array containing objects, each with a required 'heading' string and an optional 'fields' array of objects. Each field object is required to have 'field-name' and 'component-type', with 'component-type' being a string that must be one of 'textarea', 'simple', or 'complex'. The 'component-confs' property in each field can define configuration for different component types, with specific properties for 'simple', 'complex', and 'textarea'. 'properties' is an object with an optional 'additional-classes' string. The 'validation' property is an object with optional properties including 'focus-id', 'save-rules' array that can only have 'required' as its value, 'validation-id', and a 'validation-messages' object requiring a 'required' string. 'form-footer' is an optional string representing the footer template name. 'form-layout' is an object with an optional 'component-type' string defaulting to 'tabbed-wizard', a 'tabHeaderIdentifier' string defaulting to 'armsTabHeader', and a 'wizardDefPath' string defaulting to 'wizard-definitions/arms-draft.json'. No additional properties are allowed at the root level.
