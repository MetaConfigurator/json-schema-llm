The schema describes the details for a product return. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'items' is an object that contains product details, which is required. It includes the following properties: 'seq' is an integer representing the sequence number; 'codigo' is a string or null representing the code; 'descricao' is a string or null for the description; 'cpfCnpjRaiz' is a string for the root CPF or CNPJ; 'situacao' is a string indicating the situation; 'modalidade' is a string for the modality; 'ncm' is a string for the NCM code; 'codigoNaladi' is an integer or null for the Naladi code; 'codigoGPC' is an integer or null for the GPC code; 'codigoGPCBrick' is an integer or null for the GPC Brick code; 'codigoUNSPSC' is an integer or null for the UNSPSC code; 'paisOrigem' is a string for the country of origin; 'cpfCnpjFabricante' is a string or null for the manufacturer's CPF or CNPJ; 'fabricanteConhecido' is a boolean indicating whether the manufacturer is known; 'codigoOperadorEstrangeiro' is an integer or string for the foreign operator code; 'atributos' is an array or null of objects each with 'atributo' as a string and 'valor' as a string; 'codigosInterno' is an array or null of strings containing internal codes. No additional properties are allowed at the root level.
