The schema describes cell metadata for nbgrader. It is an object type. The main properties of the schema are: 'schema_version' is an integer the version of the cell metadata schema. It is required and must be exactly 3. 'grade' is a boolean whether this cell should be graded. It is required and has a default value of false. 'locked' is a boolean whether the cell should be editable. It is required and has a default value of false. 'solution' is a boolean whether this cell contains a solution. It is required and has a default value of false. 'task' is a boolean whether this cell contains a task. It is optional and has a default value of false. 'grade_id' is a string the nbgrader id of the cell, only present if one or more of grade/locked/solution is true. It is optional and must match the pattern '^[a-zA-Z0-9_\-]*$'. 'points' is a number the number of points this cell is worth; only present if grade is true. It is optional and must be at least 0. 'checksum' is a string a checksum of the cell contents; generally only present in the student version of an assignment. It is optional and must be exactly 32 characters long. 'cell_type' is a string the type the cell was when nbgrader generate_assignment was run; used for checking if the cell type has changed and should only be present in the student version of an assignment. It is optional and must be one of the following values: 'raw', 'code', 'markdown'. No additional properties are allowed at the root level.
