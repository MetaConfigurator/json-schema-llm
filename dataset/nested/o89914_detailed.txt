The schema describes a representation of packages metadata. It is an object type. It defines several reusable components under 'definitions': 'versions' is an object, a hashmap of versions and their metadata, with additional properties referencing 'version'. 'version' is an object, which can be either a reference to 'package' or 'metapackage'. 'package-base' is an object with properties including 'name', 'type', 'version', 'version_normalized' (optional), 'autoload', 'require', 'replace', 'conflict', 'provide', and 'time', allowing additional properties. 'package' is an object that includes 'package-base' and requires either 'name', 'version', 'source' or 'name', 'version', 'dist'. 'metapackage' is an object that includes 'package-base' and requires 'name', 'version', and 'type' to be 'metapackage'. 'provider' is an object with a property 'sha256', which describes a hash value for resource validation. The main properties of the schema are: 'packages' is an object or array, a hashmap of package names in the form of <vendor>/<name> with additional properties referencing 'versions'; 'providers-url' is a string, an endpoint for retrieving provider data; 'provider-includes' is an object, a hashmap of provider listings with additional properties referencing 'provider'; 'providers' is an object, a hashmap of package names in the form of <vendor>/<name> with additional properties referencing 'provider'; 'notify-batch' is a string, an endpoint to call after multiple packages have been installed; 'search' is a string, an endpoint providing search capabilities; 'warning' is a string, a message output as a warning. One of the following must be present: 'packages', 'providers', or both 'provider-includes' and 'providers-url'. No additional properties are allowed at the root level.
