The schema describes a Document associated with user filings. It is an object type. The main properties of the schema are: 'document' is an object containing required fields such as 'attachment_id', 'byte_size', 'col_24', 'comment', 'date_modified', 'date_ocr', 'date_pdf_modified', 'date_processed', 'description', 'document_id', 'edocs_link', 'file_description', 'file_name', 'file_type', 'file_type_id', 'flag_archived', 'flag_conv_problem', 'flag_migrated_from_rips', 'id', 'native_app_byte_size', 'native_app_id_file_type', 'ocr_flag', 'official_title', 'page_count', 'page_count_display', 'transmission_end_date', and 'transmission_start_date'. The 'file_type' property is an object that contains required fields 'file_extension', 'file_type_description', 'id', and 'mime_type'. No additional properties are allowed at the root level.
