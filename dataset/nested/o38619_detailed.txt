The schema describes the structure for a client configuration. It is an object type. The main properties of the schema are: 'client_name' is a string required property. 'client_uri' is a string required property expected to be in URI format. 'exp' is an integer required property. 'grant_types' is an array of strings that can include the values 'authorization_code', 'implicit', 'password', 'client_credentials', 'refresh_token'. 'iat' is an integer required property. 'initiate_login_uri' is a string property expected to be in URI format. 'iss' is a string required property. 'logo_uri' is a string property expected to be in URI format. 'redirect_uris' is an array of strings that must contain at least one item and each item is expected to be in URI format. 'scope' is a string property. 'software_id' is a string property. 'token_endpoint_auth_method' is a string property which can be one of 'none', 'client_secret_post', 'client_secret_basic'. No additional properties are allowed at the root level.
