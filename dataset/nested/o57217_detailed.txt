The schema describes configuration for a LightStep tracer. It is an object type. It defines several reusable components under 'definitions': 'endpoint' is an object, representing an address composed of a host and a port. It requires 'host' which is a string and 'port' which is an integer with a minimum value of 1 and a maximum value of 65535. The main properties of the schema are: 'access_token' is a string, required to configure the LightStep project; 'collector_host' is an optional string for the host to communicate with the LightStep collector; 'collector_plaintext' is an optional boolean indicating whether to use plaintext for communication; 'collector_port' is an optional integer, with a minimum of 1 and maximum of 65535, for communicating with the LightStep collector; 'component_name' is an optional string representing the human-readable identity of the instrumented process; 'max_buffered_spans' is an optional integer, with a minimum of 1, indicating the maximum spans to buffer; 'propagation_modes' is an optional array of strings for listing propagation modes, allowing values: "lightstep", "b3", "envoy", and "trace_context"; 'report_timeout' is an optional integer, with a minimum of 1, indicating the timeout for reports in microseconds; 'reporting_period' is an optional integer, with a minimum of 1, indicating the maximum time between sending spans, also in microseconds; 'satellite_endpoints' is an optional array of endpoints, each referencing the 'endpoint' definition; 'ssl_root_certificates' is an optional string for CA certificates in PEM format; 'tags' is an optional object for key-value pairs applied to spans; 'use_single_key_propagation' is an optional boolean to inject/extract span context as a single key; 'use_stream_recorder' is an optional boolean enabling faster span uploads; 'verbose' is an optional boolean to log at the debug level. No additional properties are allowed at the root level.
