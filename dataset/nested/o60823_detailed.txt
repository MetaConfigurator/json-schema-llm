The schema describes api-box input json schema. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'endpoints' is an array of objects containing required property 'path', with items that are objects requiring 'method' and 'result'. 'method' is a string with an enum of allowed values: 'GET', 'POST', 'PUT', 'DELETE'. 'result' is a value that can be a string, number, boolean, object, or array. 'global' is an object, optional. 'name' is a string, required. 'prefix' is a string, optional. 'version' is a string, optional. Compositional constraints require 'name' and 'endpoints' at the root level. No additional properties are allowed at the root level.
