The schema describes a performance platform for user satisfaction with multiple quantitative-based questions plus comment records. It is an object type. The main properties of the schema are: '_id' is a string unique url-friendly base64 encoded, utf8 encoded concatenation identifier, required. '_timestamp' is a string ISO8601 combined date and time in UTC e.g. '2014-12-01T00:00:00+00:00', required. 'period' is a string the period covered by each data point from the _timestamp, required. Allowed values are: 'hour', 'day', 'week', 'month', 'quarter'. 'question' is a string the identifier for the question, required. 'rating_1' is an integer the number of respondents answering with the lowest ranked score, required, minimum 0. 'rating_2' is an integer the number of respondents answering with the second lowest ranked score, required, minimum 0. 'rating_3' is an integer the number of respondents answering with the middle ranked score, required, minimum 0. 'rating_4' is an integer the number of respondents answering with the second highest ranked score, required, minimum 0. 'rating_5' is an integer the number of respondents answering with the highest ranked score, required, minimum 0. 'respondents' is an integer the number of respondents responding to the specific question - respondents may choose not to answer all survey questions, required, minimum 0. 'total_respondents' is an integer the total number of respondents undertaking the user-satisfaction questionnaire, required, minimum 0. No additional properties are allowed at the root level.
