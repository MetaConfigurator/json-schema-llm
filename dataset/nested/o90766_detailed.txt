The schema describes the configuration for a MariaDB service. It is an object type. The main properties of the schema are: 'mariadb' is an object representing MariaDB service configuration properties. It is required. It contains 'cpus' which is a number specifying CPU shares to allocate to each MariaDB node, with a minimum value of 1 and a default of 1. It includes 'mem' which is a number indicating memory to allocate to each MariaDB node, with a minimum value of 1024.0 and a default of 1024.0. 'bridge' is a boolean indicating whether to use bridge networking mode for the Docker container, with a default of false. 'host_volume' is a string specifying the location of a volume on the host to persist MariaDB data, with a default value of '/tmp'. The 'service' property is an object representing DC/OS service configuration properties. It is optional and contains 'name' which is a string for the name of the MariaDB service instance, with a default of 'mariadb'. No additional properties are allowed at the root level.
