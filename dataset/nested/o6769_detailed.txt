The schema describes metadata for a HEPData table apart from independent_variables and dependent_variables. It is an object type. The main properties of the schema are: 'name' is a string that is required and must have a maximum length of 64. 'location' is a string that is optional and must have a maximum length of 256. 'description' is a string that is required. 'keywords' is an array that is required, containing items that are objects with the properties: 'name', which is a string that is required and must have a maximum length of 128 with allowed values 'cmenergies', 'observables', 'reactions', 'phrases'; and 'values', which is an array that is required with items that can be either strings or numbers. 'data_file' is a string that is required. 'data_schema' is a string that is optional and describes the remote location of the JSON schema for the data_file. 'data_license' is an object that is optional with required properties: 'name' (string, maximum length 256) and 'url' (string, maximum length 256), and no additional properties are allowed. 'additional_resources' is an array that is optional, containing items that are objects with required property 'location' (string, maximum length 256) and optional properties 'description' (string), 'type' (string, maximum length 64), and 'license' (an object with required properties 'name' (string, maximum length 256) and 'url' (string, maximum length 256), and no additional properties are allowed). 'table_doi' is a string that is optional and describes if a user downloads YAML from an existing HEPData record. No additional properties are allowed at the root level.
