The schema describes a structure for licensing information. It is an object type. The main properties of the schema are: 'data_type' is a string that defines the type of data, with the required value 'licence'. 'source_url' is a string that represents the source URL, which is required. 'confidence' is a string that represents the confidence level of the information, which is required. 'properties' is an object that contains further details about the licence. The 'properties' object includes required fields: 'jurisdiction_code', which is a string that denotes the jurisdiction for which the licence was issued; 'jurisdiction_classification', which is an array that must contain at least one item and denotes the local classification given by the regulator; and 'category', which is a string that specifies the category of licence with allowed values 'Financial' and 'Business'. Additionally, 'licence_number' is either a string or null and is optional. 'regulator' is a string that describes the regulating body that issued the licence and is also optional. 'oc_classification' is an array that is optional. No additional properties are allowed at the root level.
