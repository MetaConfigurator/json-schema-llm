The schema describes a configuration for the OSTRICH-SWMM package. It is an object type. It defines several reusable components under 'definitions': 'extract_step' is an object, which requires the field 'type'. It may contain an optional boolean field 'enabled'. 'node_extract_step' is an object, which requires the fields 'statistics', 'nodes', and 'output_path'. It has optional fields 'event_threshold_flow_rate' and 'type', with 'type' constrained by the pattern '^node$'. It may contain an array of strings in 'nodes' and an array of 'node_extract_statistic' items in 'statistics'. 'node_extract_statistic' is an enum, allowing values: 'node_name', 'num_flow_events', 'total_flow_volume', 'total_flow_duration', 'first_flow_start', 'first_flow_end', 'first_flow_duration', 'first_flow_volume', 'last_flow_start', 'last_flow_end', 'last_flow_duration', 'last_flow_volume', 'max_volume_flow_start', 'max_volume_flow_end', 'max_volume_flow_duration', 'max_volume_flow_volume', 'max_duration_flow_start', 'max_duration_flow_end', 'max_duration_flow_duration', 'max_duration_flow_volume'. The main properties of the schema are: 'binary_output_path' is a string, optional. 'input_template_path' is a string, required, with a minimum length of 1. 'input_parameters_path' is a string, required, with a minimum length of 1. 'input_path' is a string, required, with a minimum length of 1. 'report_output_path' is a string, required, with a minimum length of 1. 'swmm_path' is a string, required, with a minimum length of 1. 'summary_dir' is a string, optional. 'extract' is an object, required, which contains the required property 'steps', an array of items, each referencing the 'extract_step' definition. No additional properties are allowed at the root level.
