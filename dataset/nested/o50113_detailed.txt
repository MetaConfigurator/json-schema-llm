The schema describes a configuration for a client application. It is an object type. The main properties of the schema are: 'schema' is an object that contains 'name' and 'validator', both required string fields. 'client' is an object that includes 'backend', 'path_processor', and 'tile_processor', all required objects. The 'backend' requires 'name', 'class', 'host', and 'protocol', all of which are string fields. The 'path_processor' and 'tile_processor' require 'class' and 'params', with 'class' being a string and 'params' being an object. 'database' is an object that requires 'collection', 'experiment', and 'channel', all of which are string fields. 'ingest_job' is an object that requires 'resolution', 'extent', and 'tile_size'. 'resolution' is an integer. 'extent' requires arrays 'x', 'y', 'z', and 't', all containing integers. 'tile_size' has required integer fields 'x', 'y', 'z', and 't'. No additional properties are allowed at the root level.
