The schema describes a feature component's schema definition. It is an object type. It defines several reusable components under 'definitions': 'left' is an object, requiring a property 'config' that is also an object with a required property 'align' limited to the value 'left'. 'center' is an object, requiring a property 'config' that is also an object with a required property 'align' limited to the value 'center'. 'right' is an object, requiring a property 'config' that is also an object with a required property 'align' limited to the value 'right'. The main properties of the schema are: 'content' is an object requiring properties 'heading', 'body', and 'image'. 'heading' is a string, 'body' is a string, and 'image' is an object requiring properties 'vp1', 'vp2', 'vp3', 'vp4', 'vp5', 'vp6', and 'alt', all of which are strings. 'badge' is an object requiring properties 'config' and 'content'. 'config' is an object requiring properties 'size' and 'type', where 'size' can be either 'small' or 'large', and 'type' can be either 'highlight' or 'lowlight'. 'content' within 'badge' is an object requiring a property 'text' which is a string. 'call-to-action' is an object containing 'content', which is an object requiring properties 'text' and 'href', both of which are strings. No additional properties are allowed at the root level.
