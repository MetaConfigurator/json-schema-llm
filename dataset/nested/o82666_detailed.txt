The schema describes an iframe block configuration. It is an object type. The main properties of the schema are: 'etag' is a string that must match the pattern '^[\w\.:-]+$'. It is optional. 'css_class' is a string that must match the pattern '^[\w\s-]+$' and is described as being appendable to the DOM elements' class attribute in rendering environments that support HTML. It is optional. 'updated_date' is a string that must be in date-time format and indicates an update that occurred on the node this block is attached to, not to the block itself. It is optional. 'aside' is a boolean that signifies if the block is a portion of the document with content indirectly related to the main content, often presented as sidebars. It is optional. 'src' is a string that must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$'. It is required. 'width' is a string that must match the pattern '^\d+(%|px)?$'. It is optional. 'height' is a string that must match the pattern '^\d+(%|px)?$'. It is optional. 'align' is a string that must match the pattern '^(center|right|left)$'. It is optional. 'scrolling_enabled' is a boolean indicating if scrolling is enabled for the iframe. It is optional. 'data' is an object with patternProperties that allow for strings matching '^[a-zA-Z_]{1}[\w\.:-]+$'. Each string must have a minimum length of 0 and a maximum length of 255, and additionalProperties are not allowed. It is optional. No additional properties are allowed at the root level.
