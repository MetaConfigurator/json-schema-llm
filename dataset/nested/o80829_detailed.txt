The schema describes information needed for an article entity. It is an object type. The main properties of the schema are: '_files' is an array that contains file objects with properties 'bucket' (a string), 'checksum' (a string), 'key' (a string), 'size' (an integer), 'uri' (a string), and 'version_id' (a string). 'author' is a string that specifies the author of the article. 'body' is an object containing 'content' (a string) and 'type' (a string). 'collections' is an array of collection objects. 'created' is a string representing the creation date. 'experiment' is a string denoting an experiment. 'featured' is an integer indicating whether the article is featured. 'id' is a string representing the article's identifier. 'related' is an array of objects, each having 'id' (a string) and 'type' (a string). 'screenshot' is a string providing the URL to a screenshot of the article. 'short_description' is an object containing 'content' (a string) and 'type' (a string). 'tags' is an array of strings for categorizing the article. 'title' is a string representing the title of the article. 'type' is an object with 'primary' (a string) and 'secondary' (an array of strings). No additional properties are allowed at the root level.
