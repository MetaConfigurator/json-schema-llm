The schema describes the result of Linguist worker. It is an object type. It defines several reusable components under 'definitions': 'linguist_details' is an object, details of Linguist run on one file. It has required fields: 'path', 'output', and 'type'. The 'path' is a string, 'output' can either be a reference to 'linguist_output' or null, and 'type' is an array of strings. 'linguist_output' is an object, Linguist output for one file. It has required fields: 'lines', 'sloc', 'type', 'language', and 'mime', all of which are strings or integers. The main properties of the schema are: 'schema' is an object that provides the schema metadata, with required fields: 'name' (string matching pattern '^[a-zA-Z0-9_]+$') and 'version' (string matching pattern '^[0-9]+-[0-9]+-[0-9]+$'), 'url' (optional string, formatted as URI). '_release' is a string, unique release id in the form of 'ecosystem:package:version' (optional). 'status' is a string, required and can be either 'success' or 'error'. 'details' is an array of references to 'linguist_details' (required). 'summary' is an array of strings (required). No additional properties are allowed at the root level.
