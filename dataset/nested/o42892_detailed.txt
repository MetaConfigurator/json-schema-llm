The schema describes a message envelope. It is an object type. The main properties of the schema are: '_schema' is a string that adheres to the specified regex pattern, with a default value of 'pbj:gdbots:pbjx::envelope:1-1-0'. It is required. 'envelope_id' is a string that matches a specific UUID pattern. It is required. 'ok' is a boolean with a default value of true. 'code' is an integer that defaults to 0, with a minimum of 0 and a maximum of 65535. 'http_code' is an integer with a default value of 200 and an enum of allowed values including 200, 404, and 500, among others. 'etag' is a string that matches the specified regex pattern. 'error_name' is a string that matches a specific pattern. 'error_message' is a string with a minimum length of 0 and a maximum length of 65535. 'message_ref' is an object with required fields 'curie' and 'id', adhering to specific regex patterns and lengths. 'message' is an object. 'derefs' is an object with a pattern property and no additional properties allowed, allowing it to include dereferenced messages. 'links' is an object with pattern properties for strings, also allowing no additional properties and referencing HATEOAS. 'metas' is an object that can include pattern properties for strings, with no additional properties allowed. No additional properties are allowed at the root level.
