The schema describes a set of parameters produced by OSTRICH to inject into SWMM input. It is an object type. It defines several reusable components under 'definitions': 'lid' is an object, representing a lid configuration with required fields for 'location', 'type', 'number', 'area', 'width', 'initSat', 'fromImp', and 'toPerv'. The 'location' field must be either a map with required 'map' coordinates or a subcatchment with required 'subcatchment'. 'roof' is an object, representing a roof configuration with required fields for 'location', 'OutID', 'area', 'slope', and 'width'. Similar to 'lid', the 'location' field must be either a map with required 'map' coordinates or a subcatchment with required 'subcatchment'. The main properties of the schema are: 'lids' is an array of references to the 'lid' definition, and 'roofs' is an array of references to the 'roof' definition. No additional properties are allowed at the root level.
