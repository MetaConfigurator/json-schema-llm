The schema describes an MCDA data source for uploaded JSON. It is an object type. The main properties of the schema are: 'id' is a string identifier of the data source, required. 'unitOfMeasurement' is an object, required, that includes 'type' which is a string with enum values of 'custom', 'percentage', or 'decimal', and 'label' which is a string. 'source' is a string that describes where the data comes from, optional. 'sourceLink' is a string that provides a link to the source, optional. 'strengthOfEvidence' is a string representing the strengths of the data, optional. 'uncertainties' is a string detailing uncertainties about the data, optional. 'scale' is an array of two items, required, where each item can be a number or null. 'pvf' is either an object or null; if an object, it requires 'direction', 'type', and 'range'. The 'direction' property is a string with enum values of 'increasing' or 'decreasing'. The 'type' property is a string with enum values of 'linear' or 'piecewise-linear'. The 'range' is an array of two numbers. If present, 'values' and 'cutoffs' are arrays of three numbers. No additional properties are allowed at the root level.
