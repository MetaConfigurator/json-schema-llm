The schema describes an API endpoint configuration. It is an object type. The main properties of the schema are: 'description' is a string with a maximum length of 32767 characters, optional. 'route' is a string with a minimum length of 1 and a maximum length of 1024 characters, required. 'enabled' is a boolean, optional. 'method' is a string that must be one of the following values: 'delete', 'get', 'options', 'patch', 'post', 'put', required. 'access' is a string that must be one of the following values: 'public', 'authenticated', 'group', optional. 'endpointTags' is an object where property names must match the pattern '^[0-9a-zA-Z_-]{1,255}$' and corresponding values must be strings with a minimum length of 1 and a maximum length of 255 characters, optional; no additional properties are allowed. 'experienceGroupIds' is an array with a maximum of 1000 items, where each item is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'staticReply' is either an object with required properties 'value' (string) and 'type' (string, must be either 'page' or 'redirect') along with 'statusCode' (number between 100 and 599), with no additional properties allowed, or null, optional. 'unauthorizedReply' is either an object with required properties 'value' (string) and 'type' (string, must be either 'page' or 'redirect') along with 'statusCode' (number between 100 and 599), with no additional properties allowed, or null, optional. Compositional constraints: 'staticReply' and 'unauthorizedReply' can be either specific objects or null values. No additional properties are allowed at the root level.
