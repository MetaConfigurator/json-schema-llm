The schema describes a collection of webhooks and related metadata. It is an object type. The main properties of the schema are: 'items' is an array containing objects, each representing a single webhook. The items must follow the schema defined for a webhook with properties including: 'id', 'webhookId', and 'applicationId' as strings matching the pattern '^[A-Fa-f\d]{24}$'; 'creationDate' and 'lastUpdated' as strings formatted as date-time; 'name' as a string with a minimum length of 1 and maximum length of 255; 'token' as a string with a minimum length of 1; 'responseCode' as an integer between 100 and 599; 'verificationType' as a string with allowed values: 'facebook', 'fitbit', 'none', 'twilio', 'alexa'; 'verificationCode' as a string with a maximum length of 32767; 'waitForReply' as a boolean; 'basicAuthUsername' and 'basicAuthPassword' as strings with a maximum length of 255. 'count' is an integer; 'totalCount' is an integer; 'perPage' is an integer; 'page' is an integer; 'filter' is a string; 'filterField' is a string; 'sortField' is a string; and 'sortDirection' as a string with allowed values: 'asc', 'desc', 'ASC', 'DESC', ''. No additional properties are allowed at the root level.
