The schema describes the notification of triggering jobs in a resultset. It is an object type. The main properties of the schema are: 'version' is a number required field with an enum value of 1. 'project' is a string required field that serves as an identifier for the treeherder project, like 'try' or 'mozilla-central'. 'resultset_id' is a string required field that represents the unique identifier for a resultset. 'times' is a number optional field that indicates the number of times to execute the command for a resultset. 'action' is a string required field with enum values of 'cancel_all', 'trigger_missing_jobs', and 'trigger_all_talos_jobs'. 'requester' is a string required field representing the requester of the action (usually an email). No additional properties are allowed at the root level.
