The schema describes getting information about the user that is making the request. It is an object type. The main properties of the schema are: 'id' is a string The unique identifier for this user (generated by shipcloud). It's required. 'email' is a string The email address associated with this user. It's required. 'first_name' is a string First name of the user. It's optional. 'last_name' is a string Last name of the user. It's optional. 'customer_no' is a string The unique identifier for this customer (generated by shipcloud). Used e.g. in support cases. It's required. 'environment' is a string Shows the corresponding environment for the currently used api key. It is required and must be one of the following values: 'production', 'sandbox'. 'subscription' is an object A container for subscription details. It is required and has the following properties: 'plan_name' is a string A key to identify the plan the user currently has. It's required. 'plan_display_name' is a string Name of the plan the user currently has. Can be used for displaying it at the frontend. It's required. 'chargeable' is a boolean true if the user did provide a chargeable payment method. It's required. No additional properties are allowed at the root level.
