The schema describes a study. It is an object type. It defines several reusable components under 'definitions': 'submittableRef' is an object, it requires either 'alias' and 'team' or 'accession', and has properties 'alias', 'accession', and 'team', each of which is a string with a minimum length of 1. The main properties of the schema are: 'alias' is a string, required, with a minimum length of 1. 'title' is a string, optional. 'description' is a string, optional. 'attributes' is an object, optional, with properties allowing keys that match the pattern '^.*$', where each key maps to an array of objects. Each object in the array must contain a required 'value' of type string with a minimum length of 1, an optional 'units' of type string with a minimum length of 1, and an optional 'terms' which is an array of objects with a required 'url' of type string formatted as a URI. 'StudyDataType' is a string, required, with allowed values: 'Sequencing', 'FunctionalGenomics', 'Metabolomics', 'Proteomics'. 'projectRef' is an object, optional, that references the 'submittableRef' definition. 'protocolRefs' is an array, optional, with items that are objects referencing the 'submittableRef' definition. No additional properties are allowed at the root level.
