The schema describes the configuration for an Instana agent. It is an object type. The main properties of the schema are: 'instana-agent' is an object Instana specific configuration properties. It is required and allows additional properties. It has the following properties: 'agent_endpoint' is a string Instana-SaaS endpoint URL or on-prem backend hostname. It is required and defaults to 'saas-us-west-2.instana.io'. 'agent_endpoint_port' is an integer Instana SaaS/on-prem monitoring endpoint port (SaaS port: 443 - on-prem default port: 1444). It is required, defaults to 443, and has a minimum value of 443. 'agent_key' is a string Your (Customer specific) Instana Agent key. It is required. 'app_id' is a string Automatically assigned Marathon application ID. It is optional and defaults to 'instana-agent'. 'cpus' is a number Number of CPU shares assigned to the Instana agent. It is optional, defaults to 1, and has a minimum value of 0.5. 'framework_name' is a string Cluster ID. It is optional and defaults to 'instana-agent'. 'instances' is an integer Number of Instana Agent instances to run. It is optional, defaults to 1, and has a minimum value of 1. 'ram' is an integer Number of memory shares assigned to the Instana agent. It is optional, defaults to 400, and has a minimum value of 300. 'tag_list' is a string Comma separated list of host-specific tags which identify the Mesos node. It is optional and defaults to an empty string. No additional properties are allowed at the root level.
