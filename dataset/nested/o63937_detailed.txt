The schema describes the CT transcode specification for watermark. It is an object type. The main properties of the schema are: 'input_bucket' is a string that specifies the bucket name and path where the watermark is located, which is required. It must have a maximum length of 63 characters, a minimum length of 3 characters, and must match the pattern '^[0-9a-zA-Z\.\-_]+$'. 'input_file' is a string that specifies the filename of the file containing the watermark, which is required. It must have a maximum length of 1024 characters, a minimum length of 1 character, and must match the pattern '^[\w /-]+\.[0-9a-zA-Z]+$'. 'opacity' is a number that allows changing the default watermark opacity. 'size' is a string that specifies the size of the watermark in the video result, formatted as [width]x[height], and must match the pattern '^[0-9]+x[0-9]+$'. 'x' is a number that allows changing the watermark position in pixels. 'y' is a number that allows changing the watermark position in pixels. No additional properties are allowed at the root level.
