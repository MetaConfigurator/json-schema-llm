The schema describes an AMQP 0-9-1 Channel Binding Object. It is an object type. It defines several reusable components under 'definitions': 'exchange' is an object, which defines the properties of the exchange used when 'is' is 'routingKey'. It has properties 'name' (a string, which must not exceed 255 characters), 'type' (a string, with allowed values 'topic', 'direct', 'fanout', 'default', 'headers'), 'durable' (a boolean), and 'autoDelete' (a boolean). 'queue' is an object, which defines the properties of the queue used when 'is' is 'queue'. It has properties 'name' (a string, which must not exceed 255 characters), 'durable' (a boolean), 'exclusive' (a boolean), and 'autoDelete' (a boolean). The main properties of the schema are: 'is' is a string that defines what type of channel it is, can be either 'routingKey' or 'queue' (default); 'exchange' is a reference to the 'exchange' definition; 'queue' is a reference to the 'queue' definition; 'bindingVersion' is a string that defines the version of this binding with allowed values '0.1.0' and 'latest'. Queue and exchange are mutually exclusive, meaning that if one is present, the other must not be. The schema has a oneOf constraint that requires either 'queue' (when 'is' is 'queue') or 'exchange' (when 'is' is 'routingKey') but not both. No additional properties are allowed at the root level.
