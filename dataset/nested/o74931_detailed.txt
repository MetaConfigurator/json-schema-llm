The schema describes a shipping request. It is an object type. It defines several reusable components under 'definitions': 'address' is an object, which represents the receiver's address with required fields: 'first_name', 'last_name', 'street', 'street_no', 'city', 'zip_code', and 'country', and it disallows additional properties. 'package' is an object, which defines package dimensions with required fields: 'width', 'height', 'length', and 'weight', and it also disallows additional properties. The main properties of the schema are: 'carrier' is a string and required, with enum values: 'dhl', 'ups', 'dpd', 'hermes', 'gls', describing the acronym of the carrier you want to use; 'to' is a required reference to the 'address' definition, describing the receiver's address; 'from' is an optional reference to the 'address' definition, which if missing uses the default sender address (if defined in your shipcloud account); 'package' is a required reference to the 'package' definition; 'service' is a string and optional, with a default value of 'standard', representing additional services such as 'standard', 'returns', 'one_day', and 'one_day_early'; 'reference_number' is an optional string, allowing a maximum of 30 characters to identify the shipment; 'create_shipping_label' is a boolean and optional, determining if a shipping label should be created at the carrier (meaning you will be charged). No additional properties are allowed at the root level.
