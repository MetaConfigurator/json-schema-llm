The schema describes a Bill. It is an object type. It defines several reusable components under 'definitions': 'Frequency' is an object, describing the frequency of billing. It has a required field 'recurring' of type boolean, and additional properties are not allowed. The main properties of the schema are: '_id' is an integer A unique ID for Bill, required. 'billHeadId' is an integer optional. 'date' is a string with format date-time optional. 'daysAutoEnter' is an integer optional. 'frequency' is an object referencing the 'Frequency' definition optional. 'transactionId' is an integer optional. No additional properties are allowed at the root level.
