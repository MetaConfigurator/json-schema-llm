The schema describes a field configuration for forms. It is an object type. The main properties of the schema are: 'name' is a string that serves as a unique identifier (within the form) for the field. This value is required, must match the pattern '^[a-zA-Z_]{1}[\w-]*$', and should NOT change once set. 'maps_to' is a string that defines the name of the schema field the answer will map to, defaulting to 'cf'. It must match the pattern '^[a-zA-Z_]{1}\w*$'. 'label' is a string providing the main text for the question/field, with a minimum length of 0 and a maximum of 255 characters. 'placeholder' is an optional string with a minimum length of 0 and a maximum of 255 characters. 'description' is a string, optional, with a minimum length of 0 and a maximum of 65535 characters, offering a brief explanation of the field. 'is_required' is a boolean indicating whether the field is mandatory. 'link_text' is an optional string with a minimum length of 0 and a maximum of 255 characters, intended to replace the token '{link}' within the label or description. 'link_url' is a string containing a URL, which must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$', also intended for the token '{link}' within the label or description. 'pii_impact' is a string that can take one of the following values: 'unknown', 'high', 'moderate', or 'low'. No additional properties are allowed at the root level.
