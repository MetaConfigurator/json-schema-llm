The schema describes a sample toolchain for continuous deployment. It is an object type. The main properties of the schema are: 'prod-region' is a string that specifies the bluemix region; 'prod-organization' is a string that specifies the bluemix org; 'prod-space' is a string that specifies the bluemix space; 'bluemix-user' is a string that specifies your Bluemix user ID; 'bluemix-password' is a string that specifies your Bluemix Password; 'bluemix-api-key' is a string that is required for Federated ID since Federated ID can't login with Bluemix user and password via Bluemix CLI; 'bluemix-cluster-account' is a string that specifies the GUID of the Bluemix account where you created the cluster; 'bluemix-cluster-name' is a string that specifies your cluster name; 'mysql-db-user' is a string that specifies your MySQL username; 'mysql-db-password' is a string that specifies your MySQL password; 'mysql-db-host' is a string that specifies your MySQL host; 'mysql-db-port' is a string that specifies your MySQL port number. These properties include required fields: 'prod-region', 'prod-organization', 'prod-space', 'prod-app-name', and 'bluemix-cluster-name'. Additionally, the schema has a conditional requirement where either 'bluemix-user', 'bluemix-password', and 'bluemix-cluster-account' must be provided or 'bluemix-api-key' must be present. No additional properties are allowed at the root level.
