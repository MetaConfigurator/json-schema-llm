The schema describes a network object. It is an object type. The main properties of the schema are: 'data' is an array List of identifiers (or URIs) of the references describing the data. 'date' is a string The time at which the network was sampled. 'description' is a string A short description of the network. 'environment' is an array List of identifiers (or URIs) of environmental measurements associated to the network. 'id' is an integer Integer data. Ex: 2673. 'interactions' is an array List of identifiers (or URIs) of the interactions in the network. 'latitude' is a string Latitude. 'longitude' is a string Longitude. 'name' is a string The name of the network. 'owner' is a string A single related resource. Can be either a URI or set of nested resource data. 'papers' is an array List of identifiers (or URIs) of the references to the papers associated with the dataset. 'public' is a boolean Whether the network can be viewed by all users. 'resource_uri' is a string Unicode string data. Ex: 'Hello World'. The schema requires 'interactions', 'owner', 'public', and 'name'. No additional properties are allowed at the root level.
