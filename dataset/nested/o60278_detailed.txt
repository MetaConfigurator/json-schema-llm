The schema describes a configuration for DC/OS. It is an object type. The main properties of the schema are: 'dcos_acs_token' is a string that describes the token generated by authenticating to DC/OS with ACS. It is optional. 'dcos_url' is a string that describes the public master IP of your DC/OS installation. It must be in URI format and is optional. 'mesos_master_url' is a string that describes the Mesos master URL. It must be set in the format: "http://host:port" and is optional. 'pagination' is a boolean that indicates whether to paginate output. It defaults to true and is optional. 'prompt_login' is a boolean that indicates whether to prompt the user to log in when the token expires; otherwise, it automatically initiates login. It defaults to false and is optional. 'reporting' is a boolean that indicates whether to report usage events to Mesosphere. It defaults to true and is optional. 'ssl_verify' is a string that describes whether to verify SSL certs for HTTPS or the path to certs. Valid values are True, False, or a path to a CA_BUNDLE. It is optional. 'timeout' is an integer that specifies the request timeout in seconds. It defaults to 5 and has a minimum value of 1. It is optional. No additional properties are allowed at the root level.
