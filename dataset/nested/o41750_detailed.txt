The schema describes a User Schema. It is an object type. The main properties of the schema are: 'username' is a string that is required. 'id' is a string that is required. 'barcode' is a string that is optional. 'active' is a boolean that is optional. 'type' is a string that is optional. 'patronGroup' is a string that is optional. 'meta' is an object that is optional. 'personal' is an object that is optional and defines several properties: 'lastName' is a string that is required. 'firstName' is a string that is optional. 'middleName' is a string that is optional. 'email' is a string that is optional. 'phone' is a string that is optional. 'mobilePhone' is a string that is optional. 'dateOfBirth' is a string formatted as date-time that is optional. 'preferredContact' is an object that is optional and defines two properties: 'value' is a string that must be one of the following values: 'EMAIL', 'MAIL', 'TEXT', 'PHONE', 'MOBILE'. 'desc' is a string that must be one of the following values: 'Email', 'Mail (Primary Address)', 'Text message', 'Phone', 'Mobile phone'. No additional properties are allowed at the root level.
