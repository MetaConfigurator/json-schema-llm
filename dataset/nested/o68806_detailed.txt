The schema describes the result from processing a (plugin, instance) pair. It is an object type. It defines several reusable components under 'definitions': 'Error' is an object type, describing the details of an exception. It has a required field 'message' of type string and includes additional optional fields 'exc', 'fname', 'func', and 'line_number', all of type string or number. 'Record' is also an object type, documenting log data. It has a required field 'message' of type string and other optional fields without defined types. The main properties of the schema are: 'duration' is an optional number describing the time in milliseconds taken to process a pair. 'error' is an optional property that can be either an 'Error' object or null. 'instance' is an optional property that can be a string or null, indicating the name of the processed instance. 'plugin' is a required string property specifying the name of the processed plug-in. 'records' is an optional array of 'Record' objects, containing the records related to the processing. 'success' is an optional boolean property denoting the processing status. No additional properties are allowed at the root level.
