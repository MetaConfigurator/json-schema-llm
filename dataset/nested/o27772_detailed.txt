The schema describes the structure of a response object. It is an object type. The main properties of the schema are: 'correlationGuid' is a string matching the pattern '^[a-f\d]{8}-[a-f\d]{4}-4[a-f\d]{3}-[89ab][a-f\d]{3}-[a-f\d]{12}$', optional. 'responderGLN' is a string with a minimum and maximum length of 13 characters, optional. 'data' is an object that is required and must contain the property 'verified'. The 'verified' property is a boolean indicating if all elements were matched, required. 'verifiedInfo' is an optional string with an enum of values: '001', '002', '003', '004', used only if 'verified' is 'false'. 'additionalInfo' is an optional string with an enum of values: '001', '002', '003', used if 'response' is 'true'. 'errorMessage' is an optional string. No additional properties are allowed at the root level.
