The schema describes an ALPS document. It is an object type. The main properties of the schema are: 'links' is an array of objects identifying a link between an ALPS element and some other resource. Each object is required to have 'rel' (a string, a [RFC5988][] approved relation type) and 'href' (a string, the URL of the resource described by the associated `rel`). 'descriptors' is an array that defines the semantics of specific data elements or state transitions that may exist in an associated representation. 'doc' is an object that describes the element, containing optional properties: 'format' (an enum with allowed values: 'text', 'html', 'asciidoc', indicating how the content should be parsed), 'href' (a string, referencing a related document), and 'content' (a string, the text contents). 'ext' is an object for author-specific extensions, required to have 'id' (a string, a document-wide unique identifier). It may also include 'href' (a string, referencing the definition of the extension) and 'value' (a string, specifying the value). 'version' is a string with a default value of '1.0'. No additional properties are allowed at the root level.
