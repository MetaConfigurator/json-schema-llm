The schema describes special events. It is an object type. The main properties of the schema are: 'special-events' is an array containing objects that represent individual events. Each event object is required to have the following fields: 'name', 'link', 'simple-text', 'image', 'html', 'priority', 'start', 'end', 'id', and 'development'. The 'name' is a string that denotes the title of the event. The 'link' is a string that directs to a website about the event. The 'simple-text' is a string that provides a short description of the event. The 'image' is a string pointing to an image for the event. The 'html' is a string for larger, HTML-formatted text describing the event and can be empty. The 'priority' is an integer indicating the event's priority, where 1000 is the highest and 0 the lowest. The 'start' is a string formatted as date-time representing when the event should start. The 'end' is also a string formatted as date-time indicating when the event should end. The 'id' is an integer that serves as the unique ID for the event. The 'development' is a boolean indicating if the event is used in development, typically set to false, though it may be true for certain environments. The 'in-app' is an optional string providing a unique identifier for the type of event. No additional properties are allowed at the root level.
