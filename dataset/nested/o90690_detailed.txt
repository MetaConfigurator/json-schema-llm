The schema describes the configuration for a schema registry. It is an object type. The main properties of the schema are: 'registry' is an object containing configuration details for the schema-registry instance. It is required and contains the following properties: 'cpus' is a number representing CPU shares to allocate to each schema-registry instance, with a default value of 1 and a minimum of 1. 'heap' is a number for JVM heap allocation (in MB) for connect worker tasks, with a default value of 256 and a minimum of 256. 'instances' is an integer indicating the number of instances to run, limited to a maximum of 1, with a default value of 1 and a minimum of 1. 'kafkastore-connection-url' is a string for the Zookeeper Connect string for service cluster, with a default value of 'master.mesos:2181/dcos-service-confluent-kafka'. 'mem' is a number for memory allocation (MB) to each schema-registry instance, with a default value of 512 and a minimum of 320. 'name' is a string for the service name of the schema-registry instance(s), with a default value of 'schema-registry'. 'role' is a string indicating the role for deploying the schema-registry, with a default value of '*'. No additional properties are allowed at the root level.
