The schema describes a trait object. It is an object type. The main properties of the schema are: 'date' is a string that describes the time at which the traits were measured. 'description' is a string that describes a longer description of the trait. 'id' is an integer data that serves as an identifier. 'latitude' is a string that specifies the latitude. 'longitude' is a string that specifies the longitude. 'name' is a string that describes the name of the measured trait. 'owner' is a string that describes a single related resource, which can be either a URI or set of nested resource data. 'resource_uri' is a string that represents Unicode string data. 'units' is a string that specifies the units in which the trait was measured. 'value' is a string that represents the value of the trait. The properties 'name', 'owner', and 'value' are required. No additional properties are allowed at the root level.
