The schema describes a Resolver Package Repository. It is an object type. The main properties of the schema are: 'name' is a string that represents the name of the repository, required. 'description' is a string that represents the description of the repository, required. 'homepage' is a string that represents the url of the repository, required. 'package_target' is a string that represents the directory where the resolver-packages.json file should be generated, optional. 'web_target' is a string that represents the directory where the generated HTML will be stored, optional. 'web_template' is a string that represents the template to use when generating HTML, optional. 'auth' is an object that holds optional authentication tokens used to request the packages. 'packages' is an array that contains the list with packages that exists, required with a minimum of 1 item, unique items, and each item is an object having 'type' as a required string representing the type of the package and 'url' as a required string representing the url of the package. No additional properties are allowed at the root level.
