The schema describes the Git LFS HTTPS Batch API Response. It is an object type. It defines several reusable components under 'definitions': 'action' is an object, which includes a required field 'href' of type string, optional fields 'header' of type object with additional properties allowed, 'expires_in' of type number that must be between -2147483647 and 2147483647, and 'expires_at' of type string. The main properties of the schema are: 'transfer' is a string that is optional. 'objects' is an array of objects, each containing required fields 'oid' of type string and 'size' of type number which must be 0 or greater. Each object can also include optional fields: 'authenticated' of type boolean, 'actions' of type object with required properties 'download', 'upload', and 'verify', each referencing the 'action' definition, and 'error' of type object with required fields 'code' of type number and 'message' of type string. 'message' is a string that is optional. 'request_id' is a string that is optional. 'documentation_url' is a string that is optional. No additional properties are allowed at the root level.
