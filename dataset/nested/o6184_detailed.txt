The schema describes a configuration for data aggregation. It is an object type. The main properties of the schema are: 'resultPath' is a string JSON path to store the results, required, with a minimum length of 1. 'deviceTags' is an array of Device Tag Pair objects, which have 'key' and 'value' as string properties. Each Device Tag Pair must contain at least one item. 'deviceIds' is an array of Device ID strings, with each string having a minimum length of 1. 'start' is a number representing milliseconds since the epoch start time, optional. 'end' is a number representing milliseconds since the epoch end time, optional. 'duration' is a number denoting the duration of aggregation in milliseconds, optional. 'aggregation' is a string with a minimum length of 1, indicating a valid aggregation function name, required. 'attributes' is an array of Device ID strings, with each string having a minimum length of 1. The schema imposes the following constraints: both 'resultPath' and 'aggregation' are required, and either both 'start' and 'end' or 'duration' must be provided. No additional properties are allowed at the root level.
