The schema describes user information for a Chef user. It is an object type. The main properties of the schema are: 'user_information' is an object containing details about the Chef user, which is required. It has the following required fields: 'username' is a string that must match the format [a-z0-9\-_]+, 'display_name' is a string that provides a human-readable name, and 'email' is a string that must be a valid email address. Additional properties include optional fields: 'first_name' is a string representing the user's first name, 'middle_name' is a string for the user's middle name, 'last_name' is a string for the user's last name, 'external_authentication_uid' is a boolean for the user's external authentication UID, and 'recovery_authentication_enabled' is a boolean that indicates if the user can log in via a recovery endpoint, with a default value of false. No additional properties are allowed at the root level.
