The schema describes a configuration structure for device management. It is an object type. It defines several reusable components under 'definitions': 'filter' is an object, representing a filter with required fields 'name' (a string) and 'type' (a string). The main properties of the schema are: 'globalSettings' is an object that contains optional properties: 'namespaceAliases' (an array of objects with required fields 'alias' and 'namespace', both strings) and 'defaultParameters' (an array of objects with required fields 'type' (a string) and 'parameters' (an object)). 'deviceGroups' is an array of objects, required at the root level, where each object has required fields: 'name' (a string), 'devices' (an array of objects with required fields 'deviceId' and 'deviceSecret', both strings), and 'jobs' (an array of objects). No additional properties are allowed at the root level.
