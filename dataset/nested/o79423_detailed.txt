The schema describes a collection of software packages. It is an object type. The main properties of the schema are: 'packages' is an array of objects representing individual packages, required fields include 'name', 'currentVersion', 'versions', 'description', and 'tags'. Each package contains 'currentVersion' as a string, 'description' as a string, 'framework' as an optional boolean defaulting to false indicating whether the package installs a new Mesos framework, 'name' as a string, 'selected' as an optional boolean defaulting to false indicating if the package is selected in search results, 'tags' as an array of strings with a pattern requiring no whitespace, and 'versions' as an object mapping software versions to their latest package, where additional properties are of type string. 'version' is a string and is required. No additional properties are allowed at the root level.
