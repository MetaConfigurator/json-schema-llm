The schema describes configuration settings for a control center application. It is an object type. The main properties of the schema are: 'control-center' is an object containing configuration options for the control center. It is required and defines several properties, including: 'bootstrap-servers' is a string specifying the seed list of brokers for connection. It has a default value of 'broker.confluent-kafka.l4lb.thisdcos.directory:9092'. 'confluent-controlcenter-connect-cluster' is a string indicating the Kafka Connect Workers, with a default of '8.0.8.3:8083'. 'confluent-controlcenter-internal-topics-partitions' is a number representing the partition count for internal topics, defaulting to 3. 'confluent-controlcenter-internal-topics-replication' is a number for the replication factor of internal topics, also defaulting to 2. 'confluent-monitoring-interceptor-topic-partitions' is a number for the partition count of topics for storing interceptor data, with a default of 3. 'confluent-monitoring-interceptor-topic-replication' is a number for the replication factor of these topics, defaulting to 2. 'cpus' is a number specifying CPU shares, required to be at least 2, with a default of 2. 'instances' is an integer for the number of instances, required to be at least 1, with a default of 1. 'mem' is a number representing memory allocation in MB, required to be at least 2048, with a default of 4096. 'name' is a string for the application name, defaulting to 'control-center'. 'role' is a string indicating the deployment role, with a default value of '*'. 'zookeeper-connect' is a string for the Zookeeper connection string, defaulting to 'master.mesos:2181/dcos-service-confluent-kafka'. No additional properties are allowed at the root level.
