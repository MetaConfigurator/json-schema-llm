The schema describes a state for the scenario. It is an object type. The main properties of the schema are: 'parameters' is a required object containing the values for the parameters, 'statement' is an optional object or null, and 'internal' is an optional object containing internal data for the ScenarioReasoner. 'parameters' must be present. 'statement' contains a required 'type' property, which is a string and must be one of the values: 'player', 'computer', 'situation', a required 'text' property, which is a string, and a required 'propertyValues' property, which is an object with required 'independent' and 'perCharacter' objects. 'internal' contains internal data for the ScenarioReasoner and is not always present in output. No additional properties are allowed at the root level.
