The schema describes options for a RetryExam test. It is an object type. It defines several reusable components under 'definitions': 'zone' is an object, which contains a required field 'questions', an array of questions in the zone, with a minimum of 1 item. 'question' is an object, which contains a required field 'points', and either a required field 'qid' or 'qids', where 'qids' is an array of possible question IDs with a minimum of 1 item. 'questionPoints' is an array of points values, with a minimum of 1 item, consisting of integers. 'questionQID' is a string that represents a Question ID. The main properties of the schema are: 'questionGroups' is an array that includes arrays of arrays of question IDs for different difficulties, it is required. 'zones' is an array that includes zone objects, which is optional. 'unlimitedVariants' is a boolean indicating whether to generate new variants for all students, with a default of false. 'variantsPerQuestion' is an integer defining the number of variants to select per question, with a minimum of 1 and a default of 3. 'availDate' is a string indicating the date after which the test is available. 'nQuestions' is an integer representing the number of questions to select for each instance of the test, which is optional. 'timeLimitMin' is an integer indicating the number of minutes allowed for the test, which is optional. 'text' is a string representing HTML text shown on the test overview page, which is also optional. The schema requires that either 'questionGroups' and 'nQuestions' are present or 'zones' is present without 'nQuestions'. No additional properties are allowed at the root level.
