The schema describes configuration settings for a rest-proxy service. It is an object type. The main properties of the schema are: 'proxy' is an object that contains configuration details for the rest-proxy instances. It is required and defines several properties. 'cpus' is a number that specifies CPU shares to allocate to each rest-proxy instance, with a default value of 2 and a minimum value of 2. 'heap' is a number indicating JVM heap allocation (in MB) for the rest-proxy task, with a default value of 768 and a minimum value of 256. 'instances' is an integer that defines the number of instances to run, with a default value of 1 and a minimum value of 1. 'mem' is a number that specifies the memory (MB) to allocate to each rest-proxy instance, with a default value of 1024 and a minimum value of 512. 'name' is a string representing the service name for the rest-proxy instance(s), with a default value of 'rest-proxy'. 'role' is a string to deploy the rest-proxy only on nodes with this role, with a default value of '*'. 'schema-registry-service' is a string for the Schema Registry service to be used by connect workers, with a default value of 'schema-registry'. 'zookeeper-connect' is a string for the Zookeeper Connect string for the service cluster, with a default value of 'master.mesos:2181/dcos-service-confluent-kafka'. No additional properties are allowed at the root level.
