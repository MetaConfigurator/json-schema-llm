The schema describes a binding for a GSLB domain service. It is an object type. The main properties of the schema are: 'cnameentry' is a string that is readonly. 'cumulativeweight' is an integer that is readonly. 'dynamicconfwt' is an integer that is readonly. 'gslbthreshold' is an integer that is readonly. 'ipaddress' is a string that is readonly. 'name' is a string that is optional. 'port' is an integer that is readonly. 'servicename' is a string that is optional. 'servicetype' is a string that is readonly with allowed values: 'HTTP', 'FTP', 'TCP', 'UDP', 'SSL', 'SSL_BRIDGE', 'SSL_TCP', 'NNTP', 'ANY', 'SIP_UDP', 'SIP_TCP', 'SIP_SSL', 'RADIUS', 'RDP', 'RTSP', 'MYSQL', 'MSSQL', 'ORACLE'. 'state' is a string that is readonly with allowed values: 'UP', 'DOWN', 'UNKNOWN', 'BUSY', 'OUT OF SERVICE', 'GOING OUT OF SERVICE', 'DOWN WHEN GOING OUT OF SERVICE', 'NS_EMPTY_STR', 'Unknown', 'DISABLED'. 'svreffgslbstate' is a string that is readonly with allowed values: 'UP', 'DOWN', 'UNKNOWN', 'BUSY', 'OUT OF SERVICE', 'GOING OUT OF SERVICE', 'DOWN WHEN GOING OUT OF SERVICE', 'NS_EMPTY_STR', 'Unknown', 'DISABLED'. 'vservername' is a string that is readonly. 'weight' is an integer that is readonly. No additional properties are allowed at the root level.
