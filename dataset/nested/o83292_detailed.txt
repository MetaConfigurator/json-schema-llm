The schema describes the configuration for an endpoint setup. It is an object type. The main properties of the schema are: 'clientId' is a string that describes the Client ID if the workflow is to proxy the client; it is optional. 'clientMethod' is a string that describes the client method if the workflow is to proxy the client; it is optional. 'endpointId' is a string that describes the name of the endpoint; it is required. 'endpointType' is a string that describes the endpoint protocol type, either http or tchannel; it is required and can be one of the following values: 'http', 'tchannel'. 'handleId' is a string that describes the name of the method; it is required. 'middlewares' is an array that contains strings describing the middlewares the endpoint method depends on; it is optional. 'thriftFile' is a string that describes the path to the endpoint thrift file, relative to the idl path; it is required. 'thriftFileSha' is a string that describes the Sha of the thrift file, reserved but currently not used; it is required. 'thriftMethodName' is a string that describes the Thrift method name, in the format of service::method; it is required. 'workflowImportPath' is a string that describes the workflow import path if the workflow type is custom; it is optional. 'workflowType' is a string that describes the workflow type, either httpClient, tchannelClient, or custom; it is required and can be one of the following values: 'custom', 'httpClient', 'tchannelClient'. No additional properties are allowed at the root level.
