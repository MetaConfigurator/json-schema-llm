The schema describes an array of user objects. It is an array type. The main properties of the schema are: 'id' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is required. 'email' is a string in email format with a maximum length of 1024 and is required. 'role' is a string with an enum of allowed values: 'admin', 'edit', 'collaborate', 'view', 'none' and is required. 'applicationRoles' is an array of objects, each with properties 'resourceId' (a required string matching the pattern '^[A-Fa-f\d]{24}$') and 'role' (a required string with an enum of allowed values: 'collaborate', 'view', 'none') and is optional, with a maximum of 1000 items. 'dashboardRoles' is an array of objects, each with properties 'resourceId' (a required string matching the pattern '^[A-Fa-f\d]{24}$') and 'role' (a required string with an enum of allowed values: 'collaborate', 'view', 'none') and is optional, with a maximum of 1000 items. 'inviteDate' is a string in date-time format and is optional. 'ttl' is a number and is optional. 'hasExpired' is a boolean and is optional. No additional properties are allowed at the root level.
