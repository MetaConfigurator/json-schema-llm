The schema describes a pattern manifest. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: '_patternplate' is an object that contains technical values saved by patternplate core. 'data' is an object that must have at least one property and represents a custom data object supplied by the user. 'demoPatterns' is an object that must have at least one property, where each property name can be any string that matches the pattern '^.+$', and its corresponding value must be a string matching the pattern '^(/)?([^/\u0000]+(/)?)+$'. 'displayName' is a string that must have at least one character and provides a human-readable name for the pattern. 'flag' is a string that must match the pattern '^alpha|beta|rc|stable$' and indicates the stability of the pattern. 'id' is a string that must have at least one character and serves as a unique identifier for the pattern. 'meta' is an object that must have at least one property and includes custom meta data supplied by the user. 'name' is a string that must have at least one character, and must match the pattern '^[[a-z]*[-]?[a-z]*]*$', representing the machine-readable name of the pattern. 'options' is an object that must have at least one property and contains custom options supplied by the user. 'overrides' is an object that must have at least one property, including: 'demo' which is an optional string providing a custom URL for demo, and 'files', which is an object that must have at least one property with any name that matches the pattern '^.+$', where each value is a string matching the pattern '^(/)?([^/\u0000]+(/)?)+$'. 'patterns' is an object that must have at least one property, where each property name matches the pattern '^.+$', and its corresponding value must be a string matching the pattern '^(/)?([^/\u0000]+(/)?)+$'. 'tags' is an array that must contain at least one item, where each item is a string with at least one character, describing the pattern, and all items must be unique. 'version' is a string that matches the pattern '^\d\.\d\.\d(-[a-z]*){0,1}$', representing the semantic version of the pattern. 'versions' is an array that must contain at least one item, where each item is a string matching the pattern '^\d\.\d\.\d(-[a-z]*){0,1}$', representing available semantic versions of the pattern. No additional properties are allowed at the root level.
