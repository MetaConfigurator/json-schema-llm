The schema describes Hookay Configuration. It is an object type. It defines several reusable components under 'definitions': 'ConfigSite' is an object, which requires the fields 'branch', 'build', 'build_path', 'name', 'publish_path', and 'repository'. It also contains optional fields 'env' and 'hugo'. The 'build' field must be one of the enum values: 'copy', 'hugo', 'jekyll'. The 'hugo' property is an object that requires 'extended' and 'version', and has no additional properties. The main properties of the schema are: '$schema' is an optional string describing the schema file for the config.json. 'secret' is a required string that serves as the secret token to validate a hook call. 'server' is a required object that requires 'host', 'path', and 'port'. 'sites' is a required array of items that reference the 'ConfigSite' definition. 'version' is a required number that must be 2 at the moment. No additional properties are allowed at the root level.
