The schema describes the structure of an AST object. It is an object type. The main properties of the schema are: 'id' is an integer A unique identifier for the current element. It is required. 'type' is a string The type category of the element (component/var/derived). It is required and must be one of the following values: 'component', 'var', 'derived', 'textnode', 'data', 'meta'. 'name' is a string The name of the element. It is optional. 'properties' is an object The properties/attributes of the element. It is optional with no additional properties allowed. 'children' is an array The children of the element. It is optional and must contain unique items. Each item can be any of the following: a reference to the root schema, a structure for a textnode which includes 'id' (integer, required), 'type' (string, required, must be 'textnode'), and 'value' (string, required); a structure for var and derived which includes 'id' (integer, required), 'type' (string, required, must be either 'var' or 'derived'), and 'properties' (an object that includes 'name' with 'type' as string and 'value' as string, both required); or a structure for data which includes 'id' (integer, required), 'type' (string, required, must be 'data'), and 'properties' (an object that includes 'name' and 'source' with their own required properties). No additional properties are allowed at the root level.
