The schema describes a Pass Data Json Format. It is an object type. The main properties of the schema are: 'passTypeIdentifier' is a string and is required. 'formatVersion' is an integer with a default value of 1. 'organizationName' is a string and is required. 'serialNumber' is a string and is optional. 'teamIdentifier' is a string and is optional. 'webServiceURL' is a string and is optional. 'authenticationToken' is a string and is optional. 'description' is a string and is required. 'foregroundColor' is a string with a default value of 'rgb(255, 255, 255)'. 'backgroundColor' is a string with a default value of 'rgb(55, 117, 50)'. 'logoText' is a string with a default value of 'iVeew'. 'locations' is an array of objects where each object contains a 'latitude' (number, default 52.497181) and a 'longitude' (number, default 13.384972). 'relevantDate' is a string with a default value of '2014-06-07'. 'barcode' is an object that contains a 'messageEncoding' (string, default 'utf-8'), 'format' (string, default 'PKBarcodeFormatQR'), and 'message' (string, default '---'). 'storeCard' is an object and is optional. No additional properties are allowed at the root level.
