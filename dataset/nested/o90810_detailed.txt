The schema describes the configuration for an OpenVPN service. It is an object type. The main properties of the schema are: 'openvpn-admin' is an object that configures OpenVPN instances, required fields are 'cpus', 'mem', and 'instances'. 'cpus' is a number that specifies CPU shares to allocate to each OpenVPN instance, with a default of 0.01 and a minimum of 0.01. 'instances' is an integer that indicates the number of OpenVPN instances to run, with a default of 1 and a minimum of 1. 'mem' is a number that allocates memory (MB) to each OpenVPN task, with a default of 128.0 and a minimum of 64.0. 'service' is an object that provides service configuration, required field is 'name'. 'name' is a string that specifies the OpenVPN service name, with a default of 'openvpn-admin'. No additional properties are allowed at the root level.
