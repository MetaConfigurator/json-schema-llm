The schema describes a Notification data model. It is an object type. It defines several reusable components under 'definitions': 'Message' is an object, a message that should be delivered. It requires the fields 'guid' (a string with a uuid format), 'title' (a string), 'text' (a string), and 'sender' (a reference to the Sender definition). 'NotifyJob' is an object, a job to deliver message. It requires the fields 'message' (a reference to the Message definition), 'type' (a string with allowed values: 'email', 'sms', 'webhook'), and 'receiver' (an array of strings). 'Sender' is an object, which service is the source of the message/job. It contains the fields 'guid' (a string with a uuid format), 'name' (a string), and 'event_id' (a string with a uuid format). The main properties of the schema are: 'Message' is an object that references the Message definition. 'NotifyJob' is an object that references the NotifyJob definition. 'Sender' is an object that references the Sender definition. No additional properties are allowed at the root level.
