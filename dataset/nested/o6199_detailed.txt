The schema describes a device configuration. It is an object type. The main properties of the schema are: 'name' is a string with a minimum length of 1 and a maximum length of 255, required; 'deviceName' is a string with a minimum length of 1 and a maximum length of 255, optional; 'description' is a string with a maximum length of 32767, optional; 'deviceDescription' is a string with a maximum length of 32767, optional; 'tags' is an array of objects, each containing 'key' (a string matching the pattern '^[0-9a-zA-Z_-]{1,255}$', required) and 'value' (a string with a minimum length of 1 and a maximum length of 255, required), no additional properties allowed in the tag objects; 'attributes' is an array of objects, each containing 'name' (a string matching the pattern '^[0-9a-zA-Z_-]{1,255}$', required) and 'dataType' (a string with an enum of allowed values: 'string', 'number', 'gps', 'boolean', required), no additional properties allowed in the attribute objects; 'deviceClass' is a string with an enum of allowed values: 'standalone', 'master', 'owned', 'floating', 'virtual', optional; 'masterId' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. No additional properties are allowed at the root level.
