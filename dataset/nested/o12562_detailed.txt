The schema describes the structure for MineMeld attributes. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'type' is a string that indicates the type of the indicator, which is required and must be one of the following: 'IPv4', 'IPv6', 'domain', 'URL', 'sha512', 'sha256', 'sha1', 'md5', 'ssdeep', 'mutex', 'windows-registry-value', 'user-agent.fragment', 'file.name', 'process.command_line', 'email-addr', 'autonomous-system'. 'direction' is a string that indicates the direction of the session, applies to IPv4, and is optional with possible values 'inbound' and 'outbound'. 'first_seen' is an integer formatted as utc-millisec, representing the time the indicator has been seen for the first time, and is optional. 'last_seen' is an integer formatted as utc-millisec, representing the time the indicator has been seen for the last time, and is optional. 'sources' is an array of strings, where each item is formatted as a uri, representing the list of sources for this indicator, and is optional. 'confidence' is an integer, representing the confidence in the indicator within the range of 0 to 100, and is optional. 'share_level' is a string that indicates the share level of the indicator, which is optional and must be one of the following: 'white', 'green', 'amber', 'red'. 'country' is a string representing the ISO country code, limited to a length of 2, applicable to IPv4 and IPv6 only, and is optional. 'AS' is a string that represents the autonomous system for IPv4 and IPv6 only, and is optional. No additional properties are allowed at the root level.
