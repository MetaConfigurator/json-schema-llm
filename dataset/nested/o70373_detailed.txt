The schema describes a data structure for managing investment accounts. It is an object type. It defines several reusable components under 'definitions': 'IDType' is a string, with a pattern that matches a 10-digit number. 'ISO8601-Date' is a string, with a pattern that conforms to the date format YYYY-MM-DD. 'ISO8601-Timestamp' is a string, with a pattern that matches ISO 8601 timestamps. 'LISAManagerReferenceNumberType' is a string, with a pattern that matches 'Z' followed by six digits. The main properties of the schema are: 'LISAManagerReferenceNumber' is a string and references the 'LISAManagerReferenceNumberType' definition. 'accountID' is a string and is required. 'closeDate' is a string and references the 'ISO8601-Date' definition. 'deathNotification' is an object, required to have 'notifiedDate' and 'DoD', which respectively reference the 'ISO8601-Timestamp' and 'ISO8601-Date' definitions. 'healthStatus' is a string, with allowed values of 'default', 'dead', and 'terminally-ill'. 'investorID' is a string and references the 'IDType' definition. 'openDate' is a string and references the 'ISO8601-Date' definition. 'status' is a string, with allowed values of 'open', 'closed', and 'transferred-out'. 'terminalIllnessNotification' is an object, required to have 'notifiedDate', which references the 'ISO8601-Timestamp' definition. The schema requires 'accountID', 'LISAManagerReferenceNumber', 'investorID', 'status', and 'openDate'. No additional properties are allowed at the root level.
