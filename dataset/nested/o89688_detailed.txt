The schema describes a structure for managing CAP files. It is an object type. The main properties of the schema are: '_buckets' is an object that contains properties 'record' and 'deposit', both of which are strings with descriptions related to bucket IDs. '_files' is an array of objects, each describing information needed for files in records and containing properties such as 'checksum', 'bucket', 'version_id', 'key', 'type', and 'size', which are all strings except for 'size', which is an integer. No additional properties are allowed at the root level.
