The schema describes an object representing a review request. It is an object type. The main properties of the schema are: 'timestamp' is a string representing the number of seconds since the Unix epoch, required, with a minimum length of 10, maximum length of 10, and must match the pattern '[0-9]{10,10}'. 'requester' is a string, required. 'baseCommit' is a string, optional. 'reviewRef' is a string used to specify a git ref that tracks the current revision under review, optional. 'targetRef' is a string used to specify the git ref that should be updated once the review is approved, optional. 'reviewers' is an array of strings, optional, where items must be strings. 'description' is a string, optional. 'v' is an integer with an enum value of 0, optional. 'alias' is a string used to specify a post-rebase commit hash for the review, optional. No additional properties are allowed at the root level.
