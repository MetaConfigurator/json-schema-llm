The schema describes an array of execution records. It is an array type. The main properties of the schema are: 'id' is a string matching the pattern '^[A-Fa-f\d]{24}$', 'notebookExecutionId' is a string matching the pattern '^[A-Fa-f\d]{24}$', 'notebookId' is a string matching the pattern '^[A-Fa-f\d]{24}$', 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$', 'status' is a string with an enum of allowed values: 'queued', 'inProgress', 'completed', 'errored', 'timeout', 'executionRelativeTo' is a string with the format 'date-time', 'maxAllowedRunMinutes' is a number, 'sourceType' is a string with an enum of allowed values: 'flow', 'user', 'device', 'apiToken', 'notebook', 'sourceId' is a string matching the pattern '^[A-Fa-f\d]{24}$', 'runQueuedAt' is a string with the format 'date-time', 'runStartedAt' is a string with the format 'date-time', 'inputsCompletedAt' is a string with the format 'date-time', 'notebookCompletedAt' is a string with the format 'date-time', 'outputsCompletedAt' is a string with the format 'date-time', 'templateContext' is a string with a maximum length of 32767, 'inputInfo' is an object with pattern properties that are objects containing 'size' (a number) and 'url' (a string), 'outputInfo' is an object with pattern properties that are objects containing 'size' (a number), 'url' (a string), and 'contentType' (a string), 'executionErrors' is an array of objects each with properties 'name' (a string) and 'message' (a string). No additional properties are allowed at the root level.
