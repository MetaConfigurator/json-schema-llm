The schema describes the results of executing a qobj. It is an object type. The main properties of the schema are: 'backend' is a string describing the backend who produced this result, required. 'id' is a string describing the job id passed back from the input qobj, required. 'execution_id' is a string describing the unique execution id. May be useful for matching with backend logfiles, etc, optional. 'date' is a string formatted as date-time describing the date/time of job execution, optional. 'result' is an array describing corresponding results for the array of experiments of the input qobj, required, containing objects with the following properties: 'success' is a boolean indicating if true, we can trust results for this experiment, required; 'status' is a string providing a human-readable description of the status of this experiment, required; 'name' is a string describing the experiment name, optional; 'seed' is a string describing the experiment-level random seed, optional; 'shots' is a string describing the experiment-level number of shots, optional; 'data' is an object with properties: 'counts' is an object providing a histogram of results, optional, with additionalProperties of type integer and minimum 0; 'snapshots' is an object providing a snapshot of the simulator internal representation, optional. 'success' is a boolean indicating true if the complete input qobj executed correctly, optional. 'status' is a string describing the human-readable status of complete qobj execution, required. No additional properties are allowed at the root level.
