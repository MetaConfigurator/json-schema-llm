The schema describes a configuration for integrating Redmine with GitLab. It is an object type. It defines several reusable components under 'definitions': 'redmine-configuration' is an object, requires 'username', 'password', and 'base-uri', and does not allow additional properties. 'git-lab-configuration' is an object, requires 'private-token' and 'base-uri', and does not allow additional properties. 'labels-mapping' is an object, does not allow additional properties, and contains properties with pattern properties for custom fields that reference 'simple-label-mapping'. 'simple-label-mapping' is an object, requires at least one item in an array associated with any string key, and does not allow additional properties. 'users-mapping' is an object, requires an integer value for keys matching '^\d+$', and does not allow additional properties. The main properties of the schema are: 'redmine' is a redmine-configuration, required. 'git-lab' is a git-lab-configuration, required. No additional properties are allowed at the root level.
