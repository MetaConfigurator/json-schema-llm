The schema describes the configuration for OAuth2 authentication. It is an object type. The main properties of the schema are: 'provider' is an object that contains the OAuth2 endpoint configuration. It is optional and has properties: 'url' is a string describing the URL of the OAuth flow provider's entry point; 'redirectUrl' is a string describing the URL the OAuth provider should redirect to; 'accessTokenUrl' is a string describing the location where the application can obtain an access token; 'clientId' is a string describing the client identifier; 'clientSecret' is a string describing the client secret; 'scope' is a string describing the authorization scopes to request from the provider; 'sessionStorageId' is a string describing a unique identifier for storing temporary information inside the browser's sessionStorage with a default value of 'default'. 'auth' is an object that is required to authenticate with the OAuth2 provider, containing properties: 'onActions' is an array of strings describing names of actions that trigger OAuth2 authentication, with a default value of an empty array; 'token' is a string describing the given OAuth2 access token; 'resource' is a string describing the name under which to publish the OAuth access token; 'flag' is a string describing the name of the flag indicating the current authentication state; 'drop' is an object describing when to drop authorization, with properties: 'onActions' as an array of strings describing names of actions that trigger de-authentication, which has a default value of an empty array. No additional properties are allowed at the root level.
