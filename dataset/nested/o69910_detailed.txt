The schema describes subscribing users. It is an object type. It defines several reusable components under 'definitions': 'user_subscription' is an object, which requires 'email', 'user_id', and 'subscribe'. It allows either 'list_id' or 'list_code' to be present but not both. The properties include 'email' as a string with email format, 'list_id' as an integer, 'list_code' as a string, 'subscribe' as a boolean, 'user_id' as an integer, 'utm_params' as an object, and 'variant_id' as an integer. The main properties of the schema are: 'users' is an array of items, each referencing the 'user_subscription' definition and is required. No additional properties are allowed at the root level.
