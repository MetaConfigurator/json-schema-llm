The schema describes a configuration for device management. It is an object type. The main properties of the schema are: 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$', required. 'deviceId' is a string matching the pattern '^[A-Fa-f\d]{24}$', required. 'deviceClass' is a string with allowed values 'standalone', 'gateway', 'peripheral', 'floating', 'edgeCompute', 'system', required. 'token' is a string with a minimum length of 1, required. 'ownerType' is a string with allowed values 'user', 'organization', optional. 'filterType' is a string that must be one of 'all', 'whitelist', 'blacklist' or 'none', optional. 'pubTopics' is an array of strings with items that have a minimum length of 1 and a maximum length of 1024, optional. 'subTopics' is an array of strings with items that have a minimum length of 1 and a maximum length of 1024, optional. No additional properties are allowed at the root level.
