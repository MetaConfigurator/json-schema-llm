The schema describes retrieving details on all exports that a user has access to within a specific identity. It is an object type. The main properties of the schema are: 'exports' is an array containing objects that are required to have fields 'id', 'name', 'list_type', 'format', and 'mime_type'. Each object in the 'exports' array must have an 'id' of type integer, a 'name' of type string, a 'list_type' that can be either a string or null with allowed values 'all', 'clipboard', or 'search', a 'format' of type string with allowed values 'Comma-Separated Values', 'Microsoft Excel', 'Website HTML', 'Fixed Width Report', or 'Comma-Separated Values in Multiple Files', and a 'mime_type' of type string with allowed values 'text/csv;charset=iso-8859-1', 'application/vnd.ms-excel', 'text/html', 'text/plain', or 'application/zip'. The 'href' is a string that must match the pattern '/api/v1/user_identities/\d+/exports'. No additional properties are allowed at the root level.
