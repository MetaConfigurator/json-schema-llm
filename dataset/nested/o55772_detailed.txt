The schema describes the structure of sblg JSON output. It is an object type. It defines several reusable components under 'definitions': 'article' is an object, representing a single article in a blog. It contains optional fields: 'src' as a string, 'base' as a string, 'stripbase' as a string, 'striplangbase' as a string, 'time' as an integer, 'title' as a reference to 'markup', 'aside' as a reference to 'markup', 'author' as a reference to 'markup', 'article' as a reference to 'markup', and 'tags' as an array of strings. The 'markup' is an object, which includes properties: 'text' as a string and 'xml' as a string, with 'xml' being a required field. The main properties of the schema are: 'version' is a string, describing the version of sblg generating the JSON. It is optional. 'articles' is an array, describing the sorted articles in the blog, with items referencing the 'article' definition. No additional properties are allowed at the root level.
