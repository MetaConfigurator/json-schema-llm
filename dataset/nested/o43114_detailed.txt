The schema describes a high level JSON description from which code is generated. It is an object type. The main properties of the schema are: 'activityName' is a string that describes the activity to the user, required, with a default value of 'Unit Activity'. 'class' is a string, required, with a default value of 'disabled', intended as an internal hack to be deleted. 'started' is a boolean indicating if the activity has started, required, with a default value of false. 'completed' is a boolean indicating if the activity has completed, required, with a default value of false. 'indice' is an object that contains properties: 'isBasic' (boolean), 'isGmap' (boolean), 'question' (string), 'response' (string), and 'index' (integer). 'mcq' is an object that contains properties: 'question' (string, describing the question of a SA MCQ activity) and 'resp' (an object with properties 'resp1', 'resp2', 'resp3', 'resp4' (all strings), and 'correct' (string)). Compositional constraints include that all of 'activityName', 'class', 'started', 'completed', and 'mcq' are required fields. No additional properties are allowed at the root level.
