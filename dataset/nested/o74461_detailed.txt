The schema describes a user profile. It is an object type. The main properties of the schema are: 'age' is an integer with a minimum value of 9 and a maximum value of 100, required. 'available_for_hire' is a boolean, required. 'email' is a string with a maximum length of 128, formatted as an email, required. 'id' is a string formatted as a UUID, required. 'interests' is an array containing strings, with a minimum of 3 and a maximum of 100 items, all unique, required. 'location' is an object containing 'country' (enum: 'US', 'CA', 'GB') and 'address' (string with a maximum length of 128), both required. 'name' is a string with a minimum length of 1, a maximum length of 64, and a specific pattern, required. 'skills' is an array containing objects with 'name' (string with a maximum length of 64 and a minimum length of 1) and 'value' (number with a minimum value of 0, maximum value of 100, and multiple of 0.25), both required, containing a maximum of 100 unique items. No additional properties are allowed at the root level.
