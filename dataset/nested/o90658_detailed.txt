The schema describes configuration properties for a connect service. It is an object type. The main properties of the schema are: 'connect' is an object that contains configuration settings for connect workers, which is required. It has the following required properties: 'cpus' is a number that allocates CPU shares to each connect worker instance, with a default value of 2 and a minimum value of 1. 'mem' is a number that specifies the memory (in MB) to allocate to each connect worker instance, with a default value of 1024 and a minimum value of 512. 'instances' is an integer that indicates the number of instances to run, with a default value of 1 and a minimum value of 1. 'name' is a string that serves as the service name for the connect worker application(s), with a default value of 'connect'. 'hdfs' is an object that provides Connect-HDFS configuration properties, which is optional. It has an optional property: 'config-url' is a string that specifies the base URL that serves HDFS config files, with no default value. No additional properties are allowed at the root level.
