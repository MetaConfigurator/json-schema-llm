The schema describes a revision document. It is an object type. The main properties of the schema are: 'id' is a number that represents the revision identifier (required). 'timestamp' is a string that represents the time that the revision was saved in YYYY-MM-DDTHH:MM:SSZ format (required). 'page' is an object that contains metadata about the page the revision was saved to (required). 'contributor' is an object or null that contains metadata about the user who saved the edit (required). 'minor' is a boolean indicating if this is a minor edit (required). 'comment' is a string or null that serves as the revision summary (optional). 'text' is a string or null that contains the content of the revision (optional). 'diff' is an array that includes a collection of operations representing the changes made in this revision (optional). 'bytes' is a number or null that indicates the size of revision content in bytes (optional). 'sha1' is a string representing a sha1 hash of the revision content (required). 'parent_id' is a number or null that indicates the revision ID of the preceding revision (optional). 'model' is a string with unspecified meaning (required). 'format' is a string with unspecified meaning (required). No additional properties are allowed at the root level.
