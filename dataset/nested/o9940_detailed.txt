The schema describes an object with user role specifications. It is an object type. The main properties of the schema are: 'applicationRoles' is an array containing objects that must have 'resourceId' (a string matching the pattern '^[A-Fa-f\d]{24}$') and 'role' (a string with possible values 'collaborate', 'view', 'none'). It has a maximum of 1000 items. 'dashboardRoles' is an array containing objects that must have 'resourceId' (a string matching the pattern '^[A-Fa-f\d]{24}$') and 'role' (a string with possible values 'collaborate', 'view', 'none'). It has a maximum of 1000 items. 'email' is a string formatted as an email, with a maximum length of 1024 characters and is required. 'role' is a string with possible values 'admin', 'edit', 'collaborate', 'view', and 'none', and is required. No additional properties are allowed at the root level.
