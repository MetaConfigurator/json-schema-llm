The schema describes proxy server configuration. It is an object type. It defines several reusable components under 'definitions': 'service' is an object, which requires a 'basePath' field formatted as a URI. It may also contain a 'processResponse' field that references the 'processResponse' definition. 'processResponse' is a definition that defines how the response from the service is processed. It can be either a string that returns only the response body if the status code is < 300 and throws an exception otherwise, with an allowed enum value of 'body', or it can be a function that uses the keyword 'typeof', which should return a result or throw an exception. The main properties of the schema are: 'services' is an object that requires at least one property, where each property name conforms to the pattern '^[A-Za-z_$][A-Za-z_$0-9]+$' and references the 'service' definition. 'jsonscript' is an optional object for options passed to the JSONScript interpreter. 'processResponse' references the 'processResponse' definition. 'Promise' is an optional property described as a function. No additional properties are allowed at the root level.
