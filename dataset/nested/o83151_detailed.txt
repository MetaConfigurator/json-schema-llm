The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': 'from' is an object, it requires a 'baseImage' field and can optionally include an 'alias' field. 'labels' is a oneOf type, where it can either be an object with string values or an array of unique strings. 'list_or_dict' is a oneOf type, capable of being an object with string, number, or null values or an array of unique strings. The main properties of the schema are: '# ' is an optional string description of a file comment. 'add' is optional and can either be an array of strings or an object. 'args' is a reference to the 'list_or_dict' definition. 'cmd' is optional and can be either a string or an array of strings. 'comment' is an optional string. 'copy' is optional and can either be an array of strings or an object. 'entrypoint' is optional and can be either a string or an array of strings. 'env' is a reference to the 'list_or_dict' definition. 'expose' is an array of unique items that can be strings or numbers formatted as 'expose'. 'labels' is a reference to the 'labels' definition. 'run' is optional and can be either a string or an array of strings. 'shell' is an optional array of strings. 'stopsignal' is an optional string. 'user' is an optional string. 'volumes' is an optional array of unique strings. 'working_dir' is an optional string. No additional properties are allowed at the root level.
