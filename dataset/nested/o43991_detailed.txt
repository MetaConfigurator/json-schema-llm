The schema describes a set charging profile request. It is an object type. The main properties of the schema are: 'connectorId' is an integer required field, and 'csChargingProfiles' is an object required field. 'csChargingProfiles' consists of several properties: 'chargingProfileId' is an integer required field, 'transactionId' is an integer optional field, 'stackLevel' is an integer required field, 'chargingProfilePurpose' is a string required field with allowed values 'ChargePointMaxProfile', 'TxDefaultProfile', 'TxProfile', 'chargingProfileKind' is a string required field with allowed values 'Absolute', 'Recurring', 'Relative', 'recurrencyKind' is a string optional field with allowed values 'Daily', 'Weekly', 'validFrom' is a string optional field formatted as a date-time, 'validTo' is a string optional field formatted as a date-time, and 'chargingSchedule' is an object required field. 'chargingSchedule' consists of: 'duration' is an integer optional field, 'startSchedule' is a string optional field formatted as a date-time, 'chargingRateUnit' is a string required field with allowed values 'A', 'W', 'chargingSchedulePeriod' is an array required field with items that are objects containing: 'startPeriod' is an integer required field, 'limit' is a number required field that is a multiple of 0.1, and 'numberPhases' is an integer optional field, and 'minChargingRate' is a number optional field that is a multiple of 0.1. Compositional constraints include that 'chargingSchedule' requires both 'chargingRateUnit' and 'chargingSchedulePeriod'. No additional properties are allowed at the root level.
