The schema describes a configuration for Markdown processing. It is an object type. The main properties of the schema are: 'markdown' is an object that tells how to obtain the Markdown-formatted text. It is required and must contain the 'parameter' property. The 'parameter' is a string describing the place parameter for the flow definition to support anchors. The 'resource' is an optional string with the format 'topic' and an axRole of 'inlet', which indicates the resource topic under which Markdown-formatted text is expected or a URL referencing a file with Markdown-formatted text. The 'attribute' is an optional string that specifies the path to the actual Markdown-formatted text. The 'url' is an optional string referring to a Markdown-formatted text. 'messages' is an object that configures messages published via didEncounterError or didValidate events. The 'i18nFailedLoadingResource' property can be either an object or a string and is the message to publish when a GET for a Markdown-formatted text source fails. It has a default message with English: 'Could not load resource from "[url]"' and German: 'Die Ressource konnte nicht von "[url]" geladen werden.' 'i18n' is an object that specifies which locale to use for displaying this widget. It has a 'locale' property which is a string with the format 'topic' and an axRole of 'inlet', indicating the topic under which to expect the locale for this widget, with a default value of 'default'. No additional properties are allowed at the root level.
