The schema describes an object structure for managing organizational data. It is an object type. The main properties of the schema are: 'id' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is optional. 'orgId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is optional. 'creationDate' is a string formatted as date-time and is optional. 'lastUpdated' is a string formatted as date-time and is optional. 'name' is a string with a minimum length of 1 and a maximum length of 255 and is optional. 'description' is a string with a maximum length of 32767 and is optional. 'members' is an array of objects where each object contains: 'userId' as a string matching the pattern '^[A-Fa-f\d]{24}$' and is optional, 'firstName' as a string and is optional, 'lastName' as a string and is optional, 'email' as a string formatted as email with a maximum length of 1024 and is optional, and 'role' as a string that can be one of the following values: 'admin', 'edit', 'view'. 'summary' is an object containing: 'appCount' as a number and is optional, and 'dashCount' as a number and is optional. No additional properties are allowed at the root level.
