The schema describes a structured representation of educational content components. It is an object type. It defines several reusable components under 'definitions': 'LOM' is an object, it is a data model to describe a learning object used to support learning, with required fields: 'difficulty', 'format', 'interactivity', 'semantic', and 'type'. 'bloom' is an object, it describes a student knowledge model based on Bloom taxonomy, requiring the fields: 'level' and 'target', which are enums with allowed values 'beginner', 'middle', 'expert' for 'level', and 'knowledge', 'comprehension', 'application', 'analysis', 'synthesis', 'evaluation' for 'target'. 'Section' is an object, it is a section of a lesson from a Botbloq catalog course, requiring 'id' and 'Lessons', where 'Lessons' is an array containing at least one unique item referenced from the 'Lesson' definition. 'Lesson' is an object, it is a lesson from a Botbloq catalog course, requiring 'id' and 'resource', where 'resource' contains references to 'LOM' and 'bloom'. 'Course' is an object, it is a course from Botbloq catalog, requiring 'id', 'name', 'content', 'objetives', 'bibliography', and 'Sections', where 'Sections' is an array containing at least one unique item referenced from the 'Section' definition. The root schema resolves to the referenced definitions. No additional properties are allowed at the root level.
