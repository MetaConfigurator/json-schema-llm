The schema describes a field configuration for a form. It is an object type. The main properties of the schema are: 'name' is a string that serves as a unique identifier (within the form) for the field. This field is required. Its value must match the pattern '^[a-zA-Z_]{1}[\w-]*$'. 'maps_to' is a string with a default value of 'cf', which indicates the schema field for the answer mapping. It must match the pattern '^[a-zA-Z_]{1}\w*$'. 'label' is a string that represents the main text for the question/field, with a minimum length of 0 and a maximum length of 255 characters. 'placeholder' is an optional string that can also have a minimum length of 0 and a maximum length of 255 characters. 'description' is a string that provides a short explanation of the field, with a minimum length of 0 and a maximum length of 65535 characters. 'is_required' is a boolean that indicates if the field is required. 'link_text' is an optional string that can have a minimum length of 0 and a maximum length of 255 characters, representing the text that will replace the token '{link}' in the label or description. 'link_url' is a string representing the URL for replacing the token '{link}', which must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$'. 'pii_impact' is a string with an enum that accepts the values 'unknown', 'high', 'moderate', 'low'. No additional properties are allowed at the root level.
