The schema describes a structure for downloading items with various configuration filters. It is an object type. It defines several reusable components under 'definitions': 'filter-definition' is an object, which contains properties for filters including '+' and '-', both referencing 'filter-items', an array type that must contain at least one string item. 'download-node' is an object, required to have 'level' as a string from the enum values 'project', 'session', 'acquisition', 'analysis', and '_id' as a string matching the pattern '^[0-9a-f]{24}$'. 'download-filter' is an object that holds properties for filters including 'tags' and 'types', both referencing 'filter-definition'. 'download-input' is an object that must have 'optional' as a boolean and 'nodes' as an array containing at least one 'download-node'. 'download-ticket' is an object, required to have 'ticket' as a string. The main properties of the schema are: 'filter-definition' is a reference to 'filter-definition'. 'filter-items' is a reference to 'filter-items'. 'download-node' is a reference to 'download-node'. 'download-filter' is a reference to 'download-filter'. 'download-input' is a reference to 'download-input'. 'download-ticket' is a reference to 'download-ticket'. No additional properties are allowed at the root level.
