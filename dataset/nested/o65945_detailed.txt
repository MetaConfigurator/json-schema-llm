The schema describes a file structure representation. It is an object type. It defines several reusable components under 'definitions': 'file_common' is an object type, defining a file with a required 'name' field of type string. 'file_not_extracted' is an object type, inheriting from 'file_common', requiring an additional 'type' field with allowed values of 'file', 'dir', 'link', or 'remote_dir', and ensuring that it does not reference 'file_extracted'. 'file_extracted' is an object type, also inheriting from 'file_common', requiring 'user' and 'group' fields of type string with a minimum length of 1, and must match one of the definitions: 'file_extracted_file', 'file_extracted_dir', or 'file_extracted_link'. 'file_extracted_file' is an object type requiring 'type' to be 'file', 'size' to be a non-negative integer, and 'mode' to be a string conforming to the pattern '^[0-7]{3,4}$'. 'file_extracted_dir' is an object type requiring 'type' to be 'dir', 'size' to be a non-negative integer, 'mode' to be a string conforming to the pattern '^[0-4]?[0-7]{3}$', and 'files' to be an integer with a minimum value of 1. 'file_extracted_link' is an object type requiring 'type' to be 'link'. 'file_remote_dir' is an object type, inheriting from 'file_common', requiring 'type' to be 'remote_dir'. The main properties of the schema are: 'extracted' is a boolean and required, 'files' is an array with a oneOf condition that allows it to be either an empty array, an array containing at least one item that matches either 'file_extracted' or 'file_remote_dir', or an array containing at least one item that matches either 'file_not_extracted' or 'file_remote_dir'. No additional properties are allowed at the root level.
