The schema describes a validation report. It is an object type. The main properties of the schema are: 'warningCount' is a number that represents the number of validation warning messages and is optional, 'errorCount' is a number that represents the number of validation error messages and is optional, 'messages' is an array of objects, which is required. Each object in the 'messages' array has properties: 'id' is a string that serves as the message identifier and must match the pattern '^C[0-9]{5}$', 'type' is a string that indicates the type of message, which must be one of: error, warning, and 'text' is a string that represents the message content. At least 'type' and 'id' are required in each message object. No additional properties are allowed at the root level.
