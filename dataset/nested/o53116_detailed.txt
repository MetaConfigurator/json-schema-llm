The schema describes resource configuration. It is an object type. The main properties of the schema are: 'resource' is a string that represents the resource name or abbreviation, required and has a maximum length of 64 characters. 'name' is a string that denotes the formal name of the resource, required and has a maximum length of 200 characters. 'description' is a string that provides a description of the resource, optional and has a maximum length of 1000 characters. 'resource_type_id' is an integer that specifies the resource type ID, optional with a minimum value of 0. 'pi_column' is a string that indicates the column from the resource manager specific table to use as the PI data source, optional. 'shared_jobs' is a boolean that states whether jobs may share nodes on this resource, optional. 'timezone' is a string representing the timezone where this resource is located, optional and has a maximum length of 30 characters. No additional properties are allowed at the root level.
