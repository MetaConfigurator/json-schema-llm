The schema describes a conference. It is an object type. The main properties of the schema are: 'acronym' is a string representing the conference acronym, optional. 'alternative_titles' is an array of strings representing alternative titles, optional, containing unique items. 'closing_date' is a string in date format representing the conference closing date, optional. 'cnum' is a string that follows the pattern 'C\d\d-\d\d(-\d\d)?(\.\d+)' representing the conference ID, optional. 'contact_email' is a string in email format representing the contact email, optional. 'contact_person' is a string representing the contact person, optional. 'field_code' is an array of strings representing field codes, optional, containing unique items with allowed values such as 'Astrophysics' and 'Phenomenology-HEP'. 'keywords' is an array of strings representing free keywords, optional, containing unique items. 'nonpublic_note' is a string representing a non-public note, optional. 'note' is a string representing a note, optional. 'opening_date' is a string in date format representing the conference opening date, optional. 'place' is a string that follows the pattern '\.+, \.+' representing the conference place, optional. 'proceedings' is a string in URL format representing the conference proceedings URL, optional. 'series' is a string representing the conference series, optional. 'series_number' is an integer representing the conference series number, optional. 'short_description' is a string representing a short description, optional. 'subtitle' is a string representing the conference subtitle, optional. 'title' is a string representing the conference title, optional. 'transparencies' is a string in URL format representing the conference transparencies URL, optional. 'url' is a string in URL format representing the conference URL, optional. No additional properties are allowed at the root level.
