The schema describes JSON Table structures. It is an object type. It defines several reusable components under 'definitions': None. The main properties of the schema are: 'fields' is an array that must contain at least one item, where each item is an object with a required 'name' property of type string. Other optional properties are 'title', 'description', 'type' with allowed values 'string', 'number', 'integer', 'date', 'time', 'datetime', 'boolean', 'binary', 'object', 'geopoint', 'geojson', 'array', 'any', 'format' as a string, and 'constraints' as an object containing optional validation properties. 'primaryKey' is a string or an array that is optional. 'foreignKeys' is an optional array containing objects, each requiring 'fields' (a string or an array) and a 'reference' object that must have 'resource' (a string) and 'fields' (a string or an array). No additional properties are allowed at the root level.
