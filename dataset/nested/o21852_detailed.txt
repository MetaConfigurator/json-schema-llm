The schema describes an API request. It is an object type. The main properties of the schema are: 'HttpMethod' is a string that is required and must be one of the following values: 'DELETE', 'GET', 'HEAD', 'OPTIONS', 'PATCH', 'POST', 'PUT'. 'ResourceId' is a string that is required and must match the pattern '^[a-z0-9]+$'. 'RestApiId' is a string that is required and must match the pattern '^[a-z0-9]{10}$'. 'StatusCode' is a string that is required and must match the pattern '^\d{3}$'. 'ResponseModels' is an object that is optional, does not allow additional properties, and has pattern properties that must be strings. 'ResponseParameters' is an object that is optional, does not allow additional properties, and has pattern properties that must be one of the following values: 'true', 'false' for keys matching the pattern '^method\.response\.header\.+$'. No additional properties are allowed at the root level.
