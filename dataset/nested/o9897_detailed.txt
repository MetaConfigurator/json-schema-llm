The schema describes the structure for a view configuration. It is an object type. The main properties of the schema are: 'name' is a string with a minimum length of 1 and a maximum length of 255, required. 'description' is a string with a maximum length of 32767, optional. 'viewType' is a string that must be one of the following values: 'layout', 'page', 'component', required. 'layoutId' is either a string or null, with a pattern that matches '^[A-Fa-f\d]{24}$', optional. 'body' is a string with a minimum length of 1 and a maximum length of 131072, required. 'viewTags' is an object where each property key matches the pattern '^[0-9a-zA-Z_-]{1,255}$' and each property value is a string with a minimum length of 1 and a maximum length of 255, no additional properties are allowed. No additional properties are allowed at the root level.
