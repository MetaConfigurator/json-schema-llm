The schema describes the structure of the api-index.json files that the documentation build can create for the Demo Kit. It is an object type. It defines several reusable components under 'definitions': 'symbol' is an object, representing a symbol with required fields 'name', 'kind', 'lib', and 'visibility'. It has properties for an array 'children' of symbols, a boolean 'deprecated', a string 'displayName' (with a minimum length of 1), an array 'extendedBy' of strings, a string 'extends', an array 'implementedBy' of strings, and a string 'kind' with enum values 'namespace', 'class', 'interface', 'enum', 'typedef', and 'function'. The 'visibility' property has a default value of 'public' and enum values 'public', 'protected', 'private', and 'restricted'. The string 'lib' has a minimum length of 1. The main properties of the schema are: '$schema-ref' is a string referencing the schema with an enum value of 'http://schemas.sap.com/sapui5/designtime/api-index.json/1.0', required to be present. 'name' is a string representing the scope of libraries (required). 'symbols' is an array of symbols (required) where each item follows the 'symbol' definition. 'version' is a string following the 'version' pattern (required). No additional properties are allowed at the root level.
