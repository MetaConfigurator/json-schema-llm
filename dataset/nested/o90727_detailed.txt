The schema describes the configuration for a Linkerd instance. It is an object type. The main properties of the schema are: [linkerd] is an object requiring the properties: admin-port, config-filename, config-uri, cpus, instances, mem, name, resource-roles, and routing-port. [admin-port] is an integer providing an administrative UI for this instance, with a default value of 9990. [config-filename] is a string representing the URI of the linkerd config file, with a default value of 'linkerd-dcos-060.yaml'. [config-uri] is a string for the URI of the linkerd config file, with a default value of 'https://s3.amazonaws.com/buoyant-dcos'. [cpus] is a number indicating CPU shares to allocate to each linkerd instance, with a minimum value of 0.25 and a default of 0.25. [instances] is an integer for the number of instances to run, with a minimum value of 1 and a default of 1. [mem] is a number for memory (MB) to allocate to each linkerd task, with a minimum value of 256.0 and a default of 256.0. [name] is a string naming the linkerd instance(s), with a default value of 'linkerd'. [resource-roles] is a string for the accepted resource roles, with a default value of '"*", "slave_public"'. [routing-port] is an integer for the routing port, with a default value of 4140. No additional properties are allowed at the root level.
