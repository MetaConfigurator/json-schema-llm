The schema describes evidence of the certificate on the blockchain, using the chainpoint v2 standard. It is an object type. The main properties of the schema are: '@context' is a string that should always be chainpoint v2 JSON LD context. It is required. 'type' is a string that represents the type of hash. Currently the only supported hash type is SHA256, with chainpoint type ChainpointSHA256v2. It is optional. 'targetHash' is a string that represents the hash of the item being verified. Currently the only supported hash type is SHA256, and the targetHash format is validated accordingly. It is required. 'merkleRoot' is a string that represents the Merkle root value, which is anchored to the blockchain. It is required. 'proof' is an array that details how to walk the Merkle tree from the target item to the Merkle root. It is required. Each item in the 'proof' array is an object containing 'left', which is a string representing the value of the left neighbor to combine into the parent hash, and 'right', which is a string representing the value of the right neighbor to combine into the parent hash. Both fields follow the validated format for SHA256. 'anchors' is an array that describes how the proof is anchored to the blockchain. It is required. Each item in the 'anchors' array is an object with a required field 'sourceId', which is a string expected to be the Bitcoin transaction id, following the validated format for SHA256. Additionally, there is a field 'type', which is a string representing the type of anchor, e.g. BTCOpReturn, where currently the only supported value is BTCOpReturn. No additional properties are allowed at the root level.
