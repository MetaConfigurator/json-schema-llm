The schema describes a collection of adventures. It is an object type. The main properties of the schema are: 'adventure' is an array, containing objects that represent each adventure. Each adventure object is required and must include 'name', 'id', 'source', 'contents', 'level', 'published', and 'storyline'. The 'contents' property is an array of objects, each of which must include 'name'. Each content object can include 'headers', which is an array that may contain strings or objects with 'depth' and 'header', both of which are required when present. The 'ordinal' property is an object that requires 'type' (which must be one of the enumerated values: 'chapter', 'appendix', 'part', 'episode', 'level') and 'identifier' (which can be either a string or an integer). The 'level' can either be an object with 'start' and 'end' (both required) or an object with 'custom' (which is required). The 'published' property is a string formatted as a date, while 'publishedOrder' is an integer used for sorting adventures. The 'coverUrl' is an optional string, and 'storyline' must be one of the following enumerated values: 'Starter Set', 'Tyranny of Dragons', 'Elemental Evil', 'Rage of Demons', 'Curse of Strahd', 'Storm King's Thunder', 'Tales from the Yawning Portal', 'Tomb of Annihilation', 'Waterdeep', 'Ravnica', 'Extra Life', 'Ghosts of Saltmarsh', 'Acquisitions Incorporated', 'Essentials Kit', 'Stranger Things', 'Baldur's Gate', 'Eberron', 'Rick and Morty', 'Wildemount'. No additional properties are allowed at the root level.
