The schema describes a test case. It is an object type. It defines several reusable components under 'definitions': 'issue' is an object, GitHub issue title and action comments (optional). Title and comments will be based on rule name in case this property is absent. It requires the field 'title'. The main properties of the schema are: 'diag' is an object that includes the test diagnostics, which contains several properties: 'issue' is a reference to the 'issue' definition, 'message' is an optional string that is not used in GitHub issues, 'messages' is an optional array of strings used as GitHub issue comments, 'repo' is a required string with a pattern matching a full repository name, 'rule' is a required string with a specific pattern for rule names, and 'severity' is an optional string used as a GitHub issue label. 'id' is a required integer that acts as a test sequential number, 'name' is a required string for the test name that should be unique across all tests, and 'ok' is a required boolean representing the test result. No additional properties are allowed at the root level.
