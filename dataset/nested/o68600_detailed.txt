The schema describes a set of domains. It is an array type. The main properties of the schema are: 'items' is an object which describes the domain structure. It requires the properties 'apiVersion', 'kind', 'spec', and 'metadata'. 'apiVersion' is a string that must be one of the following values: 'v1beta1', 'v1.0', 'v1'. 'kind' is a string that must be one of the following values: 'Domain', 'domain', and is readOnly. 'metadata' is an object that requires the property 'name', which is a string with a minimum length of 3 and a maximum length of 30. It also includes optional properties 'createdOn', 'modifiedOn', and 'ref', with 'ref' being a string that must have a minimum length of 3 and a maximum length of 24, all of which are readOnly. 'spec' is an object that requires the property 'context'. 'context' is also an object that requires the property 'domainUri', which is a string with a minimum length of 1, and is readOnly. It includes an optional 'egressPolicy' that requires both 'rule' and 'numberRef', which are strings. 'accessControlList' can contain optional properties 'allow' and 'deny', both of which are arrays of strings that must have at least one item and contain unique items. No additional properties are allowed at the root level.
