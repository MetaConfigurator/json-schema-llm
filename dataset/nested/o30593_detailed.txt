The schema describes a configuration for a network group. It is an object type. The main properties of the schema are: 'allocpolicy' is a string that specifies the allocation policy and can be one of the following values: 'PORTS', 'IPADDRS'. 'clientname' is a string that specifies the name of the client. 'ftp' is a string that specifies the FTP setting and can be either 'ENABLED' or 'DISABLED'. 'ftpcm' is a string that indicates whether FTPCM is enabled or disabled and can be either 'ENABLED' or 'DISABLED'. 'groupid' is an integer that is read-only and represents the group identifier. 'groupname' is a string for the name of the group. 'ip6profile' is a string that specifies the IPv6 profile. 'logging' is a string that indicates the logging setting and can be either 'ENABLED' or 'DISABLED'. 'nattype' is a string that specifies the NAT type and can be either 'DYNAMIC' or 'DETERMINISTIC'. 'portblocksize' is an integer that indicates the size of the port block. 'pptp' is a string that specifies whether PPTP is enabled or disabled and can be either 'ENABLED' or 'DISABLED'. 'rtspalg' is a string that specifies the RTSP ALG setting and can be either 'ENABLED' or 'DISABLED'. 'sessionlogging' is a string that indicates whether session logging is enabled or disabled and can be either 'ENABLED' or 'DISABLED'. 'sessionsync' is a string that specifies whether session synchronization is enabled or disabled and can be either 'ENABLED' or 'DISABLED'. 'sipalg' is a string that indicates whether SIP ALG is enabled or disabled and can be either 'ENABLED' or 'DISABLED'. 'snmptraplimit' is an integer that specifies the limit for SNMP traps. No additional properties are allowed at the root level.
