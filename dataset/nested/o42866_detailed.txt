The schema describes a sluggable node. It is an object type. The main properties of the schema are: '_id' is a string that MUST NOT change for the life of the node, SHOULD be globally unique, and SHOULD be generated by the app. It must match the pattern '^[\w\/\.:-]+$'. 'status' is a string with a default value of 'draft' and can be one of the following values: 'unknown', 'published', 'scheduled', 'pending', 'draft', 'expired', 'archived', 'deleted'. 'etag' is a string matching the pattern '^[\w\.:-]+$'. 'created_at' is a string matching the pattern '^[1-9]{1}[0-9]{12,15}$'. 'creator_ref' is an object with required fields 'curie' and 'id', neither of which can have additional properties. 'curie' must match the pattern '^([a-z0-9-]+):([a-z0-9\.-]+):([a-z0-9-]+)?:([a-z0-9-]+)$' and can have a maximum length of 146 characters. 'id' must match the pattern '^[A-Za-z0-9:_\-]+$' and can have a maximum length of 255 characters. 'tag' is an optional string matching the pattern '^([\w\/-]|[\w-][\w\/-]*[\w-])$' with a maximum length of 255 characters. 'updated_at' is a string matching the pattern '^[1-9]{1}[0-9]{12,15}$'. 'updater_ref' is similar to 'creator_ref' with the same constraints. 'last_event_ref' is an object with the same properties and constraints as 'creator_ref'. 'title' is an optional string with a maximum length of 255 characters. 'slug' is a string matching the pattern '^([\w\/-]|[\w-][\w\/-]*[\w-])$', with a maximum length of 255 characters, indicating that it is a secondary identifier typically used in a URL that MUST be url friendly and SHOULD be unique. No additional properties are allowed at the root level.
