The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'extensions' is an array of extension points, which contains objects that may have properties 'type' (string), 'order' (number), 'title' (string), and 'options' (array of objects with 'title' and 'category' string properties). If 'type' is 'action', it may include a 'bindings' array of objects, where each object requires a 'shortcut' (string matching pattern '^[\S]+(\s[\S]+)?$') and may optionally contain 'platform' (string) and 'keybindSets' (array of strings with enum values 'devToolsDefault' and 'vsCode'). The 'category' property is required in this case, while either 'title' or 'options' (which must have at least one item with a 'title') is also required. 'dependencies' is an array of strings, 'scripts' is an array of strings, 'modules' is an array of strings, 'resources' is an array of strings, and 'skip_compilation' is an array of strings. 'experiment' is a string. No additional properties are allowed at the root level.
