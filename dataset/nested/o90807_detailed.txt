The schema describes OpenLDAP configuration properties. It is an object type. The main properties of the schema are: 'networking' is an object OpenLDAP networking configuration properties, optional. 'openldap' is an object OpenLDAP service configuration properties, required fields are 'cpus' and 'mem'. 'service' is an object DC/OS service configuration properties, optional. 'storage' is an object OpenLDAP storage configuration properties, optional. The 'networking' property contains 'external_access', an object that represents access settings with properties 'enable' (a boolean), defaulting to false, and 'external_port' (a number), defaulting to 10389. The 'openldap' property contains several string properties including 'additional_modules', 'additional_schemas', 'admin_password', 'config_password', 'domain', and 'organization', with defaults for 'admin_password', 'config_password', 'domain', and 'organization'. It also includes a number property 'cpus' with a minimum of 0.3, and 'mem' with a minimum of 128.0. The 'service' property includes a string property 'name', defaulting to 'openldap'. The 'storage' property contains a string property 'host_volume' with a default of '/tmp', and an object 'persistence' with properties 'enable' (a boolean), 'ldap_volume_size' (a number), and 'slapd_volume_size' (a number), each with specified default values. No additional properties are allowed at the root level.
