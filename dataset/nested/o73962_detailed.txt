The schema describes a device. It is an object type. It defines several reusable components under 'definitions': "Location" is an object, which does not allow additional properties and has a required field "id" of type string. "DeviceConfigurationArchive" is an object, which does not allow additional properties and has a required field "configurations" of type string. "DeviceStatus" is an object, which does not allow additional properties and has no required fields. "DeviceInfo" is an object, which does not allow additional properties and has required fields "deviceType" of type string with enum values 'OTHER', 'STREETLIGHT_AREA', 'VARIABLE_MESSAGE_SIGN', 'PARKING_HOUSE', 'DETECTOR', 'TRAFFIC_LIGHT_CONTROLLER', "manufacturer" of type string, "designation" of type string, and "version" of type string. The main properties of the schema are: "locations" is an array of items referencing the "Location" definition, which is optional. "configuration" is an object referencing the "DeviceConfigurationArchive" definition, which is optional. "status" is an array of items referencing the "DeviceStatus" definition, which is optional. "deviceInfo" is an object referencing the "DeviceInfo" definition, which is optional. "guid" is a string, which is optional. "uri" is a string, which is optional. No additional properties are allowed at the root level.
