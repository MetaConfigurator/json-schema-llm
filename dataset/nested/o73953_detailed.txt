The schema describes a structured data format. It is an object type. It defines several reusable components under 'definitions': 'LocationsType' is an object, contains properties 'location' which is an array of 'LocationType' references, and 'numsys' which is an integer. 'LocationType' is an object, contains properties 'detectors' which is a reference to 'Detectors', 'id' which is a string, 'file' which is a string with enum values 'OK' and 'NONE', and 'path' which is a string. 'Detectors' is an object, contains a property 'detector' which is an array of 'DetectorType' references. 'DetectorType' is an object, contains properties 'id' which is a string, 'code' which is an integer, 'faze' which is an integer, 'lane' which is an integer, and 'mode' which is a string. The main properties of the schema are: 'locations' is a LocationsType reference, required; 'version' is a string, optional. No additional properties are allowed at the root level.
