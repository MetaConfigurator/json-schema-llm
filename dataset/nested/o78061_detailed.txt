The schema describes metadata for a task in BlackBox 3.0. It is an object type. The main properties of the schema are: 'author' is a string optional description of the author. 'category' is a string required field representing the category of the task. 'cost' is an integer required field with a minimum value of 0 representing the cost of the task. 'desc_en' is a string optional field with a pattern of '^.*\.md$' describing the task in English. 'desc_ru' is a string optional field with a pattern of '^.*\.md$' describing the task in Russian. 'flag' is a string required field representing a flag for the task. 'flag_comp' is a string required field representing a comparative flag for the task. 'images' is an array of objects required field with a minimum of 1 item, where each object contains a required property 'filename' (a string with a pattern of '^.*\.tar$'), an optional 'tcp_ports' (an array of integers with a minimum of 1 item and no duplicates), and an optional 'udp_ports' (an array of integers with a minimum of 1 item and no duplicates). 'schema_version' is a string required field with a pattern of '^0\.2$'. 'title_en' is a string required field for the English title. 'title_ru' is a string required field for the Russian title. 'upload_timestamp' is an integer optional field for the timestamp of the upload. 'writeup_en' is a string required field with a pattern of '^.*\.md$' for an English writeup. 'writeup_ru' is a string required field with a pattern of '^.*\.md$' for a Russian writeup. No additional properties are allowed at the root level.
