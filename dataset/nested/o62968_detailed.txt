The schema describes a representation of a state in the USA. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: '_metadata' is an object that contains required properties 'request' and 'response'. 'request' is an object that contains required properties 'ip', 'params', 'time', and 'user', where 'ip' is a string, 'params' and 'user' are null, and 'time' is a string. 'response' is an object that contains a required property 'time', which is a string. 'state' is an object that contains required properties 'id', 'state_code', and 'state_name', where 'id' is an integer, and both 'state_code' and 'state_name' are strings. No additional properties are allowed at the root level.
