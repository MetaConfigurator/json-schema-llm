The schema describes a control configuration. It is an object type. The main properties of the schema are: 'name' is a string that represents the name of the control, required. 'description' is a string that provides a short description of the purpose of this control, required. 'integration' is an object that defines how this control should be loaded by the LaxarJS runtime, optional. It has a property 'technology', which is a string indicating the underlying technology of the control, with a default value of 'plain'. 'controls' is an array that lists each control as a path that can be resolved using the module system, optional, defaulting to an empty array, with items being strings. 'templateSource' is a string that specifies the location of the control's HTML template, optional. 'styleSource' is a string that specifies the location of the control's stylesheet, optional. 'assets' is an array that contains a list of files available as raw content via the artifact provider API, optional, with items being strings. 'assetUrls' is an array that lists files available as URLs via the artifact provider API, optional, with items being strings. 'assetsForTheme' is an array that contains a list of themed files available as raw content via the artifact provider API, optional, with items being strings. 'assetUrlsForTheme' is an array that lists themed files available as URLs via the artifact provider API, optional, with items being strings. No additional properties are allowed at the root level.
