The schema describes a configuration profile for parental controls content filtering. It is an object type. The main properties of the schema are: 'PayloadDescription' is a string that provides a description of the payload. 'PayloadDisplayName' is a string that denotes the name of the payload. 'PayloadIdentifier' is a string that serves as a unique identifier for the payload, typically root PayloadIdentifier plus subidentifier. 'PayloadOrganization' is a string that describes the issuing organization of the profile, as displayed to the user. 'PayloadType' is a string that indicates the type of the payload, formatted as a reverse DNS string. 'PayloadUUID' is a string that represents a unique identifier for the payload, which must match the pattern ^[0-9A-Za-z]{8}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{12}$. 'PayloadVersion' is a number that specifies the version of the whole configuration profile. 'filterBlacklist' is an array that, if specified and restrictWeb is true, contains URLs of websites never to be allowed. 'filterWhitelist' is an array that, if specified and restrictWeb is true, designates the only allowed websites. 'restrictWeb' is a boolean that, when set to true, enables web content filters. 'siteWhitelist' is an array that, when whiteListEnabled is true, contains dictionaries defining additional allowed sites along with disallowed adult sites. Each item in 'siteWhitelist' is an object with properties 'address' (a string representing the site prefix, including http(s) scheme) and 'pageTitle' (an optional string for the page title). 'useContentFilter' is a boolean set to true to attempt to automatically filter content. 'whitelistEnabled' is a boolean set to true to utilize the filterWhiteList and filterBlackList lists. No additional properties are allowed at the root level.
