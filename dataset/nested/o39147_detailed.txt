The schema describes a vehicle's financial and operational information. It is an object type. It defines several reusable components under 'definitions': 'numberWithUnit' is an object, which requires properties 'value' (a number) and 'unit' (a string). The main properties of the schema are: 'car' is a string and required, 'comment' is a string and optional, 'cost' is an object resolving to 'numberWithUnit' and required, 'fuelType' is a string and required, 'mileage' is an object resolving to 'numberWithUnit' and required, 'missedFuelStop' is a boolean and optional, 'time' is a string in date-time format and required, 'volume' is an object resolving to 'numberWithUnit' and required. No additional properties are allowed at the root level.
