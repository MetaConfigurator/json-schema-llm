The schema describes a collection of action target templates. It is an array type. The main properties of the schema are: Each item in the array is an object that contains properties: 'id' is an integer that is required and represents a unique identifier given by the server. 'key' is a string that is required and represents a unique key generated by the server to be used in events. 'name' is a string that is required and represents a name to identify the action target template. 'public' is a boolean that is required and indicates whether the action target template is available to everybody. 'organizationId' is an integer that is required and represents the organization that offers the action target template. 'configuration' is an object that is required and provides a schema aims to help the WUI for configuration. It contains 'schema' (an object) and 'url' (a string) as required fields. 'configurationUi' is an object that is optional and provides the configuration for the dynamic UI. It contains 'schemaForm' (an array). 'target' is an object that is required and represents the target configuration. It contains 'url' (a string) as a required field. No additional properties are allowed at the root level.
