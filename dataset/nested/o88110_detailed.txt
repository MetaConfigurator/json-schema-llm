The schema describes information about the classes in a Puppet code environment. It is an object type. The main properties of the schema are: 'files' is an array of objects, representing the array of manifest files which exist in an environment. Even files that do not contain any classes will appear in the array. This property is required. Each item in the array contains: 'path' which is a string representing the fully-qualified path on the server to the manifest. This field is required. 'classes' is an array of objects, representing the classes defined in the manifest. This field is required if present. Each class object contains: 'name' which is a string representing the name of the class and is required, and 'params' is an array of objects, each containing 'name' (a required string) and optional fields 'type', 'default_source', and 'default_literal', all of which are strings. The 'params' array prohibits additional properties. If the manifest parsing encounters an error, 'error' is a string providing details about the specific error, and this field will be present instead of 'classes'. The schema defines that each object in the 'files' array must meet one of the following configurations: it must either have 'classes' as a required property or have 'error' as a required property. The property 'name' is a string representing the name of the environment and is required. No additional properties are allowed at the root level.
