The schema describes a restaurant's information. It is an object type. The main properties of the schema are: 'name' is a string, optional. 'location' is an object, required, containing: 'alias' which is a string, required; 'address' which is a string, required; and 'coordinates' which is an object, required, having: 'lat' which is a number, required; and 'lng' which is a number, required. 'contact' is an object, required, containing: 'phone' which is a string, required. 'priceRange' is an object, required, containing: 'from' which is a number, required; and 'to' which is a number, required. 'menu' is an array of objects, where each object must contain: 'name' which is a string, required; and 'price' which is a number, required. Each item in the menu can optionally have 'tags' which is an array of strings. No additional properties are allowed at the root level.
