The schema describes metadata for a HEPData table apart from independent_variables and dependent_variables. It is an object type. The main properties of the schema are: 'name' is a string and is required with a maximum length of 64. 'location' is a string and is optional with a maximum length of 256. 'description' is a string and is required. 'keywords' is an array and is required, containing items that are objects. Each keyword object must include 'name' (a string with a maximum length of 128 and an enum of allowed values: 'cmenergies', 'observables', 'reactions', 'phrases') and 'values' (an array of strings or numbers). 'data_file' is a string and is required. 'data_license' is an object and is optional, containing 'name' (a string with a maximum length of 256) and 'url' (a string with a maximum length of 256), both of which are required. 'additional_resources' is an array and is optional, containing items that are objects with a required 'location' (a string with a maximum length of 256). Each additional resource object can contain an optional 'description' (a string), an optional 'type' (a string with a maximum length of 64), and an optional 'license' (an object with required 'name' and 'url', both strings with a maximum length of 256). No additional properties are allowed at the root level.
