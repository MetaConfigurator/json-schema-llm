The schema describes AirPlay configuration settings. It is an object type. The main properties of the schema are: 'Passwords' is an array that sets passwords for known AirPlay destinations, where each item is an object containing 'DeviceName', a string describing the AirPlay destination's name, and 'Password', a string for the password of the AirPlay destination. 'PayloadDescription' is a string with a default value of 'Configures AirPlay settings', describing the payload. 'PayloadDisplayName' is a string with a default value of 'AirPlay settings', defining the name of the payload. 'PayloadIdentifier' is a string with a default value of 'com.apple.airplay', serving as a unique identifier for the payload. 'PayloadOrganization' is a string that describes the issuing organization of the profile, as displayed to the user. 'PayloadType' is a string with a default value of 'com.apple.airplay', indicating the type of the payload. 'PayloadUUID' is a string with a default value of '', representing a unique identifier for the payload, which must match the pattern '^[0-9A-Za-z]{8}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{12}$'. 'PayloadVersion' is a number with a default value of 1, indicating the version of the whole configuration profile. 'Whitelist' is an array that contains a list of visible AirPlay destinations restricted on this device, where each item is an object containing 'DeviceID', a string representing the AirPlay destination's MAC address, which must match the pattern '^([0-9A-Fa-f]{2}:){5}([0-9A-Fa-f]{2})$'. No additional properties are allowed at the root level.
