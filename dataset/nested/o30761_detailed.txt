The schema describes parameters for HTTP configuration. It is an object type. The main properties of the schema are: 'builtin' is an object that defines allowed operations with the enumerated values: 'MODIFIABLE', 'DELETABLE', 'IMMUTABLE', 'PARTITION_ALL'; it is readonly. 'conmultiplex' is a string indicating the state with allowed values: 'ENABLED', 'DISABLED'. 'dropinvalreqs' is a string indicating whether to drop invalid requests with allowed values: 'ON', 'OFF'. 'feature' is a string representing available features with a comprehensive set of allowed values including 'WL', 'WebLogging', 'SP', 'SurgeProtection', among others; it is readonly. 'http2serverside' is a string indicating the HTTP/2 server-side setting with allowed values: 'ON', 'OFF'. 'ignoreconnectcodingscheme' is a string indicating whether to ignore connection coding schemes with allowed values: 'ENABLED', 'DISABLED'. 'insnssrvrhdr' is a string indicating whether to include server headers with allowed values: 'ON', 'OFF'. 'logerrresp' is a string indicating whether to log error responses with allowed values: 'ON', 'OFF'. 'markconnreqinval' is a string indicating whether to mark connection requests as invalid with allowed values: 'ON', 'OFF'. 'markhttp09inval' is a string indicating whether to mark HTTP 0.9 requests as invalid with allowed values: 'ON', 'OFF'. 'maxreusepool' is an integer for the maximum reuse pool size. 'nssrvrhdr' is a string that could specify Server Header values. No additional properties are allowed at the root level.
