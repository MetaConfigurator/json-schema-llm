The schema describes base tags definitions. It is an object type. It defines several reusable components under 'definitions': 'tag' is an object, defining a set of properties related to a tag. The properties include 'name' as a string with a minimum length of 1 and a maximum length of 191, which must match the pattern '^([^,]|$)'. 'slug' can be a string or null with a maximum length of 191. 'description' can be a string or null with a maximum length of 500. 'feature_image', 'og_image', 'twitter_image', and 'canonical_url' can each be a string formatted as a URI reference or null, with a maximum length of 2000. 'visibility' is a string that must be either 'public' or 'internal'. The properties 'meta_title', 'og_title', 'twitter_title' can each be a string or null with a maximum length of 300. The properties 'meta_description', 'og_description', 'twitter_description' can each be a string or null with a maximum length of 500. 'codeinjection_head' and 'codeinjection_foot' can each be a string or null with a maximum length of 65535. 'accent_color' can be a string or null with a maximum length of 50. Several properties like 'id', 'parent', 'parent_id', 'created_at', 'created_by', 'updated_at', 'updated_by', and 'url' are defined with 'strip' as true. The main properties of the schema are: 'tag' is an object that resolves to the 'tag' definition, which is required. No additional properties are allowed at the root level.
