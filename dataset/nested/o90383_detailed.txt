The schema describes the configuration of a Docker runtime. It is an object type. The main properties of the schema are: 'name' is a string required field that describes the name of the docker configuration (e.g. prodDockerConfig, betaDockerConfig, etc.) with a default value of 'defaultDockerConfig'. 'kind' is a string optional field that describes the type of configuration, with an enum allowing the value 'container'. 'runtime' is a string required field that describes the type of container runtime, with an enum allowing the value 'docker' and a default value of 'docker'. 'type' is a string required field that describes the type of docker installation, with an enum allowing the values 'distro' and 'tgz', and a default value of 'distro'. 'uri' is a string optional field that describes how to obtain a tgz file containing the docker binaries if the type is tgz. No additional properties are allowed at the root level.
