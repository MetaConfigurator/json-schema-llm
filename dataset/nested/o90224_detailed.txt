The schema describes a KML Layer configuration. It is an object type. The main properties of the schema are: 'id' is a string A unique identifying string for the layer. It is required. 'itemId' is a string A string containing the item ID of the service if it's registered on ArcGIS Online or your organization's portal. It is optional. 'layerType' is a string String indicating the layer type. It has an enum value of 'KML' and is required. 'listMode' is a string To show or hide the sublayer in the layer list. It has enum values of 'hide' and 'show' and is optional. 'maxScale' is a number A number representing the maximum scale at which the layer will be visible. The number is the scale's denominator. It is optional with a minimum value of 0. 'minScale' is a number A number representing the minimum scale at which the layer will be visible. The number is the scale's denominator. It is optional with a minimum value of 0. 'opacity' is a number The degree of transparency applied to the layer on the client side, where 0 is full transparency and 1 is no transparency. It has a default value of 1 and optional with a maximum value of 1 and a minimum value of 0. 'path' is a string For offline data, a path to a KML file. A URI format is used, starting with 'file:' followed by a file system path with an extension of '.kml' or '.kmz'. A relative path must be from the file which defines the layer. It has a pattern constraint and is optional. 'refreshInterval' is a number Refresh interval of the layer in minutes. Non-zero value indicates automatic layer refresh at the specified interval. Value of 0 indicates auto refresh is not enabled. It is optional. 'showLegend' is a boolean Indicates whether to allow scene authors the ability to control what layers should be shown in a client's legend. It has a default value of true and is optional. 'title' is a string A user-friendly string title for the layer that can be used in a table of contents. It is required. 'url' is a string The URL to the layer. It is required. 'visibility' is a boolean Boolean property determining whether the layer is initially visible in the web scene. It has a default value of true and is optional. 'visibleFolders' is an array Array of numeric IDs of folders that will be made visible. It is optional with integer items and unique items constraint. The schema requires at least one of the following combinations: either 'id', 'layerType', 'title', and 'url' or 'id', 'layerType', 'path', and 'title'. No additional properties are allowed at the root level.
