The schema describes the structure of an output object for analysis. It is an object type. The main properties of the schema are: 'analyse' is an array that holds objects representing analysis results, each with a required 'code' (an enum restricted to 'UNEMPLOYED_REQUIRED', 'MISSING_REQUIRED_DEFAULT'), 'message' (a string), and 'path' (a string). 'donate' is an array of URLs, with exactly 2 unique items, allowing only 'https://pledgie.com/campaigns/27813' and 'https://gratipay.com/aminassian' as valid values. 'licence' is a string that is required. No additional properties are allowed at the root level.
