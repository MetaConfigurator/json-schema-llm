The schema describes a token introspection response. It is an object type. The main properties of the schema are: 'active' is a boolean indicating if the token is valid and active; it is required. 'aud' is an array of strings representing audience values for the token. 'client_id' is a string describing the identifier of the OAuth 2.0 client that requested the token. 'dat' is an object for optional custom authorization data. 'exp' is a number for the token expiration time in seconds since the Unix epoch. 'expires_in' is a number. 'iat' is a number for the token issue time in seconds since the Unix epoch. 'iss' is a string for the token issuer identifier. 'jti' is a string for the token identifier. 'nbf' is a number for the token use-not-before time in seconds since the Unix epoch. 'scope' is a string for the scope values of the token. 'sub' is a string for the subject of the token. 'token_type' is a string describing the type of the token, set to 'Bearer'. 'username' is a string for the username of the resource owner. No additional properties are allowed at the root level.
