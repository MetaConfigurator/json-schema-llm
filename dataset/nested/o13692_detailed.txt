The schema describes a Scheduling Block Instance. It is an object type. It defines several reusable components under 'definitions': 'assigned_resources' is an object, describing a resource assigned for the processing block with a 'status' property of type string. 'processing_blocks' is an array, containing objects that specify processing blocks. Each processing block must include 'id', 'resources_requirement', and 'workflow'. The 'id' is a string with a pattern of '^[a-z]+-[a-z]+[0-9]+$'. The 'resources_requirement' property references the 'resources_requirement' definition. The 'workflow' property references the 'workflow' definition. Additional properties for each processing block include 'priority', 'status', and 'type', with the latter two being strings, and 'priority' being a number. 'resources_requirement' is an object defined by the 'resources_requirement' which requires a 'storage_type' property of type string. 'workflow' is an object defined by the 'workflow' which requires a 'status' property of type string. The main properties of the schema are: 'date' is a string with a pattern of '^[0-9]{8}$', optional. 'id' is a string with a pattern of '^[0-9]{8}-[\w\d]+-sbi[0-9]{3}$', required. 'name' is a string, optional. 'processing_blocks' references the 'processing_blocks' definition, required. 'project' is a string, optional. 'sched_block_id' is a string with a pattern of '^[0-9]{8}-[\w|\d]+-sb[0-9]{3}$', required. 'status' is a string, optional. 'sub_array_id' is a string with a pattern of '^subarray-0[0-9]|subarray-1[0-5]$', required. 'type' is a string, optional. 'version' is a string, optional. No additional properties are allowed at the root level.
