The schema describes fabric information. It is an object type. The main properties of the schema are: 'fabric_fq_name' is an array that contains strings. It is required. 'fabric_display_name' is a string that is optional. 'management_subnets' is an array of objects that is required. Each object must have 'cidr' and 'gateway' as required fields. 'cidr' is a string that must match the pattern '^([0-9]{1,3}\.){3}[0-9]{1,3}(/([0-9]|[1-2][0-9]|3[0-2]))?$' and 'gateway' is a string in ipv4 format. 'loopback_subnets' is an array that contains strings. It is required and each string must match the pattern '^([0-9]{1,3}\.){3}[0-9]{1,3}(/([0-9]|[1-2][0-9]|3[0-2]))?$'. 'fabric_subnets' is an array that contains strings. It is required and each string must match the same pattern as 'loopback_subnets'. 'fabric_asn_pool' is an array of objects that is required. Each object must have 'asn_min' and 'asn_max' as required fields, both of which are integers. 'overlay_ibgp_asn' is an integer that defaults to 64512 and is optional. 'device_auth' is an object that is required and must have 'root_password' as a required field, which is a string. 'node_profiles' is an array of objects that is required. Each object must have 'node_profile_name' as a required field, which is a string. 'serial_nums' is an optional array of strings. 'device_count' is an integer that is required and describes the total number of devices in the fabric that needs to be zero-touch provisioned. No additional properties are allowed at the root level.
