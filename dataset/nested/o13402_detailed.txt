The schema describes a Halfling Model File Definition. It is an object type. The main properties of the schema are: 'GenNormals' is a boolean that indicates whether post-processing can automatically calculate the mesh normals. This property is optional. 'CalcTangents' is a boolean that indicates whether post-processing can automatically calculate the mesh tangents. This property is optional. 'VertexBufferUsage' is a string that specifies the usage type for the VertexBuffer. The allowed values are 'default', 'immutable', 'dynamic', and 'staging'. This property is optional. 'IndexBufferUsage' is a string that specifies the usage type for the IndexBuffer. The allowed values are 'default', 'immutable', 'dynamic', and 'staging'. This property is optional. 'MaterialDefinitions' is an array that contains objects representing the material definitions for each subset of the model. This property is required and must contain at least one item. Each item has properties: 'MaterialName', which is a string describing the name of the subset; 'HMATFilePath', which is a string describing the file path to the *.hmat file for the material; and 'TextureDefinitions', which is an array of objects that specify the textures used in the shader, with a rule that order matters. Each texture object includes 'FilePath', which is a string for the file path to the texture, and 'Sampler', which is a string with allowed values 'linear_clamp', 'linear_border', 'linear_wrap', 'point_clamp', 'point_wrap', 'anisotropic_wrap'. No additional properties are allowed at the root level.
