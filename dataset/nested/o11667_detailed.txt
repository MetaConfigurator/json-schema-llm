The schema describes data type fields. It is an object type. It defines several reusable components under 'definitions': 'itemsObject' is an object, it requires a '$ref' property which is a string. No additional properties are allowed at the root level. The main properties of the schema are: 'type' is a string, optional; '$ref' is a string, optional; 'format' is a string, optional; 'defaultValue' is not an array, object, or null; 'enum' is an array of strings, it must have at least one item and all items must be unique; 'minimum' is a string, optional; 'maximum' is a string, optional; 'items' is an object, structured according to the 'itemsObject' definition; '_uniqueItems' is a boolean, optional. The schema requires either the 'type' property or the '$ref' property to be present. The 'format' property is conditionally dependent on the value of 'type'. If 'type' is 'integer', 'format' must be either 'int32' or 'int64'. If 'type' is 'number', 'format' must be either 'float' or 'double'. If 'type' is 'string', 'format' must be either 'byte', 'date', or 'date-time'.
