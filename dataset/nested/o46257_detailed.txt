The schema describes investment information. It is an object type. It defines several reusable components under 'definitions': 'InvestmentActivity' is an object, which can have 'label' as a string, and 'type' as an integer, with no additional properties allowed. 'Security' is an object, which can have '_id' as an integer with the description 'A unique ID for Security', 'name' as a string, and 'symbol' as a string, with no additional properties allowed. 'InvestmentTransaction' is an object, which can have '_id' as an integer with the description 'A unique ID for InvestmentTransaction', 'price' as a number, and 'quantity' as a number, with no additional properties allowed. The main properties of the schema are: 'activity' is a referenced object from 'InvestmentActivity', 'security' is a referenced object from 'Security', 'transaction' is a referenced object from 'InvestmentTransaction'. No additional properties are allowed at the root level.
