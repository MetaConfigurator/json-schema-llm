The schema describes a package configuration. It is an object type. It defines several reusable components under 'definitions': 'url' is a string, representing a URI that must match the pattern '^https?://'. 'hash' is an object, required to have properties 'algo' (a string with enum value 'sha256') and 'value' (a string), with no additional properties allowed. 'cliInfo' is an object, required to have properties 'url' (a compliant URI), 'kind' (a string with enum values 'executable' and 'zip'), and 'contentHash' (an array of at least one 'hash'), with no additional properties allowed. The main properties of the schema are: 'packageName' is a string, optional. 'packageVersion' is a string, optional. 'appId' is a string, required. 'postInstallNotes' is a string, optional. 'cli' is an object, required, referring to the 'cli' definition which includes a 'binaries' property that is an object requiring at least one of 'darwin', 'linux', or 'windows', each containing a required 'x86-64' property referring to the 'cliInfo' definition. No additional properties are allowed at the root level.
