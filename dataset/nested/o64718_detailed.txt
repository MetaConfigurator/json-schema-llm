The schema describes o2r metadata. It is an object type. It defines several reusable components under 'definitions': 'creators' is an array, each item being an object with properties 'name' (a string, required), 'orcid' (a string, optional), and 'affiliation' (a string, optional). 'depends' is an array, each item being an object with properties 'operatingSystem' (an array, optional), 'identifier' (a string, required), 'packageSystem' (a string, required), and 'version' (a string, optional). 'spatial' is an object with properties 'files' (an array, optional) and 'union' (an array, required). 'temporal' is an object with properties 'begin' (a string, optional) and 'end' (a string, optional). 'depends' is an array that references the 'depends' definition, required. 'description' is a string, required. 'displayfile' is either null or a string, required. 'displayfile_candidates' is an array, optional. 'interaction' is an array, optional. 'mainfile' is a string, required. 'mainfile_candidates' is an array, optional. 'objectType' is a string, optional. 'publication_date' is a string, required. 'spatial' is an object that references the 'spatial' definition, required. 'softwarePaperCitation' is a string, optional. 'version' is a string, optional. 'title' is a string, required. 'temporal' is an object that references the 'temporal' definition, optional. 'keywords' is an array, required. 'paperLanguage' is either an array or a string, optional. No additional properties are allowed at the root level.
