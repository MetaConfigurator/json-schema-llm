The schema describes Spider Metadata. It is an object type. It defines several reusable components under 'definitions': 'enum' is an object, it requires a field 'enum' which is an array of unique strings with a minimum length of 1. 'pattern' is an object, it requires a field 'pattern' which is a string with a minimum length of 1. The main properties of the schema are: 'authenticatingFactors' is an object with pattern properties matching /^[A-Za-z0-9_\-]+$/ that reference the 'pattern' definition and disallow additional properties. 'factorDisplayNames' is an object with pattern properties matching /^[A-Za-z0-9_\-]+$/ that disallow additional properties and have nested pattern properties matching /^[A-Za-z]{0,2}$/ which must be strings with a minimum length of 1. 'factorDisplayOrder' is an array of strings with a minimum length of 1, unique values are required. 'identifyingFactors' is an object with pattern properties matching /^[A-Za-z0-9_\-]+$/ that can either be a 'pattern' or 'enum' definition and disallow additional properties. 'maxConcurrentCrawls' is an integer with a minimum value of 1 and a maximum value of 25. 'maxCrawlTimeInSeconds' is an integer with a minimum value of 5 and a maximum value of 300. 'paranoiaLevel' is a string that can be one of the following values: 'low', 'medium', 'high'. 'ttlInSeconds' is an integer with a minimum value of 60 and a maximum value of 86400. 'url' is a string that must conform to a URI format. The field 'url' is required. No additional properties are allowed at the root level.
