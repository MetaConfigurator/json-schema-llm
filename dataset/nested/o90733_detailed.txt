The schema describes a configuration for a Marathon Load Balancer. It is an object type. The main properties of the schema are: 'marathon-lb' is an object that specifies configuration parameters for the Marathon Load Balancer. It is required and contains several properties. 'bind-http-https' is a boolean that reserves ports 80 and 443 for the LB, with a default value of true. 'cpus' is a number that allocates CPU shares to each marathon-lb instance, with a minimum value of 1 and a default value of 2. 'haproxy-group' is a string that represents the HAProxy group parameter, defaulting to 'external'. 'instances' is an integer representing the number of instances to run, with a minimum value of 1 and a default value of 1. 'marathon-uri' is a string indicating the URI of the Marathon instance, defaulting to 'http://master.mesos:8080'. 'mem' is a number that allocates memory (MB) to each marathon-lb task, with a minimum value of 256.0 and a default value of 1024.0. 'name' is a string specifying the name for this LB instance, defaulting to 'marathon-lb'. 'role' is a string that defines which node role to deploy marathon-lb on, with a default value of 'slave_public'. 'ssl-cert' is a string for the TLS Cert and private key for HTTPS. 'sysctl-params' is a string containing sysctl parameters to set at startup for HAProxy, with a default value of 'net.ipv4.tcp_tw_reuse=1 net.ipv4.tcp_fin_timeout=30 net.ipv4.tcp_max_syn_backlog=10240 net.ipv4.tcp_max_tw_buckets=400000 net.ipv4.tcp_max_orphans=60000 net.core.somaxconn=10000'. 'template-url' is a string for the URL to a tarball containing templates to customize the HAProxy config, defaulting to an empty string. No additional properties are allowed at the root level.
