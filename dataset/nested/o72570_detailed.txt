The schema describes a configuration for a Kangrat. It is an object type. It defines several reusable components under 'definitions': 'page' is an object, which has required properties 'schema' (an object that allows additional properties of type string with enum values: 'string', 'number', 'object', 'array', 'boolean') and 'template' (an array of items each referencing 'templateElement', with no additional properties allowed). 'templateElement' is an object, which has a required property 'elementTag' (a string) and an optional property 'propertyBindings' (an object for bindings). 'bowerElement' is an object, which has properties 'bowerPackage' (a string) and 'elementPath' (a string), with no additional properties allowed. The main properties of the schema are: 'siteName' is a string, required, with a default value of 'My Kangrat Site', 'author' is a string, required, with a default value of 'somebody', 'basePage' is a string, required, with a default value of './basepage.html', 'pages' is an object, required, with additionalProperties referencing 'page', 'elements' is an object, required, with additionalProperties referencing 'bowerElement'. No additional properties are allowed at the root level.
