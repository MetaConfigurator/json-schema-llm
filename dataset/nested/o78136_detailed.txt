The schema describes a response structure. It is an object type. The main properties of the schema are: 'status' is a string indicating the status of the response, with allowed values 'pong' and 'fail', and it is optional. 'upstreams' is an object allowing additional properties where each property is a string that can either be 'pong' or 'fail', and it is optional. 'keygen' is an object containing properties: 'keys' is an integer describing the number of available keys and is optional; 'max' is an integer describing the maximum number of available keys and is optional; 'max_workers' is an integer describing the desired maximum number of workers generating keys and is optional; 'cur_max_workers' is an integer describing the current number of workers who can generate keys and is optional; 'inflight' is an integer describing the number of keys currently being generated and is optional; 'avail_workers' is an integer describing the current number of idle workers and is optional; it allows additional properties. No additional properties are allowed at the root level.
