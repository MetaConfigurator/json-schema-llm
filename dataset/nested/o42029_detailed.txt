The schema describes the result of CodeMetrics worker. It is an object type. It defines several reusable components under 'definitions': 'code_metrics_result' is an object, details computed by CodeMetrics worker, with a required 'languages' property that is an array of items referencing the 'languages' definition. 'code_metrics_summary' is an object, summary computed by CodeMetrics worker, with required fields 'blank_lines', 'code_lines', 'comment_lines', 'total_files', and 'total_lines', all of which are numbers and no additional properties are allowed. 'languages' is an object, generic language specific statistics, with required fields 'blank_lines', 'code_lines', 'comment_lines', 'files_count', and 'language' (all types are number or string), and allows additional properties in the 'metrics' object. The main properties of the schema are: 'schema' is an object with required properties 'name' (a string with a pattern of '^[a-zA-Z0-9_]+$') and 'version' (a string with a pattern of '^[0-9]+-[0-9]+-[0-9]+$') and an optional 'url' (a string formatted as a URI). '_release' is an optional string, 'status' is a string with enum values 'success' or 'error', 'details' references the 'code_metrics_result' definition, and 'summary' references the 'code_metrics_summary' definition. No additional properties are allowed at the root level.
