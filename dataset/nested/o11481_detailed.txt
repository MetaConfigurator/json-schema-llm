The schema describes a bibliographic record for the NCSU OLE Ingest Toolkit. It is an object type. The main properties of the schema are: 'bib_id' is a string that represents the ID generally used to refer to the record and is required. 'content' is a string that represents the MARCXML serialization of the bibliographic record and is required. 'created_by' is a string that defaults to 'olemigrator' and describes the ID or initials of the user who created the record. 'date_created' is a string that represents the timestamp when the record was originally created in ISO-8601 format and is required, with a pattern validation of 'date-time'. 'date_updated' is a string that defaults to '(current time)' and describes the timestamp for the last record alteration, with a pattern validation of 'date-time'. 'fast_add' is a boolean that defaults to false and indicates whether the record was added at a circulation desk. 'former_id' is a string that represents an identifier for the record from a previous system. 'holdings' is an array that may contain multiple entries. 'staff_only' is a boolean that defaults to false and indicates if the record is restricted for viewing to just staff. 'status' is a string that defaults to 'Catalogued' and describes the name or ID of the last cataloguing status. 'status_updated_by' is a string that defaults to 'olemigrator' and identifies the last user to update the cataloguing status of the record. 'status_updated_date' is a string that defaults to '(current time)' and describes the timestamp of the status update for this record with a pattern validation of 'date-time'. 'unique_id_prefix' is a string that defaults to 'wbm', is restricted to the enum value 'wbm', and serves as the prefix for the record type added to the bib_id property to identify the object in OLE's Solr index. 'updated_by' is a string that defaults to 'olemigrator' and identifies the last user who altered the record. No additional properties are allowed at the root level.
