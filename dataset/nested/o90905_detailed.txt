The schema describes an array of room presets. It is an array type. It defines several reusable components under 'definitions': 'direction' is an enum, it allows the values 'N', 'NE', 'E', 'SE', 'S', 'SW', 'W', 'NW'. 'directionList' is an array, it contains items that reference 'direction'. The main properties of the schema are: 'roomName' is a string, required. 'goldenDoor' is a boolean, required. 'variants' is an array, required, with a minimum of 1 item; each item is an object with required properties: 'areaCode' (string), 'contentLocations' (object), 'doorLocations' (which references 'directionList'), and 'minimap' (an object with required properties: 'rows' (integer), 'columns' (integer), and 'directions' (object with property names referencing 'direction' and additional properties as an array of 2 integers). No additional properties are allowed at the root level.
