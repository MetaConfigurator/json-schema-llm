The schema describes a Financial-Payment as a payment from government to a recipient. It is an object type. It defines several reusable components under 'definitions': 'company' is an object, a company which is the subject of a statement. It has required fields 'name' and 'jurisdiction'. 'financial-payment-data-object' is an object, representing the financial payment data. It has required fields 'data_type', 'properties', 'source_url', and 'confidence'. The 'properties' field is an object with required fields 'value', 'payee_name', 'date', and 'currency'. The main properties of the schema are: 'company' is a reference to the definition 'company', required. 'data' is an array containing items that are references to 'financial-payment-data-object', required. 'retrieved_at' is an optional string with a date format. 'sample_date' is a required string with a date format. 'source_jurisdiction' is an optional string describing the jurisdiction of the data source. No additional properties are allowed at the root level.
