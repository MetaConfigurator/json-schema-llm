The schema describes HTTP request logging information. It is an object type. The main properties of the schema are: 'body' is an object or a string that contains the request body; it is optional. 'env' is an object that contains environment information passed from the webserver; it is optional. 'headers' is an object that includes any headers sent by the requester; it is optional. 'http_version' is a string representing the HTTP version; it is optional and has a maximum length of 1024. 'method' is a string representing the HTTP method; it is required and has a maximum length of 1024. 'socket' is an object that contains information about the socket; it is optional. 'url' is an object that represents a complete URL, with various properties; it is required. 'cookies' is an object that represents parsed key-value cookies; it is optional. The 'url' object properties are: 'raw' is a string representing the raw URL; it is optional and has a maximum length of 1024. 'protocol' is a string for the URL scheme; it is optional and has a maximum length of 1024. 'hostname' is a string for the host name; it is optional and has a maximum length of 1024. 'port' is a string for the URL port; it is optional and has a maximum length of 1024. 'pathname' is a string for the URL path; it is optional and has a maximum length of 1024. 'search' is a string for the query string, expected to have values delimited by ampersands; it is optional and has a maximum length of 1024. 'hash' is a string for the URL fragment; it is optional and has a maximum length of 1024. The schema requires 'url' and 'method' to be present for a valid configuration. No additional properties are allowed at the root level.
