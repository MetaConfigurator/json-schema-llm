The schema describes a representation of item objects. It is an object type. The main properties of the schema are: 'description' is a string providing a short description of the population, optional; 'id' is an integer representing integer data, optional; 'level' is a string indicating whether the item is a single individual or a population, required, with allowed values: 'individual', 'population'; 'name' is a string for identifying the item later, required; 'owner' is a string for a single related resource, required; 'resource_uri' is a string for Unicode string data, optional; 'size' is a number for the population size or biomass, optional; 'taxa' is a string identifying the taxa object to which the item belongs, required; 'traits' is an array of string values representing identifiers (or URIs) of measured traits, optional; 'units' is a string indicating the units for population size, optional. No additional properties are allowed at the root level.
