The schema describes validation rules for spider output. It is an object type. The main properties of the schema are: '_metadata' is an object that describes the metadata of the spider output and is required. The properties of '_metadata' are: 'status' is an object that describes the status of the crawl and is required. It has required properties: 'code', which is an integer with a minimum value of 0, and 'message', which is a string. 'crawlTime' is an object that describes the crawl timing and is required. It has required properties: 'started', which is a string, and 'durationInMs', which is an integer with a minimum value of 0. 'spider' is an object that describes the spider information and is required. It has required properties: 'version', which is a string matching the pattern '^sha256:[a-f0-9]{64}$', and 'name', which is a string. 'crawlArgs' is an array of strings that must match the pattern '^sha256:[a-f0-9]{64}$'. The '_debug' property is an object that describes debugging information and is optional, containing properties: 'crawlLog', which is a string with a minimum length of 1 and a maximum length of 512; 'chromeDriverLog', which is a string with a minimum length of 1 and a maximum length of 512; and 'screenshot', which is a string with a minimum length of 1 and a maximum length of 512. No additional properties are allowed at the root level.
