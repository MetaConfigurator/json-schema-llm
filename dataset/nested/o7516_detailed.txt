The schema describes an API for knowing whether a patient's medication has been verified. It is an object type. It defines several reusable components under 'definitions': 'response' is an object, which requires the 'medication_verification' property. The 'medication_verification' property is of type object, which requires 'patient_guid', 'verified_by_uid', 'verified_by_name', and 'verified_at' properties. The main properties of the schema are: 'medication_verification' is a object that resolves to the 'medication_verification' definition. No additional properties are allowed at the root level.
