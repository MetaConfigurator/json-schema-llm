The schema describes the creation of a new, generic service definition in the given or default project. It is an object type. The main properties of the schema are: 'flat' is a boolean that indicates whether to create files at the top level of the project, which is optional and defaults to true. 'lintFix' is a boolean that, when true, applies lint fixes after generating the service, which is optional and defaults to false. 'name' is a string representing the name of the service, which is required. 'path' is a string for the path at which to create the service, relative to the workspace root; it is optional and not visible. 'project' is a string for the name of the project, which is optional. 'skipTests' is a boolean that, when true, does not create 'spec.ts' test files for the new service, which is optional and defaults to false. 'spec' is a boolean that indicates whether to generate a 'spec.ts' test file for the new service, which is optional and defaults to true but is deprecated in favor of 'skipTests'. No additional properties are allowed at the root level.
