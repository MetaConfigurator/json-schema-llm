The schema describes a document representing a collection of media assets and their tags. It is an object type. The main properties of the schema are: '_id' is a string unique identifier for this scene, required; 'name' is a string human readable name to refer to the scene by, required; 'displayDuration' is a number how many seconds atemporal assets should be on the screen for, optional; 'displayInterval' is a number how many seconds should be between the showing of assets on the screen, optional; 'transitionDuration' is a number how many seconds it should take for an asset to appear/disappear from the screen, optional; 'maximumOnScreen' is an object maximum counts of media types allowed on the screen at one time, optional, with properties 'image', 'text', 'video', and 'audio' as integers; 'themes' is an object represents sets of tags bundled together to make a coherent theme, optional, with additionalProperties being a string describing a comma separated list of tags; 'scene' is an array a collection of objects representing the various assets of a scene, required, with items being objects containing properties: '_id' as a string unique identifier for this scene, required; 'url' as a string Url pointing to media asset if necessary, optional; 'text' as a string Text to display for text media asset, optional; 'type' as a string Kind of media it is (only 'image', 'text', 'video', and 'audio' is supported), required; 'tags' as a string Semantic tags for this asset, comma separated, required. No additional properties are allowed at the root level.
