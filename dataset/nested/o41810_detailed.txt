The schema describes Firefox download sources. It is an object type. The main properties of the schema are: 'patternProperties' which defines the branch configurations that must match the regex '^[a-z]+$'. Inside 'patternProperties', each matched key can contain properties: 'branches' is an object which must have a minimum of 1 property; 'defaultBranch' is a string that must be one of 'nightly', 'devedition', 'beta', 'release', 'esr', 'unbranded-release' and is required. Each branch in 'branches' is itself an object that requires: 'defaultArch', which is a string that must be one of 'multi', 'x86', 'x86_64', 'arm-v15'; 'type', which is a string that must be one of 'classic', 'mdu', 'taskcluster'; and 'arches', which is an object with patterns matching 'multi', 'x86 (_64)?', 'arm-v15', where each architecture must have at least 2 properties: 'fileEnding' (pattern '^\.[a-z0-9.-]+$'), 'namespace' (minimum 3 characters), 'path' (minimum 3 characters, matching '^[a-z0-9/-]+$'), 'platform' (matching '^[A-Z0-9_]+$'), 'product' (matching '^[A-Z_]+$'), and 'version' (matching '^[A-Z0-9_]+$'). No additional properties are allowed at the root level.
