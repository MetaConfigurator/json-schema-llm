The schema describes a configuration for pinpoint plugin type-provider.yml files. It is an object type. It defines several reusable components under 'definitions': 'serviceType' is an object, defining a service type with required fields 'code' (an integer between 1000 and 9999) and 'name' (a string with at least 1 character). The 'desc' field is an optional string with a minimum length of 1. The 'property' field is an object with at least 1 property and no additional properties, containing optional boolean fields: 'terminal', 'queue', 'recordStatistics', 'includeDestinationId', and 'alias'. The 'matcher' field requires at least 1 property, must not have additional properties, and adheres to one of the two specified schemas: either an object with a required 'type' that must be 'args' or 'none', or an object with a required 'type' set to 'exact' and a required 'code' field (an integer). 'annotationKey' is an object, defining an annotation with required fields 'code' (an integer) and 'name' (a string with at least 1 character). The 'property' field is optional with a minimum of 1 property and no additional properties, containing an optional boolean field 'viewInRecordSet'. 'serviceTypes' is an array of serviceType definitions, and 'annotationKeys' is an array of annotationKey definitions. The main properties of the schema are: 'serviceTypes' is an array of serviceType definitions that is required if 'annotationKeys' is not present, and 'annotationKeys' is an array of annotationKey definitions that is required if 'serviceTypes' is not present. No additional properties are allowed at the root level.
