The schema describes a base structure for entity-resources, intended to be extended by domain-schemas. It is an object type. The main properties of the schema are: 'uuid' is a string that serves as a universal unique identifier, assigned by the system and must match the pattern '[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}'. 'id' is a string that serves as a resource-specific ID, assigned by the system, with a maximum length of 9 and must match the pattern '^[0-9]{9}$'. 'domain' is a string that indicates the resource type with allowed values: 'records', 'people', 'organisations', 'families', 'events', 'places', 'concepts', 'things'. 'schema' is a string that describes which JSON schema is used for the creation, display, update, and validation of this resource. 'created' is a string formatted as date-time that represents the creation date of the metadata resource. 'created_by' is a string with a maximum length of 255, indicating the username/ID of the creator/uploader of the metadata resource. 'updated' is a string formatted as date-time for the latest update of the resource. 'updated_by' is a string with a maximum length of 255, indicating the username of the last updater of the metadata resource. 'status' is a string with an enum of allowed values: 'validated', 'machine-generated', 'user-generated', with a default value of 'user-generated'. No additional properties are allowed at the root level.
