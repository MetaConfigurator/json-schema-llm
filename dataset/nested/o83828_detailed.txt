The schema describes a format used to define classes of heroes in VCMI. It is an object type. The main properties of the schema are: 'animation' is an object that contains files related to hero animation. It is required and allows no additional properties. It has two required properties: 'battle', which is an object for hero animations for battle that requires 'female' and 'male', both of which are strings formatted as 'defFile' and not allowing additional properties; and 'map', which is an object for hero animations for the adventure map, also requiring 'female' and 'male', with similar types and constraints. 'index' is a number that serves as a private field to break things, do not use. 'faction' is a string describing the faction this hero class belongs to. 'affinity' is a string indicating the affinity of the hero class, which can be either 'might' or 'magic'. 'commander' is a string that identifies the creature used as commander by heroes. 'highLevelChance' is an object detailing the chance to acquire specific primary skills upon level-up for levels starting from 10, allowing additional properties of type number. 'lowLevelChance' is an object similar to 'highLevelChance', but applicable for levels less than 10. 'name' is a string for the translatable name of the hero class. 'primarySkills' is an object detailing the initial primary skills of heroes, allowing additional properties of type number. 'secondarySkills' is also an object indicating the chance to acquire specific secondary skills upon level-up, with additional properties of type number. 'defaultTavern' is a number setting the default chance for the hero to appear in a tavern, with a minimum value of 0. 'tavern' is an object describing the chance for this hero to appear in the tavern of this faction, allowing additional properties of type number with a minimum value of 0. No additional properties are allowed at the root level.
