The schema describes an Ice cream API. It is an object type. It defines several reusable components under 'definitions': 'cow' is an object, which contains properties 'milk' (a string with an enum of allowed values: 'Cream', 'Whole', 'Organic-non-pasteurize-non-homogenized-non-gmo', 'Skim', '1%') and 'quantity' (an optional number with a minimum value of 8). 'grocer' is an object, which contains properties 'custom' (an optional string), 'ice_cream_essentials' (a string with an enum of allowed values: 'sugar', 'stevia'), and 'mixins' (a string with an enum of allowed values: 'chocolate chips', 'anchovies', 'blueberries', 'mixed fruit', 'jello', 'guar gum', 'apples', 'coffee', 'caramel', 'sea salt'). 'hen' is an object, which contains properties 'count' (an optional number) and 'egg' (a string with an enum of allowed values: 'brown/yolk only', 'scavenged (turkey)/ whole', 'stolen (crocodile)/ yolk only'). 'ice-man' is an object, which contains properties 'count' (a number with a minimum value of 1) and 'ice' (a string with an enum of allowed values: 'cube', 'sphere', 'special square'). The root schema resolves to the referenced definitions. The main properties of the schema are: 'cow' is a reference to the 'cow' definition, 'grocer' is a reference to the 'grocer' definition, 'hen' is a reference to the 'hen' definition, 'ice-man' is a reference to the 'ice-man' definition. No additional properties are allowed at the root level.
