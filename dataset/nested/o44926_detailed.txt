The schema describes a collection of adventures. It is an object type. The main properties of the schema are: 'adventure' is an array containing objects that represent individual adventures. Each adventure object is required to have 'name', 'id', 'contents', 'level', 'published', and 'storyline'. The 'contents' property is an array, where each item must be an object that has a required 'name' property. Each contents object can have an optional 'headers' property, which is an array of strings. The 'ordinal' property within the contents items must be an object containing required 'type' and 'identifier' properties. The 'type' is a string and must be one of the following: 'chapter', 'appendix', 'part', 'episode'. The 'identifier' can be either an integer or a string. The 'level' property in an adventure object can either be an object with required 'start' and 'end' integer properties or an object with a required 'custom' string property. The 'published' property is a date format string and the 'storyline' is a string that must be one of the following: 'Starter Set', 'Tyranny of Dragons', 'Elemental Evil', 'Rage of Demons', 'Curse of Strahd', 'Storm King's Thunder', 'Tales from the Yawning Portal', 'Tomb of Annihilation'. No additional properties are allowed at the root level.
