The schema describes a configuration for a build tool. It is an object type. It defines several reusable components under 'definitions': 'architect' is an object, containing properties for the build configuration, which include required properties such as 'configuration' (a string describing the named build target), 'prod' (a boolean indicating if the production target is used), and 'project' (a string for the project name to build). 'base' is an object, containing an optional property 'help' (an enum with values true, false, 'json', 'JSON' and a default of false). 'interactive' is an object, with properties 'defaults' (a boolean, defaulting to false, which disables interactive prompts for default options) and 'interactive' (a boolean, defaulting to true, which when false disables interactive prompts). 'schematic' is an object, containing properties 'dryRun' (a boolean with alias 'd', defaulting to false, for reporting without writing results) and 'force' (a boolean with alias 'f', defaulting to false, which forces overwriting existing files). The main properties of the schema are: 'architect' is an object that resolves to the 'architect' definition, 'base' is an object that resolves to the 'base' definition, 'interactive' is an object that resolves to the 'interactive' definition, and 'schematic' is an object that resolves to the 'schematic' definition. All of these properties are required. No additional properties are allowed at the root level.
