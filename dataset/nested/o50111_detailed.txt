The schema describes a template configuration. It is an object type. The main properties of the schema are: 'name' is a string that represents the template's name for display and is required. 'author' is a string that represents the author's name and is required. 'description' is a string that provides a brief description of the template to display and is required. 'web' is a string that indicates the web or repository URL of this template and is required. 'copy' is an array of strings used to determine which files to copy from the template folder and is optional with a default value of ['**/*']. 'pages' is an object that contains page-related properties and is required. The 'pages' object has mandatory properties: 'script', which is a string representing the relative path to the njk file used to render script pages, and 'folder', which is a string representing the relative path to the njk file used to render folder pages. No additional properties are allowed at the root level.
