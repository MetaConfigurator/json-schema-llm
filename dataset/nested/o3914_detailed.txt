The schema describes a list of fluids for the PC-SAFT EOS in CoolProp. It is an array type. The main properties of the schema are:  'name' is a string that specifies the name of the fluid, required. 'CAS' is a string that represents the CAS registry number of the fluid, required. 'm' is a number denoting the number of segments, required, with a minimum of 0 and a maximum of 1000. 'sigma' is a number representing the segment diameter in Angstrom, required, with a minimum of -2 and a maximum of 6. 'sigma_units' is an enum with allowed value 'Angstrom', required. 'u' is a number that indicates the dispersion energy divided by Boltzmann constant in K, required, with a minimum of 0 and a maximum of 2000. 'u_units' is an enum with allowed value 'K', required. 'uAB' is a number denoting the association energy in K, required, with a minimum of 0 and a maximum of 5000. 'uAB_units' is an enum with allowed value 'K', required. 'volA' is a number that specifies the association volume, required, with a minimum of 0 and a maximum of 1. 'volA_units' is an enum with allowed value 'Angstrom^3', required. 'dipm' is a number representing the dipole moment in Debye, required, with a minimum of 0 and a maximum of 13. 'dipm_units' is an enum with allowed value 'Debye', required. 'dipnum' is a number that indicates the number of dipole moments per molecule, required, with a minimum of 0 and a maximum of 100. 'charge' is a number representing the charge of ion, required, with a minimum of -10 and a maximum of 10. 'molemass' is a number denoting the molar mass in kg/mol, required, with a minimum of 0 and a maximum of 1. 'molemass_units' is an enum with allowed value 'kg/mol', required. 'aliases' is an array of strings allowing a minimum of 0 items. 'BibTeX' is a string for the BibTeX reference to the source from which the PC-SAFT parameters were taken. No additional properties are allowed at the root level.
