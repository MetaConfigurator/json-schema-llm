The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'extensions' is an array of extension points. Each item may contain properties such as 'type', 'order', 'title', and 'options'. If the 'type' is 'action', then either 'title' or 'options' must be present, with 'options' requiring at least one item that must include a 'title'. Each item in 'options' must contain 'title' and 'category' as required fields. 'dependencies' is an array of strings. 'scripts' is an array of strings. 'modules' is an array of strings. 'resources' is an array of strings. 'skip_compilation' is an array of strings. 'experiment' is a string. No additional properties are allowed at the root level.
