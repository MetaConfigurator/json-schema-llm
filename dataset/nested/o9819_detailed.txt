The schema describes a structure for handling data tables. It is an object type. The main properties of the schema are: 'items' is an array that contains schemas for individual Data Table Rows; each row is an object that has the properties: 'id' which is a string with a pattern of '^[A-Fa-f\d]{24}$'; 'createdAt' which is a string formatted as 'date-time'; and 'updatedAt' which is also a string formatted as 'date-time'. The row can contain additional properties that are strings, numbers, booleans, or null, constrained by pattern properties matching '^[0-9a-zA-Z_-]{1,255}$'. 'count' is an integer that represents the number of rows. 'totalCount' is an integer indicating the total number of available rows. 'limit' is an integer specifying the maximum number of rows to return. 'offset' is an integer determining the number of rows to skip. 'sortColumn' is a string indicating the column to sort by. 'sortDirection' is a string, which can be one of the following values: 'asc', 'desc', 'ASC', 'DESC', or an empty string. 'dataTableId' is a string with a pattern of '^[A-Fa-f\d]{24}$'. 'applicationId' is a string with a pattern of '^[A-Fa-f\d]{24}$'. 'query' is an object providing additional query parameters. No additional properties are allowed at the root level.
