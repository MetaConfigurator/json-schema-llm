The schema describes a shader asset. It is an object type. The main properties of the schema are: 'asset_type' is a string that must match the pattern 'shader' and is required. 'vert_source' is a string that serves as the source file for the GLSL vertex shader and defaults to 'vert.glsl'. 'frag_source' is a string that serves as the source file for the GLSL fragment shader and defaults to 'frag.glsl'. 'location_attribute_glsl_name' is a string that describes the name of the vertex location attribute and must match the pattern '\w+'. 'vertex_attributes' is an array of objects representing vertex attributes expected by the shader. Each object must have the required fields: 'name' (a string), 'glsl_id' (a string), and 'type' (a string that must match the pattern 'int|float|vec2|vec3|vec4'). 'parameters' is an array of objects representing shader parameters (uniforms in GLSL). Each object must have the required fields: 'type' (a string matching 'int|float|vec2|vec3|vec4|texture'), 'glsl_name' (a string matching '\w+'), 'name' (a string), and 'defualt'. There are additional properties for 'glsl_offset' (an integer with a minimum of 0), 'default' (which can be a string, an array of numbers with a length of 2 to 4, or a number), and 'description' (a string). No additional properties are allowed at the root level.
