The schema describes parameters accepted by the AppNexus adapter. It is an object type. The main properties of the schema are: 'placement_id' is an integer that identifies this placement of the impression; 'placementId' is an integer that is deprecated and should not be used; 'inv_code' is a string that identifies the inventory of this placement; 'invCode' is a string that is deprecated and should not be used; 'member' is a string that identifies the member selling the impression; 'keywords' is an array that must contain at least one item; each item is an object with a required 'key' string and a 'value' array with at least one string item; 'traffic_source_code' is a string that specifies the source of this impression; 'trafficSourceCode' is a string that is deprecated and should not be used; 'reserve' is a number representing the minimum acceptable bid in CPM, using US Dollars; 'position' is a string that must be either 'above' or 'below'; 'use_pmt_rule' is a boolean indicating whether to apply the relevant payment rule; 'private_sizes' is an array containing objects with required integer properties 'w' and 'h'; it describes private sizes and is experimental. The schema has compositional constraints: it requires either 'placementId' or 'placement_id' to be present, requires either 'invCode' and 'member' or 'inv_code' and 'member', and prohibits the presence of 'placementId', 'invCode', and 'member' at the same time. No additional properties are allowed at the root level.
