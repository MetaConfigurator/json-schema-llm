The schema describes the structure of a notifier result. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: '_schema' is a string that must match a specific pattern and has a default value of 'pbj:triniti:notify::notifier-result:1-0-0'. 'tags' is an object that uses pattern properties for its keys and has a description regarding its usage, with no additional properties allowed. 'ok' is a boolean that has a default value of true. 'code' is an integer that must be between 0 and 65535, with a default value of 0. 'error_name' is a string that must match a specific pattern. 'error_message' is a string with a minimum length of 0 and a maximum length of 65535. 'raw_response' is a string with a minimum length of 0 and a maximum length of 65535, describing the response from a notification. No additional properties are allowed at the root level.
