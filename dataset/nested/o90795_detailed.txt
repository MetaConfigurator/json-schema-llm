The schema describes configuration settings for an nginx application. It is an object type. The main properties of the schema are: 'nginx' is an object containing configuration settings for nginx, which is required. It includes the following required fields: 'cpus' is a number that specifies CPU shares to allocate to each nginx instance, with a default value of 1 and a minimum value of 1. 'mem' is a number that represents memory (MB) to allocate to each nginx task, with a default value of 1024.0 and a minimum value of 256.0. 'instances' is an integer that indicates the number of instances to run, with a default value of 1 and a minimum value of 1. 'name' is a string that specifies the name for this nginx application, with a default value of 'nginx'. Optional properties include: 'bridge' is a boolean that indicates whether to use BRIDGE networking mode for Docker container, with a default value of false. 'contentDir' is a string that defines the path of the directory, relative to sandbox, containing HTML code. 'contentUrl' is a string that represents the URL of content that needs to be hosted. 'maximumOverCapacity' is a number that specifies the maximum over capacity, with a default value of 0.2 and a minimum value of 0. 'minimumHealthCapacity' is a number that defines the minimum health capacity, with a default value of 0.5. 'role' is a string that specifies to deploy nginx only on nodes with this role, with a default value of '*'. No additional properties are allowed at the root level.
