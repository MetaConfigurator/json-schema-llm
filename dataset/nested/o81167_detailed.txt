The schema describes a configuration for software distribution. It is an object type. It defines several reusable components under 'definitions': 'cliInfo' is an object, it describes information about command line interface binaries. It has required fields: 'url', 'kind', and 'contentHash'. 'url' is a string, 'kind' is a string with allowed values: 'executable', 'zip', and 'contentHash' is an array containing at least one item that references the 'hash' definition. Additional properties are not allowed. 'hash' is an object, it describes cryptographic hash information. It has required fields: 'algo' and 'value'. 'algo' is a string with allowed value: 'sha256', and 'value' is a string. Additional properties are not allowed. The main properties of the schema are: 'assets' is an object, it contains properties: 'uris', which is an object allowing additional string properties, and 'container', which is an object containing 'docker', allowing additional string properties. Additional properties are not allowed. 'images' is an object, it has properties: 'icon-small', 'icon-medium', and 'icon-large', all of which are strings describing PNG icon URLs, and 'screenshots', which is an array of strings describing PNG screen URLs. Additional properties are not allowed. 'cli' is an object, it has a required property 'binaries', which is an object containing properties: 'linux', 'windows', and 'darwin', each requiring 'x86-64' which resolves to the 'cliInfo' definition. Additional properties are not allowed. No additional properties are allowed at the root level.
