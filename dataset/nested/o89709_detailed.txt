The schema describes minimum required output for a LogMessage. It is an object type. The main properties of the schema are: 'application' is a string that represents the application that raised the log and is required with a minimum length of 1. 'datetime' is a string formatted as date-time that indicates when the log event occurred and is required. 'extra' is an optional property that can be either a string or null for any extra data. 'message' is a string that contains the log message, must follow printf standards for arguments, is required, and has a minimum length of 1. 'messageArgs' is an array for arguments to format into the message, with no minimum items required. 'module' is an optional string for the module in the application that raised the log. 'severity' is a string that indicates the severity of this log message, is required, and must be one of the following values: 'emergency', 'alert', 'critical', 'error', 'warning', 'notice', 'informational', 'debug'. 'source' is a string that identifies the machine the log came from (IP, DNS name etc.), is required with a minimum length of 1. No additional properties are allowed at the root level.
