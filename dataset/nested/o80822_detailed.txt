The schema describes a claim provided by a claimant. It is an object type. The main properties of the schema are: 'arguments' is an optional object supporting the certainty of the claim, containing optional properties 'actor' (a string describing the person who assessed the certainty), 'human' (a number indicating whether the claim was done by a human (1) or an algorithm (0)), and 'role' (a string describing the role of the actor). 'certainty' is a number representing the certainty of the claim, which must be between 0 and 1.0, maximum 1, and minimum 0. 'claimant' is a string representing the short name of the claimant, with a minimum length of 1. 'created' is a string indicating the datetime in which the claim was created, with a minimum length of 1. 'object' is an object representing the subject of the claim predicate, containing required properties 'type' (a string for the type of persistent identifier) and 'value' (a string for the actual value of the identifier). 'predicate' is a string representing the type of relation between the subject and the object, with allowed values: 'is_same_as', 'is_variant_of', 'is_author_of', 'is_contributor_to', 'is_erratum_of'. 'subject' is an object representing the subject of the claim predicate, containing required properties 'type' (a string for the type of persistent identifier) and 'value' (a string for the identifier value). No additional properties are allowed at the root level.
