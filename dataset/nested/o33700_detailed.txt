The schema describes entities named. It is an object type. The main properties of the schema are: 'schemaType' is a string schema type, required. 'schemaType' is searchable, has a search mode of 'noop', and is locked. '_documentID' is a string internal document GUID, required. '_documentID' is searchable, has a search mode of 'noop', and is locked. '_corpusID' is a string internal Corpus GUID, required. '_corpusID' is searchable, has a search mode of 'noop', and is locked. 'offsets' is an array of objects, required, with unique items and at least one item. Each item has 'begin' which is an integer with a minimum of 0 and 'end' which is an integer with a minimum of 0. 'TypeEN' is a string type of named entity, optional, with an enum of 'personne', 'lieu', 'organisation', 'autre'. 'TypeEN' is searchable, has a search mode of 'basic', and is locked. 'TypeEn_Wikipedia' is an array of strings representing the entity type on Wikipedia, optional, with unique items and a minimum of 0 items. 'description' is a string describing the entity, optional, searchable, with a search mode of 'basic'. 'wikipedia' is a string for the entity's reference URL on Wikipedia, optional, searchable, with a search mode of 'basic'. No additional properties are allowed at the root level.
