The schema describes the structure for managing categories and repositories. It is an object type. The main properties of the schema are: 'categories' is an object that requires the field 'resource', which is a string formatted as 'topic' and represents the name of the data resource with categories. 'repository' is an object that requires the field 'action', which is a string formatted as 'topic' and represents the name of the action to request all data of a repository. 'expandAll' is an object that has an optional 'enabled' boolean field, defaulting to false, an optional 'action' string field formatted as 'topic', and an optional 'button' object, which has a default value with 'htmlLabel' set to 'expand all.' 'collapseAll' is an object that has an optional 'enabled' boolean field, defaulting to false, and an optional 'button' object, which has a default value with 'htmlLabel' set to 'collapse all.' No additional properties are allowed at the root level.
