The schema describes an Ingest Bulk Update. It is an object type. It defines several reusable components under 'definitions': 'CollectionIdType' is an object, containing required fields 'short-name' and 'version', both of type string. 'UpdateTypeEnum' is a string, with allowed values: 'ADD_TO_EXISTING', 'CLEAR_FIELD', 'CLEAR_ALL_AND_REPLACE', 'FIND_AND_REMOVE', 'FIND_AND_REPLACE', 'FIND_AND_UPDATE', 'FIND_AND_UPDATE_HOME_PAGE_URL'. 'UpdateFieldEnum' is a string, with allowed values: 'SCIENCE_KEYWORDS', 'LOCATION_KEYWORDS', 'DATA_CENTERS', 'PLATFORMS', 'INSTRUMENTS'. The main properties of the schema are: 'concept-ids' is an array of strings, required, with a minimum of 1 item. 'name' is a string, optional, with a minimum length of 1 and a maximum length of 255. 'collection-ids' is an array of CollectionIdType objects, required, with a minimum of 1 item. 'update-type' is an UpdateTypeEnum, required. 'update-field' is an UpdateFieldEnum, required. 'update-value' is a value that can either be an object or an array of objects, with at least 1 item if an array. 'find-value' is an object, optional. No additional properties are allowed at the root level.
