The schema describes a configuration object for devices. It is an object type. The main properties of the schema are: 'device_abstract_config' is an object JSON string of the device abstract config derived from VNC data model. 'additional_feature_params' is an object Additional feature specific parameters that are not defined in the VNC data model. These could be vendor specific parameters. It defines several properties: 'basic' is an object that allows no additional properties. It contains: 'snmp' which is an object allowing no additional properties that includes 'communities', an array of objects. Each object in 'communities' must have a string 'name' and can optionally have a boolean 'readonly' with a default value of true. 'static_routes' is an array of objects. Each object must have required fields 'subnets' and 'nexthop'. The 'subnets' field is an array of strings. The 'nexthop' field is a string. 'ip_clos' is an object that allows no additional properties and has a property 'bgp_hold_time' which is an integer with a default value of 90. 'is_delete' is a boolean that indicates a config delete operation with a default value of false. No additional properties are allowed at the root level.
