The schema describes a structured textual way of describing how to find an address, using unformatted strings for the street address. It is an object type. The main properties of the schema are: 'city' is a string that represents the city that the address is in, with a maximum length of 50 characters and is optional. 'country' is a string that represents the country that the address is in, with a maximum length of 50 characters and is required. 'formattedAddressLine1' is a string that represents a formatted address line containing a non-fielded street address, with a maximum length of 60 characters and is required. 'formattedAddressLine2' is a string that represents a formatted address line containing a non-fielded street address or sub-address, with a maximum length of 60 characters and is optional. 'locality' is a string that represents a bounded locality within a city, town or shire, with a maximum length of 50 characters and is optional. 'objectType' is a string that represents the type of the object, which must be one of the allowed values: 'formattedAddress', and is required. 'postcode' is a string that describes a postal delivery area, with a maximum length of 6 characters and is required. 'postcodeExtension' is a string that represents the four-digit extension on an American postal code, with a maximum length of 4 characters and is optional. 'stateOrProvince' is a string that represents the state or province that the address is in, with a maximum length of 50 characters and is required. No additional properties are allowed at the root level.
