The schema describes a configuration setup. It is an object type. The main properties of the schema are: 'config_type' is a string representing the configuration type, required, with a default value of 'current'. Allowed values are 'current', 'rollback 1', 'rollback 2'. 'config_filter' is a string representing the configuration filter, required, with a default value of 'all'. Allowed values include 'all', 'system', 'system/login', 'system/services', 'system/syslog', 'interfaces', 'chassis', 'services', 'snmp', 'forwarding-options', 'event-options', 'policy-options', 'class-of-service', 'firewall', 'protocols', 'routing-instances', 'routing-options', 'switch-options', 'vlans'. 'compare_configurations' is a boolean indicating whether to compare selected configuration with current, required, with a default value of false. No additional properties are allowed at the root level.
