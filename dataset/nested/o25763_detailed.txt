The schema describes a structure of data representing files and directories. It is an object type. It defines several reusable components under 'definitions': 'file' is an object, representing different types of file data with required fields 'name', 'type', 'user', and 'group'; 'type' must be one of 'file', 'link', 'dir', or 'remote_dir'. 'file_common' is an object, requiring 'name' and 'type' with 'type' having the same enum constraints. 'file_file' is an object, requiring 'type', 'size', and 'mode'; 'type' must be 'file', 'size' must be a non-negative integer, and 'mode' must match the pattern '^[0-7]{3,4}$'. 'file_dir' is an object, requiring 'type', 'size', and 'mode'; 'type' must be 'dir', 'size' must be a non-negative integer, and 'mode' must match the pattern '^[0-4]?[0-7]{3}$'. It also includes two valid configurations: one requiring 'files' and 'dirs', both non-negative integers, and another requiring 'file_objects', also a non-negative integer. 'file_link' is an object, requiring 'type' with the value 'link'. 'file_remote_dir' is an object that combines 'file_common' properties. The main properties of the schema are: '_attributes' is an object, required, with required properties 'extracted' and 'has_metadata', both of type boolean; '_elements' is an array, required, containing items that can either be of type 'file' or 'file_remote_dir'. No additional properties are allowed at the root level.
