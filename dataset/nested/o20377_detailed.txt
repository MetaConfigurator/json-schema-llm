The schema describes a configuration and data file for the TypeScript Definition Manager. It is an object type. The main properties of the schema are: 'bundle' is a string path to a <reference /> bundle, optional, matching the pattern \w+\.ts$. 'cache' is a string path to a (shared) tsd-cache, optional, with a minimum length of 1. 'installed' is an object index of installed definitions, optional, which does not allow additional properties. It has pattern properties that match ^[\w\.-]+\/[\w\.-]+\.d\.ts$ and require a 'commit' property, which is a string representing a sha1 hex matching the pattern ^[0-9a-f]{6,40}$. 'path' is a string path to the definition directory, optional, with a minimum length of 1. 'ref' is a string representing the git index ref (branch/commit), optional, matching the pattern ^[\w\.-]+(?:\/[\w\.-]+)*$. 'repo' is a string for the github repository in the format "owner/name", optional, matching the pattern ^[\w\.-]+/[\w\.-]+$. 'stats' is a boolean to enable stats tracking, optional. 'version' is a string for the config-format version, optional, matching the pattern ^v[\d]+$. The schema requires 'version', 'repo', 'ref', and 'path'. No additional properties are allowed at the root level.
