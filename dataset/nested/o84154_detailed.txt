The schema describes the format of an example defined using the x-ms-examples extension. It is an object type. It defines several reusable components under 'definitions': 'xmsParametersExample' is an object, a dictionary of parameter names and their sample values for a particular operation. Commonly used parameters like 'api-version', 'subscriptionId' and 'resourceGroupName' are defined as optional properties. It allows additional properties. 'xmsResponseExample' is an object, describes headers and body of an example HTTP response, in the context of the x-ms-examples extension. It has a 'headers' property which is a reference to 'xmsResponseHeaderExample' and a 'body' property which can be of multiple types. 'xmsResponseHeaderExample' is an object, examples of response headers. Common headers like 'location', 'azure-asyncoperation', 'retry-after' are defined as optional properties. It allows additional properties. The main properties of the schema are: 'operationId' is a string, it is optional. 'title' is a string, it is optional. 'description' is a string, it is optional. 'parameters' is a reference to 'xmsParametersExample', it is required. 'responses' is an object, it is required. It contains at least one property, no additional properties are allowed at the root level.
