The schema describes an issuer's identification structure. It is an object type. The main properties of the schema are: 'email' is a string that represents the contact address for the individual or organization. It is required. 'id' is a string that indicates the URL of the issuer's website or homepage. It is required. 'image' is a string that contains a Data URI; a base-64 encoded png image of the issuer's image. It is required and must match the pattern 'data:image/png;base64,'. 'introductionURL' is a string that denotes the URL hosting the issuer's introduction endpoint. It is required. 'issuerKeys' is an array of objects that lists issuer keys, required in descending date order (most recent first). Each object must contain 'date', which is a string representing the ISO-8601 formatted date time the key was activated, and 'key', which is a string representing the Bitcoin address that the issuer uses to issue the certificates. 'invalidated' is an optional field. 'name' is a string that states the human-readable name of the issuing entity. It is required. 'revocationKeys' is an array of objects that lists revocation keys, also required in descending date order (most recent first). Each object must include 'date' and 'key' with the same definitions as in 'issuerKeys'. 'invalidated' is an optional field here as well. 'url' is a string that signifies the URL where the issuer's certificates can be found. It is required. No additional properties are allowed at the root level.
