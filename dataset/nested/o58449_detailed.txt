The schema describes TSlint target options for Build Facade. It is an object type. The main properties of the schema are: 'tslintConfig' is a string that specifies the name of the TSLint configuration file. 'tsConfig' is a oneOf type that can be either a string or an array of strings, describing the name of the TypeScript configuration file. 'fix' is a boolean that indicates if linting errors should be fixed, with a default value of false. 'typeCheck' is a boolean that controls the type check for linting, with a default value of false. 'force' is a boolean that allows the process to succeed even if there were linting errors, with a default value of false. 'silent' is a boolean that determines whether output text is shown, with a default value of false. 'format' is a string that defines the output format and can be one of the following: 'checkstyle', 'codeFrame', 'filesList', 'json', 'junit', 'msbuild', 'pmd', 'prose', 'stylish', 'tap', 'verbose', 'vso' or have a minimum length of 1, with a default value of 'stylish'. 'exclude' is an array of strings that specifies the files to exclude from linting, with a default value of an empty array. 'files' is an array of strings that specifies the files to include in linting, with a default value of an empty array. No additional properties are allowed at the root level.
