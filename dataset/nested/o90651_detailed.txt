The schema describes the configuration for a Kafka connect worker. It is an object type. The main properties of the schema are: 'connect' is an object that specifies the configuration for the connect worker. It has required fields: 'cpus', 'mem', 'instances', and 'name'. The 'cpus' property is a number that specifies CPU shares to allocate, with a default value of 1 and a minimum of 1. The 'mem' property is a number that specifies memory in MB to allocate, with a default value of 1024.0 and a minimum of 256.0. The 'instances' property is an integer that specifies the number of instances to run, with a default value of 1 and a minimum of 1. The 'name' property is a string that specifies the name for this connect worker application, with a default value of 'connect'. No additional properties are allowed at the root level.
