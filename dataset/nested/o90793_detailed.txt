The schema describes the configuration for a namerd instance. It is an object type. The main properties of the schema are: 'namerd' is an object containing configuration details for the namerd service. It is required and includes several properties: 'admin-port' is an integer that provides an administrative UI for this instance, with a default value of 9001. 'config-filename' is a string representing the URI of the namerd config file, appending to 'config-uri', with a default value of 'namerd-dcos-060.yaml'. 'config-uri' is a string for the URI of the namerd config file, prepending to 'config-filename', with a default value of 'https://s3.amazonaws.com/buoyant-dcos'. 'cpus' is a number indicating CPU shares to allocate to each namerd instance, with a minimum value of 0.25 and a default of 0.25. 'http-port' is an integer for the httpController port that provides a REST interface to add/update dtabs, with a default of 4180. 'instances' is an integer for the number of instances to run, with a minimum value of 1 and a default of 3. 'mem' is a number indicating memory (MB) to allocate to each namerd task, with a minimum value of 256.0 and a default of 256.0. 'name' is a string for the name of the namerd instance(s), with a default value of 'namerd'. 'resource-roles' is a string for the accepted resource roles, with a default value of '"*", "slave_public"'. 'thrift-port' is an integer for the thriftNameInterpreter port, serving name data to linkerd, with a default of 4100. No additional properties are allowed at the root level.
