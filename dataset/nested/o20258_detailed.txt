The schema describes capabilities of an Earth-Observation Time-Series filter service. It is an object type. It defines several reusable components under 'definitions': 'Parameter' is an object, which includes required fields 'keyname', 'keytype', 'keymin', 'keymax', and 'default'. It has optional field 'domain', which is an array of unique string items. 'Filter' is an object, which includes required fields 'alias', 'name', 'missingdata', 'parameters', and 'sample'. The 'parameters' field is an array of items that reference the 'Parameter' definition. The main properties of the schema are: 'service' is a string, which is required, and 'filters' is an array, which is required, containing items that reference the 'Filter' definition. No additional properties are allowed at the root level.
