The schema describes an address. It is an object type. The main properties of the schema are: 'line1' is a string Line 1 of the address (e.g., Number, street, etc). 100 characters max, required. 'line2' is a string Line 2 of the address (e.g., Suite, apt #, etc). 100 characters max, optional. 'city' is a string City name. 50 characters max, required. 'country_code' is a string 2-letter country code. 2 characters max, required. 'postal_code' is a string Zip code or equivalent is usually required for countries that have them. 20 characters max, optional. 'state' is a string 2-letter code for US states, and the equivalent for other countries. 100 characters max, optional. 'phone' is a string Phone number in E.123 format. 50 characters max, optional. No additional properties are allowed at the root level.
