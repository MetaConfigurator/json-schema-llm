The schema describes Zone. It is an object type. The main properties of the schema are: 'zone' is an object representing the zone, required field that contains properties defining the zone. It has the following compositional constraints: 'oneOf' requires that 'zone' must match one of two configurations: either a primary or a secondary zone. For a primary zone, it must include the property 'type' with the value 'PRIMARY' and may not include any 'masters'. For a secondary zone, it must include properties 'type' with the value 'SECONDARY' and 'masters', which is an array of unique items formatted as 'ipandport'. No additional properties are allowed at the root level.
