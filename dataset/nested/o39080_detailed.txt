The schema describes a configuration for the go-const-ts tool. It is an object type. The main properties of the schema are: '$schema' is a string that specifies the schema version. '$tool' is a string that must be set to 'go-const-ts'. 'dist' is a string representing the output root path (relative to the pwd). 'excludeTypes' is an array of strings that allow excluding const types. 'includeTypes' is an array of strings that allow including const types only, ignoring others. 'langs' is an array of strings that specifies languages to extract; the field name will be used if a lang is not set in go const, and it must contain at least 1 item, each conforming to the pattern '^[a-z]{2}(-[A-Z]{2})?$'. 'pipePrefix' is a string serving as a prefix to the pipe name; it defaults to an empty string. 'pretties' is an object that contains directories with translated JSONs, which will overwrite the go const types. It has pattern properties for languages that conform to '^[a-z]{2}(-[A-Z]{2})?$', each allowing further nested properties for struct translates. 'prettiesRoots' is an array of strings that specifies directories containing translated JSONs to overwrite go const types. 'src' is an array of strings that specifies input go files (Glob), which must have at least 1 item and is expanded with the 'env' preprocessing option. The following properties are required: '$tool', 'src', 'dist', 'langs', and 'pipePrefix'. No additional properties are allowed at the root level.
