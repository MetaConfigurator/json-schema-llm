The schema describes a Yes-No field for a form. It is an object type. The main properties of the schema are: '_schema' is a string that must match the pattern '^pbj:([a-z0-9-]+):([a-z0-9\.-]+):([a-z0-9-]+)?:([a-z0-9-]+):([0-9]+-[0-9]+-[0-9]+)$' and has a default value of 'pbj:gdbots:forms:field:yes-no-field:1-0-0'. 'name' is a string that must match the pattern '^[a-zA-Z_]{1}[\w-]*$' and is required, with a description stating it is a unique identifier for the field that should not change. 'maps_to' is a string with a default value of 'cf' and must match the pattern '^[a-zA-Z_]{1}\w*$', describing the schema field the answer will map to. 'label' is a string with a minimum length of 0 and a maximum length of 255, serving as the main text for the question. 'placeholder' is a string with a minimum length of 0 and a maximum length of 255. 'description' is a string with a minimum length of 0 and a maximum length of 65535, providing a short description of the field. 'is_required' is a boolean. 'link_text' is a string with a minimum length of 0 and a maximum length of 255, describing text to replace the token '{link}'. 'link_url' is a string that must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$', describing the URL for the token '{link}'. 'pii_impact' is a string with an enum allowing values: 'unknown', 'high', 'moderate', 'low'. 'yes_label' is a string with a minimum length of 0 and a maximum length of 255. 'no_label' is a string with a minimum length of 0 and a maximum length of 255. No additional properties are allowed at the root level.
