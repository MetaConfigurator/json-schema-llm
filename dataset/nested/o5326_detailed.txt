The schema describes the structure of an event message. It is an object type. The main properties of the schema are: 'meta' is an object that contains required properties: 'id', 'type', 'version', and 'time'. 'id' is a string that must match the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'. 'type' is a string with an enum value of 'EiffelActivityCanceledEvent'. 'version' is a string with an enum value of '2.0.0' and has a default value of '2.0.0'. 'time' is an integer. 'tags' is an optional array of strings. 'source' is an object with properties: 'domainId', 'host', 'name', 'serializer', and 'uri', with 'serializer' having a pattern of '^pkg:' and disallowing additional properties. 'security' is an object that has a required 'sdm' property, which is an object that requires 'authorIdentity' and 'encryptedDigest', also disallowing additional properties. 'data' is an object that contains optional properties: 'reason' as a string and 'customData' as an optional array of objects, each requiring 'key' as a string and 'value' with no specific type defined while disallowing additional properties. 'links' is an array of objects, each requiring 'type' as a string and 'target' as a string that must match the same pattern as 'id', also disallowing additional properties. No additional properties are allowed at the root level.
