The schema describes the formal definition of a PCN (Process Chain Network) specification. It is an object type. The main properties of the schema are: 'domains' is an array of objects that must contain the properties 'id', 'title', and 'subtitle', all of which are strings and required. Each item in the array disallows additional items. 'steps' is an array of objects that must contain the properties 'id', 'title', 'type', and 'domain', all of which are required. The 'type' can be one of the following values: 'process', 'decision', 'wait', 'divergent_process'. Each step can have optional properties: 'emphasized' with a default value of false, 'predecessors' which is an array of objects that must include 'id' and 'type' (with possible values: 'normal_relationship', 'loose_temporal_relationship'), and 'problems' which is an array of objects. The 'domain' property within each step is an object that must include 'id' and 'region', where 'region' is another object that requires 'type' (with possible values: 'independent', 'surrogate', 'direct_leading', 'direct_shared') and 'with_domain'. The 'value_generic' and 'value_specific' have a default value of 0 and are integers. No additional properties are allowed at the root level.
