The schema describes a Patients API v3. It is an object type. It defines several reusable components under 'definitions': 'dob' is a string, describing the patients date of birth in the format 'date-time'. 'email' is a string, describing the patients email. 'first_name' is a string, describing the first name of the patient. 'guid' is a string, describing the unique identifier of the patient in the format 'uuid'. 'id' is an integer, describing the internal ID of the patient. 'identity' is a reference to 'guid'. 'last_name' is a string, describing the last name of the patient. 'middle_name' is a string, describing the middle name of the patient. 'nickname' is a string, describing the patients preferred nickname. The root schema resolves to the referenced definition 'patient'. The main properties of the schema are: 'patient' is a patient, which includes properties 'dob', 'email', 'first_name', 'guid', 'id', 'last_name', 'middle_name', and 'nickname'. No additional properties are allowed at the root level.
