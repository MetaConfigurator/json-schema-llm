The schema describes ACRSI Data Submission. It is an object type. The main properties of the schema are: '@type' is a string that provides the IRI for the JSON-LD data type, which must be 'ag:Customer'. 'fileCustomerIdentifier' is a string that acts as an internal identifier uniquely identifying a customer, required, with a minimum length of 1 and a maximum length of 2. 'interagencyCustomerIdentifier' is an optional string that serves as a system-generated unique internal identifier, with a minimum length of 1 and a maximum length of 2. 'taxIdentification' is a string used for tax authentication, with a required length of exactly 9 characters. 'taxIdentificationTypeCode' is a string that identifies the type of tax identification, required, with possible values '1' for Social Security Number and '2' for Employer Id Number, with a minimum length of 1 and maximum length of 1. 'businessName' is an optional string representing the name of a non-individual transacting business, with a minimum length of 1 and a maximum length of 100. 'middleName' is an optional string that is the middle name of an individual or entity, with a minimum length of 1 and a maximum length of 20. 'lastName' is an optional string representing the last name of an individual or entity, with a minimum length of 1 and a maximum length of 25. 'nameSuffix' is an optional string providing a free-form suffix with allowed values 'Jr', 'Sr', 'I', 'II', 'III', or 'Md', with a minimum length of 1 and a maximum length of 10. No additional properties are allowed at the root level.
