The schema describes the use of branches in git called git workflow. It is an object type. It defines several reusable components under 'definitions': 'scenarios' is an object, requiring 'name' and 'commands' fields. 'name' is a string. 'commands' is an array, containing items that must be valid according to anyOf rules referencing 'startCommand', 'finishCommand', or 'nowCommand'. 'startCommand' is an object, requiring 'name' and 'branch_start'. 'name' must be 'start', and 'branch_start' is a string. 'finishCommand' is an object, requiring 'name' and 'branch_finish'. 'name' must be 'finish', 'branch_finish' is an array of strings, and 'tagOn' is a string. 'nowCommand' is an object, requiring 'branch_start' and 'branch_finish'. 'name' must be 'now', 'branch_start' is a string, 'branch_finish' is an array of strings, and 'tagOn' is a string. The main properties of the schema are: 'workflow_name' is a string, required. 'branches' is an array, containing items that are strings, required. 'scenarios' is an array, containing items that must comply with the 'scenarios' definition, required. No additional properties are allowed at the root level.
