The schema describes a request structure. It is an object type. The main properties of the schema are: 'correlationGuid' is a string that must match the pattern '^[a-f\d]{8}-[a-f\d]{4}-4[a-f\d]{3}-[89ab][a-f\d]{3}-[a-f\d]{12}$', and it is required. 'requestorGLN' is a string with a minimum and maximum length of 13, and it is required. 'data' is an object that is required and contains the properties: 'lotNum', which is a string and is required, with the description 'Lot number for the asset to be verified', and 'expiry', which is a string that must be exactly 6 characters long with the description 'Date of expiry for the item to be looked up in format YYMMDD', and it is also required. No additional properties are allowed at the root level.
