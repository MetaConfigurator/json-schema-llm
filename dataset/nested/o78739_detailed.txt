The schema describes a sequence of semesters corresponding to a given program as recommended by Concordia University. It is an object type. It defines several reusable components under 'definitions': "semester" is an object, which contains a required array "courseList" with items that can be either a reference to a "regularCourse", an "electiveCourse", or an "orList", and a required string "isWorkTerm" with an enum value of either "true" or "false". "orList" is an array, which allows a minimum of 2 items, each of which must be either a reference to a "regularCourse" or an "electiveCourse". "regularCourse" is an object that has required properties: a string "code" matching the pattern "^\\w{4}\\s{1}\\d{3}$", a string "isElective" matching the pattern "^false$", and a string "electiveType" matching the pattern "^$". "electiveCourse" is an object that includes required properties: a string "code" matching the pattern "^$", a string "isElective" matching the pattern "^true$", and a string "electiveType" with an enum of values: "Science", "Program", "General". The main properties of the schema are: "sequenceInfo" is an object that contains required properties: a string "program" matching the pattern "\\w+", a string "option" matching the pattern "\\w+", and a string "entryType" matching the pattern "\\w+". "sourceUrl" is a string that must match the pattern "^https?://" and is required. "minTotalCredits" is a string that must match the pattern "^\\d+$" and is required. "yearList" is an array with a minimum of 3 items, each an object with required properties: "fall", "winter", and "summer" which are references to the definition "semester". No additional properties are allowed at the root level.
