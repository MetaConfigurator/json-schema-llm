The schema describes QISKit device backend calibration. It is an object type. It defines several reusable components under 'definitions': 'errorParameter' is an object, recorded error value. It requires fields 'date' of type string formatted as date-time and 'value' of type number. The main properties of the schema are: 'lastUpdateData' is a string, date of last calibration routine, optional. 'multiQubitGates' is an array of objects, where each object requires fields 'name' of type string, 'gateError' referencing the 'errorParameter' definition, 'qubits' which is an array of integers with a minimum length of 2, and 'type' of type string. 'qubits' is an array of objects, where each object requires fields 'name' of type string, 'gateError' referencing the 'errorParameter' definition, and 'readoutError' also referencing the 'errorParameter' definition. At the root level, 'lastUpdateData', 'multiQubitGates', and 'qubits' are required. No additional properties are allowed at the root level.
