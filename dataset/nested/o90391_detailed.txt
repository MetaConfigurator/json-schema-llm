The schema describes configuration for a helm chart for use in K2. It is an object type. The main properties of the schema are: 'name' is a string friendly name for the chart, required. 'namespace' is a string namespace of the chart, optional. 'repo' is a string repository to pull this chart from - referenced by helm application, optional. 'registry' is a string registry to pull this chart from - referenced by helm application, optional. 'chart' is a string actual name of chart in the repository - referenced by helm application, required. 'version' is a string version of the chart to deploy - referenced by helm application, optional. 'channel' is a string channel of the chart to deploy - referenced by helm application, optional, with allowed values: 'stable', 'beta', 'alpha'. The schema requires either 'version' or 'channel' to be present. No additional properties are allowed at the root level.
