The schema describes a package structure. It is an object type. The main properties of the schema are: 'name' is a string that describes the package name and is required. 'description' is a string that provides a short package description and is required. 'version' is a string that denotes the package version, referencing http://semver.org for valid schemes, and is required. 'update-type' is a string that specifies the package update type, which can either be 'minor', 'security', or a custom type defined by the package, and is required. 'maintainer' is a string that indicates the package maintainer name and is required. 'changelog' is an array of strings that lists all changes to be applied (changelog) and is required, with each item not allowing additional properties. 'filemapping' is an array of strings that enumerates all files that will be removed, modified, or added when upgrading the app, and is required, with each item not allowing additional properties. 'include' is a string that denotes a list of paths to include and is optional. 'time' is a string that indicates the package release date in 'YYYY-MM-DD', 'YYYY-MM-DD HH:MM:SS', or 'YYYY-MM-DDTHH:MM:SSZ' format and is optional. No additional properties are allowed at the root level.
