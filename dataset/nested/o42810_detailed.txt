The schema describes a node indexed for a content repository. It is an object type. The main properties of the schema are: '_id' is a string that represents the unique identifier for the node, required, and must adhere to the pattern '^[\w\/\.:-]+$'. 'status' is a string with a default value of 'draft' and it must be one of the following: 'unknown', 'published', 'scheduled', 'pending', 'draft', 'expired', 'archived', 'deleted'. 'etag' is a string that must adhere to the pattern '^[\w\.:-]+$'. 'created_at' is a string that must adhere to the pattern '^[1-9]{1}[0-9]{12,15}$'. 'creator_ref' is an object that serves as a fully qualified reference to what created this node. It has required fields 'curie' and 'id', and disallows additional properties. 'curie' is a string that must adhere to the pattern '^([a-z0-9-]+):([a-z0-9\.-]+):([a-z0-9-]+)?:([a-z0-9-]+)$', and 'id' is a string that must adhere to the pattern '^[A-Za-z0-9:_\-]+$'. 'tag' is an optional string that must adhere to the pattern '^([\w\/-]|[\w-][\w\/-]*[\w-])$'. 'updated_at' is a string that must adhere to the pattern '^[1-9]{1}[0-9]{12,15}$'. 'updater_ref' is an object similar to 'creator_ref' with the same structure and properties. 'last_event_ref' is an object similar to 'creator_ref' with the same structure and properties. 'title' is a string that is optional with a maximum length of 255 characters. No additional properties are allowed at the root level.
