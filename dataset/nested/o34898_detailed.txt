The schema describes a configuration for a Restreamer application. It is an object type. The main properties of the schema are: 'restreamer' is an object that contains configuration settings for the Restreamer application. It is required and must include the fields 'username', 'password', 'logger_level', and 'snapshot_refresh_interval'. 'logger_level' is an integer, with a default value of 3, representing the Logger Level. 'password' is a string, with a default value of 'datarhei', serving as the Password. 'snapshot_refresh_interval' is an integer, with a default value of 60, defining the interval to create a new Snapshot in seconds. 'username' is a string, with a default value of 'admin', representing the Username. 'service' is an object that contains the service configuration for the Restreamer application. It is required and must include the fields 'name', 'instances', 'cpus', and 'mem'. 'bridge' is a boolean, with a default value of false, indicating whether to use BRIDGE networking mode for the Docker container. 'cpus' is a number, with a default value of 1, specifying CPU shares to allocate to each Restreamer instance, and has a minimum value of 1. 'instances' is an integer, with a default value of 1, defining the number of instances to run, with a minimum value of 1. 'maximumOverCapacity' is a number, with a default value of 0.2, representing the maximum over capacity, with a minimum value of 0. 'mem' is a number, with a default value of 1024.0, which indicates memory in MB to allocate to each Restreamer task, with a minimum value of 256.0. 'minimumHealthCapacity' is a number, with a default value of 0.5, indicating the minimum health capacity, with a minimum value of 0. 'name' is a string, with a default value of 'restreamer', representing the application name. 'role' is a string, with a default value of '*', indicating the deploy role. No additional properties are allowed at the root level.
