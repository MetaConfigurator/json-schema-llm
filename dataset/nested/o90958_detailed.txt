The schema describes a configuration object. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'name' is a string that represents a unique display name (to be used instead of plugin ID). It is optional. 'fixed' is a boolean that indicates true if a fixed set is used (e.g. UI Library). It is required. 'ui' is an array that must contain at least one item. Each item is an object that contains: 'key', which is a string and is required, representing the customization key; 'description', which is a string that provides descriptive text for this key; 'source', which is a string that denotes the FQN of the underlying component (if applicable); 'deprecated', which is a boolean indicating whether this key is deprecated, with a default value of false; and 'since', which is a string marking the version in which the key was introduced. No additional properties are allowed at the root level.
