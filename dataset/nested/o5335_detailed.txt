The schema describes a structured event record. It is an object type. The main properties of the schema are: 'meta' is an object representing metadata about the event, required fields include 'id', 'type', 'version', and 'time'; 'data' is an object containing event-specific information; 'links' is an array of objects representing relationships or references related to the event. 'meta' has the following properties: 'id' is a string, required, must match the pattern: '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'; 'type' is a string, required, and can only be 'EiffelActivityStartedEvent'; 'version' is a string, required, with a default value of '3.0.0', and can only be '3.0.0'; 'time' is an integer, required; 'tags' is an optional array of strings; 'source' is an optional object with properties 'domainId', 'host', 'name', 'serializer' (which must match the pattern '^pkg:'), and 'uri'; 'security' is an optional object with a required 'sdm' object containing 'authorIdentity' and 'encryptedDigest'; 'data' contains 'executionUri' which is a string, and 'liveLogs', which is an array of objects that each must have 'name' and 'uri' as required strings; 'customData' is also an optional array of objects that each must have a 'key' as a required string and a 'value' with no specified type constraints. Lastly, 'links' is an array of objects, each required to have 'type' as a required string and 'target' as a required string that must match the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'. No additional properties are allowed at the root level.
