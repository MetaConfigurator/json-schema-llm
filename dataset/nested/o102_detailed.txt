The schema describes a configuration for services. It is an object type. The main properties of the schema are: 'provider_key' is a string optional description of the service provider. 'services' is an array of objects required, where each object is structured as follows: 'id' is an integer required unique identifier for the service. 'backend_authentication_type' is a string required indicating the type of backend authentication, with allowed values: 'provider_key', 'service_token'. 'backend_authentication_value' is a string required value for backend authentication. 'backend_version' is a string required with allowed values: '1', '2', 'oauth'. 'proxy' is an object required containing proxy configurations, with the following properties: 'api_backend' is a string required specifying the backend API. 'hosts' is an array of strings required listing the hostnames. 'proxy_rules' is an array of objects required where each object has properties: 'http_method' is a string required specifying the HTTP method, 'pattern' is a string required defining the URL pattern, 'metric_system_name' is a string required for the metric system name, and 'delta' is an integer required for the delta metrics. No additional properties are allowed at the root level.
