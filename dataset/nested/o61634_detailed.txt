The schema describes SmartCard settings. It is an object type. The main properties of the schema are: 'PayloadDescription' is a string that describes the payload. It is optional and has a default value of 'SmartCard Settings'. 'PayloadDisplayName' is a string that represents the name of the payload. It is optional and has a default value of 'SmartCard Settings'. 'PayloadIdentifier' is a string that serves as a unique identifier for the payload, dot-delimited, typically in the format of root PayloadIdentifier plus subidentifier. It is optional and has a default value of 'com.apple.smartcard'. 'PayloadOrganization' is a string that describes the issuing organization of the profile, as displayed to the user. It is optional. 'PayloadType' is a string that indicates the type of the payload, a reverse DNS string. It is optional and has a default value of 'com.apple.smartcard'. 'PayloadUUID' is a string that contains a unique identifier for the payload, formatted as 01234567-89AB-CDEF-0123-456789ABCDEF, and must match the pattern '^[0-9A-Za-z]{8}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{12}$'. It is optional and has a default value of ''. 'PayloadVersion' is a number representing the version of the whole configuration profile. It is optional and has a default value of 1. 'UserPairing' is a boolean that, if set to false, indicates users will not get the pairing dialog, although existing pairings will still work; the default is true. It is optional and has a default value of true. 'allowSmartCard' is a boolean that, if false, disables the SmartCard for logins and authorizations, while allowing other functions; a restart is required for changes to take effect. It is optional and has a default value of true. 'checkCertificateTrust' is a boolean that, if true, requires certificates on the card to meet specific validity criteria, with user overrides not allowed; it is optional and has a default value of false. 'oneCardPerUser' is a boolean that, if true, allows a user to pair with only one smart card, while existing pairings are permitted; it is optional and has a default value of false. No additional properties are allowed at the root level.
