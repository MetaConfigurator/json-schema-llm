The schema describes a JSON schema for any Grunt task. It is an object type. It defines several reusable components under 'definitions': 'dynamic' is an object, which requires the 'src' property and can include optional properties such as 'expand' (a boolean with a default value of false), 'cwd' (a string), 'dest' (a string), 'ext' (a string), 'extDot' (an enum with allowed values 'first' and 'last', defaulting to 'first'), 'filter' (an enum with allowed values 'isFile', 'isDirectory', 'isBlockDevice', 'isCharacterDevice', 'isSymbolicLink', 'isFIFO', 'isSocket'), 'flatten' (a boolean with a default value of false), 'matchBase' (a boolean with a default value of false), 'nonull' (a boolean with a default value of false), 'rename' (a string), and 'src' (an array of strings with each item having a minimum length of 1 and must be unique). 'fileFormat' is an object, which includes a 'files' property that can either be an object with additional properties of type array or string or an array of items that reference the 'dynamic' definition. The main properties of the schema are: 'options' is an object, which is optional. The schema allows additional properties that can be either of the type defined in 'fileFormat' or 'dynamic'. No additional properties are allowed at the root level.
