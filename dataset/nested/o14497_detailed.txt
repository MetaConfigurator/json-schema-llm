The schema describes rule validation results. It is an object type. The main properties of the schema are: 'matchedConditions' is an array List of conditions that matched the event. It is required and must contain at least one item. Each item is an object with a required property 'matchingBy'. 'matchingBy' is an object that groups the matching data and has required properties: 'conditionIndex', 'source', 'type', and 'function', all of which are required and have respective types specified. 'matchedActions' is an array List of transformations that matched the event. It is required and must contain at least one item. Each item is an object with required properties: 'matchingBy' and 'actionBody'. 'matchingBy' is an object that groups the matching data and has required properties: 'transformationIndex' and 'targetAndType', both of which are required and have respective types specified. 'actionBody' is an object representing the event after the transformation was applied. No additional properties are allowed at the root level.
