The schema describes data related to corporate subsidiaries. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'confidence' is a string indicating confidence in accuracy of data, and is optional with allowed values: 'HIGH', 'MEDIUM', 'LOW'. 'direct' is a boolean representing whether the control relationship is direct or indirect, and is optional. 'end_date' is a string in date format describing the latest known date this was known to be a subsidiary, and is optional. 'parent_identifier' is a string for any unique ID for the controlling company, and is optional. 'parent_jurisdiction' is a string indicating the jurisdiction where the controlling company is based, requires a minimum length of 1, and is required. 'parent_name' is a string for the name of the controlling company, requires a minimum length of 1, and is required. 'percentage_controlled' is a string representing percentage of the subsidiary that is controlled, adhering to a specific pattern, and is optional. 'retrieved_at' is a string in date format, and is optional. 'sample_date' is a string in date format describing the date on which this fact was known to be true, and is required. 'source_url' is a string for the place where this fact can be verified, requires a minimum length of 1, and is required. 'start_date' is a string in date format describing the earliest known date this was known to be a subsidiary, and is optional. 'subsidiary_identifier' is a string for any unique id for the subsidiary, and is optional. 'subsidiary_jurisdiction' is a string indicating the jurisdiction of the subsidiary, requires a minimum length of 1, and is required. 'subsidiary_name' is a string for the name of the subsidiary, requires a minimum length of 1, and is required. No additional properties are allowed at the root level.
