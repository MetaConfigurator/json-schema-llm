The schema describes a benchmark suite file that contains the commands to execute and their results. It is an object type. The main properties of the schema are: 'creation_date_utc' is a string that describes the creation time of the suite file in UTC. It is required. 'cmd_list' is an array that describes the list of the commands that makes up this benchmark suite. It is required, with items that are objects containing 'cmd', 'env', and 'label' which are all required fields of type string. Each item in 'cmd_list' also contains 'jobs', which is an array of objects that represent scheduled commands. Each job object includes required fields 'status' of type string that must match the pattern 'finished|failed|pending', and 'nruns' of type number with a minimum value of 1. Each job can include optional fields 'warmup' of type boolean, and 'results', which is an array of objects that must include the required field 'success' of type boolean. No additional properties are allowed at the root level.
