The schema describes a Revenue list. It is an array type. It defines several reusable components under 'definitions': 'code' is a string, Revenue code with a pattern of '^(9.)?([1-9].){4}[0-9]{1,2}.[0-9]{1,2}$' and an example '1.2.3.4.75.0'. 'date' is a string, Date (???) with the format 'date-time' and an example '2010-08-02'. 'description' is a string, Description of the revenue. 'id' is an integer, ID of the revenue. 'monthly_outcome' is a number, Revenue in a month. 'monthly_predicted' is a number, Revenue predicted for a month. The main properties of the schema are: 'items' is a reference to the definition 'revenue-line', which is an object containing properties: 'code' (a reference to 'code'), 'date' (a reference to 'date'), 'description' (a reference to 'description'), 'id' (a reference to 'id'), 'monthly_outcome' (a reference to 'monthly_outcome'), and 'monthly_predicted' (a reference to 'monthly_predicted'). No additional properties are allowed at the root level.
