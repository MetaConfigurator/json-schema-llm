The schema describes a sensor model. It is an object type. It defines several reusable components under 'definitions': 'base' is an object, which requires 'type' and 'properties'. 'with-id' is an object, which requires 'id' within its 'properties'. 'car-type' is an object, which restricts 'type' to the value 'car'. 'car-properties' is an object, which defines several properties: 'manufacturer' is a string, 'model' is a string, 'fuelType' is a string and can be one of 'diesel', 'gasoline', 'biodiesel', 'kerosene', 'electric', 'constructionYear' is an integer with a minimum of 0, and 'VIN' is a string. The root schema resolves to the referenced definition 'base'. The main properties of the schema are: 'type' is a string, required; 'properties' is an object, required. Valid configurations require that the schema satisfies either the 'not' condition of 'car-type' or the 'allOf' condition of 'car-type' and 'car-properties'. No additional properties are allowed at the root level.
