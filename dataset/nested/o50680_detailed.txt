The schema describes funding a platform user's account. It is an object type. The main properties of the schema are: 'account_identifier' is a string that serves as the identifier for the platform account to fund, required, with a minimum length of 5, a maximum length of 96, and must match the pattern '^[\+\w-]{5,96}$'. 'amount' is a number that indicates the amount to add to the account in cents, required, with a minimum value of 100. 'cc_token' is a string that represents the payment token issued at registration time, required. 'client_ip' is a string that provides the IP address of the person making the purchase, required. 'customer' is a string that denotes the platform's customer, required, with a minimum length of 1 and a maximum length of 96. 'security_code' is a string for the 3 or 4-digit card security code (CVV2, CVC2, or CID), required, and must match the pattern '^\d{3,4}$'. No additional properties are allowed at the root level.
