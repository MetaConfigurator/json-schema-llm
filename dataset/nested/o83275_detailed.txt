The schema describes a representation of storage resource usage. It is an object type. The main properties of the schema are: 'dt' is a string indicating the date and time usage data was recorded, required. 'file_count' is an integer representing the number of files, required, with a minimum of 0 and a maximum of 18446744073709551615. 'hard_threshold' is an integer describing the hard threshold quota measured in bytes, optional, with a minimum of 0 and a maximum of 18446744073709551615. 'logical_usage' is an integer indicating the logical file system usage measured in bytes, required, with a minimum of 0 and a maximum of 18446744073709551615. 'mountpoint' is a string representing the storage file system mountpoint directory, required, with a maximum length of 255. 'physical_usage' is an integer describing the physical file system usage measured in bytes, optional, with a minimum of 0 and a maximum of 18446744073709551615. 'pi' is a string for the storage principal investigator's system username, required, with a maximum length of 30. 'resource' is a string indicating the storage resource name, required, with a maximum length of 200. 'soft_threshold' is an integer describing the soft threshold quota measured in bytes, optional, with a minimum of 0 and a maximum of 18446744073709551615. 'user' is a string representing the storage user's system username, required, with a maximum length of 30. No additional properties are allowed at the root level.
