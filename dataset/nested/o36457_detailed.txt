The schema describes the API Reference File. It is an object type. The main properties of the schema are: 'version' is a string that indicates the API reference version, it is required and can be either 0 or '0.2.0'. '$schema' is a string that provides a link to the schema for this reference, it is required and should be in URI format. 'title' is a string that represents the API title in markdown, it is required. 'description' is a string that contains the API description in markdown, it is required. 'baseUrl' is a string that denotes the BaseUrl for all routes described in this document, it is required and should be in URI format. 'entries' is an array that contains methods in this reference, it is required and each item in the array is an object that must have the following properties: 'type' is a string that indicates the type of entry, currently only 'function', it is required; 'method' is a string that represents the HTTP method (verb) used to access the function, it is required and can be one of the following values: 'get', 'post', 'put', 'head', 'delete', 'options', 'trace', 'copy', 'lock', 'mkcol', 'move', 'purge', 'propfind', 'proppatch', 'unlock', 'report', 'mkactivity', 'checkout', 'merge', 'm-search', 'notify', 'subscribe', 'unsubscribe', 'patch', 'search'; 'route' is a string that specifies the route for the call, it is required; 'args' is an array that includes arguments from route that must be replaced, it is required and each item is a string; 'name' is a string that provides a stable identifier for use in auto-generated client libraries, it is required; 'stability' is a string that denotes the stability level of the API, it is required and can be one of the following values: 'deprecated', 'experimental', 'stable'; 'scopes' is an array that lists scope-sets of which the client must satisfy at least one, it is optional; 'input' is a string that provides a JSON schema for input, it is optional and should be in URI format; 'output' is a string that provides a JSON schema for output, it is optional and should be in URI format; 'title' is a string that indicates the title of the API entry, it is optional; 'description' is a string that describes the API entry, it is optional. No additional properties are allowed at the root level.
