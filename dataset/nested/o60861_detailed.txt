The schema describes an application definition. It is an object type. The main properties of the schema are: 'schema' is a string that is required, describing the schema identifier for payload. 'label' is a string that is required, describing the nice name of the application. 'application_dir' is a string that is required, describing the name of the directory used for application resources. 'executable' is a string that is required, describing the name of the callable executable, this is called to launch the application. 'description' is a string that is optional, describing the application. 'environment' is an object that is optional, containing key/value pairs for environment variables related to this application, which supports lists for paths, such as PYTHONPATH. 'default_dirs' is an array that is optional, with items that are strings. 'copy' is an object that is optional, with pattern properties that can match any string, where the values can be either a string or null, and it does not allow additional properties. No additional properties are allowed at the root level.
