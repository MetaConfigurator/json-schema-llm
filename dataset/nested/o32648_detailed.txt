The schema describes a product available for sale in a store. It is an object type. It defines several reusable components under 'definitions': "available_qty" is a number, The quantity of the product available for purchasing, with a default value of 0 and an example of 10. "currency" is a string, The currency of the price, with an enum value of "USD". "description" is a string, Product's description, with a maxLength of 2000. "identifier" is a string, Product SKU, with an example of "ABC-123". "image" is a string, URL for the product image, with an example of "http://static.example.com/images/product.jpg". "in_stock" is a boolean, Whether the product is in stock or not, with a default value of false and an example of true. "name" is a string, Product's name, with a maxLength of 100. "price" is a number, The price of the product, with an example of 49.99. The main properties of the schema are: "ID" is a string resolving to the "identifier" definition, required. "available_qty" is a number resolving to the "available_qty" definition, required. "currency" is a string resolving to the "currency" definition, required. "description" is a string resolving to the "description" definition, required. "image" is a string resolving to the "image" definition, optional. "in_stock" is a boolean resolving to the "in_stock" definition, required. "name" is a string resolving to the "name" definition, required. "price" is a number resolving to the "price" definition, required. No additional properties are allowed at the root level.
