The schema describes a set of rules for manipulating JSON data. It is an object type. It defines several reusable components under 'definitions': '_typeSet' is an enum, it describes a type that can only be 'set'. '_typeRemove' is an enum, it describes a type that can only be 'remove'. '_typeMove' is an enum, it describes a type that can only be 'move'. '_typeCopy' is an enum, it describes a type that can only be 'copy'. 'destination' is a string, it describes the JSON path to place result and has a minimum length of 1. 'source' is a string, it describes the JSON path of source value and has a minimum length of 1. 'valueTemplate' is a string, it has a minimum length of 1. The main properties of the schema are: 'rules' is an array of objects, it is required, must have at least 1 item, and each item can represent one of the following: 1) an object with type 'set', a destination path, and a value template (all required), 2) an object with type 'remove' and a source path (both required), 3) an object with type 'move', a destination path, and a source path (all required), or 4) an object with type 'copy', a destination path, and a source path (all required). No additional properties are allowed at the root level.
