The schema describes CVEcheck worker results. It is an object type. It defines several reusable components under 'definitions': 'cvecheck_details' is an object, detail of one CVE. It has required fields: 'cvss', 'description', 'severity', 'id', and 'references'. The 'cvss' property references another definition, while 'description', 'severity', 'id', and 'cwe' are strings, and 'references' is an array of strings formatted as URIs. The 'cvss' definition is an object, with required fields: 'score' and 'vector', both of which are a number and a string respectively. The main properties of the schema are: 'schema' is an object, with required fields: 'name' and 'version'. 'name' is a string matching the pattern '^[a-zA-Z0-9_]+$', and 'version' is a string matching the pattern '^[0-9]+-[0-9]+-[0-9]+$'. 'url' is an optional string formatted as a URI. '_release' is a string, 'status' is a string with enum values: 'success', 'error'. 'details' is an array of items referencing 'cvecheck_details'. 'summary' is an array of strings. No additional properties are allowed at the root level.
