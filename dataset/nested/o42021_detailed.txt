The schema describes a configuration for a pipeline. It is an object type. The main properties of the schema are: 'title' is a string optional description of the pipeline. 'description' is a string optional further detailing the pipeline. 'environment' is an object optional for specifying the environment settings. 'schedule' is an object optional with a 'crontab' property, which is a string representing the schedule in cron format. 'pipeline' is an array required with a minimum length of 1, containing objects that must conform to one of two valid configurations: one with a 'run' property that is a string and the other with a 'flow' property that is required. Each pipeline object may also include an optional 'parameters' object, 'cache' boolean flag, and 'validate' boolean flag. 'dependencies' is an array optional, where each item is an object with at most one property, either 'datapackage' as a string in URI format or 'pipeline' as a string. No additional properties are allowed at the root level.
