The schema describes filtered logs. It is an object type. It defines several reusable components under 'definitions': 'date-filter' is an object, it has properties 'from' and 'to', both of which are strings formatted as date-time. No additional properties are allowed in 'date-filter'. The main properties of the schema are: 'email' is a string with an email format; it is required. 'filter' is an object that applies filter on logs results; it has several date-filter properties: 'clicked_at', 'delivered_at', 'dropped_at', 'hard_bounced_at', 'opened_at', 'sent_at', and 'spam_complained_at', all of which reference the 'date-filter' definition and cannot have additional properties. 'limit' is an integer; it is optional. 'mail_template_ids' is an array of integers; it is optional. 'page' is an integer; it is optional. Dependencies exist where 'page' requires 'limit'. No additional properties are allowed at the root level.
