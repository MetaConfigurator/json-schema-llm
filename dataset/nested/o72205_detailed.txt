The schema describes claims provided to authenticate a user with a JWT. It is an object type. The main properties of the schema are: 'authentication' is a string that indicates whether the user is authenticated or unauthenticated. It is optional, defaults to 'authenticated', and allows one of the following values: 'authenticated', 'unauthenticated'. 'sub' is a string representing the user identity to be validated and is optional. 'iss' is a string for the issuer of the token and is optional. 'user' is an object that identifies the user by identity, which is optional and must include the required fields: 'id' and 'accountId'. The 'id' property is a string representing the User ID and the 'accountId' property is a string for the Account ID. The schema has a oneOf constraint requiring that either 'user' or 'sub' be present. No additional properties are allowed at the root level.
