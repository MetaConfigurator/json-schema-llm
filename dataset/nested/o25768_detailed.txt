The schema describes a file system structure. It is an object type. It defines several reusable components under 'definitions': 'file_common' is an object, it requires 'name' and 'type' fields. 'file' is an object, it requires 'name' and 'type', and may have 'user' and 'group' fields. It must match one of the defined file types: 'file', 'dir', or 'link'. 'file_file' is an object, it requires 'type' to be 'file' and may have 'size' as a non-negative integer and 'mode' matching a specific pattern. 'file_dir' is an object, it requires 'type' to be 'dir' and may have 'size' as a non-negative integer, 'mode' matching a pattern, and 'files' as a non-negative integer. 'file_link' is an object, it requires 'type' to be 'link'. 'file_remote_dir' is an object, it requires 'name' and 'type', with 'type' needing to be 'remote_dir'. The main properties of the schema are: '_attributes' is an object, it is optional and requires 'extracted' as a boolean. '_elements' is an array, it is required and contains items that can be either a 'file' or a 'file_remote_dir'. No additional properties are allowed at the root level.
