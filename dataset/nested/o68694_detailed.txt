The schema describes a list of mappings from Puli paths to filesystem paths or other Puli paths. It is an object type. It defines several reusable components under 'definitions': 'relativePath' is a string, a filesystem path relative to the base directory of the repository, with a pattern that must match '^([a-zA-Z][^:]|[^/@a-zA-Z])[^/]*(/[^/]+)*$'. 'absolutePath' is a string, an absolute filesystem path, with a pattern that must match '^([a-zA-Z]:)?/(([^/]+/)*[^/]+)?$'. 'virtualDirectory' is a null, a directory in the Puli repository that does not correspond to a real filesystem directory. 'repositoryLink' is a string, a link to another resource in the Puli repository, with a pattern that must match '^@/(([^/]+/)*[^/]+)?$'. 'singleReference' is oneOf a single reference to a file, directory, link or virtual directory, with possible types being 'relativePath', 'absolutePath', 'repositoryLink', or 'virtualDirectory'. 'combinedReference' is an array, a list of references that must contain at least 2 unique items, where each item is a 'singleReference'. The main properties of the schema are: '_order' is an object, optional, that overrides the default resolution order, with pattern properties that map to array items containing objects; each item must have a 'path' (a string with a matching pattern) and 'references' (an integer with a minimum value of 1). No additional properties are allowed at the root level.
