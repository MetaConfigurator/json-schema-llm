The schema describes the configuration structure for a system. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'config' is an object that is required and contains properties such as 'general', 'triggers', 'executors', 'notifiers', and 'global_values'. 'general' is an object that is required and contains properties such as 'planFilePath', 'plan', 'executorsPath', 'notifiersPath', 'triggersPath', 'modulesPath', 'crypto', 'api', 'servers', 'queues', and 'runnerty.io'. 'planFilePath' is an optional string. 'plan' is an optional object. 'executorsPath' is an optional string. 'notifiersPath' is an optional string. 'triggersPath' is an optional string. 'modulesPath' is an optional string. 'crypto' is an optional object with properties 'algorithm' and 'iv', both of which are optional strings. 'api' is an optional object that can be configured in various ways, including requiring either 'port' or 'unix_socket', or both 'port' and 'ssl'. 'servers' is an optional array of objects, each requiring an 'id' and 'endpoint', with additional configurations varying among types. 'queues' is an optional object with a property 'refreshInterval', which is an optional number. 'runnerty.io' is an optional object that requires 'apikey' and contains several optional properties including 'disable', 'host', 'healthChecker', 'remoteControl', and 'debug'. No additional properties are allowed at the root level.
