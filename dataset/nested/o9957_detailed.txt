The schema describes an object representing a data aggregation configuration. It is an object type. The main properties of the schema are: 'start' is a string representing the start date-time, required; 'end' is a string representing the end date-time, required; 'resolution' is a number representing the resolution, required; 'aggregation' is an object defining the aggregation method, required, which can be one of multiple structures with different configuration options; 'devices' is an object where each property is itself an object that represents a device, required. It includes a pattern that keys must match (24 hex characters), and each device object must have a 'name' string and a 'points' array of objects, where each point contains 'time' and 'data', also required. Additionally, the schema contains compositional constraints for 'aggregation', which requires either a specific string type from an enum or a structured object configuration with validation rules. No additional properties are allowed at the root level.
