The schema describes API options for connecting to a rippled server. It is an object type. The main properties of the schema are: 'trace' is a boolean that indicates if true, log rippled requests and responses to stdout. 'feeCushion' is a number with a minimum value of 1, which is a factor to multiply the estimated fee by to provide a cushion during transaction submission; it defaults to `1.2`. 'maxFeeXRP' is a string that specifies the maximum fee to use with transactions in XRP. It must be a string-encoded number and defaults to `'2'`. 'server' is a string that represents the URI for the rippled websocket port to connect to. It must start with `wss://`, `ws://`, `wss+unix://`, or `ws+unix://`, and it has a pattern validation. 'proxy' is a string that provides the URI for an HTTP/HTTPS proxy to use to connect to the rippled server. 'timeout' is an integer with a minimum value of 1, indicating the timeout in milliseconds before considering a request to have failed. 'proxyAuthorization' is a string that specifies the username and password for HTTP basic authentication to the proxy in the format **username:password**. 'authorization' is a string that represents the username and password for HTTP basic authentication to the rippled server in the format **username:password**. 'trustedCertificates' is an array of strings where each item is a PEM-formatted SSL certificate to trust when connecting to a proxy; each element must contain a single certificate as concatenated certificates are not valid. 'key' is a string that contains the private key of the client in PEM format and can be an array of keys. 'passphrase' is a string representing the passphrase for the private key of the client. 'certificate' is a string that contains the certificate key of the client in PEM format and can be an array of certificates. No additional properties are allowed at the root level.
