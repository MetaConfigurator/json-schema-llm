The schema describes a configuration object for UI components. It is an object type. The main properties of the schema are: 'name' is a string unique display name (to be used instead of plugin ID), optional; 'fixed' is a boolean true if fixed set (e.g. UI Library), required; 'ui' is an array containing at least one item, each of which is an object describing a customization key, required. Each item in the 'ui' array has the following properties: 'key' is a string representing the customization key, required; 'description' is a string providing description text for this key, optional; 'source' is a string representing the FQN of the underlying component (if applicable), optional; 'deprecated' is a boolean indicating whether this key is deprecated, with a default value of false, optional. No additional properties are allowed at the root level.
