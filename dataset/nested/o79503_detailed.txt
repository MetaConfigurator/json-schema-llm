The schema describes the logical information needed to populate a button/control at runtime. It is an object type. It defines several reusable components under 'definitions': 'gateState' is a boolean, for actions that are apply gates, true if the gate has been passed for this specific candidate + job, false if it was failed - this field should only be present when there is a result. 'name' is a string, fixed name for this action, containing only lower case alpha, digits and '-' characters, length <= 30. The main properties of the schema are: 'backgroundColor' is a string the color that the button's background should be rendered in, in 6 character hex format without leading #, e.g. 00ff00. 'disabled' is a boolean true if the button/control should appear disabled, i.e. the action can be seen but not performed. 'gateState' is a boolean resolving to the definition 'gateState'. 'imageUri' is a string the url of an image to be displayed for this action (i.e. instead of a dynamically set up button). 'label' is a string a string indicating what the action will do, used as the label on its button/control. 'name' is a string resolving to the definition 'name', fixed name for this action, containing only lower case alpha, digits and '-' characters, length <= 30. 'textColor' is a string the color that the button's text should be rendered in, in 6 character hex format without leading #, e.g. 00ff00. 'type' is an enum with allowed values: 'image', 'button'. 'uri' is a string the url that the button/control will redirect the user to when pressed, or use to populate an iframe inside a modal. The following fields are required: 'name', 'label', 'uri', 'disabled'. No additional properties are allowed at the root level.
