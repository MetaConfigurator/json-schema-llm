The schema describes Auth0 rules user profile. It is an object type. It defines several reusable components under 'definitions': 'userIdentity' is an object, which requires the fields 'connection', 'isSocial', 'provider', and 'user_id'. The main properties of the schema are: 'app_metadata' is an object optional; 'blocked' is a boolean optional with a default value of false; 'created' is a string formatted as date-time optional; 'email' is a string formatted as email optional; 'email_verified' is a boolean optional; 'identities' is an array of items referencing the 'userIdentity' definition with a minimum of 1 item; 'last_ip' is a string formatted as ipv4 optional; 'last_login' is a string formatted as date-time optional; 'logins_count' is a number with a minimum value of 0 optional; 'multifactor' is an array of object items optional; 'name' is a string optional; 'nickname' is a string optional; 'phone_number' is a string optional; 'phone_verified' is a boolean optional; 'picture' is a string formatted as uri optional; 'updated_at' is a string formatted as date-time optional; 'user_id' is a string optional; 'user_metadata' is an object optional; 'username' is a string optional. No additional properties are allowed at the root level.
