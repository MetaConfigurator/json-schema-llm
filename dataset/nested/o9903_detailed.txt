The schema describes a collection of files and their metadata. It is an object type. The main properties of the schema are: 'items' is an array that contains schemas for individual files, where each item is an object with properties such as 'id', 'applicationId', 'creationDate', 'lastUpdated', 'authorId', 'authorType', 'status', 'name', 'parentDirectory', 'type', 'fileSize', 's3etag', 'contentType', 'fileDimensions', 'moveIntoId', and 'url'. 'count' is an integer representing the number of items, 'totalCount' is an integer representing the total items available, 'perPage' is an integer for pagination, 'page' is an integer for the current page number, 'filter' is a string for filtering results, 'filterField' is a string specifying the field to filter by, 'sortField' is a string for the field to sort by, and 'sortDirection' is a string with allowable values 'asc', 'desc', 'ASC', 'DESC', and '' for sort order. The 'items' array requires each file to have properties including: 'id', 'applicationId', and 'authorId' matching a specific pattern; 'creationDate' and 'lastUpdated' formatted as date-time; 'authorType' with enum values 'flow', 'user', 'device', 'apiToken', 'notebook'; 'status' with enum values 'pending', 'moving', 'deleting', 'completed'; 'name' having a length between 1 and 1024 characters; 'parentDirectory' having a max length of 1024 characters; 'type' with enum values 'file' and 'directory'; and 'fileDimensions' as an object with 'width' and 'height' as numbers. No additional properties are allowed at the root level.
