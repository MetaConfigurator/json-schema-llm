The schema describes an object structure for managing application certificates. It is an object type. The main properties of the schema are: 'id' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'applicationCertificateAuthorityId' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'applicationId' is a string matching the pattern '^[A-Fa-f\d]{24}$', optional. 'creationDate' is a string in date-time format, optional. 'lastUpdated' is a string in date-time format, optional. 'status' is a string with allowed values: 'active', 'inactive', optional. 'name' is a string with a minimum length of 1 and a maximum length of 255, optional. 'description' is a string with a maximum length of 32767, optional. 'caBundle' is a string with a minimum length of 50 and a maximum length of 32767, optional. 'caInfo' is an array of objects, where each object has properties: 'serialNumber' (a string with a minimum length of 1 and a maximum length of 1024), 'fingerprint' (a string with a minimum length of 1 and a maximum length of 1024), 'commonName' (a string with a maximum length of 1024), 'issuerName' (a string with a maximum length of 1024), 'notValidBefore' (a string in date-time format), and 'notValidAfter' (a string in date-time format). No additional properties are allowed at the root level.
