The schema describes a configuration for deployment scenarios. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'class' is a string name of the target class, required. 'scenarios' is an array list of possible deploying scenarios, required, where each scenario is an object containing several properties. Each scenario must have 'name', 'sig', 'provide', 'cost', and 'methods' properties. 'name' is a string name of the scenario, required and must be unique in the array. 'provide' is an integer Number of functionalities provided (-1 for unbounded), required. 'cost' is an object mapping resource name to cost, where additional properties are integers. 'methods' is an array list of methods to add and remove preferences, required, where each method is an object that includes an 'add' property that is also a required object with 'name' and 'param_type' as required strings, and a 'remove' property which is an optional object with 'name' as a required string and 'param_type' as an optional string. 'sig' is an array of objects that may match any of several structures, including those requiring specific properties like 'kind' and 'type'. No additional properties are allowed at the root level.
