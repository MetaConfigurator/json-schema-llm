The schema describes a model with data for timeChart. It is an object type. It defines several reusable components under 'definitions': 'pred' is an object, a prediction for season onset. It requires a 'point' field of type number. The main properties of the schema are: 'id' is a string, optional, for a short unique identifier for the model. 'pinned' is a boolean, optional, indicating whether to pin this model in the legend. 'meta' is an object, required, containing properties: 'description' (required, string) with a text description of the model, and 'name' (required, string) for the full name of the model. 'style' is an object, optional, with properties for 'color' (string), 'point' (object), 'area' (object), and 'line' (object), which specify CSS styles. 'predictions' is an array, required, containing items that are either objects or null. Each item contains properties: 'onsetTime' (referring to 'pred'), 'peakTime' (referring to 'pred'), 'peakValue' (referring to 'pred'), 'dataVersionTime' (string, for the time of data), and 'series' (required, an array of predictions referring to 'pred'). No additional properties are allowed at the root level.
