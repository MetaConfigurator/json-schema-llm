The schema describes a structured data format. It is an object type. It defines several reusable components under 'definitions': 'metadata' is an object, allowing additional properties as an array. 'replies' is an array, where each item is an object with required property 'reply' of type string and an optional 'metadata' property that references 'metadata'. The main properties of the schema are: 'feeds' is an array, where each item is an object with required properties 'name' of type string and 'replies' that references 'replies'. 'query' is a string, required. No additional properties are allowed at the root level.
