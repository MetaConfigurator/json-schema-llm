The schema describes a container schema for all Metadata from streamsapi. It is an object type. It defines several reusable components under 'definitions': 'coreSchema' is an object, which can be either a reference to 'vibrationChild_schema' or 'vibrationParent_schema'. 'vibrationChild_schema' is an object, for a single vibration measurement. It has required fields: 'id', 'parentId', 'measurementDate', and 'additionalMeasurements'. The 'measurementDate' is a string of type date-time. The 'statistics' is an object with a default 'rms' array of [1, 1] and references 'numArray' for fields: 'kurtosis', 'mean', 'rms', 'skewness', and 'std'. 'vibrationParent_schema' is an object, for a series of vibration measurements. It has required fields: 'id', 'documentClass', 'nominalFreq', and 'trackedFlag'. The 'documentClass' is a string with an enum value of 'parentOfChunks' and 'trackedFlag' is a boolean. The main properties of the schema are: no direct properties at the root level. The root schema resolves to the 'coreSchema' definition. The schema-level constraints allow for either a single object of 'coreSchema' or an array of 'coreSchema' with a minimum of one item. No additional properties are allowed at the root level.
