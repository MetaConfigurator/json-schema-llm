The schema describes a mapping used by Glue to extract findings from a security tool report. It is an object type. The main properties of the schema are: 'task_name' is a string that describes the name of the task (tool) that produces those findings and is required. 'app_name' is a string that describes the path to the property in the report file that contains the target app name and is required. 'mappings' is an array that describes an array of mappings and is required, where each item in the array is an object that contains the property 'key', which is a string that represents the key in the JSON report to an array that contains findings, and is required. Each item also contains a 'properties' object, which describes the mapping between the items in the array and Glue's findings and is required, containing 'description', 'detail', 'source', 'severity', and 'fingerprint', all of which are strings and are required. No additional properties are allowed at the root level.
