The schema describes the result of a Github worker. It is an object type. It defines several reusable components under 'definitions': 'github_extracted_details' is an object, which details the inspection of Github. It contains optional properties: 'forks_count' as an integer, 'last_year_commits' referenced from 'github_last_year_commits_details', 'open_issues_count' as an integer, 'stargazers_count' as an integer, 'subscribers_count' as an integer, 'updated_issues' referenced from 'github_issues_details', and 'updated_pull_requests' referenced from 'github_pull_requests_details'. No additional properties are allowed. 'github_issues_details' is an object, which provides details of updated Github issues. It requires 'open' as an integer and 'closed' as an integer. No additional properties are allowed. 'github_last_year_commits_details' is an object, which gives details of last year Github commits. It requires 'sum' as an integer and 'weekly' as an array of integers. No additional properties are allowed. 'github_pull_requests_details' is an object, which contains details of updated Github pull requests. It requires 'open' as an integer and 'closed' as an integer. No additional properties are allowed. The main properties of the schema are: 'schema' is an object, which provides the details of the schema with required properties 'name' as a string (pattern: '^[a-zA-Z0-9_]+$') and 'version' as a string (pattern: '^[0-9]+-[0-9]+-[0-9]+$'). No additional properties are allowed. '_release' is an optional string, which is a unique release id in the form of 'ecosystem:package:version'. 'status' is a string with allowed values: 'success', 'error', 'unknown'. 'details' is a referenced object from 'github_extracted_details'. 'summary' is an array of strings. No additional properties are allowed at the root level.
