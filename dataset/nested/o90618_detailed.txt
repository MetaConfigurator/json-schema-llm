The schema describes the configuration for a Bitbucket application. It is an object type. The main properties of the schema are: 'bitbucket' is an object that contains properties for configuring the Bitbucket application. It is required. This object includes the following required fields: 'cpus', 'mem', 'instances', and 'name', with the following types and descriptions: 'cpus' is a number representing CPU shares to allocate to each bitbucket instance, with a default of 2 and a minimum of 2. 'mem' is a number indicating memory (MB) to allocate to each bitbucket task, with a default of 2048.0 and a minimum of 2048.0. 'instances' is an integer specifying the number of instances to run, with a default of 1 and a minimum of 1. 'name' is a string for naming the Bitbucket application, with a default of 'bitbucket'. Optional properties include 'host-volume', 'maximumOverCapacity', 'minimumHealthCapacity', 'role', and 'virtual-host', which have their own defaults but are not required. No additional properties are allowed at the root level.
