The schema describes generating a new, generic route guard definition in the given or default project. It is an object type. The main properties of the schema are: 'flat' is a boolean when true (the default), creates the new files at the top level of the current project. It is optional and defaults to true. 'implements' is an array specifies which interfaces to implement. It is required to have at least one item, which must be a unique string contained in the enum: 'CanActivate', 'CanActivateChild', 'CanDeactivate', 'CanLoad'. 'lintFix' is a boolean when true, applies lint fixes after generating the guard. It is optional and defaults to false. 'name' is a string the name of the new route guard. It is required. 'path' is a string the path at which to create the interface that defines the guard, relative to the current workspace. It is optional. 'project' is a string the name of the project. It is optional. 'skipTests' is a boolean when true, does not create "spec.ts" test files for the new guard. It is optional and defaults to false. 'spec' is a boolean when true (the default), generates a  "spec.ts" test file for the new guard. It is optional and defaults to true, but is deprecated in favor of 'skipTests'. No additional properties are allowed at the root level.
