The schema describes ED3D systems and filters list. It is an object type. The root schema resolves to the referenced definition '#/categories/items/category'. The effective root schema describes a category as an object, which has an integer field 'id' that is required, a string field 'name' that is required, and an optional string field 'color'. The main properties of the schema are: 'categories.items.category' is an object representing a category, which is required; 'systems' is an array representing a list of systems, which is required and contains objects with a required string field 'name' and a required object field 'coords' (which includes required integer fields 'x', 'y', and 'z'), an optional integer field 'type', an optional string field 'infos', and an optional array field 'cat' that contains objects with a required integer field 'id'. No additional properties are allowed at the root level.
