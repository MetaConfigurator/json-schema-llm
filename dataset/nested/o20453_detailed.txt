The schema describes a target gene web request graph for Targetscope, building a tree of individual operations, inputs and outputs. It is an object type. The main properties of the schema are: 'client_id' is a string unique identifier for the client making the request, optional; 'dtsubmission' is a string an RFC3339-formatted request graph submission timestamp in UTC time, optional; 'id' is a string a unique identifier for a target gene request graph instance, required; 'name' is a string name of the operation request graph instance, required; 'operations' is an array list of request graph Operation components, required; 'save_intermediate' is a boolean save intermediate results of the pipeline, optional; 'sets' is an array list of request graph Element and/or Interaction Set components, optional; 'version' is a string schema version associated with request, with allowed values: 'v1.0.0', 'v1.1.0', 'v1.2.0', required. The 'operations' array must contain objects where each object has 'id' and 'name' as required string properties, and 'parameters' which is an array of objects each containing 'kind' and 'arguments' as required. 'arguments' must be an array containing objects with required 'kind' and 'arguments', while 'sets' must also follow similar rules with required 'id' and 'kind'. 'sets' can only contain objects where 'kind' must be either 'element' or 'interaction'. 'kind' fields in 'arguments' must conform to specific enumerated values as defined. Additionally, 'version' must match one of the enumerated values. No additional properties are allowed at the root level.
