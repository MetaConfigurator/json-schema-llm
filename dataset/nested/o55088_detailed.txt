The schema describes page layout configuration. It is an object type. The main properties of the schema are: 'page' is an object containing page level options, which is optional. 'components' is an array of component objects that are required. Each component object must contain 'presenter' and 'page_location' as required properties. 'presenter' is a string that describes a URI for the presenter location. 'page_location' is a string indicating the key in the template for the component's location. 'fail_page_load' is an optional boolean that indicates if the failure to load this page results in a failed page load. 'opts' is an optional object for component options. No additional properties are allowed at the root level.
