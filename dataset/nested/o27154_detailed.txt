The schema describes invocation configuration for an example boutiques tool. It is an object type. The main properties of the schema are: 'config_num' is a number that must be 4. 'enum_input' is a string that can be 'val1', 'val2', or 'val3'. 'file_input' is a string that is optional. 'flag_input' is a boolean that is optional. 'list_int_input' is an array of integers with at least 1 item. 'num_input' is a number that must be greater than 0 and less than or equal to 1. 'str_input' is an array of strings with between 1 and 3 items. The schema requires 'str_input', 'list_int_input', and 'config_num'. It enforces that either 'num_input' or 'enum_input' must be present, and has dependencies where if 'enum_input' is present, 'num_input' cannot be, and if 'flag_input' is present, it must either be of type 'file_input' or have 'flag_input' set to false. No additional properties are allowed at the root level.
