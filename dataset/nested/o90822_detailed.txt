The schema describes DC/OS service configuration properties. It is an object type. The main properties of the schema are: 'service' is an object containing configuration properties for the DC/OS service. It is required and has several properties: 'cpus' is a number representing CPU shares to allocate to each service instance, with a default of 0.1 and a minimum value of 0.1. 'host_path' is a string indicating the path in the host to be mapped into spark executors, with a default of '/var/lib/tmp/spark'. 'instances' is a number specifying the number of service instances to run in the cluster, with a default of 256 and a minimum value of 1. 'mem' is a number indicating memory to allocate to each service instance, with a default of 128.0 and a minimum value of 128.0. 'name' is a string representing the name of this service instance, with a default of 'spark-shuffle'. 'port' is a number identifying the TCP port for the shuffle service, with a default of 7337. 'spark-daemon-memory' is a string denoting memory to allocate to the spark daemon, with a default of '512m'. No additional properties are allowed at the root level.
