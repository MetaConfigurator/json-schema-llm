The schema describes SSR Dev Server target options for Build Facade. It is an object type. The main properties of the schema are: 'browserTarget' is a string that specifies the browser target to build and is required. It must match the pattern '.+:.+(:.+)?'. 'serverTarget' is a string that specifies the server target to build and is required. It must also match the pattern '.+:.+(:.+)?'. 'host' is a string that specifies the host to listen on, with a default value of 'localhost'. 'port' is a number that specifies the port to start the development server at, with a default value of 4200; passing 0 will get a dynamically assigned port. 'publicHost' is a string that specifies the URL that the browser client should use to connect to the development server, useful for complex dev server setups. 'open' is a boolean that opens the URL in the default browser, with a default value of false, and has an alias 'o'. 'progress' is a boolean that logs progress to the console while building. 'inspect' is a boolean that launches the development server in inspector mode and listens on '127.0.0.1:9229', with a default value of false. No additional properties are allowed at the root level.
