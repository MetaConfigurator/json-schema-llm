The schema describes a configuration for features and contexts. It is an object type. The main properties of the schema are: 'amountOfFeatures' is an integer representing the amount of features, required. 'amountOfContexts' is an integer representing the amount of contexts, required. 'attributesPerFeature' is an array of integers representing the number of attributes per feature, required. 'attributeDomains' is an array of objects stating the domains of the attributes, required. Each object contains 'min', an integer indicating the minimal value that the attribute can take, and 'max', an integer indicating the maximal value that the attribute can take. 'contextDomains' is an array of objects stating the domains of the context, required. Each object contains 'min', an integer indicating the minimal value that the context can take, and 'max', an integer indicating the maximal value that the context can take. 'configuration' is an object containing 'selectedFeatures', an array of integers representing which features are selected; 'attributeValues', an array of integers representing the initial values of the attributes; and 'contextValues', an array of integers representing the contextual values, required. 'constraints' is an array of strings representing constraints derived from the FM, required. 'preferences' is an array of strings representing preferences derived from the user, required. No additional properties are allowed at the root level.
