The schema describes the structure of a content item. It is an object type. It defines several reusable components under 'definitions': 'guid' is a string, requiring a format that matches the UUID pattern. 'guid_list' is an array, where each item must reference 'guid' and must have unique items. The main properties of the schema are: 'bulk_publishing' is a boolean, optional. 'links' is an object, optional, and contains several properties referencing 'guid_list'. These include 'facet_groups', 'facet_values', 'finder', 'lead_organisations', 'mainstream_browse_pages', 'meets_user_needs', 'ordered_related_items', 'ordered_related_items_overrides', 'organisations', 'original_primary_publishing_organisation', 'parent' (maxItems of 1), 'policy_areas', 'primary_publishing_organisation' (maxItems of 1), 'related', 'suggested_ordered_related_items', 'taxons', 'topics', and 'world_locations'. 'previous_version' is a string, optional. No additional properties are allowed at the root level.
