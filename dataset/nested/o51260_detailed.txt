The schema describes a configuration containing targets. It is an object type. It defines several reusable components under 'definitions': 'target' is an object, representing a target configuration with required fields 'type', 'options', and 'strategy'. The 'type' must be a string with an enum value of 'aws'. The 'options' must adhere to the 'optionsAws' definition. The 'strategy' can be one of two types: either a single string from the predefined options 'clone', 'gzip', 'gzip-suffix' or an array of unique strings adhering to the same options. The main properties of the schema are: 'targets' is an array, required, with a minimum of 1 item, which must be unique, and contains references to the 'target' definition as items. No additional properties are allowed at the root level.
