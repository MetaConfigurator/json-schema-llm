The schema describes OpenStack job accounting data. It is an object type. The main properties of the schema are: 'disk_gb' is a string that represents the disk size in gigabytes. 'ephermeral_gb' is a string that indicates the ephemeral disk size in gigabytes. 'event_type' is a string that specifies the type of event. 'generated' is a string that indicates when the event was generated. 'host' is a string that represents the host name. 'instance_id' is a string that uniquely identifies the instance. 'instance_type' is a string that indicates the type of instance. 'instance_type_id' is an integer that represents the ID of the instance type. 'launched_at' is a string in the format 'full-date' that shows when the instance was launched. 'memory_mb' is an integer representing the memory size in megabytes. 'message_id' is a string that uniquely identifies the message. 'project_id' is a string that identifies the project. 'project_name' is a string that specifies the project name. 'request_id' is a string that identifies the request. 'resource_id' is a string that identifies the resource. 'root_gb' is an integer representing the root disk size in gigabytes. 'service' is a string that indicates the service involved. 'state' is a string that shows the current state. 'tenant_id' is a string that identifies the tenant. 'user_id' is a string that identifies the user. 'user_name' is a string or null that represents the name of the user. 'vcpus' is an integer representing the number of virtual CPUs. 'volume_id' is a string that identifies the volume. 'domain' is a string that represents the domain. No additional properties are allowed at the root level.
