The schema describes Protractor target options for Build Facade. It is an object type. The main properties of the schema are: 'protractorConfig' is a string representing the name of the Protractor configuration file, which is required. 'devServerTarget' is a string describing the dev server target to run tests against, with the pattern '^([^:\s]+:[^:\s]+(:[^\s]+)?)?$'. 'grep' is a string that specifies executing specs whose names match the pattern, which is internally compiled to a RegExp. 'invertGrep' is a boolean that inverts the selection specified by the 'grep' option, with a default value of false. 'specs' is an array that overrides specs in the protractor config, defaults to an empty array, with items being strings that describe spec names. 'suite' is a string that overrides the suite in the protractor config. 'elementExplorer' is a boolean that starts Protractor's Element Explorer for debugging, with a default value of false and is marked as deprecated. 'webdriverUpdate' is a boolean that tries to update webdriver, with a default value of true. 'port' is a number representing the port to use to serve the application. 'host' is a string describing the host to listen on. 'baseUrl' is a string representing the base URL for protractor to connect to. No additional properties are allowed at the root level.
