The schema describes a Mobile Device Management configuration profile. It is an object type. The main properties of the schema are: 'PayloadUUID' is a string Unique identifier for the payload (format 01234567-89AB-CDEF-0123-456789ABCDEF). It is required and has a default value of an empty string, with a pattern validation. 'ServerCapabilities' is an array Optional. An array of strings indicating server capabilities. It has an enum value of ['com.apple.mdm.per-user-connections']. 'PayloadDescription' is a string Description of the payload. It has a default value of 'Configures a Mobile Device Management server'. 'CheckInURL' is a string The URL that the device will use to check in during installation. It has a pattern constraint that requires it to match the regex '^https://.*$'. 'CheckOutWhenRemoved' is a boolean If true, the device attempts to send a CheckOut message to the check-in server when the profile is removed. Defaults to false. 'PayloadOrganization' is a string This value describes the issuing organization of the profile, as displayed to the user. 'ServerURL' is a string The URL that the device will contact to retrieve device management instructions. It has a pattern constraint matching '^https://.*$'. 'PayloadIdentifier' is a string A unique identifier for the payload, dot-delimited, with a default value of 'com.apple.mdm'. 'PayloadDisplayName' is a string Name of the payload. It has a default value of 'MDM'. 'Topic' is a string The topic that MDM will listen to for Push notifications. 'PayloadType' is a string The type of the payload, a reverse dns string, with a default value of 'com.apple.mdm'. 'SignMessage' is a boolean If set, each message coming from the device will carry the additional HTTP header Mdm-Signature. It has a default value of false. 'PayloadVersion' is a number The version of the whole configuration profile, with a default value of 1. 'IdentityCertificateUUID' is a string UUID of the certificate payload for the device's identity. It has a pattern constraint matching '^[0-9A-Za-z]{8}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{4}-[0-9A-Za-z]{12}$'. 'AccessRights' is a number Logical OR of several bit-flags. 'UseDevelopmentAPNS' is a boolean If set, will use the development APNS servers. Otherwise, the device will use the production servers. It has a default value of false. No additional properties are allowed at the root level.
