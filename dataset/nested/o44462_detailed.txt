The schema describes a payment transaction. It is an object type. The main properties of the schema are: 'sign' is an array of strings, required, with a minimum of 1 item, where all items must be unique. 'tx' is an object, required. It has required properties: 'id' is a string that must match the pattern '^[0-9a-f]{64}$', 'from' is an array of strings, with a minimum of 1 item and unique items, 'msg' is a string with a maximum length of 254, 'fee' is an integer that must be greater than 0, and 'to' is an array of objects with required properties: 'address' (a string) and 'amount' (an integer that must be greater than 0). No additional properties are allowed at the root level.
