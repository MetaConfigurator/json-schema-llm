The schema describes the configuration properties for a DC/OS service and its networking settings. It is an object type. The main properties of the schema are: 'networking' is an object describing cadvisor networking configuration properties. It contains the optional properties: 'external_access', which is an object enabling access from outside the cluster through Marathon-LB with properties 'enable', a boolean indicating whether to create a VIP for external access with a default of false, and 'external_access_port', a number indicating the port for clear communication with a default of 18080; 'influxdb_host', a string representing the hostname or IP address of the InfluxDB host with a default of 'influxdb.marathon.l4lb.thisdcos.directory'; and 'influxdb_port', a number for the TCP port where the InfluxDB host listens with a default of 8086. 'service' is an object describing DC/OS service configuration properties. It includes the optional properties: 'cpus', a number for the CPU shares allocated to each service instance with a minimum of 0.1 and a default of 0.1; 'instances', a number for the number of service instances to run, which has a minimum of 1 and a default of 256; 'mem', a number indicating the memory allocated to each service instance with a minimum of 128.0 and a default of 128.0; and 'name', a string for the service instance's name with a default of 'cadvisor'. No additional properties are allowed at the root level.
