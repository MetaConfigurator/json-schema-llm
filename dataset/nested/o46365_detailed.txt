The schema describes an error structure. It is an object type. It defines several reusable components under 'definitions': 'code' is a string, this will be in upper-case and underscore separated, aka ANGRY_SNAKE_CASE. 'message' is a string, free text. 'path' is a string, XPath, e.g. /foo/bar. 'part' is an object, containing properties 'code', 'message', and 'path', each referencing their respective definitions. The main properties of the schema are: 'code' is a string, required. 'message' is a string, required. 'path' is a string, optional. 'errors' is an array, containing items that are objects defined by 'part'. Each 'part' object must include 'code', 'message', and 'path'. No additional properties are allowed at the root level.
