The schema describes an array of possible actions. It is an array type. It defines several reusable components under 'definitions': 'purpose' is an enum, values are: createAccount, editDetails, apply. 'gateForElement' is an object, a purpose that (e.g. 'apply') that this button is a gate for. It has a required field 'purpose' which references the 'purpose' definition, and no additional properties are allowed. 'semantics' is an object, details about the semantics of this action that can be used by other apps. It has fields 'purpose' (which references the 'purpose' definition) and 'gateFor' (an array of 'gateForElement'), with no additional properties allowed. 'possibleAction' is an object, it requires 'actionName', 'availableToInternals', 'availableToExternals', and 'mayRequireSetupFlag'. It has optional fields 'description' and 'semantics' (which references the 'semantics' definition), with no additional properties allowed. No additional properties are allowed at the root level.
