The schema describes a configuration object. It is an object type. The main properties of the schema are: 'start' is a number optional status. 'end' is a number optional status. 'duration' is a number optional status. 'resolution' is a number optional status. 'aggregation' is a string optional status, with allowed values: 'FIRST', 'LAST', 'COUNT', 'MAX', 'MIN', 'MEDIAN', 'MEAN', 'SUM', 'STD_DEV', 'NONE'. 'attributes' is an array of strings optional status, with items that must match the pattern '^[0-9a-zA-Z_-]{1,255}$'. 'deviceTags' is an array of objects optional status, with a maximum of 100 items. Each object must have a 'key' which is a string that matches the pattern '^[0-9a-zA-Z_-]{1,255}$' and a 'value' which is a string with a minimum length of 1 and a maximum length of 255. No additional properties are allowed at the root level.
