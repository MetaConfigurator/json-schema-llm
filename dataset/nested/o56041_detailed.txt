The schema describes a structure for any entity. It is an object type. The main properties of the schema are: 'annotations' is an optional object that holds arbitrary string key/value pairs, where each key can match any string pattern and the associated value must be a string. 'apiversion' is a string representing the version of the entity type and is required. 'category' is a string that can take one of the following values: 'artifact', 'execution', 'container' and is required. 'create_time' is a string representing the time when the entity is created in the format of RFC3339 and is optional. 'description' is a string providing a description of this entity and is optional. 'id' is a string serving as a unique identifier for this entity and is required. 'kind' is a string that denotes the type of this entity and is required. 'name' is a string specifying the name of this entity and is required. 'namespace' is a string that indicates the namespace of the entity type and is required. 'owner' is a string signifying the owner of this entity and is optional. No additional properties are allowed at the root level.
