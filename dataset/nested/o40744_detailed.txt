The schema describes generating a new basic app definition in the 'projects' subfolder of the workspace. It is an object type. The main properties of the schema are: 'inlineStyle' is a boolean that indicates whether to include styles inline in the root component.ts file. It is optional and defaults to false. 'inlineTemplate' is a boolean that indicates whether to include the template inline in the root component.ts file. It is optional and defaults to false. 'legacyBrowsers' is a boolean that adds support for legacy browsers using differential loading. It is optional and defaults to false. 'lintFix' is a boolean that applies lint fixes after generating the application. It is optional and defaults to false. 'minimal' is a boolean that creates a bare-bones project without testing frameworks. It is optional and defaults to false. 'name' is a string representing the name of the new app. It is required and uses the first argument from the command line as its default value. 'prefix' is a string that serves as a prefix for generated selectors. It is optional and defaults to 'app'. 'projectRoot' is a string representing the root directory of the new app. It is optional and marked as not visible. 'routing' is a boolean that indicates whether to create a routing NgModule. It is optional and defaults to false. 'skipInstall' is a boolean that indicates whether to skip installing dependency packages. It is optional and defaults to false. 'skipPackageJson' is a boolean that indicates whether to not add dependencies to the 'package.json' file. It is optional and defaults to false. 'skipTests' is a boolean that indicates whether to not create 'spec.ts' test files for the app. It is optional and defaults to false. 'strict' is a boolean that creates an application with stricter build optimization options. It is optional and defaults to false. 'style' is a string that represents the file extension or preprocessor to use for style files. It is optional, defaults to 'css', and can be one of 'css', 'scss', 'sass', 'less', or 'styl'. 'viewEncapsulation' is a string representing the view encapsulation strategy to use in the new app. It is optional and can be one of 'Emulated', 'Native', 'None', or 'ShadowDom'. No additional properties are allowed at the root level.
