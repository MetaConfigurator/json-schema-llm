The schema describes meta-information about installation of an IDE based on IntelliJ Platform. It is an object type. The main properties of the schema are: 'name' is a string full name of the product without edition, e.g. 'IntelliJ IDEA' or 'PyCharm', required. 'version' is a string marketing version of the product, e.g. '2018.3', required. 'versionSuffix' is a string specifies type of this product version which is shown after the version number in the UI, e.g. 'EAP', 'EAP 2', 'Preview', etc. May be omitted (e.g. for release builds), optional. 'buildNumber' is a string build number of the product, e.g. '183.239' or '183.SNAPSHOT', required. 'productCode' is a string product code (e.g. 'IC' for IntelliJ IDEA Community Edition) as used in full build number, required. 'dataDirectoryName' is a string name of a directory used for storing settings, caches, etc. (e.g. 'IntelliJIdea2020.1' for IntelliJ IDEA 2020.1); the full path to the directory is OS-specific, optional. 'svgIconPath' is a string path to an SVG file which contains the product icon (relative to the parent directory of this file, '/' as a separator, may start with '../'), e.g. 'bin/idea.svg', optional. 'launch' is an array specifies how the IDE can be launched, required, and must contain at least one item. Each item in the array is an object type with required properties: 'os', 'launcherPath', and 'vmOptionsFilePath'. 'os' is a string operation system this launch method is supposed to be used for, with allowed values: 'Windows', 'Linux', 'macOS'. 'launcherPath' is a string path to an executable file which starts the IDE (relative to the parent directory of this file, '/' as a separator, may start with '../'), e.g. 'bin/idea.sh', required. 'javaExecutablePath' is a string path to a JRE executable file (relative to the parent directory of this file, '/' as a separator, may start with '../'), e.g. 'jre64/bin/java'. May be missing if JRE isn't bundled with the product, optional. 'vmOptionsFilePath' is a string path to a file containing JVM options which are used to start the IDE (relative to the parent directory of this file, '/' as a separator, may start with '../'), e.g. 'bin/idea64.vmoptions', required. 'startupWmClass' is a string value of 'WMClass' property used by some Linux desktop distributions to associate windows with the owning application, e.g. 'jetbrains-idea'. May be specified only if 'os' is 'Linux', optional. No additional properties are allowed at the root level.
