The schema describes the configuration files for Elm packages. It is an object type. It defines several reusable components under 'definitions': 'rangeDependency' is an object, Dependencies are specified by [author name]/[package name]: [exact version or range]. Range versions look like: [1.0.0 <= v < 2.0.0]. It allows additional properties of type string. The main properties of the schema are: 'dependencies' is an object that references the 'rangeDependency' definition and describes a list of Elm dependencies that your package needs. 'elm-version' is a string describing the version of Elm your package is targeting and is required. 'exposed-modules' is an array of strings describing all modules that your project will make available to end users and is required. 'license' is a string describing a license for your package and is required. 'repository' is a string describing the project's GitHub URL and is required. 'source-directories' is an array of strings describing a list of directories that will be used to build your project and is required. 'summary' is a string providing a helpful summary of your package, less than 80 characters, with a maximum length of 79 and a minimum length of 1, and is required. 'version' is a string for your package's version in semantic versioning format, requiring a pattern of \d+\.\d+\.\d+, and is required. No additional properties are allowed at the root level.
