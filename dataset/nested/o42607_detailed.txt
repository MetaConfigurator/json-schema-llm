The schema describes a form field configuration. It is an object type. The main properties of the schema are: 'name' is a string that serves as a unique identifier for the field within the form. It is required and must match the pattern '^[a-zA-Z_]{1}[\w-]*$'. 'maps_to' is a string that defines the name of the schema field the answer will map to, defaulting to 'cf', and must match the pattern '^[a-zA-Z_]{1}\w*$'. 'label' is a string that provides the main text for the question or field, with a maximum length of 255 characters. 'placeholder' is an optional string with a maximum length of 255 characters. 'description' is a string that provides a short explanation of the field, with a maximum length of 65535 characters. 'is_required' is a boolean indicating if the field is required. 'link_text' is an optional string that can replace the token '{link}' within the label or description, with a maximum length of 255 characters. 'link_url' is an optional string that specifies the URL for the replaced token '{link}', which must match the pattern '^(https?:\/\/)?([\da-z\.-]+)\.([a-z\.]{2,6})([\/\w \.-]*)*\/?$'. No additional properties are allowed at the root level.
