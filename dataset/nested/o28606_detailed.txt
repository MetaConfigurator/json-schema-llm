The schema describes a list of chunk comments. It is an object type. The main properties of the schema are: 'data' is an array containing chunk comments. It is required and must have at least one item. Each item in the array is an object and represents a chunk comment. The chunk comment has the following properties: 'id' is a number and is required. 'commentator' is an object representing the comment author and is required. It has required fields: 'id' (a number), 'nickname' (a string), 'created_at' (a string), and 'updated_at' (a string). 'avatar' is either a string or null and is optional. 'body' is a string and is required. 'created_at' (a string) and 'updated_at' (a string) are both required as well. No additional properties are allowed at the root level.
