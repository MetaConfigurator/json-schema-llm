The schema describes the definitions for state machines as defined by the php izzum library. It is an object type. The main properties of the schema are: 'comment' is a string or null that allows comments for the description of the file contents. It is optional. 'machines' is an array of objects, representing all machines defined here. It is required, must contain at least one item, and all items must be unique. Each machine object must have required properties: 'name', 'description', 'states', and 'transitions'. 'name' is a string that matches the pattern '^([a-z0-9])+((-)?([a-z0-9])+)*$'. 'description' is a string or null describing the machine. 'states' is an array with a minimum of two items, each being an object that defines a state, must have required properties: 'name' and 'type'. 'name' matches the pattern '^([a-z0-9])+((-)?([a-z0-9])+)*$|^(not-)?regex:(.*)$' and 'type' is a string that must be one of the following: 'initial', 'normal', 'final', 'regex'. 'transitions' is an array with a minimum of one item, each being an object that defines a transition, must have required properties: 'state_from', 'state_to', and 'event'. 'state_from' and 'state_to' match the same pattern as 'name', and 'event' is a string that matches the pattern '^[a-zA-Z0-9]+$'. No additional properties are allowed at the root level.
