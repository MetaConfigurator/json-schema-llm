The schema describes the configuration for an MRIQC invocation manifest. It is an object type. The main properties of the schema are: 'inputs' is an object that is required and contains a required property 'nifti', which is also an object with a required property 'type' that can only be 'nifti'. 'config' is an object that is required and contains a required property 'measurement', which is a string with a default value of 'auto-detect' and can have one of the following values: 'auto-detect', 'functional', 'anatomy_t1w', 'anatomy_t2w'. No additional properties are allowed at the root level.
