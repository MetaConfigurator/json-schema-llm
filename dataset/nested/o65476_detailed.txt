The schema describes a licence information structure. It is an object type. The main properties of the schema are: 'category' is a string that indicates the category of licence and is optional with allowed values 'Financial' and 'Business'. 'company_jurisdiction' is a string that specifies the jurisdiction where the company is based and is required with a minimum length of 1. 'company_name' is a string that denotes the name of the company holding the licence and is required with a minimum length of 1. 'confidence' is a string that reflects confidence in the accuracy of data and is optional with allowed values 'HIGH', 'MEDIUM', and 'LOW'. 'end_date' is a string formatted as a date and is optional. 'jurisdiction_classification' is a type that can be either a string or an array and is optional. 'licence_jurisdiction' is a string that shows the jurisdiction for which the licence is issued and is required with a minimum length of 1. 'licence_number' is a string that represents the licence number or code and is optional. 'regulator' is a string that names the regulating body that issued the licence and is optional. 'retrieved_at' is a string formatted as a date and is optional. 'sample_date' is a string formatted as a date that indicates the date on which this fact was known to be true and is required. 'source_url' is a string that indicates where this fact can be verified and is required with a minimum length of 1. 'start_date' is a string formatted as a date and is optional. 'status' is a string that represents the status of the licence and is optional. No additional properties are allowed at the root level.
