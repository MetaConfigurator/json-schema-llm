The schema describes a route data object. It is an object type. The main properties of the schema are: 'route_id' is a string that serves as the route unique identifier and is required. 'member_id' is a string or null that represents the user ID who is assigned to the route and is required. 'member_email' is a string or null formatted as an email that indicates the user email. 'vehicle_alias' is a string or null that specifies the name of the vehicle ID. 'driver_alias' is a string or null that indicates the name of the driver ID. 'route_cost' is a number or null that denotes the route cost. 'route_revenue' is a number or null that specifies the route revenue. 'net_revenue_per_distance_unit' is a number or null representing the net revenue per distance unit, calculated as (Revenue - cost) / distance units. 'created_timestamp' is an integer or null that indicates when the route was created. 'mpg' is a string or null which describes miles per gallon used to compute fuel consumption. 'trip_distance' is a number or null that indicates the route length, with the unit provided by distance_unit. 'gas_price' is a number or null that represents the price of gas used to compute route costs. 'route_duration_sec' is an integer or null that indicates the estimated drive time of the route in seconds. No additional properties are allowed at the root level.
