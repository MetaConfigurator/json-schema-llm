The schema describes an object structure for managing user tokens. It is an object type. It defines several reusable components under 'definitions': 'scope' is an array, containing items that are strings. 'status' is a string, with allowed values being 'active' and 'inactive'. The main properties of the schema are: 'id' is a string matching the pattern '^[A-Fa-f\d]{24}$'. 'accessTokenId' is a string matching the pattern '^[A-Fa-f\d]{24}$'. 'userId' is a string matching the pattern '^[A-Fa-f\d]{24}$'. 'name' is a string with a minimum length of 1 and a maximum length of 255. 'creationDate' is a string in date-time format. 'lastUpdated' is a string in date-time format. 'expirationDate' is a string in date-time format. 'scope' is an array of strings as defined in 'scope'. 'status' is a string as defined in 'status'. 'token' is a string. No additional properties are allowed at the root level.
