The schema describes a system policy rule configuration. It is an object type. The main properties of the schema are: 'Comment' is a string that describes the comment for the system policy UI; it is optional. 'Expiration' is a string formatted as date-time that indicates an expiration date for the rule(s); it is optional. 'OperationType' is a string that specifies the operation type, with allowed values 'operation:execute', 'operation:install', 'operation:lsopen'; it is optional and has a default value of 'operation:execute'. 'PayloadDescription' is a string that describes the payload; it is optional and has a default value of 'System Policy Rule Payload'. 'PayloadDisplayName' is a string that names the payload; it is optional and has a default value of 'System Policy Rule Payload'. 'PayloadIdentifier' is a string that serves as a unique identifier for the payload; it is optional and has a default value of 'com.apple.systempolicy.rule'. 'PayloadOrganization' is a string describing the issuing organization of the profile; it is optional. 'PayloadType' is a string for the type of the payload, and it is optional with a default value of 'com.apple.systempolicy.rule'. 'PayloadUUID' is a string that uniquely identifies the payload, formatted as a specific UUID pattern; it is optional and has a default value of an empty string. 'PayloadVersion' is a number indicating the version of the whole configuration profile; it is optional and has a default value of 1. 'Requirement' is a string that specifies the policy requirement, following the syntax described in Code Signing Requirement Language; it is optional. No additional properties are allowed at the root level.
