The schema describes the configuration for an endpoint. It is an object type. The main properties of the schema are: 'config' is an object that contains properties such as 'endpoints', which is an array of strings representing the names of the method config yaml(json) files, and 'rateLimit', which is an integer indicating how many requests per second are allowed. 'dependencies' is an object that includes properties like 'client', which is an array of strings naming clients this endpoint depends on, and 'middleware', which is an array of strings naming middleware this endpoint relies on. 'name' is a string that specifies the endpoint name. 'type' is a string representing the endpoint protocol, which must be either 'http' or 'tchannel'. All properties are required. No additional properties are allowed at the root level.
