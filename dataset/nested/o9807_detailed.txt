The schema describes an Event modification request. It is an object type. The main properties of the schema are: 'eventIds' is an array of strings that must match the pattern '^[A-Fa-f\d]{24}$'. 'updates' is an object that describes the body of the Event modification request and is optional. Within 'updates', 'state' is a string that must be one of the following values: 'new', 'acknowledged', 'resolved'. 'comment' is a string with a maximum length of 32767 characters. 'data' is an unspecified type and 'eventTags' is an object with properties that must match the pattern '^[0-9a-zA-Z_-]{1,255}$'; each property must be a string with a minimum length of 1 and a maximum length of 255 characters. No additional properties are allowed at the root level.
