The schema describes an API request configuration. It is an object type. The main properties of the schema are: 'HttpMethod' is a string that is required and must be one of the following values: 'DELETE', 'GET', 'HEAD', 'OPTIONS', 'PATCH', 'POST', 'PUT'. 'ResourceId' is a string that is required and must match the pattern '^[a-z0-9]+$'. 'ResponseParameters' is an object that is optional and does not allow additional properties. It may contain properties matching the pattern '^method\.response\.header\..+$', which must be strings matching the pattern '^(integration\.response\.header\..)|(integration\.response\.body\..)|('.*')+$'. 'ResponseTemplates' is an object that is optional and does not allow additional properties. It may contain properties matching the pattern '.+', which must be strings. 'RestApiId' is a string that is required and must match the pattern '^[a-z0-9]{10}$'. 'SelectionPattern' is a string that is optional. 'StatusCode' is a string that is required and must match the pattern '^\d{3}$'. No additional properties are allowed at the root level.
