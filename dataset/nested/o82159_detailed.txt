The schema describes a job configuration for a continuous delivery process. It is an object type. The main properties of the schema are: 'id' is a string which is optional. 'context' is an object which is optional and contains the required properties 'owner' and 'repo', along with optional properties 'branch', 'commit', and 'dockerRepoBase'. The 'dockerRepoBase' property is a string with a description about Docker registry base location. 'results' is an object which is optional and contains several properties: 'build', 'checkout', 'clone', and 'push', each being an object with an optional property 'status' of type string. 'startTime' is a string which is optional. 'endTime' is either a string or null which is optional. 'status' is a string which is optional. 'image' is a string which is optional. No additional properties are allowed at the root level.
