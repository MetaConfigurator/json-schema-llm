The schema describes a Note. It is an object type. The main properties of the schema are: 'note_id' is an integer the id of the note in the system. It is optional. 'route_id' is a string Route ID. It is optional. 'route_destination_id' is an integer Route Destination ID. It is required. 'upload_id' is a string the unique and randomly generated ID of the file attachment that is associated with this note. It is optional. 'ts_added' is an integer the unix timestamp when the note was added. It is optional. 'lat' is a number Latitude. It has a minimum value of -90 and a maximum value of 90. It is optional. 'lng' is a number Longitude. It has a minimum value of -180 and a maximum value of 180. It is optional. 'activity_type' is a string Activity Type. It is optional. 'contents' is a string Contents. It is optional. 'upload_type' is a string Upload Type. It has an enum of allowed values: 'DRIVER_IMG', 'VEHICLE_IMG', 'ADDRESS_IMG', 'CSV_FILE', 'XLS_FILE', 'ANY_FILE'. It is optional. 'upload_url' is a string the direct CDN URL of the attachment uploaded with a note. It is optional. 'upload_extension' is a string the extension of the attachment ('pdf', 'csv' etc). It is optional. 'device_type' is a string Device type ('web', 'phone', 'ipad', 'android phone', 'android tablet' etc). It is optional. No additional properties are allowed at the root level.
