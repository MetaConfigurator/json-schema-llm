The schema describes the initiation of the background execution of a specific export. It is an object type. The main properties of the schema are: 'export_files' is an array of objects representing export file details. Each item in the array is required and must include 'id', 'href', 'export_id', and 'status'. 'id' is an integer, describing the unique identifier of this export file. 'href' is a string, describing the hypertext reference to this resource, matching the pattern '/api/v1/exports/\d+/export_files/\d+'. 'export_id' is an integer, describing the unique identifier of the associated export. 'status' is a string, describing the current status of this export file, with allowed values: 'Initializing', 'Queued', 'In Progress', 'Available', 'Success With Errors', 'Empty List', 'Failed'. 'href' is a string, describing the hypertext reference to this resource, matching the pattern '/api/v1/user_identities/\d+/export_files'. No additional properties are allowed at the root level.
