The schema describes a collection of network rules. It is an array type. It defines several reusable components under 'definitions': 'IcmpRule' is an object, it requires fields 'ruleid' (a string), 'protocol' (a string with a constant value of 'icmp'), 'icmpcode' (an integer within the range of -1 to 255), 'icmptype' (an integer within the range of -1 to 255), and 'cidr' (a string). 'TcpUdpRule' is an object, it requires fields 'ruleid' (a string), 'protocol' (a string that can be either 'tcp' or 'udp'), 'startport' (a TcpUdpPort), 'endport' (a TcpUdpPort), and 'cidr' (a string). 'RulesArray' is an array, it allows items that can either be a 'TcpUdpRule' or an 'IcmpRule', and it requires that each item has a unique 'ruleid'. The main properties of the schema are: 'id' is a string, required, with a minimum length of 1; 'name' is a string, required, with a minimum length of 1; 'description' is a string, required; 'preselected' is a boolean, required; 'ingressrule' is a RulesArray, required; 'egressrule' is a RulesArray, required. No additional properties are allowed at the root level.
