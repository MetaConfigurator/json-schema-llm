The schema describes the result of a Github worker. It is an object type. It defines several reusable components under 'definitions': 'github_extracted_details' is an object, detailing the inspection of Github with properties for 'forks_count', 'last_year_commits', 'open_issues_count', 'stargazers_count', 'subscribers_count', 'contributors_count', 'topics', and 'license'. 'last_year_commits' is a reference to another definition, which includes 'sum' and 'weekly'. 'github_last_year_commits_details' is an object, containing required properties 'sum' as an integer and 'weekly' as an array of integers. The main properties of the schema are: 'schema' is an object, containing required properties 'name' as a string matching the pattern '^[a-zA-Z0-9_]+$' and 'version' as a string matching the pattern '^[0-9]+-[0-9]+-[0-9]+$'; 'status' is a string, required, with allowed values 'success', 'error', and 'unknown'; 'details' is a reference to 'github_extracted_details'; 'summary' is an array of strings, required. No additional properties are allowed at the root level.
