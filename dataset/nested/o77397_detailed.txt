The schema describes a Tlog message. It is an object type. The main properties of the schema are: 'ver' is a string Message format version, required. It must match the pattern '^[0-9]+(\.[0-9]+)?$'. 'host' is a string Name of the host recording was made on, required. 'rec' is a string ID of the recording on the host, required. 'user' is a string Recorded user name, required. 'term' is a string Terminal type name, required. 'session' is an integer Audit session ID, required. It must be at least 1. 'id' is an integer ID of the message within the recording, required. It must be at least 1. 'pos' is an integer Message position in the recording, milliseconds, required. It must be at least 0. 'timing' is a string Distribution of this message's events in time, required. 'in_txt' is a string Input text with invalid characters scrubbed, optional. 'in_bin' is an array Scrubbed invalid input characters as bytes, optional. The items must be integers between 0 and 255. 'out_txt' is a string Output text with invalid characters scrubbed, optional. 'out_bin' is an array Scrubbed invalid output characters as bytes, optional. The items must be integers between 0 and 255. No additional properties are allowed at the root level.
