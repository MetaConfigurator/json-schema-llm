The schema describes a set of rules configuration options for CFLint. It is an object type. It defines several reusable components under 'definitions': 'pluginMessage' is an object, representing a message associated with a rule. It has properties: 'code', which is a string describing the rule or type of issue; 'messageText', which is a string representing the message text for the issue; and 'severity', which is a string with allowed values: 'FATAL', 'CRITICAL', 'ERROR', 'WARNING', 'CAUTION', 'INFO', 'COSMETIC'. The main properties of the schema are: 'rule' is an array, allowing you to add a plugin for this folder that is not listed in the global configuration; it is optional and has a default value of an empty array, containing objects with properties: 'name', a string; 'className', a string; 'message', an array of 'pluginMessage'; and 'parameter', an array of objects with properties: 'name', a string, and 'value', which can be any type. 'excludes' is an array, describing rules to be excluded, is optional, has a default value of an empty array, and contains items of 'pluginMessage'. 'includes' is an array, describing rules to be applied, is optional, has a default value of an empty array, and contains items of 'pluginMessage'. 'inheritParent' is a boolean, configuring inheritance of global rules; it is optional and has a default value of true. 'parameters' is an object, where the keys denote RuleName.parameterName, containing pattern properties to override specific rule parameters as strings, with no additional properties allowed at the root level. No additional properties are allowed at the root level.
