The schema describes a structured event. It is an object type. The main properties of the schema are: 'meta' is an object containing metadata about the event. It is required and has the following properties: 'id' is a string that must match the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$', 'type' is a string that can only be 'EiffelAnnouncementPublishedEvent', 'version' is a string that can only be '1.1.0' and has a default value of '1.1.0', 'time' is an integer. The 'source' property is an object which has 'domainId', 'host', 'name', and 'uri' as strings, and a 'serializer' object that has required properties: 'groupId', 'artifactId', and 'version', with no additional properties allowed. The 'security' property is also an object that contains 'sdm', which must include 'authorIdentity' and 'encryptedDigest' as strings, with no additional properties allowed. 'data' is an object that contains 'heading' as a string, 'body' as a string, 'uri' as a string, 'severity' as a string that can only be one of 'MINOR', 'MAJOR', 'CRITICAL', 'BLOCKER', 'CLOSED', 'CANCELED', and a 'customData' array of objects with required properties 'key' and 'value', with no additional properties allowed. 'links' is an array of objects containing required properties 'type' as a string and 'target' as a string that must match the same UUID pattern as 'id'. No additional properties are allowed at the root level.
