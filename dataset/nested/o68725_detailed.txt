The schema describes detailed information about a specific task. It is an object type. The main properties of the schema are: 'metadata' is an object containing arbitrary keys and is optional, allowing additional properties; 'files' is an array of objects, each representing a file associated with the task and is required. Each file object has properties: 'filename' is a string representing the file name plus extension and is required; 'sha256' is a string representing the SHA256 hash of the file contents and is optional; 'size_bytes' is a number representing the file size in bytes and is optional; 'uri' is an object providing relative URI information for accessing the task contents and is required. The 'uri' object contains 'path' which is a string representing a relative URI and is required, and 'params' which is an object containing a required 'environment' string parameter, allowing additional properties. No additional properties are allowed at the root level.
