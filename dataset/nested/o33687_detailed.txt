The schema describes configuration settings for an application. It is an object type. The main properties of the schema are: 'debug' is a boolean required property that indicates whether debugging is enabled. 'router-config' is a string required property representing the router configuration. 'cache-dir' is a string optional property for the cache directory. 'logger-dir' is a string required property indicating the directory for logs. 'template-dir' is a string optional property specifying the directory for templates. 'template-file-extension' is a string optional property for the template file extension. 'serializer-model-dir' is a string required property representing the directory for serializer models. 'schema-dir' is a string required property indicating the directory for schemas. 'support-contact' is a string required property for support contact information. 'database' is an object optional property that contains database connection settings. It has the properties: 'host' as a string, 'port' as a number, 'database' as a string, 'user' as a string, and 'password' as a string. 'smtp' is an object optional property that contains SMTP settings. It has the properties: 'host' as a string, 'port' as a number, 'user' as a string, 'password' as a string, and 'security' as a string. No additional properties are allowed at the root level.
