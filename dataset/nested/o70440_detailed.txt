The schema describes a Wheresocksgo API. It is an object type. It defines several reusable components under 'definitions': 'body' is a string, describing the body of the sock and marked as readOnly. 'id' is a string, representing a unique identifier of the sock in UUID format and marked as readOnly. 'identity' is an object, which requires the id definition. 'latitude' is a number, representing the latitude of sock coordinates and marked as readOnly. 'longitude' is a number, representing the longitude of sock coordinates and marked as readOnly. 'title' is a string, representing the title of the sock and marked as readOnly. The main properties of the schema are: 'sock' is a sock type, resolving to the referenced definition. No additional properties are allowed at the root level.
