The schema describes the result of Github worker. It is an object type. It defines several reusable components under 'definitions': 'github_extracted_details' is an object, details of Github inspection with optional properties: 'forks_count' (integer), 'last_year_commits' (reference to 'github_last_year_commits_details'), 'open_issues_count' (integer), 'stargazers_count' (integer), 'subscribers_count' (integer), 'updated_issues' (reference to 'github_issues_details'), 'updated_pull_requests' (reference to 'github_pull_requests_details'), and 'contributors_count' (integer). No additional properties are allowed. 'github_issue&prs_with_time_duration' is an object, details of Github issues + prs yearly or monthly or any given date-range with required properties: 'opened' (integer) and 'closed' (integer). No additional properties are allowed. 'github_issues_details' is an object, details of updated Github issues with optional properties: 'year' (reference to 'github_issue&prs_with_time_duration') and 'month' (reference to 'github_issue&prs_with_time_duration'). No additional properties are allowed. 'github_last_year_commits_details' is an object, details of last year Github commits with required properties: 'sum' (integer) and 'weekly' (array of integers). No additional properties are allowed. 'github_pull_requests_details' is an object, details of updated Github pull requests with optional properties: 'year' (reference to 'github_issue&prs_with_time_duration') and 'month' (reference to 'github_issue&prs_with_time_duration'). No additional properties are allowed. The main properties of the schema are: 'schema' is an object with required properties: 'name' (string, matching pattern '^[a-zA-Z0-9_]+$') and 'version' (string, matching pattern '^[0-9]+-[0-9]+-[0-9]+$'), and optional property 'url' (string, formatted as a URI). '_release' is a string, unique release id in form of 'ecosystem:package:version'. 'status' is a string, required with enums: 'success', 'error', 'unknown'. 'details' is a reference to 'github_extracted_details'. 'summary' is an array of strings. No additional properties are allowed at the root level.
