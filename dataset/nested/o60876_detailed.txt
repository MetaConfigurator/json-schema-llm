The schema describes an individual version. It is an object type. The main properties of the schema are: 'schema' is a string that specifies the schema associated with this document, required, with allowed value 'avalon-core:version-2.0'. 'type' is a string that indicates the type of document, required, with allowed value 'version'. 'parent' is a string that provides a unique identifier to the parent document, optional. 'name' is a number representing the number of version, required. 'locations' is an array of strings that specifies where on the planet this version can be found, optional. 'data' is an object that contains document metadata, required, with required properties: 'families' (an array of strings with one or more families associated with this version), 'author' (a string for the user logged on to the machine at the time of publish), 'source' (a string representing the original file from which this version was made), and 'time' (a string for ISO formatted, file-system compatible time). No additional properties are allowed at the root level.
