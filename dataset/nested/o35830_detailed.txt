The schema describes the mappings for generating a GlovePIE script. It is an object type. It defines several reusable components under 'definitions': 'simpleOutput' is a string, representing an output assignment to a simple button or key. It must match the pattern '^[A-Za-z][A-Za-z0-9]*(\.[A-Za-z][A-Za-z0-9]*)*$'. 'macroOutput' is a string, representing an output assignment to a keyboard macro. It must match the pattern '^~[A-Za-z][A-Za-z0-9]*([- ][A-Za-z][A-Za-z0-9]*)*$'. 'output' is an oneOf type, which can either resolve to 'simpleOutput' or 'macroOutput'. The main properties of the schema are: 'input' is a string that describes an input prefix applied to all inputs; it is optional. 'alias' is an object that allows arbitrary GlovePIE code to be used as an input, with pattern properties requiring keys to match '^[A-Za-z][A-Za-z0-9]*$' and values being strings. 'shift' is an array of strings, describing inputs that can be used as shift buttons; items must match the pattern '^:?[A-Za-z][A-Za-z0-9]*(\.[A-Za-z][A-Za-z0-9]*)*$' and must be unique. 'axis' is an object describing mappings for analog inputs to virtual joystick axes, with pattern properties requiring keys to match '^:?[A-Za-z][A-Za-z0-9]*(\.[A-Za-z][A-Za-z0-9]*)*$' and values being strings matching the pattern '^-?[A-Za-z][A-Za-z0-9]*(\.[A-Za-z][A-Za-z0-9]*)*$'; additional properties are not allowed. 'button' is an object that maps digital inputs to virtual joystick buttons, and follows pattern properties similar to 'axis'; it has an oneOf constraint for either a simple mapping or a mapping with shift buttons where 'normal' is a required property with a reference to 'output'; additional properties are not allowed. 'throttle' is an object with properties for virtual joystick axis and buttons, requiring at least one of 'axis', 'button', 'forwardButton', or 'reverseButton' to be specified; it also has conditional requirements for forward and reverse inputs as either digital or analog. No additional properties are allowed at the root level.
