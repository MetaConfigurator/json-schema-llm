The schema describes a data structure for defining forms. It is an object type. It defines several reusable components under 'definitions': 'field' is an array, representing a collection of form field definitions, where each item is an object with properties 'name', 'label', 'info', 'placeholder', and 'validation'. Each field object must meet specific validation criteria, including a type that is required and either a 'name' alone or both 'name' and 'label', depending on the chosen type. The allowed field types include enums such as 'header', 'paragraph', 'text', 'textarea', 'richtext', 'link', 'collection', 'checkbox', 'multiselect', 'nested', 'switch', 'image', 'file', 'select', 'radio', 'buttongroup', 'number', 'slider', 'date', 'time', 'datetime', and 'group'. Each type has defined defaults: strings for text-based fields, arrays for collection types, booleans for switches, and objects for file types. The main properties of the schema are: 'label' is a string and is required. 'name' is a string and is required. 'version' is an integer with a default value of 1 and is required. 'info' is a string and is optional. 'fields' is an array that references the 'field' definition. No additional properties are allowed at the root level.
