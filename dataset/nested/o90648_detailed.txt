The schema describes a configuration for a Ceph service. It is an object type. The main properties of the schema are: 'ceph' is an object containing service configuration properties, required fields are 'cpus' and 'mem'. 'cpus' is a number representing CPU shares to allocate to each service instance, with a default value of 1 and a minimum of 0.5. 'mem' is a number representing memory to allocate to each service instance, with a default value of 2048 and a minimum of 1024. 'networking' is an object containing Ceph networking configuration properties, required fields are 'cluster_network' and 'public_network'. 'cluster_network' is a string representing the network where the Ceph nodes are allowed to live, with a default value of '0.0.0.0/0'. 'public_network' is a string representing the network that the Ceph services are exposed on, with a default value of '0.0.0.0/0'. 'external_access' is an optional object enabling access from outside the cluster through Marathon-LB. 'enable' is a boolean indicating whether to create a VIP for external access with a default value of true. 'external_access_port' is a number for external access port communication with a default value of 5000. 'virtual_host' is a string for the external load balancer's Virtual Host URL with a default value of 'ceph.example.org'. 'service' is an object containing DC/OS service configuration properties. 'name' is a string representing the name of this service instance, with a default value of 'ceph'. No additional properties are allowed at the root level.
