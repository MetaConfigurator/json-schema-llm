The schema describes a Price Floors object which is parsed and used to enforce dynamic floors depending on the properties of the json. It is an object type. The main properties of the schema are: 'schema' is an object that defines the schema of the rules. It is required and has a required property 'fields', which is an array that lists fields used for matching criteria of a bid response with a price floor. This array must have at least 1 item, must contain unique items, and does not allow additional items. Each item is a string. 'delimiter' is an optional string that specifies the character used to differentiate fields inside a single rule, with examples including '|', '^', and '~'. 'values' is an object that defines key:value pairs constituting a rule and floor. It is required and must have at least 1 property. Additional properties can only be numbers. 'currency' is an optional string representing the three-digit Currency Code, which must match the pattern '^[a-zA-Z]{3}$' with examples such as 'USD', 'EUR', and 'JPY'. 'modelVersion' is an optional string that denotes the floor modeling name for tracking, with examples like 'Prebid-Floor-Model-1.2'. 'skipRate' is an optional integer representing the skip rate with a minimum of 0 and a maximum of 100. 'default' is an optional number that serves as the default floor to use if no entry matches a bid response, with examples like 0.75. No additional properties are allowed at the root level.
