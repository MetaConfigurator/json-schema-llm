The schema describes an Ebola call centre CouchDB user. It is an object type. It defines several reusable components under 'definitions': 'locations' is an array, containing objects with properties 'name' (a string), 'id' (a string), 'level' (an integer), and 'default' (a boolean). The main properties of the schema are: '_id' is a string and is required. '_rev' is a string and is optional. 'password_scheme' is a string and is required. 'iterations' is an integer and is optional. 'name' is a string and is required. 'roles' is an array of strings and is optional. 'details' is an object containing properties 'fullName' (a string), 'role' (a string), and 'app' (a string). 'type' is a string and is required. 'doc_type' is a string and is required. 'derived_key' is a string and is required. 'salt' is a string and is required. 'locations' refers to the 'locations' definition. No additional properties are allowed at the root level.
