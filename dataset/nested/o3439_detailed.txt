The schema describes configuration for generating classes based on Salesforce SObjects. It is an object type. It defines several reusable components under 'definitions': "AuthConfig" is an object, which contains properties for authentication method, including accessToken, clientId, clientSecret, instanceUrl, oAuthHost, password, and username, all of which are strings. The version property is a number. Properties are optional. "FieldMapping" is an object, which requires properties apiName and propName, both of which are strings. "SObjectConfig" is an object, which requires the apiName property as a string and has optional properties including autoConvertNames (a boolean defaulted to true), className (a string), enforcePicklistValues (an enum with values 'ALL', 'RESTRICTED', and false), fieldMappings (an array of FieldMapping objects), and generatePicklists (a boolean). The main properties of the schema are: "auth" is an object referencing the AuthConfig definition, which is used for authentication. "enforcePicklistValues" is an enum with allowed values 'ALL', 'RESTRICTED', and false. "generatePicklists" is a boolean that is defaulted to false. "outPath" is a string describing the path to generate files. "sObjects" is an array that can contain either strings or SObjectConfig objects. The array items can be one of either referenced SObjectConfig definition or a string. No additional properties are allowed at the root level.
