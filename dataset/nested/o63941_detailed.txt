The schema describes the configuration for a workflow section in a main config file. It is an object type. The main properties of the schema are: 'domain' is a string that represents the SWF domain, required, with a minimum length of 3, a maximum length of 64, and must match the pattern '^[\w\.\- ]+$'. 'decisionTaskList' is a string that denotes the workflow decision tasklist, required, with a minimum length of 3, a maximum length of 64, and must match the pattern '^[\w\.\- ]+$'. 'name' is a string for the workflow name, required, with a minimum length of 3 and a maximum length of 64. 'version' is a string for the workflow version, required, with a minimum length of 1, a maximum length of 32, and must match the pattern '^[\w\.\- ]+$'. 'description' is a string for the workflow description, required, with a minimum length of 1 and a maximum length of 256. 'defaultExecutionStartToCloseTimeout' is an integer for the maximum time from start to close for a workflow, required, with a minimum value of 1 and a maximum value of 31536000. 'defaultTaskStartToCloseTimeout' is an integer for the maximum time from start to close for a task, required, with a minimum value of 1 and a maximum value of 31536000. 'defaultChildPolicy' is a string that indicates how to terminate children if the workflow is terminated, required, with allowed values being 'TERMINATE', 'REQUEST_CANCEL', and 'ABANDON'. No additional properties are allowed at the root level.
