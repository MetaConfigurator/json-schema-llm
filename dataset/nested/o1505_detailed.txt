The schema describes the structure of log data. It is an object type. It defines several reusable components under 'definitions': 'commonProperties' is an object, the common properties for all the logs. It has required fields: 'time', 'resourceId', 'operationName', and 'category'. The properties include 'time' as a string that represents the timestamp (in UTC) of the log, 'resourceId' as a string for the resource Id, 'operationName' as a string for the operation name, 'category' as a string for the log category, along with several optional fields such as 'operationVersion', 'resultType', 'resultSignature', 'resultDescription', 'durationMs', 'callerIpAddress', 'correlationId', and 'level'. The property 'durationMs' is a number with a minimum value of 0. The main properties of the schema are: 'commonProperties' is an object that resolves to the referenced definition 'commonProperties'. The root schema is required to have 'commonProperties'. No additional properties are allowed at the root level.
