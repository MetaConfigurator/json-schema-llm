The schema describes a resource metadata structure. It is an object type. The main properties of the schema are: 'title' is a string and required, 'contributors' is an array of strings and required, 'sources' is an array of objects and required, 'resources' is an array of objects and required, 'description' is a string or null, 'scraper_required' is a boolean and required, 'language' is a string that must match the pattern '(\w{2})' and required, 'extension' is a string that must be one of the following: '.xls', '.xlsx', '.csv', '.pdf' and required. The 'sources' array items must have a 'name' field which is a string and format as a URI. The 'resources' array items must have 'title', 'publication_date', and 'schema' as required fields. The 'publication_date' must either be a string, or null, and must satisfy the format '(\d{2}-(\d{2})-(\d{4}))'. The 'schema' property, which is an object, must contain 'fields' which is an array of objects that require a 'name' field. The 'maps_to' property in 'fields' can be a null, string, or an array, and it can only take specific enumerated values including null, 'funding_period_number', 'project_id', and many others. No additional properties are allowed at the root level.
