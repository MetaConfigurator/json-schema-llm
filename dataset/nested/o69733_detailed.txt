The schema describes the structure of a user configuration object. It is an object type. The main properties of the schema are: 'guid' is a string required field that uniquely identifies the user. 'schemaVersion' is an integer optional field representing the version of the schema. 'userIDs' is an array of objects that is a required field, where each object must contain 'uid' and 'domain', both of which are strings and required. 'lastUpdate' is a string required field indicating the last update timestamp. 'timeout' is a string required field specifying the timeout duration. 'publicKey' is a string required field for the user's public key. 'salt' is a string required field used for encryption. 'active' is an integer required field representing the user's active status. 'revoked' is an integer required field indicating if the user is revoked. 'defaults' is an object required field containing properties: 'voice', 'chat', and 'video', all of which are strings and required. 'legacyIDs' is an array of objects optional field where each object must contain 'type', 'category', and 'id', all of which are strings and required. No additional properties are allowed at the root level.
