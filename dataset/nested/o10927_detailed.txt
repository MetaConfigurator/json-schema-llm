The schema describes a media file information structure. It is an object type. The main properties of the schema are: 'created' is a string that is required, 'numFiles' is an integer that is required, 'files' is an array of objects that is required, 'infoSize' is an integer that is required, 'args' is an object that is required, and 'mkvInfoVersion' is a string that is required. Each object in 'files' has 'filePath' as a string, 'dirName' as a string, 'fileSize' as an integer, 'head' as an object (which requires 'Doc type', 'Doc type version', and 'Doc type read version'), 'segInfo' as an object (which requires 'Segment information', 'Segment UID', 'Muxing application', 'Writing application', 'Timecode scale', and 'Duration'), and 'segTracks' as an array of objects (each requiring 'number', 'uid', 'type', and 'codecId'). No additional properties are allowed at the root level.
