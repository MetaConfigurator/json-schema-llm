The schema describes content restrictions of an ANS object. It is an object type. The main properties of the schema are: 'content_code' is a string that represents the content restriction code/level/flag associated with the ANS object. 'embargo' is an object that contains the embargo configuration to enforce publishing restrictions. It has a required field 'active', which is a boolean flag indicating if the embargo is active or not. If this field is false, the embargo should be ignored. It also includes an optional 'end_time' string field in date-time format, which indicates when the embargo ends, and an optional 'description' string field for the embargo. 'geo' is an object that contains the geo-restriction configuration, including a required field 'restrictions', which is an array limited to a size of 1 containing geo-restriction objects. Each item in the array is an object specifying the _id of the restriction associated with this content, which has a required 'restriction_id' string field that represents the restriction stored in Global Settings. No additional properties are allowed at the root level.
