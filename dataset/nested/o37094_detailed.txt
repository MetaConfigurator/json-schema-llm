The schema describes an MCDA performance table entry for relative data. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'criterion' is a string required field that describes the performance criterion. 'dataSource' is a string required field that denotes the source of the data. 'performance' is an object required field that contains detailed performance metrics. It has a required 'distribution' property: 'distribution' is an object that has required properties 'type' and 'parameters'. 'type' is a string with enum values: 'relative-identity-normal', 'relative-logit-normal', 'relative-log-normal', 'relative-cloglog-normal', 'relative-smd-normal', 'relative-normal', 'relative-survival'. 'parameters' is an object with required properties 'baseline' and 'relative'. 'baseline' is an object that has required fields 'type' and 'name', with additional properties disallowed. 'type' in 'baseline' has enum values: 'dnorm', 'dsurv', 'dbeta-logit', 'dbeta-cloglog', 'dt', 'identity', 'logit', 'log', 'cloglog', 'smd'. Additional properties in 'baseline' include optional 'mu', 'sigma', 'distribution', 'title', 'property', 'alpha', 'beta', 'summaryMeasure', 'survivalAtTime', 'scale', 'std.err', 'stdErr', 'mean', and 'dof', with respective enum values for some. 'relative' is an object with required properties 'mu', 'cov', and 'type', where 'type' is a string with enum value 'dmnorm'. 'mu' is an object with additional properties as numbers. 'cov' is an object with required properties: 'colnames', 'rownames', and 'data'. 'colnames' and 'rownames' are arrays of strings, while 'data' is a 2D array of numbers. No additional properties are allowed at the root level.
