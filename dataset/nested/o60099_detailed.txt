The schema describes a collection of browser statements. It is an object type. It defines several reusable components under 'definitions': 'browser_statement' is an object, it requires 'name' and 'releases'. The 'name' field is a string that describes the browser name, and 'releases' is an object that can contain additional properties that must conform to the 'release_statement' definition. 'release_statement' is an object, it requires 'engine', 'engine_version', 'release_date', 'release_notes', and 'status'. The 'engine' field is a string that can be one of the following values: 'Blink', 'EdgeHTML', 'Gecko', 'Presto', 'Trident', 'WebKit', 'V8'. The 'release_notes' field is a string formatted as a URI. The 'status' field is a string that can be one of the following values: 'retired', 'current', 'exclusive', 'beta', 'nightly', 'esr', 'planned'. The main properties of the schema are: 'browsers' is an object referencing the 'browsers' definition. No additional properties are allowed at the root level.
