The schema describes a device configuration. It is an object type. It defines several reusable components under 'definitions': 'tags' is an array, containing objects that have a required string property 'key' with a pattern '^[0-9a-zA-Z_-]{1,255}$' and a required string property 'value' with a minLength of 1 and a maxLength of 255. 'attributes' is an array, containing objects that have a required string property 'name' with a pattern '^[0-9a-zA-Z_-]{1,255}$' and a required string property 'dataType' with an enum of allowed values: 'string', 'number', 'gps', 'boolean'. 'deviceClass' is a string with an enum of allowed values: 'standalone', 'master', 'owned', 'floating', 'virtual'. 'connectionInfo' is an object with properties 'time' as a string in date-time format and 'connected' as a number. The main properties of the schema are: 'id' is a string with a pattern '^[A-Fa-f\d]{24}$', required; 'deviceId' is a string with a pattern '^[A-Fa-f\d]{24}$', optional; 'applicationId' is a string with a pattern '^[A-Fa-f\d]{24}$', optional; 'creationDate' is a string in date-time format, optional; 'lastUpdated' is a string in date-time format, optional; 'name' is a string with a minLength of 1 and a maxLength of 255, optional; 'description' is a string with a maxLength of 32767, optional; 'tags' is an array as defined in 'tags', optional; 'attributes' is an array as defined in 'attributes', optional; 'deviceClass' is a string as defined in 'deviceClass', optional; 'masterId' is a string with a pattern '^[A-Fa-f\d]{24}$', optional; 'connectionInfo' is an object as defined in 'connectionInfo', optional. No additional properties are allowed at the root level.
