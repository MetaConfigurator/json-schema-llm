The schema describes a layer for displaying raster data. It is an object type. The main properties of the schema are: 'copyright' is a string attribution to the Raster Data Layer provider. It is displayed in the attribution on the scene. Input required by the user when the layer is added to the scene. 'id' is a string a unique identifying string for the layer. 'layerType' is a string string indicating the layer type, allowed values are: 'RasterDataLayer'. 'listMode' is a string to show or hide the sublayer in the layer list, allowed values are: 'hide', 'show'. 'maxScale' is a number a number representing the maximum scale at which the layer will be visible, with a minimum value of 0. 'minScale' is a number a number representing the minimum scale at which the layer will be visible, with a minimum value of 0. 'opacity' is a number the degree of transparency applied to the layer on the client side, with a minimum value of 0, a maximum value of 1, and a default value of 1. 'path' is a string for offline data, a path to a raster data file in URI format beginning with 'file:', with a pattern of '^file:.+$'. 'title' is a string a user-friendly string title for the layer that can be used in a table of contents. 'visibility' is a boolean Boolean property determining whether the layer is initially visible in the scene, with a default value of true. 'copyright', 'id', 'layerType', 'path', and 'title' are required fields. No additional properties are allowed at the root level.
