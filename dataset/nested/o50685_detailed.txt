The schema describes resource models for api-pls. It is an object type. The main properties of the schema are: 'name' is a string required and it represents the name of the resource model. 'plural_form' is a string optional and it represents the plural form of the resource model. 'actions' is an object optional and it defines the actions available for the resource model, with properties 'create', 'read_one', 'read_many', 'update', and 'delete', each of which is a boolean. 'pagination' is either an object or a boolean; if it is an object, it defines pagination settings with optional properties 'default_page_size' (integer, minimum 1), 'default_page_number' (integer, minimum 0), and 'enabled' (boolean). 'attributes' is an object optional, allowing for additional attributes with a pattern where keys can be any string and values can either be a string or an object, with that object having optional properties 'type' (string), 'nullable' (boolean), and 'default' (string). 'meta' is an object optional, similar to 'attributes' with the same pattern. 'relationships' is an object optional with a pattern for keys and allows values that are either a string (one of 'many-to-one', 'one-to-many', 'one-to-one', 'many-to-many') or an object that describes the relationship with properties 'resource' (string), 'cardinality' (a string one of 'many-to-one', 'one-to-many', 'one-to-one', 'many-to-many'), and 'host' (boolean). 'built_in_meta' is an object optional with additional properties 'created_at' (boolean) and 'updated_at' (boolean). No additional properties are allowed at the root level.
