The schema describes a structure for RESTful Open Annotation in JSON-LD. It is an object type. It defines several reusable components under 'definitions': 'context' is an object, used to define the short-hand names that are used throughout a JSON-LD document. It allows additional properties. 'graph' is an object, used to express a graph. It allows additional properties. Its main property '@graph' is an array or object, allowing additional items that must conform to the 'common' and 'oa' definitions. 'oa' is an object, representing the annotation details. It disallows additional properties and requires the 'target' property. It contains properties: 'body' is a string with a URI format, 'target' is a string with a URI format, 'annotatedBy' is a string with a URI format, 'serializedBy' is a string with a URI format, 'serializedAt' is a string with a date-time format, and 'annotatedAt' is a string with a date-time format. 'common' is an object, allowing additional properties that may conform to 'common'. It includes the properties: '@id' is a string with a URI format, '@value' is a string, boolean, number, or null, '@language' is a string or null, '@type' is a string or null, restricted to the enum value '@id', '@container' is a string or null, restricted to the enum values '@list', '@index', or '@set', '@list' is an object, '@set' is an object, '@reverse' is a string, object, or null, allowing additional properties that may conform to 'common', '@base' is a string or null with a URI format, and '@vocab' is a string or null with a URI format. The main properties of the schema are: There are no direct properties defined at the root level. The root schema resolves to the referenced definition. The referenced definition is 'context', 'graph', or 'common', depending on the valid configuration. No additional properties are allowed at the root level.
