The schema describes the current state of a host. It is an object type. It defines several reusable components under 'definitions': 'Instance' is an object, representing an instance which has required fields 'name' and 'data'. The 'data' field is also an object that requires 'family' and may include 'compatibleInstances', which is an array of strings. 'Plugin' is an object, representing a plug-in which has required fields 'name' and 'data'. The 'data' field is an object that requires 'families' and may include properties like 'canProcessContext' (a boolean), 'canProcessInstance' (a boolean), and others. The main properties of the schema are: 'context' is an object representing the context which includes a required 'data' field and an optional 'children' field that holds an array of 'Instance' references. 'plugins' is an array of 'Plugin' references, and both 'context' and 'plugins' are required fields. No additional properties are allowed at the root level.
