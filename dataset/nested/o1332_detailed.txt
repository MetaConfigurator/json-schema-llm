The schema describes creating a new generic component definition in the given or default project. It is an object type. The main properties of the schema are: 'changeDetection' is a string that defines the change detection strategy to use in the new component. It is optional and defaults to 'Default'; allowed values are 'Default', 'OnPush'. 'entryComponent' is a boolean indicating if the new component is the entry component of the declaring NgModule. It is optional and defaults to false. 'export' is a boolean that indicates if the declaring NgModule exports this component. It is optional and defaults to false. 'flat' is a boolean that specifies if the new files are created at the top level of the current project. It is optional and defaults to false. 'inlineTemplate' is a boolean indicating if the template is included inline in the component.ts file. It is optional and defaults to false. 'lintFix' is a boolean that specifies if lint fixes are applied after generating the component. It is optional and defaults to false. 'module' is a string representing the declaring NgModule. It is optional. 'name' is a string that specifies the name of the component. It is required. 'path' is a string describing the path at which to create the component file, relative to the current workspace. It is optional. 'prefix' is a string that applies to the generated component selector, with a maximum length of 0 or must match an HTML selector format with a minimum length of 1. It is optional. 'project' is a string that specifies the name of the project. It is optional. 'selector' is a string that indicates the HTML selector to use for this component. It is optional. 'skipImport' is a boolean indicating if this component should not be imported into the owning NgModule. It is optional and defaults to false. 'skipTests' is a boolean that specifies if "spec.ts" test files should not be created for the new component. It is optional and defaults to false. 'spec' is a boolean indicating if a "spec.ts" test file should be generated for the new component. It is optional and defaults to true but is deprecated in favor of 'skipTests'. 'viewEncapsulation' is a string that defines the view encapsulation strategy to use in the new component. It is optional and allowed values are 'Emulated', 'Native', 'None', 'ShadowDom'. No additional properties are allowed at the root level.
