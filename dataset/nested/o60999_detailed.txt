The schema describes CVEcheck worker results. It is an object type. It defines several reusable components under 'definitions': 'cve_access' is an object, which describes access requirements with required fields 'authentication', 'complexity', and 'vector', all of which must be one of the specified enum values: 'authentication' can be 'NONE', 'SINGLE', or 'MULTIPLE'; 'complexity' can be 'LOW', 'MEDIUM', or 'HIGH'; 'vector' can be 'NETWORK', 'ADJACENT_NETWORK', or 'LOCAL'. 'cve_impact' is an object, which describes impact assessment with required fields 'availability', 'confidentiality', and 'integrity', all of which must be one of the specified enum values: 'availability' can be 'NONE', 'PARTIAL', or 'COMPLETE'; 'confidentiality' can be 'NONE', 'PARTIAL', or 'COMPLETE'; 'integrity' can be 'NONE', 'PARTIAL', or 'COMPLETE'. The main properties of the schema are: 'schema' is an object describing the schema's metadata with required fields 'name', which must match the pattern '^[a-zA-Z0-9_]+$', and 'version', which must match the pattern '^[0-9]+-[0-9]+-[0-9]+$'; 'url' is a string describing the full URL of the schema, formatted as a URI; '_release' is a string describing a unique release id; 'status' is a string which must be either 'success' or 'error'; 'details' is an array of objects, each referencing the 'cvecheck_details' definition; 'summary' is an array of strings. No additional properties are allowed at the root level.
