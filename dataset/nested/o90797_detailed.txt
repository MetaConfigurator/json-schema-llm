The schema describes configuration settings for an Nginx application. It is an object type. The main properties of the schema are: 'nginx' is an object that contains detailed settings for the Nginx application. It is required. It defines several properties: 'bridge' is a boolean that indicates whether to use BRIDGE networking mode for the Docker container. By default, this is true. 'configUrl' is a string specifying the URL to the Nginx configuration; it cannot be combined with 'contentUrl' or 'contentDir'. 'contentDir' is a string that specifies the path of the directory, relative to the sandbox, containing HTML code; it cannot be combined with 'configUrl'. 'contentUrl' is a string specifying the URL of the content to host; it cannot be combined with 'configUrl'. 'cpus' is a number representing CPU shares allocated to each Nginx instance, with a minimum of 1 and a default value of 1. 'https-redirect' is a boolean indicating whether Marathon-lb should redirect HTTP traffic to HTTPS, which defaults to false. 'instances' is an integer representing the number of instances to run, with a minimum of 1 and a default value of 1. 'maximumOverCapacity' is a number indicating the maximum over capacity, with a minimum of 0 and a default value of 0.2. 'mem' is a number specifying the memory (in MB) allocated to each Nginx task, with a minimum of 256.0 and a default value of 1024.0. 'minimumHealthCapacity' is a number representing the minimum health capacity, with a minimum of 0 and a default value of 0.5. 'name' is a string for the name of this Nginx application, with a default value of 'nginx'. 'role' is a string allowing deployment of Nginx only on nodes with this role, defaulting to '*'. 'virtual-host' is a string for the virtual host address to configure for integration with Marathon-lb. No additional properties are allowed at the root level.
