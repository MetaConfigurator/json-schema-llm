The schema describes an entity representing a person or identity. It is an object type. The main properties of the schema are: '_id' is a string representing the unique identifier for the person. 'email' is a string representing the email address, which is required to be in email format. 'identityProviders' is an array of objects, each containing 'id' (a string representing the identity provider's identifier) and 'name' (a string representing the identity provider's name). 'name' is a string representing the full name, which is a required field. 'password' is a string representing the password. 'roles' is an array of strings representing the roles assigned to the person. The 'name' and 'email' fields are required. No additional properties are allowed at the root level.
