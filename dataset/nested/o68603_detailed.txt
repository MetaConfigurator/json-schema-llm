The schema describes a list of users. It is an array type. The main properties of the schema are: Each item in the array is an object representing a user with the following properties: 'apiVersion' is a string with an enum of allowed values: 'v1beta1', 'v1.0', 'v1'. This property is required. 'kind' is a string with an enum of allowed values: 'User', 'user'. This property is read-only and required. 'metadata' is an object that contains the following properties: 'name' is a string with a minimum length of 3 and a maximum length of 30. This property is required. 'createdOn' is a string that is read-only. 'modifiedOn' is a string that is read-only. 'ref' is a string that is read-only with a minimum length of 3 and a maximum length of 24. 'spec' is an object that contains the following property: 'credentials' is an object with properties: 'username' is a string that is read-only with a minimum length of 3 and a maximum length of 30. This property is required. 'secret' is a string that is write-only. The 'credentials' property itself is required. No additional properties are allowed at the root level.
