The schema describes an API for managing durations. It is an object type. It defines several reusable components under 'definitions': 'created_at' is a string, describing when the duration was created with a date-time format. 'id' is a string, describing the unique identifier of a duration with a uuid format, marked as readOnly. 'identity' is an anyOf type, which can be either a reference to 'id' or 'name'. 'name' is a string, describing the unique name of a duration, also marked as readOnly. 'updated_at' is a string, describing when the duration was updated with a date-time format.
