The schema describes a content item configuration. It is an object type. It defines several reusable components under 'definitions': 'guid' is a string, representing a valid UUID format specified by the pattern '^[a-f0-9]{8}-[a-f0-9]{4}-[1-5][a-f0-9]{3}-[89ab][a-f0-9]{3}-[a-f0-9]{12}$'. 'guid_list' is an array of 'guid', requiring unique items. The main properties of the schema are: 'bulk_publishing' is an optional boolean that indicates whether bulk publishing is enabled. 'links' is an optional object that cannot contain additional properties, consisting of various optional properties that reference the 'guid_list' definition, including 'facet_groups', 'facet_values', 'field_of_operation' (with maxItems set to 1), 'finder', 'lead_organisations', 'mainstream_browse_pages', 'meets_user_needs', 'ordered_related_items', 'ordered_related_items_overrides', 'organisations', 'original_primary_publishing_organisation', 'parent' (with maxItems set to 1), 'people', 'policy_areas', 'primary_publishing_organisation' (with maxItems set to 1), 'roles', 'suggested_ordered_related_items', 'taxons', and 'topics'. 'previous_version' is an optional string. No additional properties are allowed at the root level.
