The schema describes mapping from the Vulnerability Rating Taxonomy to CVSS v3. It is an object type. It defines several reusable components under 'definitions': 'MappingMetadata' is an object, it requires a 'default' field which must conform to the 'CVSSv3' definition. 'VRTid' is a string, it must match the pattern '^[a-z_]*$'. 'CVSSv3' is a string, it must match the pattern '^AV:[NALP]/AC:[LH]/PR:[NLH]/UI:[NR]/S:[UC]/C:[NLH]/I:[NLH]/A:[NLH]$'. 'Mapping' is an object, it requires 'id' and 'cvss_v3' fields which must conform to the 'VRTid' and 'CVSSv3' definitions respectively. No additional properties are allowed at the 'Mapping' level. 'MappingParent' is an object, it requires 'id' and 'children' fields, where 'id' must conform to the 'VRTid' definition. 'children' is an array that can contain items that either conform to the 'MappingParent' or 'Mapping' definitions. No additional properties are allowed at the 'MappingParent' level. The main properties of the schema are: 'metadata' is an object, it is required and must conform to the 'MappingMetadata' definition. 'content' is an array, it is required and can contain items that either conform to the 'MappingParent' or 'Mapping' definitions. No additional properties are allowed at the root level.
