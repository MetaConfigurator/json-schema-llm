The schema describes PX and DC/OS service configuration properties. It is an object type. The main properties of the schema are: 'portworx' is an object PX configuration properties, with required fields: 'kvdb', 'clusterid', 'storage', 'mgmtif', 'dataif', 'headers_dir', 'api_port', and 'instances'. 'portworx' has the following properties: 'api_port' is an integer API access port with a default value of 9001; 'clusterid' is a string PX Cluster ID with a default value of 'my-cluster'; 'cpus' is a number CPU shares to allocate to each Marathon instance with a minimum value of 0.0 and a default value of 1.0; 'dataif' is a string Network interface to use for Data traffic with a default value of 'enp0s3'; 'framework-name' is a string The name of the framework with a default value of 'portworx'; 'headers_dir' is a string Directory location for system headers with a default value of '/usr/src'; 'instances' is an integer Number of PX instances to run with a minimum value of 3 and a default value of 3; 'kvdb' is a string Key Value database to use for PX to store configuration parameters with a default value of 'etcd://etcd.mycompany.com:4001'; 'mem' is a number Memory (MB) to allocate to each Marathon task with a minimum value of 2048.0 and a default value of 2048.0; 'mgmtif' is a string Network interface to use for Mgmt traffic with a default value of 'enp0s3'; 'storage' is a string Local storage devices to use with a default value of '/dev/sdb'. 'service' is an object DC/OS service configuration properties with an optional property 'name' is a string Name of this service instance with a default value of 'portworx'. No additional properties are allowed at the root level.
