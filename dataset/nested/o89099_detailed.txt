The schema describes the configuration file for unit test projects using xUnit.net. It is an object type. The main properties of the schema are: 'appDomain' is a string that determines whether the runner will use an app domain to discover and run tests. It is optional, defaults to 'ifAvailable', and it can be one of the following values: 'required', 'ifAvailable', 'denied'. 'diagnosticMessages' is a boolean that enables or disables diagnostic information during test discovery and execution. It is optional, defaults to false. 'maxParallelThreads' is an integer that configures the maximum number of threads to be used when parallelizing tests within this assembly. It is optional and must be at least 1. 'methodDisplay' is a string that configures the default display name for test cases. It is optional, defaults to 'classAndMethod', and can be either 'method' or 'classAndMethod'. 'parallelizeAssembly' is a boolean that instructs the test runner that this assembly is willing to run in parallel with other assemblies. It is optional, defaults to false. 'parallelizeTestCollections' is a boolean that enables or disables tests inside this assembly from running in parallel against each other. It is optional, defaults to true. 'preEnumerateTheories' is a boolean that enables or disables pre-enumerate of theories so that there is an individual test case for each theory data row. It is optional, defaults to true. 'shadowCopy' is a boolean that enables or disables the use of shadow copying when using app domains. It is optional, defaults to true. No additional properties are allowed at the root level.
