The schema describes the configuration for a Humio application. It is an object type. The main properties of the schema are: 'humio' is an object that contains required properties 'host', 'dataspace', and 'ingestToken', all of which are strings that respectively denote the Humio URL, the name of the Humio dataspace, and the Humio ingest token. 'node' is an object that contains required properties 'cpus', 'mem', and 'datadir'. 'cpus' is a number with a minimum value of 0.1 and a default of 0.5, representing CPU shares allocated to each Filebeat instance. 'mem' is a number with a minimum value of 256.0 and a default of 512.0, indicating memory (MB) allocated to each Filebeat task. 'datadir' is a string with a default value of '/var/humio/data', specifying the path of the directory. 'service' is an object that contains a required property 'name', which is a string with a default of 'humio-agent', representing the name for this Humio application. No additional properties are allowed at the root level.
