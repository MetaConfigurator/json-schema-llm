The schema describes a Yunohost application manifest. It is an object type. It defines several reusable components under 'definitions': 'arguments' is an object, which requires an 'install' array where each item references the 'install_item' definition. 'boolean_string' is a string, which can only be either 'true' or 'false'. 'description' is an object, with additional properties allowed and requires an 'en' string. 'email' is a string that conforms to the email format. 'install_item' is an object, which requires 'name' and 'ask', allowing additional properties and containing 'choices', 'default', 'example', 'optional', and 'pattern'. 'install_item_ask' is an object, which requires an 'en' string and allows additional properties. 'install_item_choice' is an array of strings. 'install_item_type' is a string, which can be one of 'domain', 'user', 'app', 'password', or 'path'. 'maintainer' is an object, which requires 'name' and 'email', allowing no additional properties. 'multi_instance' references the 'boolean_string' definition. 'uri' is a string that conforms to the URI format. The main properties of the schema are: 'arguments' is an arguments type and is required; 'description' is a description type and is required; 'id' is a string and is required; 'maintainer' is a maintainer type and is required; 'multi_instance' references the boolean_string type and is required; 'name' is a string and is required; 'url' references the uri type and is optional. No additional properties are allowed at the root level.
