The schema describes an entity returned after successfully authenticating. It is an object type. The main properties of the schema are: 'expires' is a string that describes the expiration time of the token. 'token' is a string that describes a JSON Web Token that is valid for 15 minutes. 'user' is an object that describes the user profile of the authenticated user. The properties of 'user' are: '_id' is a string that is a unique identifier for a user with a pattern of '^[a-z0-9_-]{7,}$' and is readOnly. '_plan' is a string that describes the user's current subscription plan. 'created_at' is a string that is a timestamp when the user was created and is readOnly. 'email' is a string that describes the email address of the user, formatted as an email, and is unique. 'gravatar_id' is a string that describes the Gravatar ID, with a pattern of '^[a-f0-9]{32}$' and is readOnly. 'is_active' is a boolean that indicates whether the user is able to login, set to false to ban the user. 'name' is a string that describes the display name, initially set to the user name, with a maxLength of 30 and a minLength of 3. 'permissions' is an object that defines the set of permissions for that user and is readOnly. 'provider' is a string that describes how the user is authenticated, with enum values of 'local' and 'github', and is readOnly. 'roles' is an array that lists the roles the user belongs to, with enum values of 'root', 'admin', 'contributor', and 'member'. 'username' is a string that describes the username used when locally logging in, with a maxLength of 30 and a minLength of 3, and is unique. No additional properties are allowed at the root level.
