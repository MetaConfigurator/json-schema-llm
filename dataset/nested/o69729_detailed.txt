The schema describes a wallet data structure. It is an object type. It defines several reusable components under 'definitions': 'Transaction' is an object, representing a financial transaction with required fields 'recipient', 'source', 'date', 'value', and 'nonce'. The 'date' field must be a string formatted as 'date-time', and 'nonce' and 'value' must be integers. The main properties of the schema are: 'address' is a string and is required. 'balance' is an integer and is required. 'bonus-credit' is an integer and is required. 'created' is a string formatted as 'date-time' and is required. 'identity' is an object and is required. 'ranking' is an integer and is required. 'scheme' is a constant with the value 'walletData' and is required. 'status' is a string with an enum of allowed values 'active' and 'deleted' and is required. 'transactions' is an array of items that reference the 'Transaction' definition and is required. No additional properties are allowed at the root level.
