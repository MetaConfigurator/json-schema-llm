The schema describes a relationship of control between two companies. It is an object type. It defines several reusable components under 'definitions': 'company' is an object, a company which is the subject of a statement. It requires the fields 'name' and 'jurisdiction'. The 'subsidiary-relationship-data' is an object, containing properties that include 'data_type' (which is an enum with the value 'subsidiary_relationship'), 'properties' (which is an object requiring the field 'subsidiary') that details control characteristics of the subsidiary, including a required 'subsidiary' property that is itself an object requiring the field 'name'. The main properties of the schema are: 'company' is a reference to the 'company' definition, required. 'data' is an array of items, each referencing the 'subsidiary-relationship-data' definition, required. 'end_date' is a string representing a date, optional. 'end_date_type' is a string, optional. 'retrieved_at' is a string representing a date, optional. 'sample_date' is a string representing a date, required. 'source_jurisdiction' is a string, optional. 'start_date' is a string representing a date, optional. 'start_date_type' is a string, optional. No additional properties are allowed at the root level.
