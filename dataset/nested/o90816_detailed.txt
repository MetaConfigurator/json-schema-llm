The schema describes the configuration properties for a PostgreSQL admin service. It is an object type. The main properties of the schema are: 'networking' is an object PostgreSQL admin networking configuration properties, which is optional. It contains the properties: 'external_access' is an object Enable access from outside the cluster through Marathon-LB. NOTE: this connection is unencrypted, which is optional. It has the properties: 'enable' is a boolean Enable or disable creating a VIP for external access through a public node running Marathon-LB. Defaults to false., which defaults to false; 'external_port' is a number For external access, port number to be used for clear communication in the external Marathon-LB load balancer, which defaults to 15433; 'virtual_host' is a string For external access, Virtual Host URL to be used in the external load balancer., which defaults to 'postgresql.example.org'. 'postgres_host_port' is a number The port where the postgresql instance is listening on. Defaults to 5432, which defaults to 5432; 'postgres_location' is a string The name of the DC/OS postgresql instance to connect to., which defaults to 'postgresql'. 'postgresql-admin' is an object postgresql admin service configuration properties, which is required. It contains the properties: 'cpus' is a number CPU shares to allocate to each postgresql node., which is required and has a minimum value of 0.2 and defaults to 0.2; 'defaultdb' is a string Default database. Defaults to 'admin'., which defaults to 'admin'; 'mem' is a number Memory to allocate to each postgresql node., which is required and has a minimum value of 256.0 and defaults to 256.0. 'service' is an object DC/OS service configuration properties, which is optional. It contains the property: 'name' is a string Name of this service instance., which defaults to 'postgresql-admin'. No additional properties are allowed at the root level.
