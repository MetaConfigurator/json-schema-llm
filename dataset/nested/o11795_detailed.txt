The schema describes pagination and result information for secure data version summaries. It is an object type. The main properties of the schema are: 'has_next' is a boolean indicating if there are more results available. It is required. 'limit' is an integer representing the maximum number of results to return. It is required. 'offset' is an integer indicating the starting point of the results. It is required. 'version_count_in_result' is an integer counting the versions included in the current results. It is required. 'total_version_count' is an integer representing the total number of versions available. It is required. 'secure_data_version_summaries' is an array of objects containing the summaries of secure data versions. It is required. Each item in the array is an object with the following properties: 'id' is a string and is required. 'sdbox_id' is a string and is required. 'path' is a string and is required. 'action' is a string and is required. 'type' is a string and is required. 'size_in_bytes' is an integer and is required. 'version_created_by' is a string and is required. 'version_created_ts' is a string and is required. 'action_principal' is a string and is required. 'action_ts' is a string and is required. No additional properties are allowed at the root level.
