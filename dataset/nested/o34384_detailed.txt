The schema describes the representation of what running a Dangerfile generates. It is an object type. It defines several reusable components under 'definitions': 'Violation' is an object, the result of user doing warn, message or fail, built this way for expansion later. It has optional fields: 'file' (a string), 'icon' (a string), 'line' (a number), and 'message' (a string). The main properties of the schema are: 'fails' is an array of Violations, optional. 'markdowns' is an array of Violations, optional. 'messages' is an array of Violations, optional. 'meta' is an object with optional properties: 'runtimeHref' (a string) and 'runtimeName' (a string). 'warnings' is an array of Violations, optional. No additional properties are allowed at the root level.
