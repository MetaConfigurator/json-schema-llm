The schema describes a list of OpenTok Archives. It is an object type. It defines several reusable components under 'definitions': 'archive' is an object, an OpenTok Archive. It has required fields: 'id', 'partnerId', 'sessionId', and 'status'. The 'id' is a string with a specific pattern: '^[a-z0-9]{8}-[a-z0-9]{4}-[a-z0-9]{4}-[a-z0-9]{4}-[a-z0-9]{12}$'. The 'partnerId' and 'sessionId' are integers. The 'status' is a string with allowed values: 'available', 'started', 'stopped', 'failed', 'deleted', 'uploaded', 'expired', 'paused'. The main properties of the schema are: 'count' is an integer, required. 'items' is an array, required, with items referenced from the 'archive' definition. No additional properties are allowed at the root level.
