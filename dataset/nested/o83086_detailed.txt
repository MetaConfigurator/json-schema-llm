The schema describes a collection of test cases for JSON schemas. It is an array type. It defines several reusable components under 'definitions': 'outputItem' is an object type, representing the output detailing the validity of the schema tests and can contain the following properties: 'valid' is an optional boolean indicating if the schema is valid; 'keywordLocation' is an optional string representing the location of the keyword; 'absoluteKeywordLocation' is an optional string in URI format; 'instanceLocation' is an optional string representing the location of the instance; 'annotations' is an optional array of items, where each item is an outputItem; 'errors' is an optional array of items, where each item is also an outputItem. The main properties of the schema are: 'description' is a string required field; 'schema' is a required property without a defined type; 'tests' is an array required field that must contain at least one item, where each item is an object with: 'description' which is a string required field; 'data' which has no defined type; 'valid' which is a boolean required field; 'output' which is an object required field containing 'basic', 'detailed', and 'verbose', all required properties that each reference the outputItem definition. No additional properties are allowed at the root level.
