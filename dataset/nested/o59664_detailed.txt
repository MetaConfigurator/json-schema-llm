The schema describes a haxelib project. It is an object type. It defines several reusable components under 'definitions': 'dependancyVersion' is a oneOf type that can either be a reference to the 'semver' definition or a string with a maximum length of 0. 'haxelibTag' is a string, describing a keyword or term associated with a haxelib project, requiring a minimum length of 2 and matching the pattern '^[A-Za-z0-9_.-]{2,}$'. 'projectName' is a string, describing the name of a haxelib project, requiring a minimum length of 3 and matching the pattern '^[A-Za-z0-9_.-]{3,}$'. 'semver' is a string, matching the pattern '^([0-9]+)\.([0-9]+)\.([0-9]+)(-(alpha|beta|rc)(\.([0-9]+))?)?$'. 'userName' is a string, describing the name of a user, requiring a minimum length of 3 and matching the pattern '^[A-Za-z0-9_.-]{3,}$'. The main properties of the schema are: 'classPath' is an optional string describing the folder in the package which contains the source files for this haxelib. 'contributors' is an array of references to the 'userName' definition, requiring a minimum of 1 item and allowing unique items. 'dependencies' is an object with additional properties allowed, each referencing the 'dependancyVersion' definition. 'description' is an optional string. 'license' is a string with an enum allowing the values 'GPL', 'LGPL', 'BSD', 'Public', 'MIT'. 'name' is a reference to the 'projectName' definition and is required. 'releasenote' is a required string. 'tags' is an array of references to the 'haxelibTag' definition, allowing unique items. 'url' is an optional string formatted as a URI. 'version' is a reference to the 'semver' definition and is required. No additional properties are allowed at the root level.
