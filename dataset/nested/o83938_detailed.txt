The schema describes a clinical concept. It is an object type. The main properties of the schema are: '_comments' is a string Optional comments about the clinical concept. 'codeString' is a string The code (ex: LOINC, RxNorm) for this clinical concept. It is required. 'displayBounds' is an object Lower and upper display bounds for this concept. It requires two properties: 'lower' and 'upper', both of which are numbers. 'displayName' is a string The display name for this clinical concept, as returned by the FHIR server. It is required. 'forceDisplayBounds' is a boolean Whether the display bounds should be forced even if the data is outside of them. It defaults to false if unset. 'groupName' is a string If set, this clinical concept will appear grouped with the same groupName. If unset, it will appear independently. 'innerComponentOnly' is a boolean If set, the ResourceCode for this component will be created, but it will not be displayed unless it is an inner component of another Resource. 'showByDefault' is a boolean Whether this clinical concept should be displayed by default. It defaults to false if unset. No additional properties are allowed at the root level.
