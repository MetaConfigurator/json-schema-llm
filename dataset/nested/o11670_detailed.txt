The schema describes configuration for OAuth 2.0 flow. It is an object type. It defines several reusable components under 'definitions': 'implicit' is an object, which includes a required field 'loginEndpoint' that references 'loginEndpoint' definition. It also has an optional field 'tokenName'. 'authorizationCode' is an object, which includes required fields 'tokenEndpoint' and 'tokenRequestEndpoint', both of which reference their respective definitions. The main properties of the schema are: 'implicit' is an object that is optional and references the 'implicit' definition. 'authorization_code' is an object that is optional and references the 'authorizationCode' definition. No additional properties are allowed at the root level.
