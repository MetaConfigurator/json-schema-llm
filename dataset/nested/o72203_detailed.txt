The schema describes user metrics that establish how aggregate functions can be used to update a custom user field with respect to an event. It is an object type. It defines several reusable components under 'definitions': 'singleFieldAggregate' is an object, which requires a fieldName property, and does not allow additional properties. 'fieldName' is a string, describing the field on which this aggregate will be performed. 'filter' is an object, which requires an operator property, and does not allow additional properties. The operator must be one of 'eq', 'gt', 'gte', 'lt', 'lte', 'exists', or 'or'. The main properties of the schema are: 'id' is a string, required, and serves as a unique identifier for this metric. 'customFieldKey' is a string, required, representing the name of the custom field associated with this metric. 'aggregateId' is a string, required, specifying an unique identifier for the aggregate operation to perform, with allowed values: 'SUM', 'COUNT', 'FIRST_SEEN', 'FIRST_VALUE', 'LAST_VALUE', 'THROTTLED_LAST_NUMERIC_VALUE'. 'dateTriggeredWindow' is a string, optional, denoting an ISO 8601 interval for the aggregate operation. 'eventKey' is a string, required, indicating the event that will be aggregated by this metric. 'fieldsFilters' is an array, optional, containing items referenced by the 'filter' definition. 'filter' is a string, optional, representing a JSONata expression to filter out non-matching events. Compositional constraints require that if 'aggregateId' is 'SUM' or 'FIRST_VALUE' or 'LAST_VALUE', the schema must include an 'aggregateRules' object defined by 'singleFieldAggregate'. If 'aggregateId' is 'THROTTLED_LAST_NUMERIC_VALUE', it must include an 'aggregateRules' object with a fieldName and a diff, which is a string describing the throttle interval. No additional properties are allowed at the root level.
