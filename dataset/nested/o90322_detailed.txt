The schema describes a tiled elevation layer used for rendering elevation. It is an object type. The main properties of the schema are: 'id' is a string A unique identifying string for the layer. It is required. 'itemId' is an optional string containing the item ID of the service if it's registered on ArcGIS Online or your organization's portal. 'layerType' is a string indicating the layer type. It is required and can only be 'ArcGISTiledElevationServiceLayer'. 'listMode' is a string to show or hide the elevation layer in the layer list. It is optional and can be either 'hide' or 'show'. 'path' is a string for offline data, specifying a path to a tile package containing elevation data. It must match the pattern '^file:.+\.tpkx?$' and is required only in the second configuration. 'title' is a string providing a user-friendly title for the layer. It is required. 'url' is a string representing the URL to the layer. If the layer is not from a web service, the url property is omitted. It is required only in the first configuration. 'visibility' is a boolean property determining whether the layer is initially visible in the web scene. It defaults to true. The schema requires either an 'id', 'layerType', 'title', and 'url', or an 'id', 'layerType', 'path', and 'title' configuration. No additional properties are allowed at the root level.
