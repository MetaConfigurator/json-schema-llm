The schema describes fishing regulations. It is an object type. It defines several reusable components under 'definitions': The 'species' definition is an object, describing a species with required fields 'bova_id', 'name', and 'taxonomy'. It includes an 'aliases' array, and an 'images' array containing objects with a required 'url' property and optional 'credit' and 'name' properties. The main properties of the schema are: 'agency' is a string, required. 'best_fishing' is an object, optional. Within 'best_fishing', 'description' is a string, optional, and 'lakes' is an object allowing additional properties, each being an object with 'comment' (string), 'gnis_id' (number), 'latitude' (number), and 'longitude' (number). 'rivers' is an object, optional. 'conformsTo' is a string, required. It follows URI format. 'date_effective' is a string, required. It follows date-time format. 'date_expires' is a string, required. It follows date-time format. 'documentation' is a string, required. It follows URI format. 'fishing_techniques' is a string, optional. 'limits' is an object, optional, with an optional 'description' string. 'schema_version' is a string, required. 'species' is an object, required. Within 'species', it requires 'bova_id', 'name', and 'taxonomy'. 'state' is a string, required, with a maximum length of 2, minimum length of 2. It must match the pattern '([A-Z]{2})'. No additional properties are allowed at the root level.
