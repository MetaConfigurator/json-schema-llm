The schema describes configurations for an IntelliSense engine. It is an object type. It defines several reusable components under 'definitions': 'configurations' is an array, which contains objects that are required to have a 'name' property. Each object in the array has the following properties: 'name' is a string, which describes the platform name, with 'Mac', 'Linux', or 'Win32' as the defaults unless a custom platform is added. 'includePath' is an optional array of strings that lists paths for the IntelliSense engine to search for included headers, with non-recursive searching. 'defines' is an optional array of strings that lists preprocessor definitions for the IntelliSense engine. 'intelliSenseMode' is an optional string that must be one of the following values: 'msvc-x64', 'clang-x64', which overrides the default mode used by the IntelliSense engine based on the platform. 'browse' is an optional object with properties: 'limitSymbolsToIncludedHeaders' which is a boolean indicating whether to process only included headers, 'databaseFilename' which is a string representing the path to the generated symbol database, and 'path' which is an optional array of strings for the tag parser's search paths. The main properties of the schema are: 'configurations' is an array that references the 'configurations' definition. The root schema resolves to the referenced definition. No additional properties are allowed at the root level.
