The schema describes an internet population entity. It is an object type. The main properties of the schema are: 'id' is an integer Unique id for each entry. It is optional. 'place' is an integer Position in the top list. It is required and must be between 1 and 64. 'region' is a string Name of the region. It is required, must have a minimum length of 3 characters, a maximum length of 64 characters, and must match the pattern '[A-z]+'. 'population' is an integer Complete number of population. It is required. 'users' is an integer Number of internet users. It is required. 'worldUsers' is a number Percentage users of the world. It is required. 'datetime' is a string Date when the entity was created. It is optional and must be in date-time format. No additional properties are allowed at the root level.
