The schema describes a document field in a form. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: '_schema' is a string that follows a specific pattern, with a default value of 'pbj:gdbots:forms:field:document-field:1-0-0'. 'name' is a string that must match a specific pattern and is required; it serves as a unique identifier for the field within the form. 'maps_to' is a string that defaults to 'cf' and matches a specific pattern, describing the schema field the answer will map to. 'label' is a string with a maximum length of 255 characters, serving as the main text for the question/field. 'placeholder' is a string with a maximum length of 255 characters. 'description' is a string with a maximum length of 65535 characters, providing a short description of the field. 'is_required' is a boolean indicating if the field is required. 'link_text' is a string with a maximum length of 255 characters that will replace the token '{link}' within the label or description. 'link_url' is a string matching a specific URL pattern, used for the replaced token '{link}' within the label or description. 'pii_impact' is a string that can be one of the following values: 'unknown', 'high', 'moderate', or 'low'. 'max_bytes' is an integer that defaults to 0, with a minimum of 0 and a maximum of 4294967295. No additional properties are allowed at the root level.
