The schema describes a journal message structure. It is an object type. It defines several reusable components under 'definitions': 'disallowed' is an object, representing schemas that disallow certain types such as array, boolean, integer, number, null, object, and string. The main properties of the schema are: '$schemaRef' is a string, required. 'header' is an object, required, which contains properties: 'gatewayTimestamp' (optional, string, with date-time format), 'softwareName' (string, required), 'softwareVersion' (string, required), and 'uploaderID' (string, required, must be included). 'message' is an object, required and contains properties: 'ActiveFine' (disallowed), 'BoostUsed' (disallowed), 'CockpitBreach' (disallowed), 'Factions' (optional, array of objects containing disallowed '_Localised$' properties), 'FuelLevel' (disallowed), 'FuelUsed' (disallowed), 'JumpDist' (disallowed), 'Latitude' (disallowed), 'Longitude' (disallowed), 'StarPos' (array of numbers, must have exactly 3 items), 'StarSystem' (string, must be at least 1 character), 'SystemAddress' (integer, optional), 'Wanted' (disallowed), 'event' (enum: 'Docked', 'FSDJump', 'Scan', 'Location', 'SAASignalsFound', 'CarrierJump', required), 'timestamp' (string, with date-time format, required). No additional properties are allowed at the root level.
