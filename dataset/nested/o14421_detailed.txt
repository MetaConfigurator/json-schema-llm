The schema describes a template for a sensor data point. It is an object type. It resolves to the referenced definition 'datos' as the effective root schema. The main properties of the schema are: 'Sensor' is a string, required. No additional properties are allowed at the root level.

The 'datos' definition is an object, defining the structure of the sensor data. It includes required fields: 'timestamp', 'assetId', 'measure', and 'geometry'. The 'timestamp' property is an object, required, with a required field '$date' which is a string in date-time format. The 'assetId' is a string, optional. The 'measure' is a number, optional. The 'sensorMeasure' is a number, optional. The 'correctedMeasure' is a number, optional. The 'geometry' is an object, required, with required fields: 'coordinates' and 'type'. The 'coordinates' is an array with exactly two items, the first item is a number with a minimum of -90 and a maximum of 90, and the second item is a number with a minimum of -180 and a maximum of 180. The 'type' is a string with an enum value of 'Point'. No additional properties are allowed in 'datos'.
