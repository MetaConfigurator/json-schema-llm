The schema describes a collection of experience slugs. It is an object type. The main properties of the schema are: 'items' is an array of objects, where each object represents a single Experience Slug. Each object must contain the following properties: 'id' is a string that must match the pattern '^[A-Fa-f\d]{24}$'; 'experienceSlugId' is a string that must match the pattern '^[A-Fa-f\d]{24}$'; 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$'; 'creationDate' is a string formatted as date-time; 'lastUpdated' is a string formatted as date-time; 'slug' is a string with a minimum length of 4, a maximum length of 63, and must match the pattern '^[0-9a-z-]*$'; 'version' is a string with a minimum length of 1 and a maximum length of 255. 'count' is an integer and 'totalCount' is an integer. 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$'. No additional properties are allowed at the root level.
