The schema describes a collection of books. It is an object type. The main properties of the schema are: 'book' is an array containing objects, each representing a book. Each book object is required to have 'name' (a string), 'id' (a string), and 'contents' (an array). The 'contents' array must contain objects, each required to have 'name' (a string). Each content object can have an optional 'headers' array containing strings and a required 'ordinal' object. The 'ordinal' object must have 'type' (a string) which can be one of the following values: 'chapter', 'appendix', 'part', 'episode', and 'identifier' which can be either an integer or a string. No additional properties are allowed at the root level.
