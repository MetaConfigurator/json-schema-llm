The schema describes taxa objects. It is an object type. The main properties of the schema are: 'bold' is an integer The BOLD identifier of the taxa. 'description' is a string A short description of the taxa. 'eol' is an integer The EOL identifier of the taxa. 'gbif' is an integer The GBIF identifier of the taxa. 'id' is an integer Integer data. Ex: 2673. 'name' is a string The scientific name of the taxa. 'ncbi' is an integer The NCBI Taxonomy identifier of the taxa. 'owner' is a string URI of the taxa owner. When submitting from the R package, this field is populated automatically. 'resource_uri' is a string Unicode string data. Ex: "Hello World". 'status' is a string The taxonomic status of the taxa. It is required. Allowed values are: 'confirmed', 'trophic species', 'morphospecies', 'nomen dubium', 'nomen oblitum', 'nomen nudum', 'nomen novum', 'nomen conservandum', 'species inquirenda'. 'traits' is an array A list of traits values identifiers (or URIs) that were measured on this taxa. Good for taxa-level traits. 'tsn' is an integer The TSN identifier. 'vernacular' is a string The vernacular name of the taxa, in English. No additional properties are allowed at the root level.
