The schema describes configuration settings for a project. It is an object type. It defines several reusable components under 'definitions': 'EditySettings' is an object, describing the settings class with the following properties: 'ReadFromCurrentDirectory' is a boolean, optional, describing whether to read only from the current directory. 'DetailedErrors' is a boolean, optional, describing whether to get detailed errors. 'BaseUrl' is either null or a string, optional, describing the base url for the website, which can be null unless hosted in a subfolder. 'ProjectMode' is a string, required, describing the mode to handle the project's files, restricted to the enumerated values: 'OneRepo', 'OneRepoPerUser'. 'Publisher' is a string, required, describing the publisher to use, restricted to the enumerated values: 'Direct', 'RoundRobin'. 'OutputPath' is either null or a string, optional, describing the output folder for published files. 'ProjectPath' is either null or a string, optional, describing the path to the project. 'EdityCorePath' is either null or a string, optional, describing the path to the edity core client-side files. 'SitePath' is either null or a string, optional, describing the path to this site's client-side files. 'OverrideVars' is either null or an object, optional, describing the override variables for the project, allowing any string key with a string value. No additional properties are allowed at the root level.
