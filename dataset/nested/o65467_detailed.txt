The schema describes a subsidiary relationship. It is an object type. The main properties of the schema are: 'data_type' is a string that is required and must be one of the following values: 'subsidiary_relationship'. 'properties' is an object that is required. It defines several reusable components under 'definitions': 'subsidiary' is an object representing a controlled company, with a required field 'name' among other properties, and does not allow additional properties. The other properties include 'direct' which is a boolean that indicates if the control is direct, 'percentage_controlled' which is a string representing the percentage controlled, and 'significant' which is a boolean indicating if the control is significant. No additional properties are allowed at the root level.
