The schema describes the structure of an event document. It is an object type. The main properties of the schema are: 'meta' is an object containing metadata about the event. It is required and has the following required fields: 'id', which is a string matching the pattern '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'; 'type', which is a string with an enum value of 'EiffelFlowContextDefinedEvent'; 'version', which is a string with an enum value of '1.1.0' and a default of '1.1.0'; and 'time', which is an integer. 'source' is an optional object with properties 'domainId', 'host', 'name', 'serializer', and 'uri', where 'serializer' is an object requiring 'groupId', 'artifactId', and 'version' as strings and does not allow additional properties. 'security' is an optional object with a required 'sdm' property, which contains 'authorIdentity' and 'encryptedDigest' as required strings and does not allow additional properties. 'data' is an object with optional properties: 'product', 'project', 'program', 'track', and 'version', all as strings, and 'customData', which is an array of objects that each must contain a required 'key' string and 'value' of any type and does not allow additional properties. 'links' is an array of required objects that must have 'type' as a string and 'target' as a string matching the specified UUID pattern, and does not allow additional properties. No additional properties are allowed at the root level.
