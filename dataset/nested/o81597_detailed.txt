The schema describes an order for beverages. It is an object type. The main properties of the schema are: 'order' is an object that contains details about the order. It is required and defines the following properties: 'orderId' is an integer that is required. 'customerName' is a string that is required. 'drinkType' is a string that has an enum of allowed values: 'Latte', 'Espresso', 'Cappuccino', 'Chai', 'Tea', 'Steamer', 'Hot Chocolate'. It is required. 'flavor' is a string that has an enum of allowed values: 'Carmel', 'Chocolate', 'Hazelnut', 'Vanilla', 'Peppermint', 'White Chocolate'. It is optional. 'size' is a string that has an enum of allowed values: 'Small', 'Medium', 'Large', 'Extra-Large', 'Bucket'. It is required. 'date' is a string that is required. 'time' is a string that is required. 'milk' is a string that has an enum of allowed values: 'Non-Fat', 'Whole', 'Breve', 'Soy', 'Almond'. It is required. 'espressoConCard' is a string that is required. 'totalPrice' is a number that is required. No additional properties are allowed at the root level.
