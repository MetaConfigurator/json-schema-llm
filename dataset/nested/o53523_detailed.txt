The schema describes a set of rules. It is an object type. It defines several reusable components under 'definitions': The main properties of the schema are: 'url' is a string that is optional. 'forceHTTPS' is a boolean that is optional. 'updater' is an object that is required and must include 'x86'. 'version_extractor' is a string that is required. 'version_source' is a string that is optional. The 'updater' object requires 'x86', which itself is an object that requires 'rule_type'. The 'x86' object can contain 'url', 'rule_type', 'selector', 'baselink', and 'filter', all of which are optional except 'rule_type'. Similarly, 'x86_64' is an object within 'updater', which also requires 'rule_type'. There are no additional properties allowed at the root level.
