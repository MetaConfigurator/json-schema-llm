The schema describes a collection of application certificate authorities. It is an object type. The main properties of the schema are: 'items' is an array containing objects that represent single application certificate authorities, 'count' is an integer representing the number of items, 'totalCount' is an integer indicating the total number of items, 'perPage' is an integer for pagination, 'page' is an integer for the current page, 'filter' is a string for filtering results, 'filterField' is a string for the field to filter on, 'sortField' is a string for the field to sort by, and 'sortDirection' is a string indicating the direction of sorting, which can be one of the following values: 'asc', 'desc', 'ASC', 'DESC', or an empty string. 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$'. Each object in the 'items' array has the following properties: 'id' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'applicationCertificateAuthorityId' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'applicationId' is a string that must match the pattern '^[A-Fa-f\d]{24}$', 'creationDate' is a string in date-time format, 'lastUpdated' is a string in date-time format, 'status' is a string that can be either 'active' or 'inactive', 'name' is a string with a minimum length of 1 and a maximum length of 255, 'description' is a string with a maximum length of 32767, 'caBundle' is a string with a minimum length of 50 and a maximum length of 32767. 'caInfo' is an array of objects, each containing properties: 'serialNumber' (string, minLength of 1, maxLength of 1024), 'fingerprint' (string, minLength of 1, maxLength of 1024), 'commonName' (string, maxLength of 1024), 'issuerName' (string, maxLength of 1024), 'notValidBefore' (string in date-time format), and 'notValidAfter' (string in date-time format). No additional properties are allowed at the root level.
