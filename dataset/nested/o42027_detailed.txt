The schema describes the result of Binwalk worker. It is an object type. It defines several reusable components under 'definitions': 'binwalk_details' is an object, details of Binwalk run on one file. It requires the fields 'path' and 'output', where 'path' is a string and 'output' is an array of strings. No additional properties are allowed in 'binwalk_details'. The main properties of the schema are: 'schema' is an object containing the properties 'name' (required string with pattern '^[a-zA-Z0-9_]+$') and 'version' (required string with pattern '^[0-9]+-[0-9]+-[0-9]+$'), both of which disallow additional properties; '_release' is an optional string describing a unique release id in the form of 'ecosystem:package:version'; 'status' is a string which must be either 'success' or 'error'; 'details' is an array of objects defined by the 'binwalk_details' definition; 'summary' is an array of strings. All properties at the root level are required except for '_release'. No additional properties are allowed at the root level.
