The schema describes loot tables in Minecraft Add-Ons. It is an object type. It defines several reusable components under 'definitions': 'conditions' is an array, which describes types of conditions that cause loot tables to activate. The items in this array are objects with required fields: 'condition' (a string describing the condition that triggers loot), 'default_chance' (a number indicating the chance of loot in default difficulty), 'peaceful' (a number for the chance of loot in peaceful difficulty), and 'hard' (a number for the chance of loot in hard difficulty). 'rolls' is a type that can be either an array or an integer, intended to represent one unknown number or an array of 'min' and 'max'. 'pools' is an array, containing objects with required field 'rolls' (a reference to the 'rolls' definition) and additional properties for 'conditions' (a reference to the 'conditions' definition). The main properties of the schema are: 'pools' is a required property that resolves to the referenced 'pools' definition. No additional properties are allowed at the root level.
