The schema describes postal addresses for Canada, USA and Mexico. It is an object type. It defines several reusable components under 'definitions': 'CAN_province' is a string, allowing enum values 'AB', 'BC', 'MB', 'NB', 'NL', 'NS', 'NT', 'NU', 'ON', 'PE', 'QC', 'SK', 'YT'. 'CAN_postalCode' is a string, validating against the pattern '^[A-Z][0-9][A-Z]( )?[0-9][A-Z][0-9]$'. 'USA_state' is a string, allowing enum values 'AL', 'AK', 'AR', 'AS', 'AZ', 'CA', 'CO', 'CT', 'DC', 'DE', 'FL', 'FM', 'GA', 'GU', 'HI', 'IA', 'ID', 'IL', 'IN', 'KS', 'KY', 'LA', 'MA', 'MD', 'ME', 'MH', 'MI', 'MN', 'MO', 'MP', 'MS', 'MT', 'NC', 'ND', 'NE', 'NH', 'NJ', 'NM', 'NV', 'OH', 'OK', 'OR', 'PA', 'PR', 'PW', 'RI', 'SC', 'SD', 'TN', 'TX', 'UT', 'VA', 'VI', 'VT', 'WA', 'WI', 'WV', 'WY', 'AA', 'AE', 'AP'. 'USA_zipCode' is a string, validating against the pattern '^[0-9]{5}(-[0-9]{4})?$'. 'MEX_state' is a string, allowing enum values 'AGS', 'BC', 'BCS', 'CAM', 'COAH', 'COL', 'CHIH', 'CHIS', 'DF', 'DGO', 'GTO', 'GRO', 'HGO', 'JAL', 'MEX', 'MICH', 'MOR', 'NAY', 'NL', 'OAX', 'PUE', 'QRO', 'Q ROO', 'SLP', 'SIN', 'SON', 'TAB', 'TAMPS', 'TLAX', 'VER', 'YUC', 'ZAC'. 'MEX_postalCode' is a string, validating against the pattern '^[0-9]{5}$'. The main properties of the schema are: 'address' is an object, required with fields: 'name', 'number', 'street', 'city', 'national'. 'name' is a string, required. 'number' is a string, required. 'street' is a string, required. 'city' is a string, required. 'national' is an object, required, which can be one of three configurations based on 'country': for 'CAN', it requires 'country' (enum value 'CAN'), 'province' (refers to 'CAN_province'), and 'postalCode' (refers to 'CAN_postalCode'); for 'USA', it requires 'country' (enum value 'USA'), 'state' (refers to 'USA_state'), and 'zipCode' (refers to 'USA_zipCode'); for 'MEX', it requires 'country' (enum value 'MEX'), 'state' (refers to 'MEX_state'), and 'postalCode' (refers to 'MEX_postalCode'). No additional properties are allowed at the root level.
