The schema describes user parameters for optimizing and customizing system behaviors. It is an object type. It defines several reusable components under 'definitions': 'eyeData' is an object, which includes optional properties: 'pupilDistance', a number describing the distance from the bridge of the nose to the pupil when infinity-focused (in mm); 'dominant', a boolean indicating which eye is dominant; and 'correction', an object for vision correction information containing optional properties: 'distance', an object with properties 'spherical' (a number with default 0 for spherical correction in diopters), 'cylindrical' (a number with default 0 for astigmatism correction in diopters), and 'axis' (a number for astigmatism measured in degrees), with dependencies that require 'axis' if 'cylindrical' is present, and vice versa; and 'addNear', an object with an optional property 'spherical' (a number with default 0 for near vision correction in diopters). The main properties of the schema are: 'personalSettings' is an object, required. It includes properties: 'name', a string; 'gender', a string which may provide defaults for some anthropometric data; 'eyes', an object that requires properties 'left' and 'right', both of which reference the definition 'eyeData'; and 'anthropometric', an object containing optional properties: 'standingEyeHeight', a number; 'seatedEyeHeight', a number; and 'eyeToNeck', a number. No additional properties are allowed at the root level.
