The schema describes the reward API data input structure. It is an object type. It defines several reusable components under 'definitions': 'rewardUnit' is a string, the unit of credit associated with this reward (i.e. months or usd). 'rewardName' is a string or null type. 'rewardDateExpires' is an integer or null type, the date this reward will expire. 'rewardCancellable' is a boolean or null type, whether this reward can be cancelled. 'rewardDateScheduledFor' is an integer or null type, if the reward is created in a PENDING state this is the date we will attempt to 'give' it. 'integrationSettings' is an object type with properties: 'templateId', which is a string, 'giftId', which is a string, and 'utid', which is a string. The main properties of the schema are: 'type' is a string required property that must be one of the following values: 'PCT_DISCOUNT', 'FUELTANK', 'CREDIT', 'INTEGRATION'. There are dependencies based on the value of 'type'. If 'type' is 'PCT_DISCOUNT', 'discountPercent' is a required integer property with an exclusive minimum of 0 and a maximum of 100, 'name', 'dateExpires', and 'cancellable' are required properties that reference definitions. If 'type' is 'FUELTANK', 'fuelTankCode' is a required string property, 'fuelTankType' is a required string property that must be one of the following values: 'PCT_DISCOUNT', 'CREDIT', 'amount' is a required string property. If 'type' is 'CREDIT', 'unit' is a required property that references a definition, and 'assignedCredit' is a required integer property with an exclusive minimum of 0. If 'type' is 'INTEGRATION', 'unit' is a required property that references a definition, 'valueInCents' is a required string property, and 'integrationSettings' is a required property that references a definition. No additional properties are allowed at the root level.
