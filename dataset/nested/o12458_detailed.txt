The schema describes model-level pipette specifications. It is an object type. It defines several reusable components under 'definitions': 'positiveNumber' is a number, with a minimum value of 0. 'channels' is an enum, which accepts the values 1 and 8. 'displayCategory' is a string, with allowed values 'GEN1' and 'GEN2'. The main properties of the schema are: 'channels' is a required property which is a reference to the 'channels' definition. 'defaultAspirateFlowRate' is a required property, containing nested properties 'value', 'min', and 'max', each referencing the 'positiveNumber' definition. 'defaultDispenseFlowRate' is a required property, structured similarly to 'defaultAspirateFlowRate'. 'defaultBlowOutFlowRate' is a required property, also structured like 'defaultAspirateFlowRate'. 'displayName' is a required string property. 'displayCategory' is a required property which is a reference to the 'displayCategory' definition. 'maxVolume' is a required property which is a reference to the 'positiveNumber' definition. 'minVolume' is a required property which is a reference to the 'positiveNumber' definition. 'smoothieConfigs' is an optional property which is an object, containing properties 'stepsPerMM', 'homePosition', and 'travelDistance', each being a number. No additional properties are allowed at the root level.
