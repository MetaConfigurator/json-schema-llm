The schema describes a Cube. It is an object type. It defines several reusable components under 'definitions': 'attribute_base' is an object, it requires the field 'name'. The 'attribute_base' has properties 'description', 'format', 'info', 'label', 'missing_value', and 'order', where 'missing_value' can be a string, number, null, or boolean, and 'order' can be either 'asc' or 'desc'. The main properties of the schema are: 'aggregates' is an array containing items that can either be a string or an object conforming to 'attribute_base' and including properties 'expression', 'formula', 'function', and 'measure'. 'browser_options' is an optional object. 'category' is a string. 'description' is a string. 'details' is an array containing items that can either be a string or an object conforming to 'attribute_base' and including an array property 'locales'. 'dimensions' is an array of items that can be either strings or objects, titled 'Dimension name'. 'fact' is a string. 'hierarchies' is an object with pattern properties that allow for arrays of strings. 'implicit_aggregates' is a boolean. 'info' is an optional object. 'joins' is an array. 'key' is a string. 'label' is a string. 'mappings' is an optional object. 'measures' is an array containing items that can either be a string or an object conforming to 'attribute_base' and including properties 'aggregates', 'expression', and 'nonadditive'. 'name' is a string. 'store' is a string. No additional properties are allowed at the root level.
