The schema describes a set of courses offered at Concordia with information about each one. It is an array type. It defines several reusable components under 'definitions': 'courseCodeString' is a string, it must match the pattern '^[A-Z]{4} [0-9]{3,4}$'. 'arrayOfCourseCodeOrLists' is an array, it contains items that are arrays which must have a minimum of 1 item, and each item must be a reference to 'courseCodeString'. 'alphaNumPunctString' is a string, it must match the pattern '^[\wñüúáàíÎöóòëéÉèÈ ,'“”‘’"--­‐‑‒–—/.:;?§()\[\]]+$'. 'posNumRoundedToHundredths' is a string, it must match the pattern '^[0-9](\.[0-9]{1,2})?$'. The main properties of the schema are: 'code' is a string, it is required and must be a reference to 'courseCodeString'. 'name' is a string, it is required and must be a reference to 'alphaNumPunctString'. 'credits' is a string, it is required and must be a reference to 'posNumRoundedToHundredths'. 'description' is a string, it is required and must be a reference to 'alphaNumPunctString'. 'requirements' is an object, it is required and contains properties: 'prereqs' which is a reference to 'arrayOfCourseCodeOrLists', and 'coreqs' which is also a reference to 'arrayOfCourseCodeOrLists'. 'lectureHours' is a string, it is optional. 'tutorialHours' is a string, it is optional. 'labHours' is a string, it is optional. 'note' is a string, it is optional. No additional properties are allowed at the root level.
