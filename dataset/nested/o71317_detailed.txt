The schema describes a configuration object. It is an object type. The main properties of the schema are: 'interface_type' is a string that specifies the type of interface, required to be either 'physical' or 'logical', with a default value of 'physical'. 'interface_names' is an array of strings that represents valid interface names, which are required. Each item in the array must match the pattern '^([a-z]{2,3}-[0-9]{1,}/[0-9]{1,}/[0-9]{1,}(.[0-9]{1,})?)$', and examples include 'ge-0/0/1' and 'xe-0/0/0'. No additional properties are allowed at the root level.
