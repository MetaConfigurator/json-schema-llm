The schema describes a structure for logs and operators. It is an object type. The main properties of the schema are: 'operators' is an array of objects representing operators, and it is required. Each object in the array must contain 'id' (a number) and 'name' (a string), both of which are required. 'logs' is an array of objects representing logs, and it is required. Each log object must include 'description' (a string), 'key' (a string), 'maximum_merge_delay' (a number with a minimum value of 0), 'operated_by' (an array of numbers representing operator numeric ids), and 'url' (a string), all of which are required. Additionally, the 'logs' array must contain at least one item. No additional properties are allowed at the root level.
