The schema describes details about a specific app. It is an object type. It defines several reusable components under 'definitions': 'developer' is an object, which includes required fields 'shortCode', 'name', and 'blurb', and optional fields 'description'. It does not allow additional properties. The main properties of the schema are: 'shortCode' is a string that describes the app's shortcode and is required. 'name' is a string that describes the app's name and is required. 'blurb' is a string that describes the app's blurb and is required. 'description' is a string that describes the app's description and is optional. 'logoUrl' is a string formatted as a URL that describes the app's logo and is optional. 'learnMoreUrl' is a string that describes the app's learnMoreUrl and is optional. 'principalType' is a string that describes the app's principal type (if it is SSO) and is optional. 'developer' is a reference to the 'developer' definition, which is a required field. No additional properties are allowed at the root level.
