The schema describes the structure of an operational status object. It is an object type. It defines several reusable components under 'definitions': 'app' is a string, representing the application name. 'elapsedTime' is an integer, representing elapsed time of the operation in milliseconds. 'tenantEvent' is an object, requiring the 'kind' property which can be one of the following enum values: 'patchSetApplyRebootStart', 'bumpSecurityGenerationRebootStart', 'simpleRebootStart', 'tenantPreHaltStart', 'tenantPreHaltComplete', 'tenantHaltStart', 'tenantHaltComplete', 'appPatchesApplyStart', 'appPatchesApplyComplete', 'bumpSecurityGenerationComplete', 'tenantPreStartStart', 'tenantPreStartComplete', 'tenantStartStart', 'tenantStartComplete', 'patchSetApplyRebootComplete', 'bumpSecurityGenerationRebootComplete', 'simpleRebootComplete'. 'appEvent' is an object, requiring the 'kind' and 'app' properties; 'kind' can be one of: 'appPreHaltStart', 'appPreHaltError', 'appPreHaltTimeout', 'appPreHaltComplete', 'appHaltStart', 'appHaltError', 'appHaltTimeout', 'appHaltComplete', 'appPreStartStart', 'appPreStartError', 'appPreStartTimeout', 'appPreStartComplete', 'appStartStart', 'appStartError', 'appStartTimeout', 'appStartComplete', 'appInstallStart', 'appInstallError', 'appInstallTimeout', 'appInstallComplete', 'appUninstallStart', 'appUninstallError', 'appUninstallTimeout', 'appUninstallComplete', 'appEditComplete'. 'app' references the 'app' definition. 'tenantStatusEvent' is an object, requiring the 'kind', 'incarnation', and 'securityGeneration' properties; 'kind' must be 'tenantStatusEvent', and both 'incarnation' and 'securityGeneration' are integers. 'orderedEvent' is an object, requiring the 'seq', 'time', and 'event' properties; 'seq' is an integer, 'time' is a string formatted as date-time, and 'event' can be one of the defined event types: 'tenantEvent', 'appEvent', or 'tenantStatusEvent'. The main properties of the schema are: 'status' is an optional enum with values 'notLaunched', 'inProcess', 'complete'; 'latestEvent' is an optional integer; 'continuePage' is an optional string, describing the URI for redirection after a reboot; 'events' is an optional array of items, each referencing the 'orderedEvent' definition. No additional properties are allowed at the root level.
