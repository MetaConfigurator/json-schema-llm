The schema describes a community structure. It is an object type. It defines several reusable components under 'definitions': 'identifier' is an object, a generic identifier. It has required fields 'scheme' and 'identifier'. The main properties of the schema are: '$schema' is a string that describes the JSONSchema for the community and is required. 'id' is a string that serves as the community identifier and is required. 'title' is a string representing the community title and is required. 'description' is a string that gives the community description and is optional. 'type' is a string that represents the community type and is required, with allowed values: 'organization', 'event', 'topic', 'project'. 'alternate_identifiers' is an array that contains alternate identifiers for the community, where the items must follow the 'identifier' definition, and unique items are enforced. 'curation_policy' is a string that indicates the community curation policy and is optional. 'page' is a string that represents the community page and is optional. 'website' is a string that provides the community website and is optional. 'domains' is an array that holds community domains, where the items must be strings, and unique items are enforced. 'funding' is an array for community funding, containing unique string items. 'verified' is a boolean indicating if the community is verified and is optional. 'visibility' is a string representing community visibility, with allowed values: 'public', 'private', 'hidden', and is optional. 'member_policy' is a string that defines the community member policy, with allowed values: 'open', 'closed', and is optional. 'record_policy' is a string that shows the community record policy, with allowed values: 'open', 'closed', 'restricted', and is optional. 'archived' is a boolean indicating if the community is archived and is optional. 'created_by' is an integer representing the community creator and is optional. No additional properties are allowed at the root level.
