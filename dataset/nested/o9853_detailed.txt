The schema describes a configuration for device flows. It is an object type. The main properties of the schema are: 'flowId' is a string that must match the pattern '^[A-Fa-f\d]{24}$' and is required. 'version' is a string or null, must have a minimum length of 1 and a maximum length of 255, and is required. 'deviceIds' is an array of strings that must match the pattern '^[A-Fa-f\d]{24}$', with a maximum of 1000 items. 'deviceTags' is an array of objects, each containing properties 'key' which is a string matching the pattern '^[0-9a-zA-Z_-]{1,255}$' and 'value' which is a string with a minimum length of 1 and a maximum length of 255. Each object in 'deviceTags' disallows additional properties and is limited to a maximum of 100 items. No additional properties are allowed at the root level.
