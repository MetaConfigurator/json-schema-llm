The schema describes a payment option configuration. It is an object type. The main properties of the schema are: 'accountHolderName' is a string representing the account holder's name, required, with a minimum length of 1 and a maximum length of 200. 'bankAccount' is a string representing the bank account number, optional, with a minimum length of 1 and a maximum length of 20. 'currencies' is an array of strings representing permitted currencies, optional, requiring at least 1 item, with items being either 'USD' or 'GBP'. 'iban' is a string representing the International Bank Account Number, optional, with a minimum length of 12, maximum length of 39, and a specific pattern for validation. 'name' is a string representing a short name for this payment option, required, with a minimum length of 1 and a maximum length of 20. 'notes' is a string providing any additional information, optional. 'sortCode' is a string representing the bank or sort code, optional, with a minimum length of 1 and maximum length of 10. 'type' is a string indicating the type of payment, readonly, with allowed values of 'Online', 'Bank', or 'InPerson'. No additional properties are allowed at the root level.
