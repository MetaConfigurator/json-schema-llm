The schema describes list items configuration for a single CSV list. It is an object type. It defines several reusable components under 'definitions': 'listItem' is an object, single list item definition (hierarchical). It has required fields: 'value', 'label', 'inactive', and 'children'. The 'value' is a string that must have a minimum length of 1 and a maximum length of 255. The 'label' is a string that must have a minimum length of 1. The 'inactive' is a boolean with a default value of false. The 'children' is an array of items, where each item must conform to the 'listItem' definition. No additional properties are allowed at the root level. The main properties of the schema are: 'items' is an array of items for a given list, which is required, must have at least 1 item, and must contain unique items, where each item conforms to the 'listItem' definition.
