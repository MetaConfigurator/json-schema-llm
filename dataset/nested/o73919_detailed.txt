The schema describes JSON schema for Entry Point Regulation manifest files. It is an object type. It defines several reusable components under 'definitions': 'behavior' is a string type, allowing values: 'block', 'allow', 'noAuth'. 'rule' is an object type, with properties: 'path', a string (optional, relative path to resource); 'regex', a string (optional, a regular expression for identifying paths to resources); 'types', an array of strings (optional, with allowed values: 'navigation', 'image', 'stylesheet', 'script', 'xhr', 'other'); 'allowData', a boolean (optional, determines if data is allowed at this path). The main properties of the schema are: 'site' is a string (required, a fully qualified URL of your website, must be a valid URI); 'maxAge' is an integer (optional, set the max age HTTP cache expiration); 'reportUrl' is a string (optional, must be a valid URI); 'defaultNavBehavior' is a string (optional, references 'behavior'); 'defaultResBehavior' is a string (optional, references 'behavior'); 'rules' is an array (required, items reference 'rule'). No additional properties are allowed at the root level.
