The schema describes the JSON Schema for the Static Web Apps CLI (SWA CLI) configuration file. The main properties of the schema are: 'configuration' is an object that specifies settings used by the CLI to run and manage a local development environment for static web applications and their APIs, it allows additional properties and must satisfy a combination of schema rules defined using allOf. Within each configuration object, several properties are ddefined: apiLocation, a string representing the API folder or Azure Functions emulator address; apiPort, a number defining the API backend port; apiPrefix, a string restricted to the value "api"; appArtifactLocation, a string specifying the location of the build output directory relative to the application location; appLocation, a string representing the location of the static app source code; build, a boolean indicating whether the project should be built; customUrlScheme, a string specifying a custom URL scheme; devserverTimeout, a number representing the time in milliseconds to wait for the development server to start; host, a string defining the CLI host address; overridableErrorCode, an array of numbers representing error codes that can be overridden; port, a number specifying the CLI port; run, a string describing a command to run at startup; ssl, a boolean indicating whether the app and API should be served over HTTPS; sslCert, a string specifying the SSL certificate file used for HTTPS; sslKey, a string specifying the SSL key file; swaConfigFilename, a string restricted to the value "staticwebapp.config.json"; swaConfigFilenameLegacy, a string restricted to the value "routes.json"; swaConfigLocation, a string specifying the location of the configuration file; verbose, a string controlling verbose output; and context, an additional optional string property representing the configuration context. No additional properties are allowed at the root level.
