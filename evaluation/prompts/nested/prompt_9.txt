The schema describes JSON schema for djLint's configuration file. It is an object type. It defines several reusable components under 'definitions': 'css-beautify' is an object, which provides beautification settings for CSS. 'js-beautify' is an object, which provides beautification settings for JavaScript. The main properties of the schema are: '$schema' is a string optional, 'blank_line_after_tag' is a string optional, 'blank_line_before_tag' is a string optional, 'close_void_tags' is a boolean optional, 'css' is a css-beautify optional, 'custom_blocks' is a string optional, 'custom_html' is a string optional, 'exclude' is a string optional, 'extend_exclude' is a string optional, 'extension' is a string optional, 'files' is an array of strings optional, 'format_attribute_template_tags' is a boolean optional, 'format_css' is a boolean optional, 'format_js' is a boolean optional, 'ignore' is a string optional, 'ignore_blocks' is a string optional, 'ignore_case' is a boolean optional, 'include' is a string optional, 'indent' is a number optional, 'js' is a js-beautify optional, 'line_break_after_multiline_tag' is a boolean optional, 'linter_output_format' is a string optional, 'max_attribute_length' is a number optional, 'max_blank_lines' is a number optional, 'max_line_length' is a number optional, 'no_function_formatting' is a boolean optional, 'no_line_after_yaml' is a boolean optional, 'no_set_formatting' is a boolean optional, 'per-file-ignores' is an object optional, 'preserve_blank_lines' is a boolean optional, 'preserve_leading_space' is a boolean optional, 'profile' is a string required with enum values 'html', 'django', 'jinja', 'nunjucks', 'handlebars', 'golang', 'angular'. 'require_pragma' is a boolean optional, 'use_gitignore' is a boolean optional. No additional properties are allowed at the root level.
