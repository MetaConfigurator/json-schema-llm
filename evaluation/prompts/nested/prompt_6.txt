The schema describes configuration for a Grunt task. It is an object type. It defines several reusable components under 'definitions': 'dynamic' is an object with properties : 'expand' which is a boolean with a default of false, 'cwd' which is a string, 'dest' which is a string, 'ext' which is a string, 'extDot' which is an enum with allowed values 'first' and 'last', defaulting to 'first', 'filter' is an enum with allowed values 'isFile', 'isDirectory', 'isBlockDevice', 'isCharacterDevice', 'isSymbolicLink', 'isFIFO', 'isSocket', 'flatten' is a boolean defaulting to false, 'matchBase' is a boolean defaulting to false, 'nonull' is a boolean defaulting to false, 'rename' is a string and 'src' which is a required array of strings that must contain at least one unique item . 'fileFormat' is an object with a property 'files' which can either be an object with additional properties of type array or string (with unique items and minimum of one), or an array of items referring to the 'dynamic' definition. The main properties of the schema are: 'options' is an optional object. The schema allows for additional properties at the rool level that should be either definition fileFormat or definition dynamic .
