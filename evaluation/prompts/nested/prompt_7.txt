The schema describes the JSON Schema for the Embrace mobile monitoring SDK configuration file. It defines a configuration object using JSON Schema Draft-04. The root element is an object that allows additional properties and includes a title, description, and schema ID URL corresponding to version 1.0.0 of the configuration specification. The configuration requires two properties: app_id, which is a string exactly five characters long representing the application identifier, and api_token, which is a string exactly thirty-two characters long representing a hexadecimal API token. The schema also defines an optional boolean property ndk_enabled, which enables native crash capture and documents that the default behavior is false. Another optional property, sdk_config, is an object that allows additional properties and must contain at least one property if present; it groups advanced SDK configuration sections. Within sdk_config, several optional subsections are defined, each as an object allowing additional properties and requiring at least one property if included. The app section contains a boolean report_disk_usage property describing whether disk usage collection is enabled, with documentation noting that it defaults to true. The crash_handler section includes a boolean enabled property controlling whether the SDK connects to the uncaught exception handler, defaulting to true. The networking section defines a boolean capture_request_content_length (default true), an array disabled_url_patterns consisting of unique string items with at least one entry representing regular expression patterns used to exclude URLs from capture, a boolean enable_native_monitoring (default true), and a string trace_id_header describing the header name used for trace IDs with its default value explained in the description. The session section includes a boolean async_end (default false) and an integer max_session_seconds that enables automatic session termination and must be at least sixty seconds if specified. The startup_moment section defines a boolean automatically_end (default true) and a boolean take_screenshot (default false). The taps section includes a boolean capture_coordinates property controlling whether tap coordinates are captured, defaulting to true. The webview section defines a boolean capture_query_params (default true) and a boolean enable (default true) controlling web view data capture, with documentation explaining that query parameter capture has no effect if web view monitoring is disabled. Each property includes descriptive documentation explaining behavior and default values, and the overall schema represents a valid JSON object definition.
