The schema describes a rule-set and config for the Solidarity JSON checker. It is an object type. It defines several reusable components under 'definitions': 'cli', 'dir', 'file', 'env', 'shell', and 'custom'. 'cli' is an object, representing a CLI Rule, with fields 'rule' (required with the enum value 'cli'), 'binary' (required and of type string), 'semver' (optional and of type string), 'version' (optional and of type string), 'line' (optional and of type string or integer), 'matchIndex' (optional and of type integer), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows" ), 'error' (optional string type) and 'ci' (optional boolean type). 'dir' is an object, representing a Dir Rule, with fields 'rule' (required with enum values 'dir' and 'directory'), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows"), 'error' (optional string type) and 'ci' (optional boolean type). 'file' is an object, representing a File Rule, with fields 'rule' (required with the enum value 'file'), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows"), 'error' (optional string type) and 'ci' (optional boolean type). 'env' is an object, representing an ENV Rule, with fields 'rule' (required with the enum value 'env'), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows"), 'error' (optional string type) and 'ci' (optional boolean type). 'shell' is an object, representing a Shell Rule, with fields 'rule' (required with the enum value 'shell'), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows"), 'error' (optional string type), 'ci' (optional boolean type) and 'match' (required and of type string). 'custom' is an object, representing a Custom Rule, with fields 'rule' (required with the enum value 'custom'), 'plugin' (required and of type string), 'name' (required and of type string), 'platform' (optional with allowed values "darwin","macos","freebsd","linux","sunos","win32","windows"), 'error' (optional string type), 'ci' (optional boolean type) and 'match' (optional and of type string). The main properties of the schema are: 'config' is an object representing the configuration settings, with an optional property 'output' (of type string) with allowed values 'moderate', 'verbose', 'silent'. 'requirements' is an object representing a list of requirement rules, which is required, and contains an array of items that must each match one of the defined rules ('cli', 'dir', 'file', 'env', 'shell', 'custom'), with a minimum of 1 item and uniqueness enforced. No additional properties are allowed at the root level.
